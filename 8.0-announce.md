# FreeBSD 8.0-RELEASE 发行公告

日期：2009 年 11 月 25 日

FreeBSD 发行工程团队很高兴地宣布 FreeBSD 8.0-RELEASE 现已可用。此版本开启了新的 8-STABLE 分支，提升了 FreeBSD 7.X 的功能并引入许多新特性。部分亮点：

* Xen Dom-U、VirtualBox 主客机、分层 Jail
* NFSv3 GSSAPI 支持，实验性 NFSv4 客户端和服务器
* 802.11s D3.03 无线网状网络和虚拟接入点支持
* ZFS 不再处于实验性状态
* USB 子系统从头重写，包括 USB 目标模式支持
* 多个领域持续进行 SMP 可扩展性改进，尤其是 VFS
* 修订的网络链路层子系统
* 实验性 MIPS 架构支持

[新闻稿](8.0-pressrelease.md) 包含关于此版本的更多信息。

关于新功能和已知问题的完整列表，请参阅在线发行说明和勘误列表，地址如下：

* <http://www.FreeBSD.org/releases/8.0R/relnotes.html>

## 可用性

FreeBSD 8.0-RELEASE 现已可用于 amd64、i386、ia64、pc98、powerpc 和 sparc64 架构。

FreeBSD 8.0 可以从可引导 ISO 镜像或通过网络安装。某些架构（目前为 amd64 和 i386）还支持从 USB 闪存盘安装。所需文件可通过 FTP 或 BitTorrent 下载，如下节所述。虽然一些较小的 FTP 镜像可能不包含所有架构，但它们通常都会包含较常见的架构，如 amd64 和 i386。

发行版 ISO 镜像的 MD5 和 SHA256 校验和附于本消息底部。

发行版中所提供镜像的用途如下：

**dvd1：**

包含安装 FreeBSD 基本操作系统所需的全部内容、一组预构建的软件包以及文档。它还支持引导进入基于“livefs”的救援模式。如果你能刻录并使用 DVD 介质的媒体，这应该能满足你的所有需求。

**disc1：**

包含适用于 CDROM 介质的 FreeBSD 基本操作系统和文档包。不包含其他软件包。

**livefs：**

包含对引导进入基于“livefs”的救援模式的支持，但不支持从 CD 本身进行安装。它旨在帮助救援现有系统，但如有必要也可用于进行基于网络的安装。

**bootonly：**

支持使用 CDROM 驱动器引导机器，但不包含从 CD 本身安装 FreeBSD 的支持。你需要从 CD 引导后进行基于网络的安装（例如从 FTP 服务器）。

**memstick：**

可写入 USB 闪存盘（闪存盘）并用于在能够从 USB 驱动器引导的机器上进行安装。它还支持引导进入基于“livefs”的救援模式。提供文档包，但不提供其他软件包。

作为使用 memstick 镜像的一个示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似以下命令应该可行：

```sh
# dd if=8.0-RELEASE-amd64-memstick.img of=/dev/da0 bs=10240 conv=sync
```

请务必确保目标（`of=`）正确。

## FreeBSD Update

[freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+8.0-RELEASE) 工具支持对运行早期 FreeBSD 发行版的 i386 和 amd64 系统进行二进制升级。运行 7.[012]-RELEASE、8.0-BETA[1234] 或 8.0-RC[123] 的系统可按以下方式升级：

```sh
# freebsd-update upgrade -r 8.0-RELEASE
```

在此过程中，FreeBSD Update 可能会要求用户协助合并一些配置文件，或确认自动执行的合并是否正确完成。

```sh
# freebsd-update install
```

在继续之前，必须使用新安装的内核重新启动系统。

```sh
# shutdown -r now
```

重新启动后，需要再次运行 `freebsd-update` 以安装新的用户空间组件：

```sh
# freebsd-update install
```

此时，从 FreeBSD 8.0-BETA2 或更早版本升级的系统用户将被 `freebsd-update` 提示重新构建所有第三方应用程序（例如从 ports 树安装的 Port），原因是系统库的更新。请参阅：

* <http://www.daemonology.net/blog/2009-07-11-freebsd-update-to-8.0-beta1.html>

了解更多详情。在更新已安装的第三方应用程序之后（并且仅在 `freebsd-update` 打印消息指示需要这样做时），再次运行 `freebsd-update`，以便它可以删除旧的（不再使用的）系统库：

```sh
# freebsd-update install
```

最后，重新启动进入 8.0-RELEASE：

```sh
# shutdown -r now
```
