本文档列出了 FreeBSD 4.5-RELEASE 的勘误项，包含发行后才发现的重要信息。这些信息包括安全公告，以及与可能影响其运行或可用性的软件或文档相关的消息。在安装此版本 FreeBSD 之前，应始终查阅本文档的最新版本。

FreeBSD 4.5-RELEASE 的勘误文档将维护至 FreeBSD 4.6-RELEASE 发布。

## 1 引言

本勘误文档包含有关 FreeBSD 4.5-RELEASE 的“最新消息”。在安装此版本之前，请务必查阅本文档，以了解发行后才发现的或已经发现并修复的问题。

随发行版实际分发的任何勘误文档（例如 CDROM 发行版中的）按定义已过时，但其他副本会在互联网上保持更新，应作为此版本的“当前勘误”查阅。这些其他勘误副本位于 [http://www.FreeBSD.org/releases/](http://www.freebsd.org/releases/)，以及任何保持此位置最新镜像的站点。

FreeBSD 4-STABLE 的源代码和二进制快照也包含本文档的最新副本（截至快照时的版本）。

有关所有 FreeBSD CERT 安全公告的列表，请参见 [http://www.FreeBSD.org/security/](http://www.freebsd.org/security/) 或 [ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT/](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/)。

## 2 安全公告

存在一个竞态条件，文件可能在 [fstatfs(2)](https://man.freebsd.org/cgi/man.cgi?query=fstatfs&sektion=2&manpath=FreeBSD+4.5-stable) 调用和文件被访问之间被删除，导致内核 panic。已知仅 [procfs(5)](https://man.freebsd.org/cgi/man.cgi?query=procfs&sektion=5&manpath=FreeBSD+4.5-stable) 文件系统容易受到此攻击。此 bug 已在 FreeBSD 4.5-RELEASE 中修复，但描述该 bug 的安全公告在发行之后才发布。更多信息（包括变通方案和修复）请参见安全公告 [FreeBSD-SA-02:09](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:09.fstatfs.asc)。

OpenSSH 多路复用代码中的“差一”bug 已修复。此 bug 可能允许经过认证的远程用户使 [sshd(8)](https://man.freebsd.org/cgi/man.cgi?query=sshd&sektion=8&manpath=FreeBSD+4.5-stable) 以超级用户权限执行任意代码，或允许恶意 SSH 服务器以客户端用户权限在客户端系统上执行任意代码。基本系统和 Ports 中的 OpenSSH 版本的各种变通方案和修复可在安全公告 [FreeBSD-SA-02:13](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:13.openssh.asc) 中找到。

zlib 中的编程错误可能导致多次释放内存。FreeBSD 中使用的 [malloc(3)](https://man.freebsd.org/cgi/man.cgi?query=malloc&sektion=3&manpath=FreeBSD+4.5-stable)/[free(3)](https://man.freebsd.org/cgi/man.cgi?query=free&sektion=3&manpath=FreeBSD+4.5-stable) 例程不易受此错误影响，但接收特制无效压缩数据块的应用程序可能运行异常或中止。此 zlib bug 已修复。变通方案和解决方案请参见安全公告 [FreeBSD-SA-02:18](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:18.zlib.v1.2.asc)。

TCP SYN 缓存（“syncache”）和 SYN cookie（“syncookie”）实现中的 bug 已修复，这些 bug 可能导致合法的 TCP/IP 流量使机器崩溃。变通方案和补丁请参见安全公告 [FreeBSD-SA-02:20](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:20.syncache.asc)。

路由表内存泄漏已修复，该漏洞可能允许远程攻击者耗尽目标机器的内存。变通方案和补丁可在安全公告 [FreeBSD-SA-02:21](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:21.tcpip.asc) 中找到。

内存映射 I/O 中的 bug 已修复，该 bug 可能导致系统崩溃。有关解决方案的更多信息请参见安全公告 [FreeBSD-SA-02:22](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:22.mmap.asc)。

安全漏洞已修复，该漏洞中 SUID 程序可能通过操纵其标准 I/O 文件描述符读取或写入不当文件。有关解决方案的信息可在安全公告 [FreeBSD-SA-02:23](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:23.stdio.asc) 中找到。

与 [su(1)](https://man.freebsd.org/cgi/man.cgi?query=su&sektion=1&manpath=FreeBSD+4.5-stable) 不同，[k5su(8)](https://man.freebsd.org/cgi/man.cgi?query=k5su&sektion=8&manpath=FreeBSD+4.5-stable) 在尝试成为超级用户时不要求调用用户是 `wheel` 组成员。这可能导致某些意外行为。为避免此情况，[k5su(8)](https://man.freebsd.org/cgi/man.cgi?query=k5su&sektion=8&manpath=FreeBSD+4.5-stable) 现在默认以非 SUID 方式安装（实际上禁用了它）。更多信息可在安全公告 [FreeBSD-SA-02:24](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:24.k5su.asc) 中找到。

[bzip2(1)](https://man.freebsd.org/cgi/man.cgi?query=bzip2&sektion=1&manpath=FreeBSD+4.5-stable) 工具中发现多个漏洞，可能允许文件被无警告覆盖或允许本地用户意外访问文件。这些问题已通过新导入的 bzip2 修复。更多信息请参见安全公告 [FreeBSD-SA-02:25](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:25.bzip2.asc)。

## 3 系统更新信息

某些 SSH 客户端在尝试连接到 FreeBSD 4.5-RELEASE 服务器时，会意外地出现 S/Key 提示，即使服务器上的 [sshd(8)](https://man.freebsd.org/cgi/man.cgi?query=sshd&sektion=8&manpath=FreeBSD+4.5-stable) 未明确配置 S/Key 认证。这是因为客户端的默认设置已更改（例如使用以前未使用的 SSH 协议版本 2），或客户端旧的默认认证序列发生变化。有多种方法可禁用此行为：

- 在较新的 OpenSSH 客户端上，将以下行添加到你的 **~/.ssh/config** 文件：

  ```sh
  PreferredAuthentications publickey,password,keyboard-interactive
  ```

- 对于 PuTTY 客户端，无法更改认证序列顺序，但可在设置中禁用键盘交互认证。

- 要在服务器上禁用键盘交互认证，请取消注释服务器主机上 **/etc/ssh/sshd_config** 文件中的以下行：

  ```sh
  ChallengeResponseAuthentication no
  ```

此问题已在 FreeBSD 4.6-RC 中修复。

发行说明中提到了新的 sbni 设备驱动程序，但给出了 FreeBSD Ports 中用于配置该驱动程序的程序的错误引用。该 port 的正确文件名是 **sysutils/sbniconfig**。

Linux 模拟现在要求内核配置中包含 `options SYSVSEM`。此依赖关系在 FreeBSD 4.5-RELEASE 之前引入。

包含 KDE 某些可选组件的软件包被意外地从 ISO 镜像（因此也从官方 4 CD 套装）中遗漏。在之前的发行版中，这些软件包可使用 **x11/kde2** 软件包安装。这些组件可使用 FreeBSD Ports 安装，或从某个 FreeBSD FTP 服务器下载二进制软件包。受影响的 port 有：**games/kdegames2**、**misc/kdeutils2**、**editors/koffice**、**net/kdenetwork2**、**graphics/kdegraphics2** 和 **audio/kdemultimedia2**。请注意，这些其他组件经常需要的 **x11/kdelibs2** 和 **x11/kdebase2** 软件包**确实**包含在官方 4 CD 套装的第 1 张光盘中。[1]

包含 Samba 的二进制软件包被意外地从 ISO 镜像中遗漏。此软件可使用 FreeBSD Ports 中的 **net/samba** port 安装，或从某个 FreeBSD FTP 服务器下载并安装其二进制软件包。

soft updates 中的 bug 已修复，该 bug 可能在执行大量文件系统活动（如安装新内核或其他软件）后立即关闭系统时导致偶尔的文件系统损坏。系统关闭时无法刷新所有缓冲区并会报告此情况。变通方案是在重启前运行几次 [sync(8)](https://man.freebsd.org/cgi/man.cgi?query=sync&sektion=8&manpath=FreeBSD+4.5-stable)，或通过更新到最近的 FreeBSD 4.6-RC 快照来解决。

ciss 驱动程序被意外地从 FreeBSD 4.5-RELEASE 的 `GENERIC` 内核中遗漏，因此无法（或至少非常困难）初始安装到连接到此控制器的磁盘。此问题已在 FreeBSD 4.6-RC 中修复。

[1] 完整的 FreeBSD 软件包集合目前可装满九张 CDROM。因此官方 4 CD 套装仅包含可用软件包的子集。几家 FreeBSD 供应商提供的发行版包含更完整的软件包集；更完整的集合也可在 FreeBSD FTP 站点上找到。
