# FreeBSD 3.3-RELEASE 勘误

文件 ERRATA.TXT 包含 3.3 发行后的勘误信息，在报告本发行版的问题之前，应始终将其视为首先查阅的权威来源。此文件也会随着新问题的报告而定期更新，因此即使你最近已查看过此文件，在提交错误报告之前也请再次查阅。对此文件的任何更改也会自动发送电子邮件至：

`freebsd-stable@FreeBSD.org`

有关所有 FreeBSD 安全公告，请参见：

**ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT/**

以获取最新信息。

---- 安全公告：

当前针对 3.3 的有效安全公告数量：无

---- 系统更新信息：

X 桌面菜单中的 fvwm 桌面选项无法使用。

修复：从 **/usr/ports/x11-wm/fvwm2** 安装 fvwm，而不是使用桌面菜单项，并将“fvwm”写入你的 **$HOME/.xinitrc** 和 **$HOME/.xsession** 文件以供 `startx`/`xdm` 使用。你也可以从 3.3-STABLE 源代码构建并重新安装 **/usr/src/release/sysinstall** 来修复有问题的 X 桌面菜单项。

lo0（环回）设备在启动时未配置，导致 `mountd` 等实用程序失败。

修复：假设你确实遇到了此问题，请编辑 **/etc/rc.conf** 并搜索变量 `network_interfaces` 的设置位置。在其值中，将“auto”改为“lo0”，因为 auto 关键字无法正确启动环回设备，其原因尚待充分查明。由于初始安装后你的其他接口已经设置在变量 `network_interfaces` 中，因此只需在 rc.conf 中将 auto 替换为 lo0 然后继续即可。

3.3 ISO 镜像（以及 Walnut Creek CDROM 的 3.3 CDROM #1）在 ATAPI CDROM 设备上莫名其妙地无法引导，但在使用 SCSI CDROM 时可正常工作（需适配器支持可引导 CD）。

修复：使用引导软盘安装（参见 floppies/README.TXT）而非从 CDROM 引导，或从以下地址获取更新后的 ISO 镜像：

**ftp://ftp.FreeBSD.org/pub/FreeBSD/releases/i386/ISO-IMAGES/**

另请参阅该目录中的 CHECKSUM.MD5 文件，以验证你拥有的是“旧”镜像还是新镜像——某些镜像站点在获取 ISO 9660 安装镜像的未压缩版和 gzip 版时可能较慢。一如既往，Walnut Creek CDROM 也会在 3.3-RELEASE 产品购买者要求时（一旦可用）提供替换 CD。

此问题由 mkisofs 中的一个错误引起，我们仍在追查，但目前已暂时规避。

**ppp(8)** 在直接模式下无法正确检测载波。

修复：从以下地址下载并安装最新版本的 **ppp(8)**：

- <http://www.FreeBSD.org/~brian/（美国>）
- <http://www.Awfulhak.org/~brian/（英国>）

或将系统升级至 -stable。

如果你不想重新编译内核，可以编辑键映射文件使其不会导致 panic。在 **/usr/share/syscons/keymap** 中找到你的键映射文件。用编辑器程序打开它，找到以下行：

```sh
  057   ' '  ' '  nul  ' '  ' '  ' '  susp ' '  O
                                          ~~~~
```

将其改为：

```sh
  057   ' '  ' '  nul  ' '  ' '  ' '  ' '  ' '  O
                                          ~~~ (a quoted space)
```

你还必须修改下一行：

```sh
  104   slock saver slock saver susp nop  susp nop  O
                                    ~~~~      ~~~~
```

将其编辑为：

```sh
  104   slock saver slock saver nop  nop  nop  nop  O
```
