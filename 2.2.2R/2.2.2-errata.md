# FreeBSD 2.2.2-RELEASE 勘误

最后一刻的勘误：

- 以 root 身份登录时，系统控制台会显示 “login_getclass: unknown class ‘root’”。

  修复：如果你已安装源码发行版，只需执行：

  ```sh
  cp /usr/src/etc/login.conf /etc
  ```

  否则，请从 FreeBSD FTP 站点使用以下 URL 获取：
  `ftp://ftp.FreeBSD.org/pub/FreeBSD/FreeBSD-current/src/etc/login.conf`
  只需切换到 **/etc** 目录，然后使用提供的 URL 运行 `fetch(1)` 即可。

- 如果再次运行 `sysconfig` 会打乱 `rc.conf`。

  修复：从 `RELENG_2_2` 分支获取更新的 **/usr/src**，构建 **/usr/src/release/sysinstall**，然后将新的二进制文件复制到 **/stand**。

  如果你没有足够的空间存放源码，也可以使用更高版本 `2.2-YYMMDD-RELENG` 发行版的 boot/fixit 软盘组合，直接挂载你的根分区（使用 Fixit 选项），然后将软盘上的 **/stand/sysinstall** 复制到根文件系统的 **/stand**。

- 安装软盘完全无法引导——而 2.2.1 的软盘工作正常。在它尝试将根设备更改为 `fd0c` 后，会立即出现 “panic: double fault”。

  修复：问题在于你的内存为 48MB，而 FreeBSD 在 2.2.1 和 2.2.2 之间发生了某种非常神秘的变化，导致恰好在该内存大小下失败。鉴于 16MB SIMM 的流行程度，这也解释了为什么我们谁都没遇到过这个问题，因为我们的系统通常配备 16MB、32MB 或 64MB 内存。:)

  我们正在努力查找并修复此问题，但在此之前，48MB 系统可使用以下变通方案：

  1. 使用 2.2.2 引导软盘进行引导，当出现第一个菜单询问你是否要进入内核配置编辑器时，选择“仅限专家”的 CLI 模式选项。然后输入：

     ```sh
     iosize npx0 32768
     visual
     ```

     <然后像平常一样进行可视化内核配置，完成后退出>

     如果你能进入安装程序，跳到第 3 步。

  2. 如果上述方法无效，从你的机器中物理拆除内存，仅保留 32MB，然后使用引导软盘进行引导。除非你的问题完全离奇且是我们从未见过的，否则你现在应该可以继续第 3 步。

  3. 完成安装，然后从硬盘引导。此次引导应该正常工作，因为你不再使用安装程序所用的内存文件系统，而该文件系统似乎与这些内存大小问题交互不良，从而导致了你所遇到的故障。

     你还需要在某个时刻使用 `-c` 标志引导，并输入 “iosize npx0 0” 以恢复全部内存的使用，因为 32768 这个旧值在初始安装期间已保存到磁盘。如果你已计划构建自定义内核，则可跳过此步骤，因为该值无论如何都会重置。
