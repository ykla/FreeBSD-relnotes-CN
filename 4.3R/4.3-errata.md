# FreeBSD 4.3-RELEASE 勘误

如果在安装此版本 FreeBSD 之前你没有阅读任何其他文档，那么至少务必**阅读本发行版的勘误**，以免在已发现并修复的问题上绊倒。此 ERRATA.TXT 文件按定义显然已经过时，但网络上的其他副本会保持更新，应作为你的发行版的“当前勘误”查阅。这些其他勘误副本位于：

1. <http://www.FreeBSD.org/releases/>
2. **ftp://ftp.FreeBSD.org/pub/FreeBSD/releases/<your-release>/ERRATA.TXT**（以及任何保持此位置最新镜像的站点）。

对此文件的任何更改也会自动发送电子邮件至：

`freebsd-current@FreeBSD.org`

有关所有 FreeBSD 安全公告，请参见：

<http://www.FreeBSD.org/security/>

以获取最新的安全事件信息。

---- 安全公告：

安全公告 FreeBSD-SA-01:39 中记录的漏洞在 FreeBSD 4.3-RELEASE 中修复完成。发行说明中提到了该修复，但未提及安全公告。

**fts(3)** 例程（应用程序用于递归遍历文件系统）中的漏洞可能允许程序对预期目录层级之外的文件进行操作。此 bug 及其修复在安全公告 FreeBSD-SA-01:40 中有所描述。

某个缺陷允许某些信号处理程序在子进程从父进程 exec 之后仍然保持有效。这使得攻击者能够在 setuid 二进制文件的上下文中执行任意代码。更多细节及修复请参见安全公告 FreeBSD-SA-01:42。

**tcpdump(1)** 中的远程缓冲区溢出可通过向目标机器发送特定数据包来触发。更多细节及修复请参见安全公告 FreeBSD-SA-01:48。

**telnetd(8)** 中的远程缓冲区溢出可能导致任意代码在目标机器上运行。更多细节及修复请参见安全公告 FreeBSD-SA-01:49。

远程攻击者可能耗尽目标网络缓冲池的漏洞已关闭。更多细节及修复请参见安全公告 FreeBSD-SA-01:52。

**ipfw(8)** 中存在某缺陷，即 `me` 过滤规则除了匹配预期的本地 IP 地址外，还会匹配点对点接口的远程 IP 地址。更多细节及修复请参见安全公告 FreeBSD-SA-01:53。

**procfs(5)** 中的漏洞可能允许进程从另一进程的内存空间读取敏感信息。更多细节及补丁信息请参见安全公告 FreeBSD-SA-01:55。

tcp_wrappers 中的 PARANOID 主机名检查无法正常工作。更多细节及补丁信息请参见安全公告 FreeBSD-SA-01:56。

**sendmail(8)** 存在本地 root 漏洞。更多细节及补丁信息请参见安全公告 FreeBSD-SA-01:57。

**lpd(8)** 包含可被远程利用的缓冲区溢出。更多细节及修复请参见安全公告 FreeBSD-SA-01:58。

**rmuser(8)** 存在竞态条件，会短暂暴露全局可读的 **/etc/master.passwd**。更多细节、变通方案及解决方案请参见安全公告 FreeBSD-SA-01:59。

---- 系统更新信息：

发行说明中关于 ESS Maestro-3/Allegro 声音驱动程序的条目给出了通过 **/boot/loader.conf** 加载该驱动程序的错误命令。正确的命令是：

```sh
snd_maestro3_load="YES"
```

**ssh(1)** 不再以 root 身份运行 SUID。此更改的主要表现是 `.shosts` 认证可能无法“开箱即用”。临时和永久修复方案在以下 FAQ 中描述：

[http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/faq/admin.html#SSH-SHOSTS](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/faq/admin.html#SSH-SHOSTS)
