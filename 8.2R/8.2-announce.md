# FreeBSD 8.2-RELEASE 发行公告

日期：2011 年 2 月 24 日

FreeBSD 发行工程团队很高兴地宣布 FreeBSD 8.2-RELEASE 现已可用。这是 8-STABLE 分支的第三个发行版本，提升了 FreeBSD 8.1 的功能并引入一些新特性。部分亮点：

* FreeBSD/amd64 中的 Xen HVM 支持和 FreeBSD/i386 中的 Xen PV 支持已改进
* ZFS 磁盘格式更新至版本 15
* 针对 Intel AESNI 加密指令集的 [aesni(4)](http://www.freebsd.org/cgi/man.cgi?query=aesni&sektion=4&manpath=FreeBSD+8.2-RELEASE) 驱动
* BIND 和 OpenSSL 更新
* GNOME 更新至 2.32.1
* KDE 更新至 4.5.5
* 许多其他改进和错误修复

关于新功能和已知问题的完整列表，请参阅在线发行说明和勘误列表，地址如下：

* <http://www.FreeBSD.org/releases/8.2R/relnotes.html>
* <http://www.FreeBSD.org/releases/8.2R/errata.html>

关于 FreeBSD 发行工程活动的更多信息，请参阅：

* <http://www.FreeBSD.org/releng/>

## 可用性

FreeBSD 8.2-RELEASE 现已可用于 amd64、i386、ia64、pc98、powerpc 和 sparc64 架构。

FreeBSD 8.2 可以从可引导 ISO 镜像或通过网络安装。某些架构（目前为 amd64 和 i386）还支持从 USB 闪存盘安装。所需文件可通过 FTP 或 BitTorrent 下载，如下节所述。虽然一些较小的 FTP 镜像可能不包含所有架构，但它们通常都会包含较常见的架构，如 amd64 和 i386。

发行版 ISO 镜像的 MD5 和 SHA256 校验和附于本消息底部。

发行版中所提供镜像的用途如下：

**dvd1：**

包含安装 FreeBSD 基本操作系统所需的全部内容、一组预构建的软件包以及文档。它还支持引导进入基于“livefs”的救援模式。如果你能刻录并使用 DVD 介质的媒体，这应该能满足你的所有需求。

**disc1：**

包含适用于 CDROM 介质的 FreeBSD 基本操作系统和文档包。不包含其他软件包。

**livefs：**

包含对引导进入基于“livefs”的救援模式的支持，但不支持从 CD 本身进行安装。它旨在帮助救援现有系统，但如有必要也可用于进行基于网络的安装。

**bootonly：**

支持使用 CDROM 驱动器引导机器，但不包含从 CD 本身安装 FreeBSD 的支持。你需要从 CD 引导后进行基于网络的安装（例如从 FTP 服务器）。

**memstick：**

可写入 USB 闪存盘（闪存盘）并用于在能够从 USB 驱动器引导的机器上进行安装。它还支持引导进入基于“livefs”的救援模式。提供文档包，但不提供其他软件包。

作为使用 memstick 镜像的一个示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似以下命令应该可行：

```sh
# dd if=FreeBSD-8.2-RELEASE-amd64-memstick.img of=/dev/da0 bs=10240 conv=sync
```

请务必确保目标（`of=`）正确。

## FreeBSD Update

[freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+8.2-RELEASE) 工具支持对运行早期 FreeBSD 发行版的 i386 和 amd64 系统进行二进制升级。运行 7.[01234]-RELEASE、8.[01]-RELEASE、8.2-BETA1 或 8.2-RC[123] 的系统可按以下方式升级：

```sh
# freebsd-update upgrade -r 8.2-RELEASE
```

在此过程中，FreeBSD Update 可能会要求用户协助合并一些配置文件，或确认自动执行的合并是否正确完成。

```sh
# freebsd-update install
```

在继续之前，必须使用新安装的内核重新启动系统。

```sh
# shutdown -r now
```

重新启动后，需要再次运行 `freebsd-update` 以安装新的用户空间组件：

```sh
# freebsd-update install
```

此时，从 FreeBSD 7.4-RELEASE 或更早版本升级的系统用户将被 `freebsd-update` 提示重新构建所有第三方应用程序（例如从 Ports 树安装的 Port），原因是系统库的更新。

在更新已安装的第三方应用程序之后（并且仅在 `freebsd-update` 打印消息指示需要这样做时），再次运行 `freebsd-update`，以便它可以删除旧的（不再使用的）系统库：

```sh
# freebsd-update install
```

最后，重新启动进入 8.2-RELEASE：

```sh
# shutdown -r now
```
