# FreeBSD 7.2-RELEASE 发行公告

日期：2009 年 5 月 4 日

FreeBSD 发行工程团队很高兴地宣布 FreeBSD 7.2-RELEASE 现已可用。这是 7-STABLE 分支的第三个发行版，改进了 FreeBSD 7.1 的功能并引入了一些新功能。部分亮点：

* 支持对应用内存完全透明地使用 superpages
* 支持每个 Jail 使用多个 IPv4 和 IPv6 地址
* [csup(1)](http://www.freebsd.org/cgi/man.cgi?query=csup&sektion=1&manpath=FreeBSD+7.2-RELEASE) 现在支持 CVSMode 来获取完整的 CVS 仓库
* GNOME 更新至 2.26，KDE 更新至 4.2.2
* sparc64 现在支持 UltraSPARC-III 处理器

关于新功能和已知问题的完整列表，请参阅在线发行说明和勘误列表，地址如下：

* <http://www.FreeBSD.org/releases/7.2R/relnotes.html>
* <http://www.FreeBSD.org/releases/7.2R/errata.html>

关于 FreeBSD 发行工程活动的更多信息，请参阅：

* <http://www.FreeBSD.org/releng/>

## 可用性

FreeBSD 7.2-RELEASE 现已可用于 amd64、i386、ia64、pc98、powerpc 和 sparc64 架构。

FreeBSD 7.2 可以从可引导 ISO 镜像或通过网络安装。所需文件可通过 FTP 或 BitTorrent 下载，如下节所述。虽然一些较小的 FTP 镜像可能不包含所有架构，但它们通常都会包含较常见的架构，如 i386 和 amd64。

发行版 ISO 镜像的 MD5 和 SHA256 校验和附于本消息底部。

发行版中所提供 ISO 镜像的用途如下：

**dvd1：**

包含安装 FreeBSD 基本操作系统所需的全部内容、一组预构建的软件包以及文档。它还支持引导进入基于“livefs”的救援模式。如果你能刻录并使用 DVD 介质的媒体，这应该能满足你的所有需求。

**disc1、disc2、disc3、livefs、docs：**

**disc1** 包含 FreeBSD 基本操作系统和一些预构建的软件包。**disc2** 和 **disc3** 包含更多预构建的软件包。这三张光盘可刻录至 CDROM 介质的媒体，应该能满足常规安装的全部需求。**livefs** 包含对引导进入基于“livefs”的救援模式的支持，但不支持从 CD 本身进行安装。你需要执行基于网络的安装。**docs** 包含文档。

**bootonly：**

支持使用 CDROM 驱动器引导机器，但不包含从 CD 本身安装 FreeBSD 的支持。你需要从 CD 引导后进行基于网络的安装（例如从 FTP 服务器）。

> **注意**
>
> 在测试周期后期发现，部分机器无法将 i386 disc1 识别为可引导介质（它们只是直接跳过，尝试从下一个引导设备引导）。所有受影响的机器都能将其他光盘识别为可引导介质。如果你的机器存在此问题，从 bootonly 或 livefs 引导，然后在 sysinstall 启动后换入 disc1 应该可以工作。

## FreeBSD Update

[freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+7.2-RELEASE) 工具支持对运行早期 FreeBSD 发行版的 i386 和 amd64 系统进行二进制升级。运行 7.0-RELEASE、7.1-RELEASE、7.2-BETA、7.2-RC1 或 7.2-RC2 的系统可按以下方式升级：

```sh
# freebsd-update upgrade -r 7.2-RELEASE
```

在此过程中，FreeBSD Update 可能会要求用户协助合并一些配置文件，或确认自动执行的合并是否正确完成。

```sh
# freebsd-update install
```

在继续之前，必须使用新安装的内核重新启动系统。

```sh
# shutdown -r now
```

重新启动后，需要再次运行 `freebsd-update` 以安装新的用户空间组件，然后再次重新启动系统：

```sh
# freebsd-update install
# shutdown -r now
```

早期 FreeBSD 发行版（FreeBSD 6.x）的用户也可以使用 `freebsd-update` 升级到 FreeBSD 7.2，但在第二次执行“freebsd-update install”之后，需重新构建所有第三方应用程序（例如从 Ports 树安装的任何内容），以处理 FreeBSD 6.x 和 FreeBSD 7.x 之间系统库的差异。

关于使用 FreeBSD Update 从 FreeBSD 6.x 升级的更多信息，请参阅：

* <http://www.daemonology.net/blog/2007-11-11-freebsd-major-version-upgrade.html>
