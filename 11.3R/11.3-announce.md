# FreeBSD 11.3-RELEASE 发行公告

日期：2019 年 7 月 9 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 11.3-RELEASE 现已发布。这是 stable/11 分支的第四个版本。

部分亮点如下：

* `clang`、`llvm`、`lld`、`lldb` 和 `compiler-rt` 工具以及 `libc++` 已更新至上游版本 8.0.0。
* ELF 工具链已更新至版本 r3614。
* OpenSSL 已更新至版本 1.0.2s。
* ZFS 文件系统已更新，实施并行挂载。
* [loader(8)](https://www.freebsd.org/cgi/man.cgi?query=loader&sektion=8&manpath=freebsd-release-ports) 已更新，扩展了对所有架构的 [geli(8)](https://www.freebsd.org/cgi/man.cgi?query=geli&sektion=8&manpath=freebsd-release-ports) 支持。
* [pkg(8)](https://www.freebsd.org/cgi/man.cgi?query=pkg&sektion=8&manpath=freebsd-release-ports) 工具已更新至版本 1.10.5。
* KDE 桌面环境已更新至版本 5.15.3。
* GNOME 桌面环境已更新至版本 3.28。
* 内核在记录进程退出时，现在会一并记录 [jail(8)](https://www.freebsd.org/cgi/man.cgi?query=jail&sektion=8&manpath=freebsd-release-ports) ID。
* 用户空间应用程序的多项功能新增和更新。
* 多个网络驱动程序固件更新。
* 对于未来版本中将弃用的功能，现在会在所有 FreeBSD 版本中打印警告。
* 为 RFC 8221 中弃用的 IPSec 算法添加了警告。
* 在创建 [geli(8)](https://www.freebsd.org/cgi/man.cgi?query=geli&sektion=8&manpath=freebsd-release-ports) 提供者时，添加了对较弱算法的弃用警告。
* 以及更多……

完整的新功能和已知问题列表，请参阅在线发行说明和勘误列表，地址如下：

* <https://www.FreeBSD.org/releases/11.3R/relnotes>
* <https://www.FreeBSD.org/releases/11.3R/errata>

关于 FreeBSD 发布工程活动的更多信息，请参阅：

## 可用性

FreeBSD 11.3-RELEASE 现已支持 amd64、i386、powerpc、powerpc64、sparc64、armv6 和 aarch64 架构。

FreeBSD 11.3-RELEASE 可从可启动 ISO 镜像或通过网络安装。部分架构还支持从 USB 闪存盘安装。所需文件可按以下说明下载。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/11.3R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/11.3R/announce.asc>

发行版所提供镜像的用途如下：

**dvd1**

包含安装 FreeBSD 基本系统、文档和少量预构建软件包（旨在启动图形工作站）所需的全部内容。还支持启动到基于“livefs”的救援模式。若你能刻录并使用 DVD 介质，这应是你所需的全部。

**disc1**

包含 FreeBSD 基本系统。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

**bootonly**

支持使用 CDROM 驱动器启动机器，但不包含用于从 CD 本身安装 FreeBSD 的安装发行版。你需要从 CD 启动后执行基于网络的安装（例如从 FTP 服务器）。

**memstick**

可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-11.3-RELEASE-amd64-memstick.img \
    of=/dev/da0 bs=10240 conv=sync
```

务必确保目标（`of=`）正确。

**mini-memstick**

可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-11.3-RELEASE-amd64-mini-memstick.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**FreeBSD/arm SD 卡镜像**

可写入 SD 卡并用于启动受支持的 arm 系统。SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 512Mb 的 SD 卡上。

为方便没有控制台访问权限的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 [ssh(1)](https://www.freebsd.org/cgi/man.cgi?query=ssh&sektion=1&manpath=freebsd-release-ports) 访问。此外，`root` 用户密码设置为 `root`，强烈建议在获得系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，请使用 [dd(1)](https://www.freebsd.org/cgi/man.cgi?query=dd&sektion=1&manpath=freebsd-release-ports) 工具，将 *KERNEL* 替换为系统对应的内核配置名称。

```sh
# dd if=FreeBSD-11.3-RELEASE-arm-armv6-KERNEL.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。
