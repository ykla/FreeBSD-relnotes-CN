本文档列出了 FreeBSD 4.8-RELEASE 的勘误项，包含发行后才发现或发行周期后期才确定而未能纳入发行文档的重要信息。这些信息包括安全公告，以及与可能影响其运行或可用性的软件或文档相关的消息。在安装此版本 FreeBSD 之前，应始终查阅本文档的最新版本。

FreeBSD 4.8-RELEASE 的勘误文档将维护至 FreeBSD 4.9-RELEASE 发布。

## 1 引言

本勘误文档包含有关 FreeBSD 4.8-RELEASE 的“最新消息”。在安装此版本之前，请务必查阅本文档，以了解发行后才发现的或已经发现并修复的问题。

随发行版实际分发的任何勘误文档（例如 CDROM 发行版中的）按定义已过时，但其他副本会在互联网上保持更新，应作为此版本的“当前勘误”查阅。这些其他勘误副本位于 [http://www.FreeBSD.org/releases/](http://www.freebsd.org/releases/)，以及任何保持此位置最新镜像的站点。

FreeBSD 4-STABLE 的源代码和二进制快照也包含本文档的最新副本（截至快照时的版本）。

有关所有 FreeBSD CERT 安全公告的列表，请参见 [http://www.FreeBSD.org/security/](http://www.freebsd.org/security/) 或 [ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT/](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/)。

## 2 安全公告

旧版本的 **sendmail** 在标头解析中存在缓冲区溢出。它可能允许远程攻击者构造一封特殊邮件，导致 [sendmail(8)](https://man.freebsd.org/cgi/man.cgi?query=sendmail&sektion=8&manpath=FreeBSD+4.8-stable) 以运行它的用户（通常为 `root`）的权限执行任意代码。更多信息（包括补丁指针）可参见安全公告 [FreeBSD-SA-03:07](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:07.sendmail.asc)。此问题在 FreeBSD 4.8-RELEASE 中已通过厂商补丁修复，并在 FreeBSD 4.9-RC 中通过导入新版 **sendmail** 修复。然而，这些变更可能未在发行文档中注明。

[realpath(3)](https://man.freebsd.org/cgi/man.cgi?query=realpath&sektion=3&manpath=FreeBSD+4.8-stable) 函数的实现包含一个单字节缓冲区溢出 bug。这可能产生各种影响，取决于使用 [realpath(3)](https://man.freebsd.org/cgi/man.cgi?query=realpath&sektion=3&manpath=FreeBSD+4.8-stable) 的应用程序及其他因素。此 bug 已在 4.8-RELEASE 安全修复分支和 4-STABLE 开发分支上修复。更多信息请参见安全公告 [FreeBSD-SA-03:08](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:08.realpath.asc)。

内核中存在一个 bug，可能允许其尝试传递无效信号，从而导致内核恐慌。此 bug 已在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多信息请参见安全公告 [FreeBSD-SA-03:09](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:09.signal.asc)。

iBCS2 仿真模块中的一个 bug 可能导致内核内存内容泄露。（注意，此模块在 FreeBSD 中默认未启用。）此 bug 已在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多信息可参见安全公告 [FreeBSD-SA-03:10](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:10.ibcs2.asc)。

**sendmail** 实现其“DNS maps”功能时的编程错误可能导致 [sendmail(8)](https://man.freebsd.org/cgi/man.cgi?query=sendmail&sektion=8&manpath=FreeBSD+4.8-stable) 子进程崩溃或行为异常。此错误已在 4.8-RELEASE 安全修复分支上通过补丁修复，并在 4-STABLE 开发分支上通过导入新版 **sendmail** 修复。（注意，FreeBSD 附带的默认配置文件不使用 DNS maps 功能。）更多信息可参见安全公告 [FreeBSD-SA-03:11](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:11.sendmail.asc)。

**OpenSSH** 在其缓冲区管理代码中包含一个可能导致其崩溃的 bug。此 bug 已通过厂商提供的补丁在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多详细信息请参阅安全公告 [FreeBSD-SA-03:12](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:12.openssh.asc)。

**sendmail** 包含一个可远程利用的缓冲区溢出。此 bug 已通过厂商提供的补丁在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多详细信息可参见安全公告 [FreeBSD-SA-03:13](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:13.sendmail.asc)。

FreeBSD ARP 代码中包含一个 bug，可能允许内核导致资源耗尽，最终导致系统恐慌。此 bug 已在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多信息可参见安全公告 [FreeBSD-SA-03:14](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:14.arp.asc)。

**OpenSSH** PAM 认证代码中的若干 bug 可能产生从认证错误到栈损坏的各种影响。这些已通过厂商提供的补丁修复；详细信息可参见安全公告 [FreeBSD-SA-03:15](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:15.openssh.asc)。

[readv(2)](https://man.freebsd.org/cgi/man.cgi?query=readv&sektion=2&manpath=FreeBSD+4.8-stable) 系统调用的实现包含一个可能导致系统崩溃或权限提升的 bug。此 bug 已在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多信息可参见安全公告 [FreeBSD-SA-03:16](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:16.filedesc.asc)。

[procfs(5)](https://man.freebsd.org/cgi/man.cgi?query=procfs&sektion=5&manpath=FreeBSD+4.8-stable) 和 [linprocfs(5)](https://man.freebsd.org/cgi/man.cgi?query=linprocfs&sektion=5&manpath=FreeBSD+4.8-stable) 的实现包含一个可能导致内核内存内容泄露的 bug。此 bug 已在 4-STABLE 开发分支和 4.8-RELEASE 安全修复分支上修复。更多信息可参见安全公告 [FreeBSD-SA-03:17](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:17.procfs.asc)。

**OpenSSL** 包含若干 bug，可能允许远程攻击者使使用 **OpenSSL** 的应用程序崩溃或以应用程序的权限执行任意代码。这些 bug 已在 4-STABLE 开发分支上通过导入新版 **OpenSSL** 修复，并在 4.8-RELEASE 安全修复分支上通过厂商提供的补丁修复。注意，只有使用 **OpenSSL** 的 ASN.1 或 X.509 处理代码的应用程序受影响（例如 **OpenSSH** 不受影响）。更多信息可参见安全公告 [FreeBSD-SA-03:18](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:18.openssl.asc)。

## 3 最新消息

由于在发布周期后期才发现的一些问题，FreeBSD 4.8-RELEASE/i386 的 ISO 镜像和 FTP 安装目录需要重新生成并重新上传至 FTP 镜像站点。供参考，最终 ISO 镜像通过 [md5(1)](https://man.freebsd.org/cgi/man.cgi?query=md5&sektion=1&manpath=FreeBSD+4.8-stable) 计算的校验和如下：

```sh
MD5 (4.8-RELEASE-i386-disc1.iso) = c4e34b6a6be5cd1977ca206bf821c7fc
MD5 (4.8-RELEASE-i386-disc2.iso) = 93b09f97c01deead302557d7d24f87cb
MD5 (4.8-RELEASE-i386-mini.iso) = 5f0d2576dbb56d6ec85d49ac9fa4bbf9
```

文档的某些部分可能错误地将 FreeBSD 4.8-RELEASE 的发布日期列为 2003 年 3 月，而非 2003 年 4 月。

FreeBSD 4.8-RELEASE 恢复了从安装介质安装到 [mly(4)](https://man.freebsd.org/cgi/man.cgi?query=mly&sektion=4&manpath=FreeBSD+4.8-stable) 设备的能力。（此功能在 FreeBSD 4.7-RELEASE 中已损坏。）

安装 **GNOME** 后，默认终端字体可能显示乱码。如果是这种情况，请安装 `x11-fonts/bitstream-vera` port，然后重启 **GNOME**。新字体应自动生效。如果未生效，请编辑当前 gnome-terminal 配置文件并选择 Bitstream Vera Sans Mono 字体。

由于空间限制，[awi(4)](https://man.freebsd.org/cgi/man.cgi?query=awi&sektion=4&manpath=FreeBSD+4.8-stable) 驱动程序已从 1.44MB `kern.flp` i386 启动软盘所使用的内核中移除。由于 FreeBSD 4.8-RELEASE 中没有为此驱动程序提供模块，这意味着通常无法通过 [awi(4)](https://man.freebsd.org/cgi/man.cgi?query=awi&sektion=4&manpath=FreeBSD+4.8-stable) 网络安装 FreeBSD 4.8-RELEASE。

由于空间限制，ATAPI 软盘支持以及 DEC AlphaServer 8200 和 8400（“TurboLaser”）机器的支持已从 1.44MB `kern.flp` alpha 启动软盘所使用的内核中移除。

FreeBSD 4.8-RELEASE 内核中的一个 bug 阻止其在 Intel 80386 处理器上启动。此问题已在 4.8-RELEASE 安全修复分支和 4-STABLE 开发分支上修复。

FreeBSD 支持登录能力数据库的哈希形式，存储在 `/etc/login.conf.db` 中。这是从 `/etc/login.conf` 文本文件生成的。如果存在哈希数据库，[login(1)](https://man.freebsd.org/cgi/man.cgi?query=login&sektion=1&manpath=FreeBSD+4.8-stable) 将优先使用它而非文本文件的内容。FreeBSD 4.8-RELEASE 是首个在发行介质上实际包含 `/etc/login.conf.db` 的发行版；因此，修改 `/etc/login.conf` 的用户需要记住使用 [cap_mkdb(1)](https://man.freebsd.org/cgi/man.cgi?query=cap_mkdb&sektion=1&manpath=FreeBSD+4.8-stable) 重新生成数据库。执行源代码升级的用户通常不受此变更影响，因为 [mergemaster(8)](https://man.freebsd.org/cgi/man.cgi?query=mergemaster&sektion=8&manpath=FreeBSD+4.8-stable) 提供了在升级期间重新生成 `/etc/login.conf.db` 的选项。[login.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=login.conf&sektion=5&manpath=FreeBSD+4.8-stable) 提供了有关登录能力数据库格式和用法的更多详细信息。

当使用 [sysinstall(8)](https://man.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+4.8-stable) 安装 Ports 集合时，`multimedia/gstreamer-plugins` port 的一部分文件可能显示文件名已损坏。这不应影响构建 port 或安装相应的软件包。但是，建议重命名该文件，以防止已安装 Ports 集合的未来更新出现问题：

```sh
# cd /usr/ports/multimedia/gstreamer-plugins/files
# mv patch-gst-libs_ext_ffmpeg_ffmpeg_libavcodec_alpha_simple_i \
patch-gst-libs_ext_ffmpeg_ffmpeg_libavcodec_alpha_simple_idct_alpha.c
```

最近，邮件列表从 majordomo 更改为当前使用的 Mailman 列表服务器。有关使用新邮件列表的更多信息，请访问 [FreeBSD Mailman 信息页面](http://www.freebsd.org/mailman/listinfo/)。

[dc(4)](https://man.freebsd.org/cgi/man.cgi?query=dc&sektion=4&manpath=FreeBSD+4.8-stable) 驱动程序无法通过 Davicom DC9102 卡正确传输数据。此问题自 FreeBSD 4.5-RELEASE 起存在，已在 FreeBSD 4.9-RC 中修复。
