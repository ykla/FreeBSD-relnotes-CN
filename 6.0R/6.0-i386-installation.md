## 摘要

本文简要说明如何安装 FreeBSD/i386 6.0-RELEASE，并着重介绍获取 FreeBSD 发行版的方法。同时提供了一些故障排除和常见问题解答。

## 1 安装 FreeBSD

本节介绍安装全新 FreeBSD 发行版的过程。这些说明着重于获取 FreeBSD 6.0-RELEASE 发行版以及开始安装过程。FreeBSD 手册中的 [“安装 FreeBSD”](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/install.html) 章节提供了关于安装程序本身的更深入信息，包括带截图的引导式操作演示。

如果你是从早期版本的 FreeBSD 升级，请参阅 [第 3 节](#3-升级-freebsd) 获取升级说明。

### 1.1 准备工作

最重要的预安装步骤可能是阅读 FreeBSD 提供的各种说明文档。关于本次 FreeBSD 发行版的文档路线图可在 **README.TXT** 中找到，该文件通常与本文档位于同一位置；大多数文档（如发行说明和硬件兼容列表）也可在安装程序的“文档”菜单中访问。

如果你有互联网连接，还可以从 [FreeBSD 项目网站](http://www.FreeBSD.org/) 获取 FreeBSD [FAQ](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/faq/) 和 [手册](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/) 的在线版本。

这套文档看起来可能令人生畏，但花时间阅读它们将节省你大量时间。熟悉可用的资源在安装过程中遇到问题时也会有所帮助。

即使最周密的计划有时也会出错，因此如果遇到问题，请参阅 [第 4 节](#4-故障排除)，其中包含有价值的故障排除信息。你还应该在安装前阅读最新的 **ERRATA.TXT** 副本，因为它会提醒你注意特定发行版在发布后报告的任何问题。

>**重要**
>
> 尽管 FreeBSD 尽力防止数据意外丢失，但如果操作失误，仍然极有可能 *清除整个磁盘*。在进入最终的 FreeBSD 安装菜单之前，请确保已充分备份所有重要数据。

### 1.2 硬件要求

FreeBSD/i386 需要 486 或更好的处理器以及至少 24 MB 的内存。最小安装至少需要 150 MB 的可用硬盘空间。参见下文了解缩小现有 DOS 分区以安装 FreeBSD 的方法。

如果你不熟悉为 FreeBSD 配置硬件，务必阅读 **HARDWARE.TXT** 文件；其中包含 FreeBSD 支持哪些硬件的重要信息。

### 1.3 软盘镜像说明

根据你选择的 FreeBSD 安装方式，可能需要创建一组软盘（通常是三张）来启动安装过程。本节简要说明如何从 CDROM 安装或从互联网创建这些软盘。注意，在从 CDROM 启动安装的常见情况下，如果你的机器支持可启动 CDROM，本节所述步骤不需要，可以跳过。

对于大多数 CDROM 或网络安装，你需要从 **floppies/** 目录复制到实际软盘上的只有 **boot.flp** 和 **kernX.flp** 镜像（用于 1.44MB 软盘）。

通过网络获取这些镜像很简单。只需从 [ftp://ftp.FreeBSD.org/](ftp://ftp.freebsd.org/) 或手册的 [FTP 站点](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/mirrors-ftp.html) 部分列出的众多镜像之一，或 [http://mirrorlist.FreeBSD.org/](http://mirrorlist.FreeBSD.org/) 网页上获取 `release`/floppies/boot.flp 和所有 `release`/floppies/kernX.flp 文件。

准备几张空白、刚格式化的软盘，将 **boot.flp** 镜像复制到一张软盘，将 **kernX.flp** 文件复制到其他软盘。这些镜像 *不是* DOS 文件。你不能简单地将它们作为常规文件复制到 DOS 或 UFS 软盘上，需要使用 DOS 下的 `fdimage.exe`（参见 CDROM 上的 **tools** 目录或 FreeBSD FTP 镜像）或 UNIX 中的 [dd(1)](http://www.freebsd.org/cgi/man.cgi?query=dd&sektion=1&manpath=FreeBSD+6.0-RELEASE) 命令进行“镜像”复制。

例如，从 DOS 创建内核软盘镜像，可以执行类似以下操作：

```sh
C> fdimage boot.flp a:
```

假设你已将 `fdimage.exe` 和 `boot.flp` 复制到某个目录。当然，对 **kernX.flp** 文件也是如此。

如果从 UNIX 机器创建启动软盘，可能会发现以下命令有效：

```sh
# dd if=floppies/boot.flp of=/dev/rfd0
```

或：

```sh
# dd if=floppies/boot.flp of=/dev/fd0
```

或：

```sh
# dd if=floppies/boot.flp of=/dev/floppy
```

具体取决于你的硬件和操作系统环境（不同版本的 UNIX 对软盘驱动器有不同的命名）。

### 1.4 从 CDROM 或互联网安装 FreeBSD

最简单的安装方式是从 CDROM 安装。如果你有受支持的 CDROM 驱动器和 FreeBSD 安装 CDROM，有两种方式可以从它启动安装：

- 如果你的系统支持可启动 CDROM 媒体的“CDBOOT”标准，并且已启用从 CD 启动，只需将 FreeBSD 安装 CD 放入 CDROM 驱动器并启动系统即可开始安装。
- 从每个 FreeBSD 发行版的 **floppies/** 目录构建一组 FreeBSD 启动软盘。阅读 [第 1.3 节](#13-软盘镜像说明) 了解在不同操作系统下创建可启动软盘的更多信息。然后从第一张软盘启动，很快就会进入 FreeBSD 安装程序。

如果你没有 CDROM（或你的电脑不支持从 CDROM 启动），并希望使用 PPP、SLIP 或专用连接通过网络安装，应使用 [第 1.3 节](#13-软盘镜像说明) 中的说明，从 **floppies/boot.flp** 和 **floppies/kernX.flp** 文件构建一组 FreeBSD 启动软盘来开始安装。使用 **boot.flp** 软盘重启计算机；在提示时根据需要插入其他软盘。然后请参阅 [第 1.5.5 节](#155-通过网络使用-ftp-或-nfs-安装) 获取通过 FTP 或 NFS 安装的额外提示。

### 1.5 各种安装类型的详细说明

一旦你以某种方式进入初始安装屏幕，应该能够按照各种菜单提示进行操作。如果你以前从未使用过 FreeBSD 安装程序，也建议阅读“文档”子菜单中的一些文档以及第一个菜单上的通用“Usage”说明。

>**注意**
>
> 如果在某个屏幕卡住，按 **F1** 键可获取该特定章节的在线文档。

如果你从未安装过 FreeBSD，建议使用“Standard”安装模式，因为它确保你会经过所有重要的检查清单项。如果你对 FreeBSD 安装过程非常熟悉并且 *确切* 知道要做什么，可使用“Express”或“Custom”安装选项。如果你要升级现有系统，使用“Upgrade”选项。

FreeBSD 安装程序支持直接使用软盘、DOS、磁带、CDROM、FTP、NFS 和 UFS 分区作为安装媒体；以下列出每种媒体类型的安装提示。

#### 1.5.1 从网络 CDROM 安装

如果你只想从本地 CDROM 驱动器安装，请参阅 [第 1.4 节](#14-从-cdrom-或互联网安装-freebsd)。如果你的系统没有 CDROM 驱动器，想使用另一台你有网络连接的机器的 CDROM 驱动器中的 FreeBSD 发行版 CD，有几种方式：

- 如果你可以直接从某台 FreeBSD 机器的 CDROM 驱动器通过 FTP 安装 FreeBSD，操作很简单：确保 FTP 服务器正在运行，然后使用 [vipw(8)](http://www.freebsd.org/cgi/man.cgi?query=vipw&sektion=8&manpath=FreeBSD+6.0-RELEASE) 命令在密码文件中添加以下行：

```sh
ftp:*:99:99::0:0:FTP:/cdrom:/sbin/nologin
```

在运行安装的机器上，进入 Options 菜单并将 Release Name 设置为 `any`。然后选择 `FTP` 作为 Media 类型，并在 ftp 站点菜单中选择“URL”后输入 `ftp://machine`。

>**警告**
>
> 这可能允许本地网络（或互联网）上的任何人对此机器进行“匿名 FTP”连接，这可能不是所希望的。

- 如果你更愿意使用 NFS 直接将 CDROM 导出给你要安装的机器，需要先在 **/etc/exports** 文件中（在有 CDROM 驱动器的机器上）添加条目。以下示例允许 `ziggy.foo.com` 在安装期间通过 NFS 直接挂载 CDROM：

```sh
/cdrom          -ro             ziggy.foo.com
```

有 CDROM 的机器还必须配置为 NFS 服务器。如果你不确定如何操作，那么 NFS 安装可能不是最佳选择，除非你愿意阅读 [rc.conf(5)](http://www.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&manpath=FreeBSD+6.0-RELEASE) 并进行适当配置。假设这部分顺利完成，在目标机器安装时应该能够输入 `cdrom-host:/cdrom` 作为 NFS 安装路径，例如 `wiggy:/cdrom`。

#### 1.5.2 从软盘安装

如果必须从软盘安装（无论是由于硬件不受支持还是因为你喜欢用困难的方式做事），必须先为安装准备一些软盘。

首先，按照 [第 1.3 节](#13-软盘镜像说明) 的说明制作启动软盘。

其次，仔细阅读 [第 2 节](#2-发行版格式) 并特别注意“Distribution Format”部分，因为它描述了哪些文件需要放到软盘上，哪些可以安全跳过。

接下来，你至少需要足够多的 1.44MB 软盘来容纳 **bin**（二进制发行版）目录中的所有文件。如果在 DOS 下准备这些软盘，这些软盘 *必须* 使用 MS-DOS `FORMAT` 命令格式化。如果使用 Windows，使用 Windows 文件管理器的格式化命令。

>**重要**
>
> 软盘通常以“出厂预格式化”形式销售。虽然方便，但过去用户报告的许多问题都源于使用不当格式化的媒体。请自行重新格式化以确保无误。

如果从另一台 FreeBSD 机器创建软盘，格式化仍然是个好主意，尽管不需要在每个软盘上放置 DOS 文件系统。可以使用 [disklabel(8)](http://www.freebsd.org/cgi/man.cgi?query=disklabel&sektion=8&manpath=FreeBSD+6.0-RELEASE) 和 [newfs(8)](http://www.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&manpath=FreeBSD+6.0-RELEASE) 命令在软盘上放置 UFS 文件系统，如下列命令序列所示：

```sh
# fdformat -f 1440 fd0
# disklabel -w fd0 floppy3
# newfs -i 65536 /dev/fd0
```

为 DOS 或 UFS 格式化软盘后，需要将文件复制到它们上面。发行版文件的大小使得一张软盘可容纳一个文件。每个发行版应放在软盘上各自的子目录中，例如：**a:\bin\bin.inf**、**a:\bin\bin.aa**、**a:\bin\bin.ab**……

>**重要**
>
> **bin.inf** 文件也需要放在 **bin** 集的第一张软盘上，因为安装程序会读取它以确定在获取和拼接发行版时还要查找多少个附加部分。将发行版放到软盘上时，**distname.inf** 文件 *必须* 占据每个发行版集的第一张软盘。

到达安装的 Media 屏幕时，选择“Floppy”，系统会提示你完成其余步骤。

#### 1.5.3 从 DOS 分区安装

要从 MS-DOS 分区准备安装，只需将发行版文件复制到主 DOS 分区（**C:**）上名为 **FREEBSD** 的目录中。例如，要使用从 CDROM 复制的文件从 DOS 进行 FreeBSD 的最小安装，可以执行以下操作：

```sh
C:\> MD C:\FREEBSD
C:\> XCOPY /S E:\BASE C:\FREEBSD\BASE
```

假设 **E:** 是你的 CD 挂载位置。

对于你想从 DOS 安装的每个发行版（并且有可用空闲空间），将每个发行版安装在 **C:\FREEBSD** 下的目录中——**BIN** 发行版只是最低要求。

复制目录后，可以像通常一样从软盘启动安装，并在适当时候选择“DOS”作为媒体类型。

#### 1.5.4 从 QIC/SCSI 磁带安装

从磁带安装时，安装程序期望文件简单地以 tar 格式存储在磁带上。因此，在获取你感兴趣的发行版的所有文件后，使用 [tar(1)](http://www.freebsd.org/cgi/man.cgi?query=tar&sektion=1&manpath=FreeBSD+6.0-RELEASE) 将它们写入磁带，命令类似于：

```sh
# cd /where/you/have/your/dists
# tar cvf /dev/sa0 dist1 .. dist2
```

进行安装时，还应确保在某个临时目录（你可以选择）中留出足够空间以容纳所创建磁带的 *全部* 内容。由于磁带的非随机访问特性，这种安装方法需要相当多的临时存储空间。你应该预期需要的临时存储空间与你写入磁带的内容量相当。

>**注意**
>
> 进行安装时，必须在从启动软盘启动 *之前* 将磁带放入驱动器。否则安装“探测”可能无法找到它。

现在按照 [第 1.3 节](#13-软盘镜像说明) 的说明创建启动软盘，然后继续安装。

#### 1.5.5 通过网络使用 FTP 或 NFS 安装

按照第一节所述制作启动软盘后，可以使用三种连接类型之一通过网络加载其余安装内容：串口、并口或以太网。

##### 1.5.5.1 串口

SLIP 支持相当原始，主要限于硬连线链路，例如两台计算机之间的串行电缆。该链路必须是硬连线的，因为 SLIP 安装目前不提供拨号功能。如果需要使用调制解调器拨号或以其他方式与链路对话后再连接，应使用 PPP 工具。

如果使用 PPP，确保手头有互联网服务提供商的 IP 地址和 DNS 信息，因为你在安装过程的早期就需要知道这些信息。你可能还需要知道自己的 IP 地址，不过 PPP 支持动态地址协商，如果你的 ISP 支持，可能可以直接从 ISP 获取此信息。

你还需要了解如何使用特定品牌调制解调器的各种“AT 命令”进行拨号，因为 PPP 拨号器仅提供非常简单的终端仿真程序。

##### 1.5.5.2 并口

如果有到另一台 FreeBSD 或 Linux 机器的硬连线连接，还可以考虑通过“laplink”式并口电缆安装。并口的数据传输率远高于串行线路通常可能的速率（最高 50k/sec），因此安装更快。以这种方式使用点对点并口电缆时，通常不需要使用“真实”IP 地址，一般可以只使用 RFC 1918 风格的地址作为链路两端（例如 **10.0.0.1**、**10.0.0.2** 等）。

>**重要**
>
> 如果使用 Linux 机器而非 FreeBSD 机器作为 PLIP 对端，还需在 TCP/IP 设置屏幕的“ifconfig 附加选项”字段中指定 `link0`，以兼容 Linux 略有差异的 PLIP 协议。

##### 1.5.5.3 以太网

FreeBSD 支持大多数常见的以太网卡；支持的网卡列表作为 FreeBSD 硬件说明的一部分提供（参见启动软盘上“文档”菜单或 CDROM 顶级目录中的 **HARDWARE.TXT**）。如果你使用的是受支持的 PCMCIA 以太网卡之一，还要确保在笔记本电脑开机 *之前* 插入网卡。遗憾的是，FreeBSD 目前不支持在安装期间“热插拔”PCMCIA 卡。

你还需要知道你在网络上的 IP 地址、子网的 `netmask` 值以及你的机器名称。系统管理员可以告诉你哪些值适合你的特定网络设置。如果你想通过名称而非 IP 地址引用其他主机，还需要名称服务器和可能的网关地址（如果使用 PPP，则是你的提供商的 IP 地址）来进行通信。如果你想通过 HTTP 代理进行 FTP 安装（见下文），还需要代理的地址。

如果你不知道这些问题的答案，那么在尝试此类安装之前应该先与系统管理员讨论。在活动网络上使用随机选择的 IP 地址或子网掩码几乎肯定会失败，并且可能会受到系统管理员的训斥。

一旦某种网络连接正常工作，安装就可以通过 NFS 或 FTP 继续。

##### 1.5.5.4 NFS 安装提示

NFS 安装相当直接：只需将你想要的 FreeBSD 发行版文件复制到某个服务器上，然后将 NFS 媒体选择指向它。

如果此服务器仅支持“特权端口”访问（这通常是 Sun 和 Linux 工作站的默认设置），你可能需要在 Options 菜单中设置此选项才能继续安装。

如果你的以太网卡质量较差，传输速率很慢，可能还需要切换相应的 Options 标志。

为使 NFS 安装正常工作，服务器还必须支持“子目录挂载”，例如如果你的 FreeBSD 发行版目录位于 **wiggy:/usr/archive/stuff/FreeBSD**，那么 `wiggy` 必须允许直接挂载 **/usr/archive/stuff/FreeBSD**，而不仅仅是 **/usr** 或 **/usr/archive/stuff**。

在 FreeBSD 的 **/etc/exports** 文件中，这由 `-alldirs` 选项控制。其他 NFS 服务器可能有不同的约定。如果你从服务器收到 `Permission Denied` 消息，很可能是你没有正确启用此功能。

##### 1.5.5.5 FTP 安装提示

可以从任何包含相当新的 FreeBSD 版本的镜像站点进行 FTP 安装。安装期间的 FTP 站点菜单提供了几乎适用于世界上任何位置的合理选择列表。

如果你要从此菜单中未列出的其他 FTP 站点安装，或者在配置名称服务器时遇到问题，还可以通过在该菜单中选择“URL”选项来指定自己的 URL。URL 可以包含主机名或 IP 地址，因此在没有名称服务器的情况下，类似以下的内容也可以工作：

```sh
ftp://216.66.64.162/pub/FreeBSD/releases/i386/4.2-RELEASE
```

有三种 FTP 安装模式可供使用：

- **FTP**：此方法使用标准的“Active”模式进行传输，服务器向客户端发起连接。这在大多数防火墙下无法工作，但通常在不支持被动模式的旧 FTP 服务器上效果最佳。如果被动模式连接挂起，尝试此模式。
- **FTP Passive**：此模式设置 FTP“Passive”模式，阻止服务器向客户端打开连接。此选项最适合需要通过不允许在随机端口地址上进行传入连接的防火墙的用户。
- **通过 HTTP 代理的 FTP**：此选项指示 FreeBSD 使用 HTTP 连接到代理进行所有 FTP 操作。代理将转换请求并发送到 FTP 服务器。使用户能够穿越不允许 FTP 但提供 HTTP 代理的防火墙。你必须在 FTP 服务器之外指定代理的主机名。

在极少数情况下，如果你的 FTP 代理不通过 HTTP，可以将 URL 指定为类似以下形式：

```sh
ftp://foo.bar.com:port/pub/FreeBSD
```

在上面的 URL 中，`port` 是代理 FTP 服务器的端口号。

#### 1.5.6 串口控制台用户提示

如果你想在仅使用串口的机器上安装 FreeBSD（例如你没有或不想使用 VGA 卡），请按照以下步骤操作：

1. 将某种 ANSI（vt100）兼容终端或终端仿真程序连接到你要安装 FreeBSD 的 PC 的 **COM1** 端口。
2. 拔下键盘（是的，没错！），然后尝试从软盘或安装 CDROM 启动，具体取决于你拥有的安装媒体类型，键盘保持拔下状态。
3. 如果串口控制台上没有任何输出，请重新插入键盘。如果从 CDROM 启动，听到蜂鸣声后即进入第 5 步。
4. 如果从软盘启动，当磁盘访问停止时，插入第一张 **kernX.flp** 软盘并按 **Enter**。当此磁盘访问完成时，插入下一张 **kernX.flp** 软盘并按 **Enter**，重复直到所有 **kernX.flp** 软盘都插入完毕。当磁盘活动完成时，重新插入 **boot.flp** 软盘并按 **Enter**。
5. 听到蜂鸣声后，按数字键 **6**，然后输入：

```sh
boot -h
```

你现在应该可以在串口上看到所有内容。如果仍然不起作用，请检查串口电缆以及终端仿真程序或实际终端设备的设置。它应设置为 9600 波特、8 位、无奇偶校验。

### 1.6 i386 架构用户问答

#### 1.6.1 救命！我没有空间！需要先删除所有内容吗？

如果你的机器已经在运行 DOS 且几乎没有可用空间用于 FreeBSD 安装，并非毫无希望！你可能会发现 FreeBSD CDROM 上 **tools/** 子目录或各种 FreeBSD FTP 站点提供的 **FIPS** 工具非常有用。

**FIPS** 允许你将现有的 DOS 分区拆分为两部分，保留原始分区并允许你在第二部分空闲空间上安装。首先使用 DOS 6.xx `DEFRAG` 工具或 Norton Disk Tools 对 DOS 分区进行“碎片整理”，然后运行 FIPS。它会提示你输入所需的其余信息。之后，你可以重启并在新分区上安装 FreeBSD。还要注意，FIPS 会创建第一个分区的“克隆”，因此你会看到原来一个 DOS 主分区现在变成两个。不要惊慌！你可以简单地删除多余的 DOS 主分区（通过检查其大小确保是正确的那个）。

**FIPS** 目前不支持 NTFS 风格的分区。要拆分此类分区，需要商用产品如 Partition Magic。抱歉，如果你的 Windows 分区占用整个磁盘且不想从头重新安装，只能如此。

**FIPS** 不支持扩展 DOS 分区。Windows 95/98/ME FAT32 主分区受支持。

#### 1.6.2 我可以从 FreeBSD 使用压缩的 DOS 文件系统吗？

不可以。如果你使用 Stacker 或 DoubleSpace 等工具，FreeBSD 只能使用你保留未压缩的文件系统部分。文件系统的其余部分将显示为一个大文件（stacked/dblspaced 文件！）。*不要删除该文件*，否则你可能会非常后悔！

最好创建另一个未压缩的 DOS 扩展分区，并用于 DOS 和 FreeBSD 之间的通信（如果你需要的话）。

#### 1.6.3 我可以挂载我的 DOS 扩展分区吗？

可以。DOS 扩展分区映射到 FreeBSD 中其他“slice”的末尾，例如你的 **D:** 驱动器可能是 **/dev/da0s5**，**E:** 驱动器是 **/dev/da0s6**，依此类推。此示例假设你的扩展分区位于 SCSI 驱动器 0。对于 IDE 驱动器，将 `ad` 替换为 `da`。挂载扩展分区与挂载任何其他 DOS 驱动器完全相同，例如：

```sh
# mount -t msdos /dev/da0s5 /dos_d
```

#### 1.6.4 我可以在 FreeBSD 下运行 DOS 二进制文件吗？

BSDI 的 [doscmd(1)](http://www.freebsd.org/cgi/man.cgi?query=doscmd&sektion=1&manpath=FreeBSD+6.0-RELEASE) 工具的持续开发工作在许多情况下足够使用，尽管仍有一些粗糙之处。[emulators/doscmd](http://www.freebsd.org/cgi/url.cgi?ports/emulators/doscmd/pkg-descr) port/软件包可在 FreeBSD Ports 中找到。如果你对此感兴趣，请发送邮件至 [FreeBSD-emulation 邮件列表](http://lists.FreeBSD.org/mailman/listinfo/freebsd-emulation) 并表明你有兴趣加入此项持续进行的工作！

FreeBSD Ports 中的 [emulators/pcemu](http://www.freebsd.org/cgi/url.cgi?ports/emulators/pcemu/pkg-descr) port/软件包模拟 8088 和足够的 BIOS 服务来运行 DOS 文本模式应用程序。它需要 X Window System（XFree86）才能运行。

#### 1.6.5 我可以在 FreeBSD 下运行 Microsoft Windows 应用程序吗？

FreeBSD Ports 中有多个 port/软件包可以启用许多 Windows 应用程序。[emulators/wine](http://www.freebsd.org/cgi/url.cgi?ports/emulators/wine/pkg-descr) port/软件包在 FreeBSD 之上提供兼容层，允许许多 Windows 应用程序在 X Windows（XFree86）中运行。

#### 1.6.6 我可以在 FreeBSD 下运行其他操作系统吗？

FreeBSD Ports 中有多个 port/软件包模拟“虚拟机”并允许其他操作系统在 FreeBSD 之上运行。[emulators/bochs](http://www.freebsd.org/cgi/url.cgi?ports/emulators/bochs/pkg-descr) port/软件包允许 Microsoft Windows、Linux 甚至其他 FreeBSD 副本在 FreeBSD 桌面上的窗口中运行。[emulators/vmware2](http://www.freebsd.org/cgi/url.cgi?ports/emulators/vmware2/pkg-descr) 和 [emulators/vmware3](http://www.freebsd.org/cgi/url.cgi?ports/emulators/vmware3/pkg-descr) port/软件包允许在 FreeBSD 上运行商用 VMware 虚拟机软件。

## 2 发行版格式

FreeBSD 发行版以目录结构组织。典型的 FreeBSD 发行版目录如下所示（具体细节可能因版本、架构和其他因素而异）：

```sh
ERRATA.HTM      README.TXT      compat1x        dict            kernel
ERRATA.TXT      RELNOTES.HTM    compat20        doc             manpages
HARDWARE.HTM    RELNOTES.TXT    compat21        docbook.css     packages
HARDWARE.TXT    base            compat22        filename.txt    ports
INSTALL.HTM     boot            compat3x        floppies        proflibs
INSTALL.TXT     catpages        compat4x        games           src
README.HTM      cdrom.inf       crypto          info            tools
```

这些相同文件也包含在多盘套装的第一张 CDROM 中，但在磁盘上的布局略有不同。在大多数架构上，安装 CDROM 除发行版文件外还包含“live filesystem”。live filesystem 在修复或故障排除现有 FreeBSD 安装时很有用（参见 [第 4 节](#4-故障排除)）。

**floppies** 目录对无法从 CDROM 媒体启动（但能通过其他方式读取 CDROM）的用户特别有用。从 **floppies** 目录生成一组 1.44MB 启动软盘很容易（参见 [第 1.3 节](#13-软盘镜像说明) 了解操作说明），并使用这些软盘从 CDROM、FTP 或 NFS 启动安装。安装期间所需的其余数据将根据你的选择自动获取。

以下是对其中一些项目的更详细说明：

- **\*.TXT** 和 **\*.HTM** 文件包含文档（例如，本文档同时包含在 **INSTALL.TXT** 和 **INSTALL.HTM** 中），应在开始安装前阅读。**\*.TXT** 文件是纯文本，而 **\*.HTM** 文件是可由几乎所有 Web 浏览器读取的 HTML 文件。有些发行版可能还包含 PDF 或 PostScript 等其他格式的文档。
- **docbook.css** 是层叠样式表（CSS）文件，某些 Web 浏览器用它来格式化 HTML 文档。
- **base**、**catpages**、**crypto**、**dict**、**doc**、**games**、**info**、**manpages**、**proflibs** 和 **src** 目录包含 FreeBSD 本身的主要发行版组件，并拆分为较小的文件以便装到软盘上（如有必要）。
- **compat1x**、**compat20**、**compat21**、**compat22**、**compat3x** 和 **compat4x** 目录包含与旧版本兼容的发行版，作为单个 gzip 压缩的 tar 文件分发——可在发行时或稍后通过运行其 **install.sh** 脚本安装。
- **floppies/** 子目录包含软盘安装镜像；有关使用它们的更多信息可在 [第 1.3 节](#13-软盘镜像说明) 中找到。
- **packages** 和 **ports** 目录包含 FreeBSD 软件包和 Ports。可通过运行以下命令从 packages 目录安装软件包：

```sh
# /stand/sysinstall configPackages
```

软件包也可以通过将 **packages**/ 中的各个文件名传递给 [pkg_add(1)](http://www.freebsd.org/cgi/man.cgi?query=pkg_add&sektion=1&manpath=FreeBSD+6.0-RELEASE) 命令来安装。

Ports 可以像任何其他发行版一样安装，解压后约需 190MB 空间。有关 Ports 的更多信息可从 [http://www.FreeBSD.org/ports/](http://www.FreeBSD.org/ports/) 获取，或在安装了 **doc** 发行版后从本地 **/usr/share/doc/handbook** 获取。

- **tools** 目录包含各种 DOS 工具，用于发现磁盘几何参数、安装启动管理器等。它是纯可选的，仅为用户方便而提供。

典型的发行版目录（例如 **info** 发行版）内部如下所示：

```sh
CHECKSUM.MD5    info.ab         info.ad         info.inf        install.sh
info.aa         info.ac         info.ae         info.mtree
```

**CHECKSUM.MD5** 文件包含每个文件的 MD5 签名，以备怀疑数据损坏时使用，仅用于参考。实际安装不使用它，也不需要与其余发行版文件一起复制。**info.a\*** 文件是拆分的 gzip 压缩 tar 文件，其内容可通过以下命令查看：

```sh
# cat info.a* | tar tvzf -
```

安装期间，它们会由安装程序自动拼接和解压。

**info.inf** 文件也是必需的，因为安装程序会读取它以确定在获取和拼接发行版时还要查找多少个分片。将发行版放到软盘上时，**.inf** 文件 *必须* 占据每个发行版集的第一张软盘！

**info.mtree** 文件是另一个非必需文件，仅供用户参考。它包含 *解压后* 发行版文件的 MD5 签名，可稍后与 [mtree(8)](http://www.freebsd.org/cgi/man.cgi?query=mtree&sektion=8&manpath=FreeBSD+6.0-RELEASE) 程序一起使用，以验证安装权限和校验和是否被修改。与 **base** 发行版一起使用时，这是检测系统特洛伊木马攻击的绝佳方法。

最后，**install.sh** 文件供希望在安装后安装发行版的用户使用。例如，要在系统安装后从 CDROM 安装 info 发行版，可以执行：

```sh
# cd /cdrom/info
# sh install.sh
```

## 3 升级 FreeBSD

这些说明描述从旧版本 FreeBSD 进行二进制升级的过程。

>**警告**
>
> 尽管 FreeBSD 升级过程尽力防止数据意外丢失，但仍然极有可能通过此安装 *清除整个磁盘*！在未充分备份所有重要数据文件之前，请不要接受最终确认请求。

>**重要**
>
> 这些说明假设你使用的是与你要升级到的 FreeBSD 版本一起提供的 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 版本。使用不匹配的 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 版本几乎肯定会出问题，并且已知会使系统处于不可用状态。最常见的此类错误是使用现有安装中的旧 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 副本升级到较新版本的 FreeBSD。这 *不* 推荐。

>**警告**
>
> 目前不支持从 FreeBSD 4-STABLE 二进制升级到 FreeBSD 6.0-RELEASE。FreeBSD 4-STABLE 中存在某些文件可能会造成干扰，但二进制升级不会删除它们。一个值得注意的例子是旧的 **/usr/include/g++** 目录会导致 C++ 程序编译错误（或根本无法编译）。

### 3.1 概述

升级过程将用户选择的发行版替换为与新版 FreeBSD 对应的发行版。它保留标准系统配置数据，以及用户数据、已安装的软件包和其他软件。

考虑升级的管理员应在开始升级之前完整阅读本节。否则可能导致升级失败或数据丢失。

#### 3.1.1 升级概述

发行版的升级是通过在先前版本之上解压新版本组件来执行的。属于旧发行版的文件不会被删除。

系统配置通过保留并恢复以下文件的先前版本来进行保护：**Xaccel.ini**、**XF86Config**、**adduser.conf**、**aliases**、**aliases.db**、**amd.map**、**crontab**、**csh.cshrc**、**csh.login**、**csh.logout**、**cvsupfile**、**dhclient.conf**、**disktab**、**dm.conf**、**dumpdates**、**exports**、**fbtab**、**fstab**、**ftpusers**、**gettytab**、**gnats**、**group**、**hosts**、**hosts.allow**、**hosts.equiv**、**hosts.lpd**、**inetd.conf**、**localtime**、**login.access**、**login.conf**、**mail**、**mail.rc**、**make.conf**、**manpath.config**、**master.passwd**、**motd**、**namedb**、**networks**、**newsyslog.conf**、**nsmb.conf**、**nsswitch.conf**、**pam.conf**、**passwd**、**periodic**、**ppp**、**printcap**、**profile**、**pwd.db**、**rc.conf**、**rc.conf.local**、**rc.firewall**、**rc.local**、**remote**、**resolv.conf**、**rmt**、**sendmail.cf**、**sendmail.cw**、**services**、**shells**、**skeykeys**、**spwd.db**、**ssh**、**syslog.conf**、**ttys**、**uucp**。

与新版本对应的这些文件版本被移动到 **/etc/upgrade/**。系统管理员可以审阅这些新版本并根据需要合并组件。请注意，这些文件中许多是相互依赖的，最佳合并过程是将所有站点特定数据从当前文件复制到新文件中。

在升级过程中，系统管理员会被提示输入一个位置，用于保存 **/etc/** 中的所有文件。如果对其他文件进行了本地修改，随后可从该位置检索这些文件。

### 3.2 升级步骤

本节详述升级过程。特别关注与正常安装有重大差异的项目。

#### 3.2.1 备份

在升级之前应备份用户数据和系统配置。虽然升级过程尽力防止意外失误，但仍可能部分或完全破坏数据和配置信息。

#### 3.2.2 挂载文件系统

进入磁盘标签编辑器时，会列出指定磁盘的文件系统设备。在开始升级之前，管理员应记下设备名称和对应的挂载点。应在此处输入这些挂载点。*不要* 为任何文件系统设置“newfs flag”，因为这会导致数据丢失。

#### 3.2.3 选择发行版

选择发行版时，没有必须选择哪些的约束。作为一般规则，应选择 `base` 发行版进行更新，如果已安装手册页，则选择 `man` 发行版。如果管理员希望添加额外功能，可以选择超出最初安装范围的其他发行版。

#### 3.2.4 安装后

安装过程完成后，系统管理员会被提示检查新的配置文件。此时应进行检查以确保系统配置有效。特别应检查 **/etc/rc.conf** 和 **/etc/fstab** 文件。

### 3.3 从源代码升级

对允许更多灵活性和复杂性的升级方法感兴趣的用户，可参阅 FreeBSD 手册中的 [The Cutting Edge](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/cutting-edge.html)。此过程涉及从源代码重新构建整个 FreeBSD。它需要可靠的网络连接、额外的磁盘空间和时间，但对于网络和其他更复杂的安装具有优势。这与用于跟踪 -STABLE 或 -CURRENT 开发分支的过程大致相同。

**/usr/src/UPDATING** 包含从源代码更新 FreeBSD 系统的重要信息。它列出了 FreeBSD 变更导致的可能影响升级的各种问题。

## 4 故障排除

### 4.1 修复现有 FreeBSD 安装

FreeBSD 在 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 安装程序的顶部菜单中提供“fixit”选项。它提供一个带有 FreeBSD 基本系统中常见程序的 shell；此环境对修复或故障排除现有 FreeBSD 安装很有用。要使用 fixit 模式，你还需要 **fixit.flp** 软盘（以与启动软盘相同的方式生成）或“live filesystem”CDROM。在多盘 FreeBSD 发行版中，live filesystem 镜像通常位于安装盘上。请注意，使用 fixit 选项需要一些 UNIX 系统管理经验。

通常有两种调用 fixit 模式的方式。能从 FreeBSD 安装 CDROM 启动的用户应这样做，然后从主 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 菜单中选择“fixit”项。然后从 fixit 菜单中选择“CDROM/DVD”选项。

无法从 CDROM 启动但能从软盘启动的用户需要更多步骤。除安装所需的 **boot.flp** 和 **kernX.flp** 软盘外，还需以与其他软盘相同的方式创建 **fixit.flp** 软盘。按照从软盘启动安装程序的说明操作，直到到达主 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 菜单。此时从主 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.0-RELEASE) 菜单中选择“fixit”项。然后从 fixit 菜单中选择“Floppy”选项，并在提示时插入 **fixit.flp** 软盘。

CDROM 和软盘 fixit 环境相似但不完全相同。两者都提供一个 shell，其中包含可用于检查、修复和检查文件系统及其内容的各种命令。CDROM 版本的 fixit 通过 live filesystem 提供 FreeBSD 基本系统中可用的所有命令和程序。相比之下，由于空间限制，软盘 fixit 环境只能提供命令的子集。

在软盘版本的 fixit 中，一些独立实用程序可在 **/stand** 或 **/mnt2/stand** 中找到。在 CDROM 版本的 fixit 中，这些相同程序可在 **/stand** 或 **/mnt2/rescue** 中找到（以及 live filesystem 中的其余程序，可在 **/mnt** 下找到）。

### 4.2 i386 架构用户常见安装问题

#### 4.2.1 我的系统在启动时探测硬件时挂起，或在安装期间表现异常，或软盘驱动器未被探测到。

FreeBSD 5.0 及更高版本在 i386、amd64 和 ia64 平台上广泛使用系统 ACPI 服务来辅助系统配置（如果在启动期间检测到）。遗憾的是，ACPI 驱动程序和系统主板及 BIOS 中仍存在一些缺陷。可以通过在第三阶段引导加载程序中设置 `hint.acpi.0.disabled` 提示来禁用 ACPI：

```sh
set hint.acpi.0.disabled="1"
```

每次系统启动时此设置都会重置，因此需要将 `hint.acpi.0.disabled="1"` 添加到 **/boot/loader.conf** 文件中。有关引导加载程序的更多信息，请参阅 FreeBSD 手册。

#### 4.2.3 我的旧式 ISA 设备在以前版本的 FreeBSD 中能被识别，但现在不能了。发生了什么？

一些设备驱动程序（如 matcd）由于缺乏维护或其他原因随时间推移被移除。其他驱动程序仍然存在，但由于其侵入式硬件探测例程而被禁用。以下 ISA 设备驱动程序属于此类，可从第三阶段引导加载程序重新启用：aha、ahv、aic、bt、ed、cs、sn、ie、fe、le 和 lnc。为此，在加载程序的 10 秒倒计时期间停止加载程序，并在提示符下输入：

```sh
unset hint.foo.0.disabled
```

其中 `foo` 是要重新启用的驱动程序名称。可以通过编辑 **/boot/device.hints** 文件并删除相应的“disabled”条目来永久设置。

#### 4.2.4 安装 FreeBSD 后首次从硬盘启动时，内核加载并探测硬件，但停在以下消息处：

```sh
changing root device to ad1s1a panic: cannot mount root
```

出了什么问题？我能做什么？

显示的 `bios_drive:interface(unit,partition)kernel_name` 是什么意思？

如果启动磁盘不是系统中的第一个磁盘，存在一个长期存在的问题。BIOS 使用与 FreeBSD 不同的编号方案，且很难确定哪些编号对应哪些磁盘。

如果启动磁盘不是系统中的第一个磁盘，FreeBSD 可能需要一些帮助来找到它。这里有两种常见情况，在这两种情况下，你都需要告诉 FreeBSD 根文件系统的位置。通过指定 BIOS 磁盘编号、磁盘类型和该类型的 FreeBSD 磁盘编号来完成。

第一种情况是你有两块 IDE 磁盘，每块都配置为各自 IDE 总线的主盘，并且想从第二块磁盘启动 FreeBSD。BIOS 将它们视为磁盘 0 和磁盘 1，而 FreeBSD 将它们视为 `ad0` 和 `ad2`。

FreeBSD 位于 BIOS 磁盘 1 上，类型为 `ad`，FreeBSD 磁盘编号为 2，因此你应输入：

```sh
1:ad(2,a)kernel
```

注意，如果主总线上有从盘，则上述操作不必要（实际上是错误的）。

第二种情况涉及当系统中有一块或多块 IDE 磁盘时从 SCSI 磁盘启动。在这种情况下，FreeBSD 磁盘编号低于 BIOS 磁盘编号。如果你有两块 IDE 磁盘以及一块 SCSI 磁盘，SCSI 磁盘是 BIOS 磁盘 2，类型 `da`，FreeBSD 磁盘编号 0，因此你应输入：

```sh
2:da(0,a)kernel
```

告诉 FreeBSD 你要从 BIOS 磁盘 2 启动，这是系统中的第一块 SCSI 磁盘。如果只有一块 IDE 磁盘，则使用 `1:` 代替。

确定正确的值后，可以使用标准文本编辑器将命令原样放入 **/boot.config** 文件中。除非另有指示，FreeBSD 将使用此文件的内容作为 `boot:` 提示符的默认响应。

#### 4.2.5 安装 FreeBSD 后首次从硬盘启动时，启动管理器提示在启动菜单处每次只打印 `F?`，但启动无法继续。

在安装 FreeBSD 时，分区编辑器中的硬盘几何参数设置不正确。返回分区编辑器并指定硬盘的实际几何参数。必须使用正确的几何参数从头重新安装 FreeBSD。

如果完全无法确定机器的正确几何参数，这里有一个提示：在磁盘开头安装一个小的 DOS 分区，然后在其后安装 FreeBSD。安装程序会看到 DOS 分区并尝试从中推断正确的几何参数，这通常有效。

以下提示不再推荐，但保留在此供参考：

> 如果你正在设置一台真正专用的 FreeBSD 服务器或工作站，不关心（未来）与 DOS、Linux 或其他操作系统的兼容性，你还可以选择使用整个磁盘（分区编辑器中的“A”），选择 FreeBSD 从第一个到最后一个扇区占用整个磁盘的非标准选项。这将忽略所有几何参数考虑，但有些局限性，除非你永远不会在磁盘上运行除 FreeBSD 以外的任何东西。

#### 4.2.6 [mcd(4)](http://www.freebsd.org/cgi/man.cgi?query=mcd&sektion=4&manpath=FreeBSD+6.0-RELEASE) 驱动程序不断认为它找到了设备，这阻止了我的 Intel EtherExpress 卡工作。

在第三阶段引导加载程序中设置提示 `hint.mcd.0.disabled="1"` 和 `hint.mcd.1.disabled="1"` 以禁用 `mcd0` 和 `mcd1` 设备的探测。一般来说，你应该只在内核中保留将要使用的设备为启用状态。

#### 4.2.7 系统找到我的 [ed(4)](http://www.freebsd.org/cgi/man.cgi?query=ed&sektion=4&manpath=FreeBSD+6.0-RELEASE) 网卡，但我不断收到设备超时错误。

你的网卡可能使用了与 **/boot/device.hints** 文件中指定的不同的 IRQ。ed 驱动程序默认不使用“软”配置（在 DOS 中使用 EZSETUP 输入的值），但如果你在接口的 hints 中指定 `-1`，它将使用软件配置。

要么将卡上的跳线移到硬配置设置（必要时更改内核设置），要么通过设置提示 `hint.ed.0.irq="-1"` 将 IRQ 指定为 `-1`。这将告诉内核使用软配置。

另一种可能是你的卡使用 IRQ 9，它由 IRQ 2 共享，经常是问题的根源（尤其是当你的 VGA 卡使用 IRQ 2 时！）。如果可能，不应使用 IRQ 2 或 9。

#### 4.2.8 我在 IBM ThinkPad 笔记本电脑上启动安装软盘，键盘完全混乱了。

旧款 IBM 笔记本电脑使用非标准键盘控制器，因此必须告诉键盘驱动程序 (atkbd0) 进入适用于 ThinkPad 的特殊模式。设置提示 `hint.atkbd.0.flags="4"`，应该就能正常工作。

#### 4.2.9 我的系统找不到我的 Intel EtherExpress 16 卡。

必须将 Intel EtherExpress 16 卡设置为内存映射地址 0xD0000，并使用 Intel 提供的 `softset.exe` 程序将映射内存量设置为 32K。

#### 4.2.10 在 EISA HP Netserver 上安装时，我的板载 AIC-7xxx SCSI 控制器未被检测到。

这是一个已知问题，希望未来能修复。为使系统至少能安装，在第三阶段加载程序中设置提示 `hw.eisa_slots="12"`。

#### 4.2.11 我有一台 Panasonic AL-N1 或 Rios Chandler Pentium 机器，我发现系统现在在进入安装之前就挂起了。

由于某种原因，你的机器不喜欢新的 `i586_copyout` 和 `i586_copyin` 代码。要禁用此功能，设置提示 `hint.npx.0.flags="1"`。

#### 4.2.12 我有一个据说有缺陷的 CMD640 IDE 控制器。

FreeBSD 不支持此控制器。

#### 4.2.13 在 Compaq Aero 笔记本电脑上，尝试从软盘安装时出现消息“No floppy devices found! Please check ...”。

Compaq 总是与其他系统有些不同，它们不在 Aero 笔记本电脑的 CMOS RAM 中通告软盘驱动器。因此，软盘驱动器程序假设没有配置驱动器。设置提示 `hint.fdc.0.flags="1"`，这会向驱动程序假装第一个软盘驱动器（作为 1.44 MB 驱动器）存在，而不询问 CMOS。

#### 4.2.14 在 Dell Poweredge XE 上安装时，Dell 专有 RAID 控制器 DSA（Dell SCSI Array）未被识别。

使用 EISA 配置实用程序将 DSA 配置为使用 AHA-1540 仿真。之后 FreeBSD 将 DSA 检测为 Adaptec AHA-1540 SCSI 控制器，irq 为 11，端口为 340。在仿真模式下，系统将使用 DSA RAID 磁盘，但你无法使用 DSA 特定功能（如监控 RAID 健康状态）。

#### 4.2.15 我有一块 IBM EtherJet PCI 卡，[fxp(4)](http://www.freebsd.org/cgi/man.cgi?query=fxp&sektion=4&manpath=FreeBSD+6.0-RELEASE) 驱动程序正确检测到了它，但卡上的指示灯不亮，也无法连接到网络。

我们不明白为什么会发生这种情况。IBM 也不明白（我们问过他们）。这块卡实际上是标准的 Intel EtherExpress Pro/100，只是贴了 IBM 标签，这些卡通常工作正常。你可能只在某些 IBM Netfinity 服务器中看到这些症状。唯一的解决方案是安装不同的以太网适配器。

#### 4.2.16 在 IBM Netfinity 3500 上安装期间配置网络时，系统冻结。

Netfinity 3500 的板载以太网存在问题，我们目前无法确定原因。可能与系统的 SMP 功能配置错误有关。你必须安装另一个以太网适配器，并避免在任何时候尝试配置板载适配器。

#### 4.2.17 当我安装到由 Mylex PCI RAID 控制器管理的驱动器上时，系统无法启动（例如出现 `read error` 消息）。

Mylex 驱动程序存在缺陷，导致它忽略 BIOS 中的“8GB”几何模式设置。请改用 2GB 模式。
