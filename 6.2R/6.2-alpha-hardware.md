### 2.2 在 Alpha 上运行 FreeBSD 的基本要求

显然，你需要一台 FreeBSD 了解的 Alpha 机器。Alpha 机器*不像* PC。各种核心逻辑芯片组和主板设计之间存在相当大的差异。这意味着内核在运行之前需要了解特定机器的详细信息。向未知硬件抛出某个 **GENERIC** 内核几乎肯定会惨败。

对于要考虑用于 FreeBSD 的机器，请确保已安装 SRM 控制台固件。或者至少确保特定机器类型有 SRM 控制台固件可用。如果 FreeBSD 当前不支持你的机器类型，假设 SRM 可用，这很可能在某个时候会改变。当 SRM 控制台固件不可用时，一切都不确定。

带有 ARC 或 AlphaBIOS 控制台固件的机器是为 WindowsNT 设计的。有些机器在系统 ROM 中有 SRM 控制台固件可用，你只需选择（通过 ARC 或 AlphaBIOS 菜单）。在其他情况下，你需要用 SRM 代码重新刷新 ROM。请查看 [http://ftp.digital.com/pub/DEC/Alpha/firmware](http://ftp.digital.com/pub/DEC/Alpha/firmware) 了解特定系统可用的固件。无论如何：没有 SRM 就意味着*没有* FreeBSD（NetBSD、OpenBSD、Tru64 Unix 或 OpenVMS 也是如此）。随着 WindowsNT/alpha 的消亡，许多以前的 NT 机器在二手市场上出售。当它们仅支持 NT 时，几乎没有或没有折旧价值。所以，如果价格看起来太好的话，要小心。

如果你在选择 SRM 时遇到问题（即选择了 SRM 控制台后系统又回到 AlphaBIOS），是时候检查 CMOS 时钟/参数 RAM 的电池是否耗尽了。如果是的话，更换电池很可能会解决问题。

已知的非 SRM 机器有：

- Digital XL 系列
- Digital XLT 系列
- Samsung PC164UX（“Ruffian”）
- Samsung 164B

有 SRM 但 FreeBSD 不支持的机器有：

- DECpc 150（“Jensen”）
- DEC 2000/300（“Jensen”）
- DEC 2000/500（“Culzean”）
- AXPvme 系列（“Medulla”）

再让事情复杂一点：Digital 曾经有所谓“white-box”Alpha 机器，定位为仅用于 NT，以及“blue-box”Alpha 机器，定位用于 OpenVMS 和 Digital Unix。这些名称基于机箱的颜色，分别为“FrostWhite”和“TopGunBlue”。虽然你可以在 whitebox 上安装 SRM 控制台固件，但 OpenVMS 和 Digital Unix 会拒绝在它们上面启动。FreeBSD 在 post-4.0-RELEASE 之后可以在 white 和 blue-box 两种变体上运行。在有人问之前：白色的那种有一个相当不同的（即更便宜的）Digital 价格标签。

作为 SRM 的一部分，你会得到所谓的 OSF/1 PAL 代码（OSF/1 是 Digital 在 Alpha 上提供 UNIX 的初始名称）。PAL 代码可以看作是硬件和操作系统之间的软件抽象层。它使用常规 CPU 指令以及少量专用于 PAL 的特权指令。PAL 不是微代码。ARC 控制台固件包含不同的 PAL 代码，面向 WinNT，完全不适合 FreeBSD 使用（或者更一般地说：Unix 或 OpenVMS）。在有人问之前：Linux/alpha 自带 PAL 代码，使其能在 ARC 和 AlphaBIOS 上启动。有各种原因说明为什么这在 *BSD 人士看来不是个好主意。我不想在这里详细讨论。如果你对血淋淋的细节感兴趣，请搜索 FreeBSD 和 NetBSD 网站。

前面还有另一个陷阱：你需要一个 SRM 控制台固件能识别的磁盘适配器，才能从磁盘启动。遗憾的是，SRM 接受什么作为启动适配器高度依赖于系统和 SRM 版本。对于较旧的基于 PCI 的机器，这意味着你需要基于 NCR/Symbios 53C810 的适配器，或基于 Qlogic 1020/1040 的适配器。有些机器在主板上嵌入了 SCSI 芯片。较新的机器设计和 SRM 版本能够与更现代的 SCSI 芯片/适配器配合工作。请查看下面的机器特定信息。请注意，本讨论的其余部分仅提及 Symbios 芯片，这意在包括那些仍然标有 NCR 的较旧芯片。Symbios 在某个时候收购了 NCR。

这个问题可能会困扰那些机器最初是 WindowsNT 机器的用户。ARC 或 AlphaBIOS 知道它可以启动的*其他*适配器类型比 SRM 多。例如，你可以用 ARC/AlphaBios 从 Adaptec 2940UW 启动，但（通常）不能用 SRM。一些较新的机器类型引入了 Adaptec 启动支持。详情请参阅机器特定章节。

大多数不能启动的适配器对于仅数据磁盘来说工作良好。SRM 和 ARC 之间的差异也可能让你在某些（前 WindowsNT）系统中获得预装的 IDE CDROM 和硬盘。存在可以从 IDE 磁盘和 CDROM 启动的 SRM 版本（取决于机器类型）。详情请参阅机器特定章节。

为了能够启动，根分区（分区 a）必须位于磁盘驱动器的偏移 0 处。这意味着你必须使用安装程序的分区菜单，并从将分区 a 分配在偏移 0 处的根分区开始。然后按照你的喜好布局其余分区。如果不遵守此规则，安装会顺利进行，但系统将无法从新安装的磁盘启动。注意，在 Alpha 上运行的 FreeBSD 不使用 fdisk，磁盘直接使用 disklabel 标记。

如果你没有/不想要本地磁盘驱动器，可以通过以太网启动。这需要一个 SRM 控制台能识别的以太网适配器/芯片。一般来说，这归结为基于 21040 或 21142 或 21143 的以太网接口。较旧的机器或 SRM 版本可能无法识别 21142 / 21143 快速以太网芯片，此时你只能使用 10Mbit 以太网进行网络启动。基于上述芯片的非 DEC 卡通常可以工作（但不保证）。注意，Intel 在收购 Digital Semiconductor 时接管了 21x4x 芯片。所以现在你可能会在它们上面看到 Intel 标志。最近的机器设计有对 Intel 8255x 以太网芯片的 SRM 支持。

Alpha 机器可以在图形控制台或串行控制台上运行 SRM。如有需要，ARC 也可以在串行控制台上运行。带有 8 位控制的 VT100 仿真至少应该能让你从 ARC/AlphaBIOS 切换到 SRM 模式，而无需先安装显卡。

如果你想在没有显示器/显卡的情况下运行 Alpha 机器，只需不要将键盘/鼠标连接到机器上。而是将串行终端[模拟器]连接到串行端口 #1。SRM 会以 9600N81 与你通信。这对于调试目的也非常实用。注意：某些/大多数（？）SRM 也会在串行端口 #2 上为你提供控制台提示符。但是，启动中的内核会在串行端口 #1 上显示启动消息，并将控制台放在那里。*这可能非常令人困惑。*

大多数基于 PCI 的 Alpha 可以使用普通的 PC 型 VGA 卡。SRM 足够智能使其工作。但这并不意味着街上的每块 PCI VGA 卡都能在 Alpha 机器上工作。S3 Trio64、Mach64 和 Matrox Millennium 通常可以工作。旧的基于 ET4000 的 ISA 卡也为我工作过。但购买前先打听一下。

大多数来自 PC 世界的 PCI 设备也可以在 FreeBSD 基于 PCI 的机器上工作。查看 **/sys/alpha/conf/GENERIC** 文件了解这方面的最新信息。如果你想使用上面有 PCI 桥接芯片的 PCI 卡，请查看相应机器类型的讨论。在某些情况下，你可能会遇到 PCI 卡不能正确处理 PCI 奇偶校验的问题。这可能导致 panic。可以使用以下 SRM 命令禁用 PCI 奇偶校验：

```sh
>>> SET PCI_PARITY OFF
```

这不是 FreeBSD 的问题，在 Alpha 硬件上运行的所有操作系统都需要此变通方法。

如果你的系统（也）包含 EISA 扩展槽，在安装 EISA 卡或升级控制台固件后，你需要运行 EISA 配置实用程序（ECU）。

对于 Alpha CPU，你会发现有多代。最初的 Alpha 设计是 21064。它在一个称为 MOS4 的芯片工艺中生产，在此工艺中制造的芯片被昵称为 EV4。较新的 CPU 有 21164、21264 等。你会看到 EV4S、EV45、EV5、EV56、EV6、EV67、EV68 等代号。双位数字的 EV 是略有改进的版本。例如，EV45 与它所基于的 EV4 相比，有改进的 FPU 和 16 kByte 片上独立 I & D 缓存。经验法则：“EV”后面紧跟的数字越高，越理想（即更快/更现代）。不建议在低于 EV5 的任何东西上花钱。当然，免费获得的硬件另当别论。

对于内存，你至少需要 32 Mbyte。多年前我曾在 16 Mbyte 系统上运行过 FreeBSD，但你不会喜欢那样的。当我升级到 32 Mbyte 时，内核构建时间减半。注意，SRM 控制台会从总系统内存中占用 2Mbyte（并保留它）。对于更严肃的工作，建议 128 Mbyte 或更多。虽然 Alpha 机器通常可以容纳大到非常大的物理内存，但 FreeBSD 限于 1 或 2 Gbyte（取决于核心芯片组）的 RAM。此限制是由于 VM 系统的当前实现。

在内存话题上：密切注意你的机器使用的内存类型。各种机器有非常不同的内存配置和要求。

最后一句：我期望以上内容对首次使用 Alpha 的用户来说听起来有点令人生畏。不要太被吓倒。如果在阅读本文档后有什么不清楚的地方，请随时提问。

### 2.4 支持的硬件概述

需要提醒的是：FreeBSD 的装机量远不如 FreeBSD/Intel。这意味着 PCI/ISA 扩展卡的巨大多样性在 alpha 上被测试的机会比在 Intel 上小得多。这并不是说它们注定会失败，只是遇到从未测试过的东西的机会要高得多。**GENERIC** 只包含已知能在 Alpha 上工作的东西。

PCI 和 ISA 扩展总线得到完全支持。Turbo Channel 不在 **GENERIC** 中，支持有限（参见相关机器型号信息）。不支持 MCA 总线。不支持 EISA 总线用于 EISA 扩展卡，因为缺乏 EISA 支持代码。据报道，EISA 插槽中的 ISA 卡可以工作。Compaq Qvision EISA VGA 卡以 ISA 模式驱动，作为控制台工作正常。

支持 1.44 Mbyte 和 1.2 Mbyte 软盘驱动器。Alpha 机器中有时发现的 2.88 Mbyte 驱动器最高支持到 1.44Mbyte。

ATA 和 ATAPI（IDE）设备通过 `ata`(4) 驱动程序框架支持。由于大多数人用 SCSI 磁盘运行他们的 Alpha，它的测试不如 SCSI 充分。注意 IDE 磁盘的可启动性限制。参见机器特定信息。

通过 CAM 层完全支持 Adaptec 2940x（基于 AIC7xxx 芯片）、Qlogic 系列和 Symbios 的 SCSI。对 U160 SCSI 感兴趣的人可以看看 Adaptec 39160 双通道 LVD U160 适配器。Compaq 称之为 KZPEA 适配器。最近的 Alpha 型号有可以从它们启动的 SRM 版本。一般来说，要注意各种适配器型号的机器特定可启动性问题。已知的情况下，它们在各个机器描述中列出。

Qlogic QL2x00 FibreChannel 主机适配器得到完全支持。

如果你想通过以太网启动你的 Alpha，显然需要一块 SRM 控制台能识别的以太网卡。这通常意味着你需要一块带 21x4x 以太网芯片的板卡，因为那是 Digital 使用的。这些芯片由 FreeBSD `de`(4)（较旧的驱动程序）或 `dc`(4)（较新的驱动程序）驱动。已知一些新的 SRM 版本能识别由 FreeBSD `fxp`(4) 驱动程序驱动的 Intel 8255x 以太网芯片。但要注意：据报道 `fxp`(4) 驱动程序在 FreeBSD/alpha 上不能正确工作（尽管它在 FreeBSD/x86 上工作得非常好）。

一般来说，SRM 控制台在 PCI VGA 卡上模拟 VGA 兼容模式。然而，Compaq/DEC 并不保证它对每种卡类型都能工作。当 SRM 认为 VGA 可接受时，FreeBSD 就能使用它。控制台驱动程序的工作方式与 FreeBSD/intel 机器上一样。请注意，Alpha 不支持 VESA 模式，所以你只能用 80x25 控制台。

在某些 Alpha 机器上，你会发现基于 TGA 芯片的视频适配器。FreeBSD 中的 TGA 支持并不像它应该的那样健壮。出现问题时，建议尝试串行控制台或普通 VGA 卡。

大多数 Alpha 上的“PC 标准”串口得到支持。

FreeBSD/alpha 不支持 ISDN（i4b）。
