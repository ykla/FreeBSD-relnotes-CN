# FreeBSD 11.1-RELEASE 发行公告

日期：2017 年 7 月 26 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 11.1-RELEASE 现已发布。这是 stable/11 分支的第二个版本。

部分亮点如下：

* `clang`、`llvm`、`lld`、`lldb` 和 `compiler-rt` 工具以及 `libc++` 已更新至版本 4.0.0。
* 多项第三方（贡献）软件更新，如 Elf Tool Chain、ACPICA、[libarchive(3)](https://man.freebsd.org/cgi/man.cgi?query=libarchive&sektion=3&format=html)、[ntpd(8)](https://man.freebsd.org/cgi/man.cgi?query=ntpd&sektion=8&format=html)、[unbound(8)](https://man.freebsd.org/cgi/man.cgi?query=unbound&sektion=8&format=html) 等。
* OpenSSH 新增对 [blacklistd(8)](https://man.freebsd.org/cgi/man.cgi?query=blacklistd&sektion=8&format=html) 的支持。
* 新增 [zfsbootcfg(8)](https://man.freebsd.org/cgi/man.cgi?query=zfsbootcfg&sektion=8&format=html) 工具，为 [zfsboot(8)](https://man.freebsd.org/cgi/man.cgi?query=zfsboot&sektion=8&format=html) 提供类似 [boot.config(5)](https://man.freebsd.org/cgi/man.cgi?query=boot.config&sektion=5&format=html) 的一次性选项。
* 新增 [efivar(8)](https://man.freebsd.org/cgi/man.cgi?query=efivar&sektion=8&format=html) 工具，提供管理 UEFI 变量的接口。
* 新增对 Microsoft(R) Hyper-V(TM) 第二代虚拟机的支持。
* 新增 [ena(4)](https://man.freebsd.org/cgi/man.cgi?query=ena&sektion=4&format=html) 驱动程序，提供对 Amazon(R) EC2(TM) 平台上"下一代"增强网络的支持。
* NFS 客户端现在支持 Amazon(R) 弹性文件系统(TM)（EFS）。
* EFI 加载器现在除了 NFS 外，还可以通过 TFTP 访问远程文件，作为运行时配置选项。
* ZFS 现在将压缩数据存储在缓存中，提高缓存命中率和性能。
* 多项改进以提供构建可重现性。

完整的新功能和已知问题列表，请参阅在线发行说明和勘误列表，地址如下：

* <https://www.FreeBSD.org/releases/11.1R/relnotes/>
* <https://www.FreeBSD.org/releases/11.1R/errata/>

关于 FreeBSD 发布工程活动的更多信息，请参阅：

* <https://www.FreeBSD.org/releng/>

## 获取途径

FreeBSD 11.1-RELEASE 现已支持 amd64、i386、powerpc、powerpc64、sparc64、armv6 和 aarch64 架构。

FreeBSD 11.1-RELEASE 可从可启动 ISO 镜像或通过网络安装。部分架构还支持从 USB 闪存盘安装。所需文件可按以下说明下载。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/11.1R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/11.1R/announce.asc>

>**注意**：从 11.1-RC2 在 VirtualBox(TM) 中升级的说明：
>
> 如果从 11.1-RC1 升级到 11.1-RC2 时遇到系统崩溃，并且通过本地构建 `emulators/virtualbox-ose-additions-nox11` port 作为解决方案，则从 11.1-RC2 升级到 11.1-RELEASE 时需要重新构建该 port，或者使用以下任一命令从 `pkg(8)` 镜像重新安装软件包：
>
> ```sh
> # pkg install -f virtualbox-ose-additions
> ```
>
> 或：
>
> ```sh
> # pkg install -f virtualbox-ose-additions-nox11
> ```
>
> 为确保系统在重启进入更新后的内核后不会崩溃，建议在重启系统之前（如果可能）禁用 [rc.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&format=html) 中的 `vboxguest` 服务，或使用 `pkg(8)` 强制重新安装该软件包。
>
> 从 11.1-RC1 及更早版本以及 11.1-RC3 升级到 11.1-RELEASE 的系统应不受影响。

发行版所提供镜像的用途如下：

dvd1::
包含安装 FreeBSD 基础操作系统、文档、调试发行版和少量预构建 package（旨在启动图形工作站）所需的全部内容。
还支持启动到基于「livefs」的救援模式。
若你能刻录并使用 DVD 介质，这应是你所需的全部。

disc1::
包含 FreeBSD 基础操作系统。
还支持启动到基于「livefs」的救援模式。
不含预构建的 package。

bootonly::
支持使用 CDROM 驱动器启动机器，但不包含从 CD 本身安装 FreeBSD 所需的安装发行版。
从 CD 启动后，需要执行基于网络的安装（例如从 FTP 服务器）。

memstick::
可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。
还支持启动到基于「livefs」的救援模式。
不含预构建的 package。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-11.1-RELEASE-amd64-memstick.img \
    of=/dev/da0 bs=10240 conv=sync
```

务必确保目标（`of=`）正确。

mini-memstick::
可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。
还支持启动到基于「livefs」的救援模式。
不含预构建的 package。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-11.1-RELEASE-amd64-mini-memstick.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

FreeBSD/arm SD 卡镜像::
可写入 SD 卡并用于启动受支持的 arm 系统。
SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 512Mb 的 SD 卡上。

为方便没有控制台访问权限的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 [ssh(1)](https://man.freebsd.org/cgi/man.cgi?query=ssh&sektion=1&format=html) 访问。
此外，`root` 用户密码设置为 `root`，强烈建议在获得系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，请使用 [dd(1)](https://man.freebsd.org/cgi/man.cgi?query=dd&sektion=1&format=html) 工具，将 _KERNEL_ 替换为系统对应的内核配置名称。

```sh
# dd if=FreeBSD-11.1-RELEASE-arm-armv6-KERNEL.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。
