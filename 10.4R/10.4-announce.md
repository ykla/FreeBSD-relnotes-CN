# FreeBSD 10.4-RELEASE 发行公告

日期：2017 年 10 月 3 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 10.4-RELEASE 现已可用。这是 stable/10 分支的第五个发行版，基于 10.3-RELEASE 的稳定性和可靠性构建，并引入了新功能。

部分亮点如下：

* 10.4-RELEASE 是首个全面支持 eMMC 存储的 FreeBSD 发行版，包括 eMMC 分区、TRIM 以及高达 HS400 的总线速度模式。但请注意，DDR52、HS200 和 HS400 模式的可用性尤其需要实际的 **sdhci(4)** 前端以及所用硬件的支持。另请注意，Intel Apollo Lake 芯片组的 SDHCI 控制器部分受若干严重的硅片缺陷影响。显然，目前已有的变通方案是否足以避免在附加 **sdhci(4)** 时出现超时，取决于特定的 Apollo Lake 平台。
* 在使用 GPT 磁盘标签时，**fsck_ffs(8)** 工具现在能够找到备用超级块。
* **aesni(4)** 驱动不再在多个线程的多个会话之间共享单个 FPU 上下文，解决了使用 **aesni(4)** 加速 **ipsec(4)** 时出现的问题。
* 新增对 Intel i219(4)/ i219(5) 设备的 Kaby Lake 代支持至 **em(4)** 驱动。
* **em(4)** 驱动现在能够为 Intel i217、i218 和 i219 芯片启用局域网唤醒（WOL）。注意，此前为这些设备启用 WOL 的失败尝试所留下的失效接口配置现在将实际生效。例如，`\`ifconfig em0 wol\`` 会激活所有 WOL 变体（包括 wol_mcast），这可能并非所愿。
* 新增对 **igb(4)** 驱动的 WOL 支持，此前该驱动无法在任何设备上激活此功能。关于失效 WOL 配置的同样说明也适用于 **em(4)** 驱动。
* 用户空间核心转储现在可触发事件，例如通过 **devd(8)** 生成人类可读的崩溃报告。此功能默认关闭。
* **qlxgbe(4)** 驱动附带的固件已更新至 5.4.66 版本。此外，该驱动还获得了一些 TSO 和锁定修复、性能优化，以及提供 MAC、RX 和 TX 统计信息的 SYSCTL。
* Mellanox ConnectX-4 系列适配器现在由新增的 **mlx5ib(4)** 驱动支持。
* OpenSSH 更新至 7.3p1 版本。
* GNOME 更新至 3.18 版本。
* Xorg-Server 更新至 1.18.4 版本。
* 以及更多...

关于新功能和已知问题的完整列表，请参阅在线发行说明和勘误列表，地址如下：

## 可用性

FreeBSD 10.4-RELEASE 现已支持 amd64、i386、ia64、powerpc、powerpc64、sparc64 和 armv6 架构。

FreeBSD 10.4-RELEASE 可从可引导 ISO 镜像或通过网络安装。部分架构还支持从 USB 闪存盘安装。所需文件可按以下章节的说明下载。

本消息底部包含了发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 哈希值。

发行版镜像的 PGP 签名校验和也可在以下地址获取：

* <https://www.FreeBSD.org/releases/10.4R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/10.4R/announce.asc>

amd64（x86_64）架构还提供了额外的支持 UEFI 的镜像。

作为发行版一部分提供的镜像用途如下：

**dvd1**：
包含安装 FreeBSD 基本操作系统、文档以及一组旨在使图形工作站运行起来的预构建软件包所需的全部内容。它还支持引导进入基于“livefs”的救援模式。如果你能刻录和使用 DVD 大小的介质，这应是你所需的全部。

**disc1**：
包含 FreeBSD 基本操作系统。它还支持引导进入基于“livefs”的救援模式。不包含预构建的软件包。

**bootonly**：
支持使用 CDROM 驱动器引导机器，但不包含从 CD 本身安装 FreeBSD 的安装发行版集合。从 CD 引导后，需要执行基于网络的安装（例如从 FTP 服务器）。

**memstick**：
可写入 USB 闪存盘（闪存驱动器）并用于在能够从 USB 驱动器引导的机器上执行安装。它还支持引导进入基于“livefs”的救援模式。不包含预构建的软件包。

作为如何使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似以下命令应该可行：

```sh
# dd if=FreeBSD-10.4-RELEASE-amd64-memstick.img of=/dev/da0 bs=10240 conv=sync
```

请务必确保目标（of=）正确。

**mini-memstick**：
可写入 USB 闪存盘（闪存驱动器）并用于引导机器，但不包含介质本身的安装发行版集合，类似于 bootonly 镜像。它还支持引导进入基于“livefs”的救援模式。不包含预构建的软件包。

作为如何使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似以下命令应该可行：

```sh
# dd if=FreeBSD-10.4-RELEASE-amd64-mini-memstick.img of=/dev/da0 bs=1m conv=sync
```

请务必确保目标（of=）正确。

**FreeBSD/arm SD 卡镜像**：
可写入 SD 卡并用于引导受支持的 arm 系统。SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 512Mb 的 SD 卡上。

为方便没有控制台访问权限的用户，默认提供用户名为 `freebsd`、密码为 `freebsd` 的用户用于 **ssh(1)** 访问。此外，`root` 用户密码设为 `root`，强烈建议在获得系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，请使用 **dd(1)** 工具，将 *KERNEL* 替换为系统对应的内核配置名称。

```sh
# dd if=FreeBSD-10.4-RELEASE-arm-armv6-KERNEL.img of=/dev/da0 bs=1m conv=sync
```

请务必确保目标（of=）正确。
