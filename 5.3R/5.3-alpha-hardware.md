### 2.2 在 Alpha 上运行 FreeBSD 的基本要求

显然，你需要一台 FreeBSD 了解的 Alpha 机器。Alpha 机器*不像* PC。各种核心逻辑芯片组和主板设计之间存在相当大的差异。这意味着内核在运行之前需要了解特定机器的详细信息。向未知硬件抛出某个 **GENERIC** 内核几乎肯定会惨败。

对于要考虑用于 FreeBSD 的机器，请确保已安装 SRM 控制台固件。或者至少确保特定机器类型有 SRM 控制台固件可用。如果 FreeBSD 当前不支持你的机器类型，假设 SRM 可用，这很可能在某个时候会改变。当 SRM 控制台固件不可用时，一切都不确定。

带有 ARC 或 AlphaBIOS 控制台固件的机器是为 WindowsNT 设计的。有些机器在系统 ROM 中有 SRM 控制台固件可用，你只需选择（通过 ARC 或 AlphaBIOS 菜单）。在其他情况下，你需要用 SRM 代码重新刷新 ROM。请查看 <http://ftp.digital.com/pub/DEC/Alpha/firmware> 了解特定系统可用的固件。无论如何：没有 SRM 就意味着*没有* FreeBSD（NetBSD、OpenBSD、Tru64 Unix 或 OpenVMS 也是如此）。随着 WindowsNT/alpha 的消亡，许多以前的 NT 机器在二手市场上出售。从控制台固件角度来看，当它们仅支持 NT 时，几乎没有或没有折旧价值。因此，如果价格看起来好得难以置信，请保持怀疑。

已知非 SRM 机器有：

- Digital XL 系列
- Digital XLT 系列
- Samsung PC164UX（“Ruffian”）
- Samsung 164B

有 SRM 但 FreeBSD 不支持的机器有：

- DECpc 150（“Jensen”）
- DEC 2000/300（“Jensen”）
- DEC 2000/500（“Culzean”）
- AXPvme 系列（“Medulla”）

为了使事情更复杂一些：Digital 曾经有所谓的“白箱”Alpha 机器（专为 NT-only 设计）和“蓝箱”Alpha 机器（专为 OpenVMS 和 Digital Unix 设计）。这些名称基于机箱的颜色，分别为“FrostWhite”和“TopGunBlue”。虽然你可以在白箱上安装 SRM 控制台固件，但 OpenVMS 和 Digital Unix 会拒绝在它们上启动。FreeBSD 在 4.0-RELEASE 之后的版本可以在白箱和蓝箱变体上运行。在有人问之前：白箱有一个相当不同的（即更便宜的）Digital 价格标签。

作为 SRM 的一部分，你会得到所谓的 OSF/1 PAL 代码（OSF/1 是 Digital 在 Alpha 上的 UNIX 产品的初始名称）。PAL 代码可以被认为是硬件和操作系统之间的软件抽象层。它使用普通 CPU 指令以及少量专用于 PAL 使用的特权指令。PAL 不是微代码。ARC 控制台固件包含不同的 PAL 代码，面向 WinNT，完全不适合 FreeBSD 使用（更一般地说：Unix 或 OpenVMS）。在有人问之前：Linux/alpha 自带 PAL 代码，允许它在 ARC 和 AlphaBIOS 上启动。\*BSD 人士认为这不是一个好主意，原因有很多。我不想在这里详述。如果你对那些复杂的细节感兴趣，请搜索 FreeBSD 和 NetBSD 网站。

还有另一个陷阱：你需要 SRM 控制台固件识别的磁盘适配器才能从磁盘启动。遗憾的是，SRM 接受什么作为启动适配器高度依赖于系统和 SRM 版本。对于较旧的基于 PCI 的机器，这意味着你需要基于 NCR/Symbios 53C810 的适配器，或基于 Qlogic 1020/1040 的适配器。有些机器在主板上嵌入了 SCSI 芯片。较新的机器设计和 SRM 版本将能够与更现代的 SCSI 芯片/适配器一起工作。查看下面的机器特定信息。请注意，本讨论的其余部分仅涉及 Symbios 芯片，这包括仍然印有 NCR 的较旧芯片。Symbios 在某个时候收购了 NCR。

这个问题可能会咬到那些机器最初是 WindowsNT 机器的人。ARC 或 AlphaBIOS 知道它可以从中启动的*其他*适配器类型，而 SRM 则不同。例如，你可以使用 ARC/AlphaBios 从 Adaptec 2940UW 启动，但（通常）不能使用 SRM。一些较新的机器类型引入了 Adaptec 启动支持。请查阅机器特定部分以获取详细信息。

大多数不能启动的适配器对于仅数据磁盘工作正常。SRM 和 ARC 之间的差异也可能在某些（前 WindowsNT）系统中为你带来预封装的 IDE CDROM 和硬盘。存在可以从 IDE 磁盘和 CDROM 启动的 SRM 版本（取决于机器类型）。查看机器特定部分以获取详细信息。

FreeBSD 4.0 及以后版本可以从发行版 CDROM 启动。早期版本需要从 2 张软盘启动。

为了可启动，根分区（分区 a）必须位于磁盘驱动器的偏移 0 处。这意味着你必须使用安装程序的分区菜单，并从将分区 a 分配给根分区（偏移 0）开始。随后根据你的喜好布局其余分区。如果你不遵守此规则，安装将顺利进行，但系统将无法从新安装的磁盘启动。

如果你没有/不想要本地磁盘驱动器，你可以通过以太网启动。这假定 SRM 控制台识别以太网适配器/芯片。一般来说，这意味着你需要带有 21040 或 21142 或 21143 芯片的以太网接口。较旧的机器或 SRM 版本可能不识别 21142 / 21143 快速以太网芯片，你将被限制为使用 10Mbit 以太网进行网络启动。基于上述芯片的非 DEC 卡通常可以（但不保证）工作。请注意，Intel 在收购 Digital Semiconductor 时接管了 21x4x 芯片。所以你现在可能会在它们上看到 Intel 标志。最近的机器设计有对 Intel 8255x 以太网芯片的 SRM 支持。

Alpha 机器可以在 SRM 上运行图形控制台或串行控制台。如果需要，ARC 也可以在串行控制台上运行。带有 8 位控件的 VT100 仿真至少应该允许你从 ARC/AlphaBIOS 切换到 SRM 模式，而无需先安装显卡。

如果你想在没动显示器/显卡的情况下运行 Alpha 机器，只需不连接键盘/鼠标到机器。相反，将串行终端 [模拟器] 连接到串行端口 #1。SRM 将以 9600N81 与你通信。这对于调试目的也非常实用。注意：一些/大多数（？）SRM 也会在串行端口 #2 上为你提供控制台提示。但是，启动内核将在串行端口 #1 上显示启动消息，并将控制台放在那里。*这可能会非常令人困惑。*

大多数基于 PCI 的 Alpha 可以使用普通 PC 型 VGA 卡。SRM 包含足够的智能使其工作。然而，这并不意味着街上的每个 PCI VGA 卡都能在 Alpha 机器中工作。像 S3 Trio64、Mach64 和 Matrox Millennium 这样的卡通常可以工作。旧的基于 ET4000 的 ISA 卡对我来说也工作过。但在购买之前先问一下。

来自 PC 世界的多数 PCI 设备也可以在 FreeBSD 基于 PCI 的机器中工作。查看 **/sys/alpha/conf/GENERIC** 文件以获取这方面的最新信息。如果你想使用上面有 PCI 桥芯片的 PCI 卡，请查看相应机器类型的讨论。在某些情况下，你可能会遇到 PCI 卡不能正确处理 PCI 奇偶校验的问题。这可能导致 panic。可以使用以下 SRM 命令禁用 PCI 奇偶校验：

```sh
>>> SET PCI_PARITY OFF
```

这不是 FreeBSD 的问题，在 Alpha 硬件上运行的所有操作系统都需要这种变通方法。

如果你的系统（还）包含 EISA 扩展插槽，在安装 EISA 卡或升级控制台固件之后，你需要运行 EISA 配置实用程序（ECU）。

对于 Alpha CPU，你会发现多代产品。最初的 Alpha 设计是 21064。它采用名为 MOS4 的芯片工艺生产，在此工艺中制造的芯片昵称为 EV4。较新的 CPU 有 21164、21264 等。你会看到诸如 EV4S、EV45、EV5、EV56、EV6、EV67、EV68 之类的代号。双位数 EV 是略有改进的版本。例如，EV45 相比其所基于的 EV4，改进了 FPU 并拥有 16 kByte 片上独立 I & D 缓存。经验法则：“EV”后面的数字越大越可取（即：更快/更现代）。

对于内存，你至少需要 32 Mbyte。我曾在一台 16 Mbyte 系统上运行过 FreeBSD，但你不会享受那种体验。当我升级到 32 Mbyte 时，内核构建时间减半。请注意，SRM 控制台会从总系统内存中占用 2Mbyte（并保留它）。对于更严肃的工作，建议 64 Mbyte 或更多。

在内存方面：请密切注意你的机器使用的内存类型。各种机器的内存配置和要求非常不同。

最后一句：我期望上述内容对首次 Alpha 用户来说听起来有点令人生畏。不要太气馁。如果阅读本文档后仍有不清楚的地方，请随时提问。

### 2.3 系统特定信息

下面是 FreeBSD 运行的硬件概述。此列表肯定会增长，查看 **/sys/alpha/conf/GENERIC** 可能会有所启发。

Alpha 机器通常以其项目代号最为人知。已知代号在下面括号中列出。

#### 2.3.1 AXPpci33 (“NoName”)

NoName 是一块基于 21066 LCA（Low Cost Alpha）处理器的 baby-AT 主板。NoName 最初是为 OEM 用途设计的。LCA 芯片几乎包含了驱动 PCI 总线和内存子系统的所有逻辑。所有这些使得它成为一种低价设计。

由于有限的内存接口，在缓存未命中时系统不是特别快。只要你在片上缓存内操作，CPU 就可与 21064（第一代 Alpha）相媲美。这些主板现在应该非常便宜就能获得。它是完整的 64 位 CPU，只是不要指望速度上有奇迹。

特性：

- 166 MHz 的 21066 Alpha CPU 或 233MHz 的 21066A CPU。21068 CPU 也有可能，但更慢。
- 板载 Bcache / L2 缓存：0、256k 或 1 Mbyte（使用 DIL 芯片）
- PS/2 鼠标和键盘端口或 5pin DIN 键盘（2 种主板型号）
- 内存：
  - 总线宽度：64 位
  - PS/2 型 72 针 36 位 Fast Page Mode SIMM
  - 70ns 或更好
  - 成对安装
  - 4 个 SIMM 插槽
  - 使用 ECC
- 512kB 闪存 ROM 用于控制台代码。
- 2 个 16550A 串口
- 1 个并口
- 软驱接口
- 1 个嵌入式 IDE 接口
- 扩展：
  - 3 个 32 位 PCI 插槽（1 个与 ISA 共享）
  - 5 个 ISA 插槽（1 个与 PCI 共享）
- 使用 Symbios 53C810 芯片的嵌入式 Fast SCSI

NoName 的闪存 ROM 中可以有 SRM *或* ARC 控制台固件。闪存 ROM 不够大，无法同时容纳 ARC 和 SRM 并允许软件选择备用控制台代码。但你只需要 SRM。

NoName 的缓存是 15 或 20 ns DIL 芯片。对于 256 kByte 缓存，你可以检查你废弃的 486 主板。1 Mbyte 缓存的芯片遗憾的是比较罕见。建议至少获取 256kByte 缓存以获得性能。无缓存时它们真的很慢。

NoName 主板有一个 PC/AT 标准电源连接器。它还有一个 3.3 伏电源连接器。不需要急于购买新电源。3.3 伏仅在运行 3.3 伏 PCI 扩展板时才需要。这些相当罕见。

IDE 接口由 FreeBSD 支持，需要在内核配置文件中添加如下行：

```ini
device ata
```

ATA 接口使用 irq 14。

遗憾的是，SRM 控制台*无法从* IDE 磁盘启动。这意味着你必须使用 SCSI 磁盘作为启动设备。

NoName 在串行控制台方面有些顽固。它需要

```sh
>>> SET CONSOLE SERIAL
```

才会切换到串行控制台。仅拔掉键盘是不够的，不像在大多数其他 Alpha 型号上那样。要返回图形控制台，需要在串行控制台上输入：

```sh
>>> SET CONSOLE GRAPHICS
```

有报告说有时你需要按 **Ctrl**-**Alt**-**Del** 来吸引 SRM 的注意。我自己从未见过这种情况，但如果开机后你面对空白屏幕，值得一试。

确保你使用真正的 36 位 SIMM，并且只用 FPM（Fast Page Mode）DRAM。EDO DRAM 或假奇偶校验的 SIMM *无法工作*。主板使用 4 个额外位进行 ECC。33 位 FPM SIMM 出于同样原因也无法工作。

如果有选择，请获取 PS/2 变体的主板。除了给你一个鼠标端口作为额外功能外，如果你曾经想或需要运行 Tru64 Unix，它可以直接支持。“DIN 插头”变体对 FreeBSD 应该也能正常工作。

[OEM 手册](ftp://ftp.digital.com/pub/DEC/axppci/design_guide.ps) 是推荐阅读材料。

NoName 内核的内核配置文件必须包含：

```ini
options    DEC_AXPPCI_33
cpu EV4
```

#### 2.3.2 Universal Desktop Box（UDB 或“Multia”）

> **注意**
>
> Multia 可以是基于 Intel 或 Alpha CPU 的。出于显而易见的原因，我们这里假设基于 Alpha 的型号。

Multia 是一台小型桌面机箱，旨在作为某种个人工作站。它们有相当多的变体，请仔细检查你得到的型号。

特性：

- 166 MHz 的 21066 Alpha CPU 或 233MHz 的 21066A CPU
- 板载 Bcache / L2 缓存：COAST 式 256 kByte 缓存模块；233MHz 型号有 512kByte 缓存；166MHz 型号有焊接的 256kB 缓存
- PS/2 鼠标和键盘端口
- 内存：
  - 总线宽度：64 位
  - PS/2 型 72 针 36 位 Fast Page Mode SIMM
  - 70ns 或更好
  - SIMM 成对安装
  - 4 个 SIMM 插槽
  - 使用 ECC
- 2 个 16550A 串口
- 1 个并口
- 软驱接口
- Intel 82378ZB PCI 到 ISA 桥
- 1 个嵌入式基于 21040 的 10Mbit 以太网，AUI 和 10base2 连接器
- 扩展：
  - 1 个 32 位 PCI 插槽
  - 2 个 PCMCIA 插槽
- 板载 Crystal CS4231 或 AD1848 声音芯片
- 使用 PCI 转接卡上的 Symbios 53C810[A] 芯片的嵌入式 Fast SCSI

Multia 有足够的闪存 ROM 同时存储 SRM 和 ARC 代码，并允许软件选择其中之一。

嵌入式 TGA 视频适配器目前*不可*用作 FreeBSD 控制台。你需要使用串行控制台。

Multia 只有一个 32 位 PCI 插槽用于扩展，且只适合小型 PCI 卡。通过牺牲 PCI 插槽空间，你可以安装 3.5 英寸硬盘。安装配件可能随你的 Multia 一起提供。由于电源功率有限和系统机箱极其勉强的散热，添加 3.5 英寸磁盘*不*是推荐的升级方式。

Multia 还有 2 个 PCMCIA 扩展插槽。FreeBSD 目前不支持这些。

CPU 可能是插座的也可能不是，在考虑 CPU 升级方案之前请先检查。低端 Multia 有焊接的 CPU。

Multia 有 2 个串口，但通过单个 25 针 sub-D 连接器将两者连接到外部。Multia FAQ 解释了如何制作你自己的 Y 型电缆以允许两个端口同时使用。

虽然 Multia SRM 支持从软盘启动，但这可能有问题。典型的错误如下：

```sh
*** Soft Error - Error #10 - FDC: Data overrun or underrun
```

这不是 FreeBSD 的问题，而是 SRM 的问题。安装 FreeBSD 的最佳变通方法是从 SCSI CDROM 启动。

有报告说有时你需要按 **Ctrl**-**Alt**-**Del** 来吸引 SRM 的注意。我自己从未见过这种情况，但当你开机后面对空白屏幕时值得一试。

使用 `pcm(4)` 驱动程序以及内核配置文件中针对 Crystal CS4231 芯片的如下行，声音工作正常：

```ini
device pcm
```

声音设备位于端口 0x530，使用 irq 9 和 drq 3。你还需要在 **device.hints** 文件中指定 flags 0x15。

我还没有成功让我的带 AD1848 的 Multia 播放任何声音。

在验证回放时，我被提醒了 166MHz CPU 缺乏算力。MP3 只能使用 22kHz 降采样才能可接受地播放。

Multia 因中暑死亡而有些臭名昭著。非常紧凑的机箱并不真正允许冷却空气进入。请使用 Multia 的垂直支架，不要水平放置（“披萨方式”）。用能推动更多空气的风扇替换原来的风扇确实值得推荐。你还可以切断风扇速度传感器的一根线。一旦切断，风扇以（响亮的）全速运行。当心高功耗的 PCI 卡。如果你的系统已经死了，你可能需要查看 [NetBSD 网站](http://www.netbsd.org/) 上的 Multia-Heat-Death 页面以获取恢复帮助。

Intel 82378ZB PCI 到 ISA 桥使得 IDE 磁盘可以使用。这需要在内核配置文件中添加如下行：

```ini
device ata
```

ATA 接口使用 IRQ 14。

IDE 连接器引脚间距是为 2.5 英寸笔记本磁盘设计的。3.5 英寸 IDE 磁盘无论如何也放不进机箱。至少在不牺牲你唯一的 PCI 插槽的情况下。遗憾的是，SRM 控制台不知道如何从 IDE 磁盘启动。你需要使用 SCSI 磁盘作为启动磁盘。

如果你想更换内部硬盘：从 PCI 转接板到 2.5 英寸硬盘的内部排线比标准 SCSI 排线间距更细。否则它无法安装在 2.5 英寸驱动器上。也有转接卡带有标准间距 SCSI 电缆连接的，可以适合普通 SCSI 磁盘。

再次建议不要试图在内部塞入替换硬盘。使用外部 SCSI 连接器并将磁盘放在外部机箱中。Multia 本身已经够热了。在大多数情况下你会有外部高密度 50 针 SCSI 连接器，但一些 Multia 型号出厂时没有磁盘，可能缺少该连接器。购买前需要检查。

Multia 内核的内核配置文件必须包含：

```ini
options    DEC_AXPPCI_33
cpu EV4
```

关于 Multia 的推荐阅读可在 <http://www.netbsd.org/Ports/alpha/multiafaq.html> 或 <http://www.brouhaha.com/~eric/computers/udb.html> 找到。

#### 2.3.3 Personal Workstation (“Miata”)

Miata 是一台旨在放在桌下的小型塔式机器。有多种 Miata 变体。最初的 Miata 是 MX5 型号。由于它受到一些硬件设计缺陷的困扰，进行了重新设计，产生了 MiataGL。遗憾的是，从机箱外部不容易一眼区分这些变体。一个简单的检查是看机器背面是否有两个 USB 连接器。如果有，就是 MiataGL。MX5 型号在二手系统市场上往往更常见。

系统命名看起来像“Personal Workstation 433a”。Personal Workstation 有点拗口，通常缩写为 PWS。这意味着它有 433 MHz CPU，并且最初是作为 WinNT 工作站（尾部的“a”）。从第一天起就指定运行 Tru64 Unix 或 OpenVMS 的系统会有“433au”。WinNT-Miata 可能预装了 IDE CDROM 驱动器。因此，系统通常命名为 PWS[433,500,600]a[u]。

还有一种由 Kryotech 提供特殊 CPU 冷却系统的 Miata 型号。Kryotech 有特殊的冷却系统，housed 在不同的机箱中。

特性：

- 433、500 或 600MHz 的 21164A EV56 Alpha CPU
- 21174 核心逻辑（“Pyxis”）芯片
- 板载 Bcache / L3 缓存：0、2 或 4 Mbytes（使用缓存模块）
- 内存：
  - 总线宽度：128 位宽，ECC 保护
  - 无缓冲 72 位宽 SDRAM DIMM，成对安装
  - 6 个 DIMM 插槽
  - 最大内存 1.5 GBytes
- 板载快速以太网：
  - MX5 使用 21142 或 21143 以太网芯片，取决于 PCI 转接卡的版本
  - MiataGL 有 21143 芯片
  - 接口卡可以是 10/100 Mbit UTP，或 10 Mbit UTP/BNC
- 2 个板载 [E]IDE 磁盘接口，基于 CMD646（MX5）或 Cypress 82C693（MiataGL）
- 1 个 Ultra-Wide SCSI Qlogic 1040 [仅 MiataGL]
- 2 个 64 位 PCI 插槽
- 3 个 32 位 PCI 插槽（在 DEC PCI-PCI 桥芯片后面）
- 3 个 ISA 插槽（与 32 位 PCI 插槽物理共享，通过 Intel 82378IB PCI 到 ISA 桥芯片）
- 2 个 16550A 串口
- 1 个并口
- PS/2 键盘和鼠标端口
- USB 接口 [仅 MiataGL]
- 基于 ESS1888 芯片的嵌入式声卡

Miata 逻辑分为两块印刷电路板。机器底部的下板有 PCI 和 ISA 插槽以及声音芯片等。上板有 CPU、Pyxis 芯片、内存等。请注意，MX5 和 MiataGL 使用不同的 PCI 转接板。这意味着你不能只升级到 MiataGL CPU 板（带有更新的 Pyxis 芯片），还需要不同的转接板。显然，MX5 转接板配 MiataGL CPU 板可以工作，但绝对不是支持或测试过的配置。其他所有部分（机箱、接线等）MX5 和 MiataGL 完全相同。

当 DMA 通过 2 个 64 位 PCI 插槽跨越页面边界时，MX5 有问题。32 位插槽没有这个问题，因为 PCI-PCI 桥芯片不允许有问题的传输。SRM 代码知道这个问题，如果在 64 位插槽中有它不知道的 PCI 卡，会拒绝启动系统。SRM“已知良好”的卡可以在 64 位插槽中使用。

如果你想愚弄 SRM，可以在 SRM 提示符下输入 `set pci_device_override`。只是如果你的数据神秘地被损坏时不要抱怨。

完整的命令是：

```sh
>>> SET PCI_DEVICE_OVERRIDE <vendor_id><device_id>
```

例如：

```sh
>>> SET PCI_DEVICE_OVERRIDE 88c15333
```

最激进的方法是使用：

```sh
>>> SET PCI_DEVICE_OVERRIDE -1
```

这完全禁用 PCI ID 检查，因此你可以安装任何 PCI 卡而无需检查其 ID。为此你需要相当新的 SRM 版本。

> **重要**
>
> 此操作风险自负。

FreeBSD 内核在发现有缺陷的 Pyxis 芯片时会报告：

```sh
Sep 16 18:39:43 miata /kernel: cia0: Pyxis, pass 1
Sep 16 18:39:43 miata /kernel: cia0: extended capabilities: 1<BWEN>
Sep 16 18:39:43 miata /kernel: cia0: WARNING: Pyxis pass 1 DMA bug; no bets...
```

MiataGL 探测为：

```sh
Jan  3 12:22:32 miata /kernel: cia0: Pyxis, pass 1
Jan  3 12:22:32 miata /kernel: cia0: extended capabilities: 1<BWEN>
Jan  3 12:22:32 miata /kernel: pcib0: <2117x PCI host bus adapter> on cia0
```

MiataGL 没有 MX5 的 DMA 问题。使 MX5 SRM 在 64 位插槽中安装时窒息的 PCI 卡，MiataGL SRM 可毫无问题地接受。

MX5 的最新主板修订版包含该 bug 的硬件变通方法。SRM 不知道此 ECO，仍会像以前一样抱怨未知卡。FreeBSD 内核也是如此。

Miata SRM 可以从 IDE CDROM 驱动器启动。IDE 硬盘启动已知对 MiataGL 和 MX5 磁盘都能工作，因此你可以从 IDE 磁盘根目录运行 FreeBSD。假设有合适的驱动器，MX5 上的速度约为 14 Mbytes/sec。Miata 的 CMD646 芯片支持高达 WDMA2 模式，因为该芯片太不稳定无法用于 UDMA。

Miata MX5 通常使用基于 Qlogic 1040 的 SCSI 适配器。这些可由 SRM 控制台启动。请注意，Adaptec 卡*不可*由 Miata SRM 控制台启动。

MiataGL 在 PCI 转接卡上有比某些 MX5 转接卡版本更快的 PCI-PCI 桥芯片。一些 MX5 转接卡有与 MiataGL *相同*的芯片。总而言之有相当多的变体。

并非所有 VGA 卡都能在 PCI-PCI 桥后面工作。这表现为完全没有视频。变通方法是将 VGA 卡放在桥“前面”，即 64 位 PCI 插槽之一。使用 64 位插槽的图形性能通常要好得多。

MX5 和 MiataGL 都有板载声音芯片 ESS1888。它模拟 SoundBlaster，可以通过在内核配置文件中添加以下内容来启用：

```ini
device pcm
device  sbc
```

如果你的 Miata 安装了可选缓存板，请确保其牢固就位。观察到略有松动的缓存会导致奇怪的崩溃（显然不奇怪，但在故障排除时可能不那么明显）。缓存模块在 MX5 和 MiataGL 之间是通用的。

安装 2Mb 缓存模块可实现 10-15% 的速度提升（基于 buildworld 经过时间），但 64 位 PCI 卡的 PCI DMA 读取带宽会*下降*。在 64 位 Myrinet 卡上的基准测试显示从 149 Mbytes/sec 下降到 115 Mbytes/sec。在使用 64 位 PCI 适配器进行真正高速操作时需要记住这一点。

虽然硬件允许你安装高达 1.5Gbyte 的内存，但 FreeBSD 限制为 1Gbyte，因为 DMA 代码不能正确处理 1Gbyte 以上的内存。

升级到更快的 CPU 非常简单，更换 CPU 芯片并将时钟倍频 dipswitch 设置为新 CPU 的速度即可。

如果你在停止 FreeBSD 后遇到如下 SRM 错误

```sh
ERROR: scancode 0xa3 not supported on PCXAL
```

你应该将 SRM 固件更新到 V7.2-1 或更高版本。此 SRM 版本首先在固件更新 CD V5.7 上提供，或在 <http://www.compaq.com/> 上提供。此 SRM 问题在 Miata MX5 和 Miata GL 上均已修复。

USB 由 FreeBSD 4.1 及以后版本支持。

拆卸机器前请断开电源线，软电源开关在机器关闭时*甚至*保持部分逻辑供电。

Miata 内核的内核配置文件必须包含：

```ini
options    DEC_ST550
cpu EV5
```

#### 2.3.4 Evaluation Board 64 family

在试图普及 Alpha CPU 的过程中，DEC 生产了多款所谓的评估板。该系列的成员有 EB64、EB64+、AlphaPC64（代号“Cabriolet”）。该系列的非 DEC 成员是 Aspen Alpine。EB64 系列评估板具有以下特性集：

- 150 至 275 MHz 的 21064 或 21064A CPU
- 内存：
  - 内存总线宽度：128 位
  - PS/2 型 72 针 33 位 Fast Page Mode SIMM
  - 70ns 或更好
  - 以 4 个一组安装
  - 8 个 SIMM 插槽
  - 使用奇偶校验内存
- Bcache / L2 缓存：0、512 kByte、1 Mbyte 或 2 Mbytes
- 21072（“APECS”）芯片组
- Intel 82378ZB PCI 到 ISA 桥芯片（“Saturn”）
- 双 16550A 串口
- 并行打印机端口
- Symbios 53C810 Fast-SCSI（不在 AlphaPC64 上）
- IDE 接口（仅在 AlphaPC64 上）
- 嵌入式 10 Mbit 以太网（不在 AlphaPC64 上）
- 2 个 PCI 插槽（AlphaPC64 有 4 个插槽）
- 3 个 ISA 插槽

Aspen Alpine 略有不同，但与 EB64+ 足够接近，可以运行 EB64+ SRM EPROM（我的可以..）。Aspen Alpine 没有嵌入式以太网，有 3 个而不是 2 个 PCI 插槽。它带有 2 Mbytes 的缓存已焊接在主板上。它有跳线可选择使用 60、70 或 80ns SIMM 速度。

36 位 SIMM 工作正常，3 位简单地保持未使用。请注意系统使用 Fast Page Mode 内存，不是 EDO 内存。

EB64+ SRM 控制台代码 housed 在 UV 可擦除 EPROM 中。EB64+ 没有简单的闪存 SRM 升级。EB64+ 可用的最新 SRM 版本相当古老。

EB64+ SRM 可以启动 53C810 和 Qlogic1040 SCSI 适配器。Qlogic 的陷阱是 SRM 下载到 Qlogic 芯片的固件非常旧。EB64+ SRM 没有可用更新。所以你也受困于旧的 Qlogic 位。当我想在 Alpine 上使用 Qlogic 的 Ultra-SCSI 驱动器时遇到了相当多的问题。FreeBSD 内核可以编译包含更新的 Qlogic 固件修订版。这不是默认的，因为它给内核增加了数百 kBytes 的臃肿。在 FreeBSD 4.1 及以后版本中，isp 固件包含在内核可加载模块中。所有这些可能意味着你需要使用非 Qlogic 适配器来启动。

AlphaPC64 板通常带有 ARC 控制台固件。SRM 控制台代码可以从软盘加载到闪存 ROM 中。

AlphaPC64 的 IDE 接口不能从 SRM 控制台启动。启用它需要在内核配置文件中添加如下行：

```ini
device ata
```

ATA 接口使用 irq 14。

请注意这些板需要一个为 CPU 提供 3.3 伏的电源。

对于 EB64 系列机器，内核配置文件必须包含：

```ini
options    DEC_EB64PLUS
cpu EV4
```

#### 2.3.5 Evaluation Board 164（“EB164, PC164, PC164LX, PC164SX”）family

EB164 是基于 21164A CPU 的较新设计评估板。此设计已用于“衍生”多个变体，其中一些被 OEM 制造商/装配商使用。Samsung 做了自己的 PC164LX，只有 32 位 PCI，而 Digital 变体有 64 位 PCI。

- 21164A，多种速度变体 [EB164, PC164, PC164LX]
- 21164PC [仅在 PC164SX 上]
- 21171 (Alcor) 芯片组 [EB164]

  21172 (Alcor2) 芯片组 [PC164]

  21174 (Pyxis) 芯片 [164LX, 164SX]
- Bcache / L3 缓存：EB164 使用特殊缓存 SIMM
- 内存总线：128 位 / 256 位
- 内存：
  - PS/2 型 SIMM，以 4 或 8 个一组
  - 36 位，Fast Page Mode，使用 ECC [EB164 / PC164]
  - SDRAM DIMM，以 2 个一组，可使用非 ECC 或 ECC [PC164SX]
  - SDRAM DIMM，以 2 个一组，使用 ECC [PC164LX]
- 2 个 16550A 串口
- PS/2 型键盘和鼠标
- 软驱控制器
- 并口
- 32 位 PCI
- 64 位 PCI [某些型号]
- 通过 Intel 82378ZB PCI 到 ISA 桥芯片的 ISA 插槽

使用 8 个 SIMM 实现 256 位宽内存可以比 4 个 SIMM/128 位宽内存产生有趣的加速。显然所有 8 个 SIMM 必须是相同类型才能使其工作。系统必须明确设置为使用 8 个 SIMM 内存排列。你必须有 8 个 SIMM，4 个 SIMM 分布在 2 个 bank 上无法工作。对于 AlphaPC164，你最多可以有 1Gbyte 的 RAM，使用 8 个 128Mbyte SIMM。手册指示最大为 512 Mbyte。

SRM 可以从 Qlogic 10xx 板或 Symbios 53C810[A] 启动。较新的 Symbios 810 修订版如 Symbios 810AE 不被 PC164 上的 SRM 识别。PC164 SRM 似乎不识别基于 Symbios 53C895 的主机适配器（用 Tekram DC-390U2W 测试过）。另一方面，有报告称某无品牌 Symbios 53C985 板可以工作。像 Tekram DC-390F（基于 Symbios875）这样的卡已确认在 PC164 上工作正常。遗憾的是这似乎取决于芯片/板的实际版本。

Symbios 53C825[a] 也可作为启动适配器。Diamond FirePort 虽然基于 Symbios 芯片，但不可由 PC164SX SRM 启动。据报道 PC164SX 可以用基于 Symbios825、Symbios875、Symbios895 和 Symbios876 的卡正常启动。此外，据报道 Adaptec 2940U 和 2940UW 可用于启动（在 SRM V5.7-1 上验证过）。Adaptec 2930U2 和 2940U2[W] 不工作。

带有 SRM 固件版本 5.8 或更高版本的 164LX 和 164SX 可以从 Adaptec 2940 系列适配器启动。用 Adaptec 3940UW 测试表明那个不受支持。164SX SRM 识别 Intel 8255x 以太网卡，显示为 eia。使用这样的 NIC 允许网络启动。

总结：这系列机器在 SCSI 适配器方面“受赐”了具有挑战性的兼容性。

在 164SX 上你最多可以有 1 Gbyte 的 RAM。据报道 4 个常规（PC100 或 PC133）256MB DIMM 工作正常。512MB DIMM 是否也能工作目前未知。你可以使用 ECC 或非 ECC DIMM。非 ECC 的与 PC 中常见的一样。遗憾的是 164SX 对它喜欢哪些 DIMM 相当挑剔，所以准备好测试和实验。

PCI 桥芯片有时不被 164SX 欣赏，它们会导致 SRM 错误和内核 panic。这似乎取决于卡是否被 SRM 控制台识别并因此正确初始化。164SX 的板载 IDE 接口相当慢，Promise 卡可提供 3-4 倍的速度提升。

在 PC164 上，SRM 有时似乎会丢失其变量设置。“对于 PC164，当前迷信说法是，为避免丢失设置，你首先要降级到 SRM 4.x，然后升级到 5.x。”观察到的一个示例错误是：

```sh
ERROR: ISA table corrupt!
```

降级到 SRM4.9，然后

```sh
>>> ISACFG -INIT
```

接着

```sh
>>> INIT
```

使问题消失。一些 PC164 所有者报告他们从未见过此问题。

在 PC164SX 上，AlphaBIOS 允许你选择在下一次开机时使用 SRM 作为控制台。这个选择似乎没有任何效果。换句话说，无论你选择什么都会得到 AlphaBIOS。解决方法是用 PC164SX 的 SRM 代码重新刷新控制台 ROM。这将覆盖 AlphaBIOS 并让你得到你想要的 SRM 控制台。SRM 代码可在 Compaq 网站上找到。

164LX 的闪存 ROM 中可以有 SRM 控制台代码或 AlphaBIOS 代码，因为闪存 ROM 太小，无法同时容纳两者。

PC164 可以从 IDE 磁盘启动，假设你的 SRM 版本足够新。

EB164 需要一个提供 3.3 伏的电源。PC164 不实现 ATX 电源切换所需的 PS_ON 信号。一个简单的开关将此信号拉到地允许你运行标准 ATX 电源。

对于 EB164 类机器，内核配置文件必须包含：

```ini
options         DEC_EB164
cpu             EV5
```

#### 2.3.6 AlphaStation 200（“Mustang”）和 400（“Avanti”）系列

Digital AlphaStation 200 和 400 系列系统是早期低端基于 PCI 的工作站。200 和 250 系列是桌面机箱，400 系列是桌面旁迷你塔。

特性：

- 166 至 333 MHz 的 21064 或 21064A CPU
- DECchip 21071-AA 核心逻辑芯片组
- Bcache / L2 缓存：512 Kbytes（200 和 400 系列）或 2048KBytes（250 系列）
- 内存：
  - 64 位总线宽度
  - 8 至 384 MBytes RAM
  - 70 ns 或更好的 Fast Page DRAM
  - 以三对安装（200 和 400 系列）
  - 以两个四元组安装，即四个一组（250 系列）
  - 内存子系统使用奇偶校验
- PS/2 键盘和鼠标端口
- 两个 16550 串口
- 并口
- 软驱接口
- 32 位 PCI 扩展插槽（AS400 系列 3 个，AS200 和 250 系列 2 个）
- ISA 扩展插槽（AS400 系列 4 个，AS200 和 250 系列 2 个）（一些 ISA/PCI 插槽物理共享）
- 嵌入式基于 21040 的以太网（200 和 250 系列）
- 嵌入式 Symbios 53c810 Fast SCSI-2 芯片
- Intel 82378IB（“Saturn”）PCI-ISA 桥芯片
- 图形为嵌入式 TGA 或 PCI VGA（取决于型号）
- 16 位声音（200 和 250 系列）

系统使用奇偶校验内存 SIMM，但这些不需要 36 位宽 SIMM。33 位宽 SIMM 即可，36 位 SIMM 也可接受。EDO 或 32 位 SIMM 不工作。支持 4、8、16、32 和 64 Mbyte SIMM。

据报道 AS200 和 AS250 声音硬件工作正常，假设你的内核配置文件中有以下行：

```ini
device pcm
```

声音设备使用端口 0x530、IRQ 10 和 drq 0。你还需要在 **device.hints** 文件中指定 flags 0x10011。

AlphaStation 200 和 250 系列有自动 SCSI 终端器。这意味着一旦你将电缆插入外部 SCSI 连接器，系统的内部终端器就会禁用。这也意味着你不应留下未终止的电缆插在机器上。

AlphaStation 400 系列有一个控制终端的 SRM 变量。如果你连接了外部 SCSI 设备，你必须使用以下命令设置此 SRM 变量

```sh
>>> SET CONTROL_SCSI_TERM EXTERNAL
```

如果只有内部 SCSI 设备，使用：

```sh
>>> SET CONTROL_SCSI_TERM INTERNAL
```

对于 AlphaStation-[24][05]00 机器，内核配置文件必须包含：

```ini
options    DEC_2100_A50
cpu EV4
```

#### 2.3.7 AlphaStation 500 和 600（EV5 的“Alcor”和“Maverick”，EV56 的“Bret”）

AS500 和 600 曾是高端基于 EV5 / PCI 的工作站。基于 EV6 的机器已在此期间取代了它们的前沿地位。AS500 是深蓝色机箱（TopGun blue）的桌面机，AS600 是坚固的桌面旁机箱。AS600 有一个漂亮的 LCD 面板用于观察 SRM 启动的早期阶段。

特性：

- 21164 EV5 CPU，AS500 为 266、300、333、366、400、433、466 或 500 MHz，AS600 为 266、300 或 333 MHz
- 21171 (Alcor) 或 21172 (Alcor2) 核心逻辑芯片组
- 缓存：
  - 2 或 4 Mb L3 / Bcache（AS600 266 MHz）
  - 4 Mb L3 / Bcache（AS600 300 MHz）
  - 2 或 8 Mb L3 / Bcache（8 Mb 仅 500 MHz 版本）
  - 2 至 16 Mb L3 / Bcache（AS600；3 个缓存 SIMM 插槽）
- 内存总线宽度：256 位
- AS500 内存：
  - 业界标准 72 位宽缓冲 Fast Page Mode DIMM
  - 8 个 DIMM 插槽

机器使用专用内存板。这些板位于与 CPU 板共享的 128 位 C-bus 上。DemiSable 支持高达 1GB，Sable 高达 2GB。其中一个内存总线插槽可以放 CPU 或内存卡。4 CPU 机器最多可以有 2 块内存板。

一些内存板型号容纳 SIMM。这些称为 SIMM 载体。也有焊接内存芯片而非 SIMM 的内存模块。这些称为“flat memory modules”。

SIMM 板以 8 个 72 针 36 位 FPM 内存为一组使用，70ns 或更快。支持的 SIMM 类型为 1M x36 bit（4 Mbyte）、2M x36bit（8 Mbyte）和 4M x36 bit（16 Mbyte）。每块内存板可容纳 4 个 bank 的 SIMM。单块内存板上不能混合 SIMM 尺寸。第一块内存模块必须先填满 SIMM 才能开始填充下一块内存模块。请注意插槽之间的间距不大，所以确保你的 SIMM 物理上合适（在购买它们之前..）

Lynx 和 Sable 在串行控制台方面都有些顽固。它们需要

```sh
>>> SET CONSOLE SERIAL
```

才会切换到串行控制台。仅拔掉键盘是不够的，不像在许多其他 Alpha 型号上那样。要返回图形控制台，需要

```sh
>>> SET CONSOLE GRAPHICS
```

在串行控制台上。在 Lynx 上，将 VGA 卡保持在主 PCI 插槽之一。EISA VGA 卡对插槽不敏感。

机器配备小型 OCP（操作员控制面板）LCD 屏幕。在此屏幕上显示系统初始化期间的自检消息。你可以使用 SRM 在上面放置你自己的小文本：

```sh
>>> SET OCP_TEXT "FreeBSD"

```

SRM 命令

```sh
>>> SHOW FRU
```

生成配置概览，包括模块序列号、硬件修订版和错误日志计数。

Sable、DemiSable 和 Lynx 都有基于 Symbios 810 的板载 Fast SCSI。检查它是否设置为 Fast SCSI 速度

```sh
>>> SHOW PKA0_FAST
```

当设置为 1 时，它正在协商 Fast 速度。

```sh
>>> SET PKA0_FAST 1
```

启用 Fast SCSI 速度。

AS2100[A] 配备 StorageWorks 7 插槽 SCSI 笼。机柜内可添加第二个笼。AS2000 有单个 7 插槽 SCSI 笼，无法扩展额外的笼。请注意这些笼中的插槽位置映射到 SCSI ID 的方式与标准 StorageWorks 架不同。使用单总线配置时，从上到下的插槽 ID 为 0、4、1、5、2、6、3。

笼也可以设置为提供两条独立 SCSI 总线。这用于嵌入式 RAID 控制器如 KZPSC（Mylex DAC960）。分体总线的插槽 ID 分配，从上到下为：0A、0B、1A、1B、2A、2B、3A、3B。其中 A 和 B 表示一条 SCSI 总线。在单总线配置中，SCSI 笼背面的终端器模块位于顶部。跳线模块位于底部。对于分体总线操作，这两个模块反转。终端器可通过注意终端器上的芯片来区分。跳线上没有任何有源组件。

DemiSable 有 7 个 EISA 插槽和 3 个 PCI 插槽。Sable 有 8 个 EISA 和 3 个 PCI 插槽。Lynx 较新，有 8 个 PCI 和 3 个 EISA 插槽。Lynx PCI 插槽以 4 个为一组分组。最靠近 CPU/内存插槽的 4 个 PCI 插槽是主插槽，因此逻辑上在 PCI 桥芯片之前。请注意与预期相反，主 PCI 插槽是编号最高的（PCI4 - PCI7）。

在 EISA 插槽中添加/更改扩展卡或升级控制台固件后，请确保运行 EISA 配置实用程序（从软盘）。这通过插入 ECU 软盘并输入

```sh
>>> RUNECU
```

完成

> **注意**
>
> EISA 插槽目前不受支持，但 Compaq Qvision EISA VGA 适配器被视为 ISA 设备。因此它作为控制台工作正常。

曾计划用于 C-bus 的特殊扩展 I/O 模块。它们是否见过天日未知。无论如何，FreeBSD 从未用 ExtIO 模块验证过。

机器可配备冗余电源。请注意机箱配备互锁开关，当机箱打开时断电。系统的冷却风扇是速度控制的。当机器有超过 2 个 CPU 和超过 1 块内存板时，双电源是强制性的。

内核配置文件必须包含：

```ini
options    DEC_2100_A500
cpu EV4 #取决于安装的 CPU 型号
cpu EV5 #取决于安装的 CPU 型号
```

#### 2.3.13 AlphaServer 4x00（“Rawhide”）

AlphaServer 4x00 机器旨在作为小型企业服务器。预计有 30 英寸高的立式机柜或同样的系统箱在 19 英寸机架中。Rawhide 是多 CPU 机器，单台机器最多可有 4 个 CPU。基本磁盘存储位于立柱底部的一个或两个 StorageWorks 架中。面向 NT 市场的 Rawhide 命名为 DIGITAL Server 7300（5/400 CPU）、DIGITAL Server 7305（5/533 CPU）。部件号尾部的 R 表示机架式变体。

特性：

- 266、300、333 MHz 的 21164 EV5 CPU 或 400、466、533、600 Mhz 的 21164A EV56 CPU
- 缓存：每个 CPU 4 Mbytes。EV5 300 MHz 也有无缓存版本。EV56 600Mhz 有 8 Mbytes
- 内存总线：128 位带 ECC
- 嵌入式软驱控制器
- 2 个串口
- 1 个并口
- PS/2 型键盘和鼠标端口

Rawhide 可配备多种 CPU 模块。CPU 模块有带和不带外部缓存的版本。在所有情况下，安装的 CPU 模块必须始终为相同速度。NT-only 和完整 Tru64/VMS CPU 的混合工作正常。然而这会导致系统向操作系统报告自己为 Digital Server 730x（即 NT-only 变体）。FreeBSD 不在乎，但这样的系统不允许 Tru64 或 VMS 运行。

Rawhide 最多使用 8 个 RAM 模块。这些模块成对使用并向总线提供 72 位（包括 ECC 位）。内存可以是 EDO RAM 或同步 DRAM。完全填充的 AS4100 有 4 对内存模块。AS4000 型号限制为 2 对内存模块。如果有选择，使用 SDRAM 以获得最佳性能。最高容量的内存板必须在标记为 MEM0L 和 MEM0H 的内存插槽中。允许混合内存板尺寸。EDO 和 SDRAM 的混合也可工作（假设你不会在单个模块对中混合 EDO 和 SDRAM）。EDO 和 SDRAM 的混合导致*整个*内存子系统以较慢的 EDO 时序运行。

Rawhide 有嵌入式 Symbios 810 芯片，为你提供窄 Fast-SCSI 总线。通常只有 SCSI CDROM 由该接口驱动。

Rawhide 有 8 个 64 位 PCI / 3 个 EISA 插槽扩展背板（称为“Saddle”模块）。有 2 条独立 PCI 总线，PCI0 和 PCI1。PCI0 有 1 个专用 PCI 插槽和（共享）3 个 PCI/EISA 插槽。PCI0 还有 PCI/EISA 桥驱动串口和并口、键盘/鼠标等。PCI1 有 4 个 PCI 插槽和 Symbios 810 SCSI 芯片。VGA 控制台卡必须安装在连接到 PCI0 的插槽中。

当前 FreeBSD 实现在处理 PCI 桥方面有问题。目前有有限的修复措施，允许单级、单设备 PCI 桥。该修复允许使用 Digital 提供的位于 21054 PCI 桥芯片后面的 Qlogic SCSI 卡。

> **注意**
>
> EISA 插槽目前不受支持，但 Compaq Qvision EISA VGA 适配器被视为 ISA 设备。因此它作为控制台工作。如果你在机器中使用 EISA 选项，你必须从软盘运行 EISA 配置实用程序（ECU）。帮自己一个忙，使用 Tru64/OpenVMS ECU，而不是 WindowsNT ECU。

Rawhide 采用基于 I2C 的电源控制器系统。如果你想确保系统所有电源已移除，请从系统中拔掉所有主电源线。

Rawhide 具有 RCM 功能，这意味着你可以远程开关机、重置等。另请参阅本文档 DS10 部分中 RMC 的描述。RCM 与 RMC 不是打字错误，我查阅的各种文档交替使用这两个首字母缩写。请注意如果你想要远程开关机功能，你需要连接一个小型 DC 适配器到机器以给 RCM 逻辑供电。你需要在键盘连接器旁边的小型入口处提供 9-12V DC。

内核配置文件必须包含：

```ini
options    DEC_KN300
cpu EV5
```

#### 2.3.14 AlphaServer 1200（“Tincup”）和 AlphaStation 1200（“DaVinci”）

AlphaServer 1200 机器是 AlphaServer 1000A 的继任者。它使用与 1000A 相同的机箱，但逻辑基于 AlphaServer 4000 设计。这些是多 CPU 机器，单台机器最多可有 2 个 CPU。基本磁盘存储位于 StorageWorks 架中。面向 NT 市场的 AS1200 命名为 DIGITAL Server 5300（5/400 CPU）和 DIGITAL Server 5305（5/533 CPU）。

特性：

- 400 或 533 Mhz 的 21164A EV56 CPU
- 缓存：每个 CPU 4 Mbytes
- 内存总线：128 位带 ECC，两块内存子卡上的 DIMM 内存
- 嵌入式软驱控制器
- 2 个串口
- 1 个并口
- PS/2 型键盘和鼠标端口

AS1200 使用 2 块内存子卡。每块卡上有 8 个 DIMM 插槽。DIMM 必须成对安装。最大内存尺寸为 4 GBytes。插槽必须按顺序填充，如果使用不同尺寸的 DIMM，插槽 0 必须包含最大尺寸的 DIMM。AS1200 对 DIMM 采用固定起始地址，每对 DIMM 从 512 Mbyte 边界开始。这意味着如果使用小于 256 Mbyte 的 DIMM，系统的物理内存映射将包含“空洞”。支持的 DIMM 尺寸为 64 Mbytes 和 256 Mbytes。DIMM 基于 72 位 SDRAM，因为系统采用 ECC。

#### 2.3.15 AlphaServer 8200 和 8400（“TurboLaser”）

AlphaServer 8200 和 8400 机器是企业级服务器。预计有高 19 英寸机柜（8200）或宽 19 英寸机架（8400）。这是大型机，不是爱好者系统。TurboLaser 是多 CPU 机器，单台机器最多可有 12 个 CPU。TurboLaser 系统总线（TLSB）允许 AS8400 上 9 个节点和 AS8200 上 5 个节点。TLSB 为 256 位数据、40 位地址，允许 2.1 GBytes/sec。TLSB 上的节点可以是 CPU、内存或 I/O。TLSB 上最多支持 3 个 I/O 端口。

基本磁盘存储位于 StorageWorks 架中。AS8400 使用三相电源，AS8200 使用单相电源。

特性：

- 高达 467 MHz 的 21164 EV5/EV56 CPU 或高达 625 MHz 的 21264 EV67 CPU
- 每个 CPU 模块一个或两个 CPU
- 缓存：每个 CPU 4Mbytes B-cache
- 内存总线：256 位带 ECC
- 内存：插入 TLSB 的大内存模块，其上安装特殊 SIMM 模块。内存模块有不同尺寸，每块高达 4 GBytes。使用 ECC（每 64 位数据 8 位）。AS8400 最多 7 个内存模块，AS8200 最多 3 个模块。最大内存 28 GBytes。
- 扩展：3 个系统“I/O 端口”，每个允许多达 12 个 I/O 通道，每个 I/O 通道可连接到 XMI、Futurebus+ 或 PCI 箱

FreeBSD 支持（并已测试）TurboLaser 上高达 2 GBytes 的内存。在 TLSB 插槽被内存模块占用和被 CPU 模块占用之间需要权衡。例如，你可以同时拥有 28GBytes 内存但只有 2 个 CPU（1 个模块）。

FreeBSD 仅支持 PCI 扩展。XMI 或 Futurebus+（仅 AS8400 有）均不受支持。

I/O 端口模块命名为 KFTIA 或 KFTHA。I/O 端口模块提供所谓的“hoses”，连接到多达 4 条 PCI 总线（KFTHA）或 1 条 PCI 总线（KFTIA）。KFTIA 有嵌入式双 10baseT 以太网、单 FDDI、3 条 SCSI Fast Wide Differential SCSI 总线和一条 Fast Wide Single Ended SCSI 总线。FWSE SCSI 专用于 CDROM。

KFTHA 可通过其 4 条 hoses 中的每一条驱动一个 DWLPA 或 DWLPB 箱。DWLPx 容纳 12 插槽 32 位 PCI 背板。物理上 12 个插槽是 3 条 4 插槽总线，但对软件来说显示为单条 12 插槽 PCI 总线。完全扩展的 AS8x00 可以有 3（I/O 端口）乘以 4（hoses）乘以 12（PCI 插槽/DWLPx）= 144 个 PCI 插槽。每个 KFTHA 的最大带宽为 500 Mbytes/second。DWLPA 还可容纳 8 块 EISA 卡，2 个插槽仅 PCI，2 个插槽仅 EISA。12 个插槽中有 2 个始终被 I/O 和连接器模块占用。DWLPB 是首选的 I/O 箱。

为获得最佳性能，将高带宽（FibreChannel、千兆以太网）分布到多条 hoses 和/或多个 KFTHA/KFTIA 上。

目前，包含 PCI 桥的 PCI 扩展卡无法与 FreeBSD 一起使用。目前不要使用它们。

KFTIA 上的单端窄 SCSI 总线将显示为*第四条* SCSI 总线。KFTIA 的 3 条 fast-wide 差分 SCSI 总线先于它。

AS8x00 通常使用串行控制台运行。一些较新的机器可能有某种图形控制台，但 FreeBSD 仅在串行控制台上测试过。

对于串行控制台使用，要么更改 **/etc/ttys** 使其包含：

```ini
console "/usr/libexec/getty std.9600"   unknown   on secure
```

作为控制台条目，或添加

```ini
zs0     "/usr/libexec/getty std.9600"   unknown   on secure
```

对于 AlphaServer 8x00 机器，内核配置文件必须包含：

```ini
options    DEC_KN8AE   # Alpha 8200/8400 (Turbolaser)
cpu EV5
```

与预期相反，内核配置文件中没有定义 `cpu EV6`。`cpu EV5` 是强制性的，以使 `config(8)` 满意。

#### 2.3.16 Alpha Processor Inc. UP1000

UP1000 是一块基于 21264a CPU 的 ATX 主板，CPU 本身位于 Slot B 模块中。通常 housed 在 ATX 塔式机箱中。

特性：

- 600 或 700 MHz 的 21264a Alpha CPU，位于 Slot B 模块中（包括冷却风扇）
- 内存总线：到 L2 缓存 128 位，从 Slot B 到 AMD-751 64 位
- 板载 Bcache / L2 缓存：2MB（600Mhz）或 4MB（700Mhz）
- AMD AMD-751（“Irongate”）系统控制器芯片
- Acer Labs M1543C PCI-ISA 桥控制器 / super-IO 芯片
- PS/2 鼠标和键盘端口
- 内存：168 针 PC100 无缓冲 SDRAM DIMM，3 个 DIMM 插槽，支持 64、128 或 256 Mb DIMM
- 2 个 16550A 串口
- 1 个 ECP/EPP 并口
- 软驱接口
- 2 个嵌入式 Ultra DMA33 IDE 接口
- 2 个 USB 端口
- 扩展：
  - 4 个 32 位 PCI 插槽
  - 2 个 ISA 插槽
  - 1 个 AGP 插槽

Slot B 是一个盒状外壳，容纳 CPU 和缓存的子板。它有 2 个小型冷却风扇。嘈杂的风扇。

机器需要支持 ECC 的 DIMM，即 72 位的。这似乎没有在 UP1000 文档中记录。系统通过 SM 总线访问 DIMM 上的串行 EEPROM。请注意，如果只使用单个 DIMM，它必须安装在插槽 *2* 中。这有点反直觉。

根据制造商的说法，UP1000 需要 400W ATX 电源。从主板和 CPU 的功耗来看，这可能有点过于保守/悲观。但一如既往，你必须考虑你的扩展卡和外围设备。M1543C 芯片包含电源管理功能和温度监控（通过 I2C / SM 总线）。

你的 UP1000 可能默认只带有 AlphaBios。SRM 控制台固件可从 Alpha Processor Inc. 网站获取。目前有 beta 版本，在 FreeBSD 移植到 UP1000 期间已成功使用。

嵌入式 Ultra DMA EIDE 端口可由 SRM 控制台启动。

UP1000 SRM 可以从 Adaptec 294x 适配器启动。在高 I/O 负载条件下，使用 Adaptec 294x 观察到机器锁定。基于 Symbios 875 的卡使用 sym 驱动程序工作正常。基于 sym 驱动程序支持的 Symbios 芯片的其他卡很可能也能工作。

USB 接口被 SRM 控制台禁用，尚未用 FreeBSD 测试。

对于 UP1000，内核配置文件必须包含：

```ini
options    API_UP1000  # UP1000, UP1100 (Nautilus)
cpu EV5
```

- ALI M1543C Ultra DMA66 IDE 接口
- 嵌入式双 Intel 82559 10/100Mbit 以太网
- 嵌入式 Symbios 53C1000 Ultra160 SCSI 控制器
- 扩展：2 个 64 位 PCI 插槽（2/3 长度）

CS20 标配 SRM 控制台代码。SRM 位于 2Mbytes 闪存 ROM 中。

CS20 需要支持 ECC 的 DIMM。请注意它使用*缓冲* DIMM。

CS20 有基于 I2C 的内部监控系统，用于监控温度、风扇、电压等。I2C 还支持“wake on LAN”。

每个 PCI 插槽连接到 Tsunami 上其自己的独立 PCI 总线。

嵌入式 Ultra DMA EIDE 端口可由 SRM 控制台启动。

CS20 有一个嵌入式超薄 IDE CD 驱动器。前面板可访问的托架用于带 SCA 连接器的 1 英寸高 3.5 英寸 SCSI 硬盘驱动器。

请注意没有软盘驱动器（或添加一个的连接器）。

内核配置文件必须包含：

```ini
options    DEC_ST6600
cpu EV5
```

与预期相反，内核配置文件中没有定义 `cpu EV6`。`cpu EV5` 是强制性的，以使 `config(8)` 满意。

#### 2.3.19 Compaq AlphaServer ES40（“Clipper”）

ES40 是一个 SMP 系统，可以有 1 - 4 个 21264 Alpha CPU。在最大 32GB 内存的配置下，这些系统通常部署为重型数据库服务器，也用于 HPTC 计算农场环境。

特性：

- 500（EV6）、667（EV67）或 833 MHz（EV68）的 21264 Alpha CPU（最多 4 个 CPU）
- 内存总线：256 位宽
- 21272 核心逻辑芯片组
- PS/2 鼠标和键盘端口
- 内存：200 针 JEDEC 标准 SDRAM DIMM，最大 32 GBytes 内存
- 2 个 16550A 串口
- 1 个 ECP/EPP 并口
- ALI M1543C Ultra DMA66 IDE 接口
- 扩展：2 条 64 位 PCI 总线

ES40 标配 SRM 控制台代码。

ES40 配有 ATA CDROM 驱动器，但使用 SCSI 硬盘。通常的 Symbios 和 Qlogic 适配器可启动，KZPEA 即 Adaptec 39160 双通道 LVD U160 适配器也可启动。

内存分为 4 个内存阵列，每个阵列包含一组 4 个 SDRAM DIMM。每个 DIMM 宽 72 位，100MHz 速度变体。一个阵列可以包含 2 组，因此每个阵列最多 8 个 DIMM。DIMM 位于内存主板（MMB）上。有 2 种 MMB 型号，分别有 4 和 8 个 DIMM 插座。每块 MMB 提供到 CPU 的 256 位内存总线宽度的一半。鉴于内存配置的众多选项，建议查看系统文档以获取最佳内存配置。

根据型号变体，ES40 有 6 或 10 个 64 位 PCI 插槽。这基本上意味着相同的背板但安装的连接器更少。

ES40 有与 DS10 和 DS20 相同的 RMC 远程电源控制。请参阅本文档 DS10 部分中 RMC 的描述。ES40 的大多数变体有多个电源，允许 N+1 冗余。安装 CPU 卡时必须拔掉所有电源线，CPU 卡接收来自电源的待机电源。最大内存配置需要比默认数量更多的电源。

内核配置文件必须包含：

```ini
options   DEC_ST6600
cpu     EV5
```

与预期相反，内核配置文件中没有定义 `cpu EV6`。`cpu EV5` 是强制性的，以使 `config(8)` 满意。

### 2.4 支持的硬件概述

一句警告：FreeBSD 的装机量远不及 FreeBSD/Intel。这意味着市面上 PCI/ISA 扩展卡的巨大多样性在 alpha 上测试的机会比在 Intel 上少得多。这并不是说它们注定会失败，只是遇到从未测试过的东西的几率要高得多。**GENERIC** 包含已知在 Alpha 上工作的内容。

PCI 和 ISA 扩展总线得到完全支持。Turbo Channel 不在 **GENERIC** 中，支持有限（参见相关机器型号信息）。MCA 总线不受支持。EISA 总线不支持用于 EISA 扩展卡，因为缺少 EISA 支持代码。EISA 插槽中的 ISA 卡据说可以工作。Compaq Qvision EISA VGA 卡以 ISA 模式驱动，作为控制台工作正常。

支持 1.44 Mbyte 和 1.2 Mbyte 软驱。Alpha 机器中有时发现的 2.88 Mbyte 驱动器支持到 1.44Mbyte。

ATA 和 ATAPI（IDE）设备通过 `ata(4)` 驱动程序框架支持。由于大多数人用 SCSI 磁盘运行 Alpha，它不如 SCSI 测试充分。请注意 IDE 磁盘的可启动性限制。参见机器特定信息。

通过 CAM 层完全支持 Adaptec 2940x（基于 AIC7xxx 芯片）、Qlogic 系列和 Symbios 的 SCSI。那些对 U160 SCSI 感兴趣的人可能想看看 Adaptec 39160 双通道 LVD U160 适配器。Compaq 称之为 KZPEA 适配器。最近的 Alpha 型号有可从中启动的 SRM 版本。一般而言，请注意各种适配器型号的机器特定可启动性问题。已知的问题在各机器描述中列出。

Qlogic QL2x00 FibreChannel 主机适配器得到完全支持。

如果你想通过以太网启动 Alpha，你显然需要 SRM 控制台识别的以太网卡。这通常意味着你需要带有 21x4x 以太网芯片的板卡，因为那是 Digital 使用的。这些芯片由 FreeBSD `de`（较旧的驱动程序）或 `dc`（较新的驱动程序）驱动。一些新的 SRM 版本已知可以识别由 FreeBSD `fxp` 驱动程序驱动的 Intel 8255x 以太网芯片。但要注意：据报道 `fxp` 驱动程序在 FreeBSD/alpha 上不能正确工作（虽然它在 FreeBSD/x86 上工作得很好）。

DEC DEFPA PCI FDDI 网络适配器在 alpha 上受支持。

通常，SRM 控制台在 PCI VGA 卡上模拟 VGA 兼容模式。然而，Compaq/DEC 并不保证这适用于每种卡类型。当 SRM 认为 VGA 可接受时，FreeBSD 就能使用它。控制台驱动程序与 FreeBSD/Intel 机器上一样工作。请注意，Alpha 上不支持 VESA 模式，因此你只能使用 80x25 控制台。

在一些 Alpha 机器中，你会发现基于 TGA 芯片的视频适配器。普通 TGA 适配器不模拟 VGA，因此不能用作 FreeBSD 控制台。TGA2 卡有基本 VGA 兼容模式，可作为 FreeBSD 控制台正常工作。

大多数 Alpha 上的“PC 标准”串口受支持。

ISDN（i4b）在 FreeBSD/alpha 上不受支持。
