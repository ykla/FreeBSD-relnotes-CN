# FreeBSD 10.3-RELEASE 发行公告

日期：2016 年 4 月 4 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 10.3-RELEASE 现已发布。这是 stable/10 分支的第四个版本，在 FreeBSD 10.2-RELEASE 的稳定性基础上进行了改进并引入了一些新功能。

部分亮点如下：

* UEFI 引导加载程序进行了多项改进：现在会读取 /boot/config 和 /boot.config 文件，多设备引导支持可用并解析命令行参数。此外，其帧缓冲驱动通过 GOP（Graphics Output Protocol）和 UGA（Universal Graphics Adapter）处理得到增强，允许在使用其中一种方法的系统上设置当前图形模式。而且，ZFS 引导能力也已添加到 UEFI 引导加载程序中，包括对多个 ZFS 引导环境（BE）的支持，例如 `sysutils/beadm` 提供的环境。
* CAM Target Layer `ctl(4)` 现在支持高可用性配置。
* Linux 兼容层经过大幅改进，现在能够在 amd64（x86_64）上运行 64 位应用程序，支持 1:1 线程、VDSO 以及足以适用于大多数程序的 `epoll(7)` 系列子集。
* `em(4)` 和 `igb(4)` 驱动分别更新至版本 7.6.1 和 2.5.3。其中带来了对 Intel Skylake 代及更新芯片组中 i219/i219(2)/i219(3) 硬件的支持。
* `isp(4)` 驱动更新和改进：现在还支持 16 Gbps Fibre Channel 适配器，改进了目标模式支持并完成了多 ID（NPIV）功能。
* `ixgbe(4)` 驱动更新至 Intel FreeBSD Networking Group 版本 3.1.13-k，并新增对 X552 和 X550T 的支持。
* `reroot` 支持的初步实现已添加到 `reboot(8)` 工具中，允许从临时源文件系统挂载根文件系统，而无需完全重启系统。
* `bsdinstall(8)` 工具更新，允许在基于 UEFI 的系统上以自动模式创建根置于 ZFS 的安装。
* `ifconfig(8)` 工具现在在指定 `-v` 参数且 NIC 驱动提供此类信息时报告 SFP/SFP+ 光模块数据，即 `cxgbe(4)`、`ixgbe(4)`、`mlx5en(4)` 和 `sfxge(4)`。
* `jail(8)` 工具更新，新增 `-l` 参数，可确保在使用时目标 Jail 环境干净。此外，`jail(8)` 在未指定命令时默认会在目标 Jail 内运行一个 shell。
* `mkimg(1)` 工具更新，支持在 GPT 和 MBR 分区方案中使用 NTFS 文件系统。
* `xz(1)` 工具更新至版本 5.2.2，提供对多线程压缩的支持。
* GNOME 更新至版本 3.16.2。
* TeXLive 更新至 TL2015。
* Xorg-Server 更新至版本 1.17.4。
* 以及更多……

完整的新功能和已知问题列表，请参阅在线发行说明和勘误列表，地址如下：

* <https://www.FreeBSD.org/releases/10.3R/relnotes/>
* <https://www.FreeBSD.org/releases/10.3R/errata/>

## 获取途径

FreeBSD 10.3-RELEASE 现已支持 amd64、i386、ia64、powerpc、powerpc64、sparc64 和 armv6 架构。

FreeBSD 10.3-RELEASE 可从可启动 ISO 镜像或通过网络安装。部分架构还支持从 USB 闪存盘安装。所需文件可通过 FTP 下载，详见下文。虽然部分较小的 FTP 镜像可能不包含所有架构，但通常都会包含较常见的架构，如 amd64 和 i386。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA256 和 MD5 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/10.3R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/10.3R/announce.asc>

amd64（x86_64）架构还提供额外的支持 UEFI 的镜像。

发行版所提供镜像的用途如下：

**dvd1**

包含安装 FreeBSD 基本系统、文档和少量预构建软件包（旨在启动图形工作站）所需的全部内容。还支持启动到基于“livefs”的救援模式。若你能刻录并使用 DVD 介质，这应是你所需的全部。

**disc1**

包含 FreeBSD 基本系统。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

**bootonly**

支持使用 CDROM 驱动器启动机器，但不包含从 CD 本身安装 FreeBSD 所需的安装发行版。从 CD 启动后，需要执行基于网络的安装（例如从 FTP 服务器）。

**memstick**

可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-10.3-RELEASE-amd64-memstick.img of=/dev/da0 bs=10240 conv=sync
```

务必确保目标（`of=`）正确。

**mini-memstick**

可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-10.3-RELEASE-amd64-mini-memstick.img of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**FreeBSD/arm SD 卡镜像**

可写入 SD 卡并用于启动受支持的 arm 系统。SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 512Mb 的 SD 卡上。

为方便没有控制台访问权限的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 `ssh(1)` 访问。此外，`root` 用户密码设置为 `root`，强烈建议在获得系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，请使用 `dd(1)` 工具，将 *KERNEL* 替换为系统对应的内核配置名称。

```sh
# dd if=FreeBSD-10.3-RELEASE-arm-armv6-KERNEL.img of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

## FTP

FreeBSD 10.3-RELEASE 可从以下站点通过 FTP 下载：

* <ftp://ftp.freebsd.org/pub/FreeBSD/releases/ISO-IMAGES/10.3/>

但在尝试此站点之前，请先检查你的区域镜像：

* `ftp://ftp.<your_country_code>.FreeBSD.org/pub/FreeBSD`

任何额外的镜像站点将标记为 ftp2、ftp3 等。

关于 FreeBSD 镜像站点的更多信息可从以下地址找到：

* <http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/mirrors-ftp.html>

FreeBSD 10.3-RELEASE 虚拟机镜像可从以下地址通过 FTP 下载：

* <ftp://ftp.freebsd.org/pub/FreeBSD/releases/VM-IMAGES/10.3-RELEASE/>

关于安装 FreeBSD 或将现有机器更新至 10.3-RELEASE 的说明，请参阅：

* <https://www.FreeBSD.org/releases/10.3R/installation/>
