# FreeBSD 7.0-RELEASE 发行公告

日期：2008 年 2 月 27 日

发件人：Ken Smith <kensmith@FreeBSD.org>

收件人：[freebsd-announce@FreeBSD.org](mailto:freebsd-announce@FreeBSD.org)

主题：[FreeBSD-Announce] FreeBSD 7.0-RELEASE 现已可用

FreeBSD 发行工程团队很高兴地宣布 FreeBSD 7.0-RELEASE 现已可用。这是 7-STABLE 分支的第一个发行版，引入了许多新功能并改进了早期分支中已有的功能。部分亮点：

- 各种数据库和其他基准测试表明，性能和 SMP 可扩展性有显著提升，在某些情况下，与 FreeBSD 6.X 相比，在正常负载下峰值性能提升高达 350%，在高负载下提升高达 1500%。与性能最佳的 Linux 内核（2.6.22 或 2.6.24）相比，性能提高 15%。这些结果来自用于分析和改进系统性能的基准测试，你的特定工作负载下的结果可能有所不同。促成此改进的部分变更包括：
  - 1:1 libthr 线程模型现为默认。
  - 更细粒度的 IPC、网络和调度器锁定。
  - 重点优化了在 5.x 和 6.x 分支中引入的 SMP 架构。

  部分基准测试显示出最多 8 个 CPU 的线性扩展。许多工作负载在多核系统上获得了显著的性能提升。
- ULE 调度器得到了大幅改进，提供更好的性能和交互响应（4BSD 调度器在 7.0 中仍是默认，但 ULE 可能在 7.1 中成为默认）。
- 对 Sun 的 ZFS 文件系统的实验性支持。
- gjournal 可用于设置日志文件系统，gvirstor 可用作虚拟化存储提供程序。
- 对 XFS 文件系统的只读支持。
- unionfs 文件系统修复完成。
- iSCSI 发起端。
- 部分网络驱动程序的 TSO 和 LRO 支持。
- 实验性的 SCTP（流控制传输协议）支持（FreeBSD 是参考实现）。
- 大幅改进的无线（802.11）支持。
- 从 OpenBSD 导入的网络链路聚合/中继（[lagg(4)](http://www.freebsd.org/cgi/man.cgi?query=lagg&sektion=4&manpath=FreeBSD+7.0-RELEASE)）。
- JIT 编译将 BPF 转换为本地代码，提升数据包捕获性能。
- 大幅改进了对基于 ARM 架构板卡的嵌入式系统开发的支持。
- jemalloc，一款全新的、高度可扩展的用户级内存分配器。
- [freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+7.0-RELEASE) 除了安全修复和勘误补丁外，还提供官方支持的到新发行版的二进制升级。
- X.Org 7.3、KDE 3.5.8、GNOME 2.20.2。
- GNU C 编译器 4.2.1。
- BIND 9.4.2。

## 可用性

FreeBSD 7.0-RELEASE 现已可用于 amd64、i386、ia64、pc98 和 powerpc 架构。sparc64 架构的版本将在几天内可用。部分软件包的构建仍在进行中。

FreeBSD 7.0 可以从可引导 ISO 镜像或通过网络安装。所需文件可通过 FTP 或 BitTorrent 下载，如下节所述。虽然一些较小的 FTP 镜像可能不包含所有架构，但它们通常都会包含较常见的架构，如 i386 和 amd64。

发行版 ISO 镜像的 MD5 和 SHA256 校验和附于本消息底部。

发行版中所提供的 ISO 镜像的内容对大多数架构发生了变更。以 i386 架构为例，存在名为 `bootonly`、`disc1`、`disc2`、`disc3`、`livefs` 和 `docs` 的 ISO 镜像。`bootonly` 镜像适用于引导机器通过 FTP 或 NFS 进行基于网络的安装。`disc1`、`disc2` 和 `disc3` 镜像用于进行包含基本软件包集的完整安装，且在安装过程中不需要通过网络访问 FTP 或 NFS 服务器。要引导进入“基于 live CD 的文件系统”和系统救援模式，需要 `disc1` 和 `livefs`。`docs` 镜像包含所有支持语言的全部文档。大多数人会发现，如果你想在初始安装过程中安装一些软件包，`disc1`、`disc2` 和 `disc3` 就是你所需的全部；如果你倾向于在初始安装完成后再安装软件包，则只需 `disc1`。

## 更新现有系统

将任何现有系统升级到 FreeBSD 7.0-RELEASE 构成主版本升级，因此无论你使用哪种方法更新旧系统，都应重新安装机器上已安装的任何 Ports。这样可以避免在未来的 Ports 升级中，当重建某个 Port 但未重建链接到它的其他 Port 时，二进制文件链接到不一致的库集合。可以在更新系统后使用以下命令完成：

```sh
# portupgrade -faP
```

注意，用于协助此操作或执行下方 FreeBSD Update 说明的部分工具默认未安装（例如 portupgrade、gpg 或 portmaster 等类似工具）。

## FreeBSD Update

从 FreeBSD 6.3 开始，[freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+7.0-RELEASE) 工具支持对运行早期 FreeBSD 发行版、候选发布版和测试版的 i386 和 amd64 系统进行二进制升级。从早期发行版（特别是早于 7.0-RC1 的版本）升级到 FreeBSD 7.0 的用户需要下载支持升级到新发行版的更新版本的 [freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+7.0-RELEASE)。

```sh
# fetch http://people.freebsd.org/~cperciva/freebsd-update-upgrade.tgz
```

强烈建议下载并验证该 tar 包的数字签名（由 FreeBSD 安全官的 PGP 密钥签名）。

```sh
# fetch http://people.freebsd.org/~cperciva/freebsd-update-upgrade.tgz.asc
# gpg --verify freebsd-update-upgrade.tgz.asc freebsd-update-upgrade.tgz
```

然后可以按以下方式解压并运行新的 [freebsd-update(8)](http://www.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+7.0-RELEASE)：

```sh
# tar -xf freebsd-update-upgrade.tgz
# sh freebsd-update.sh -f freebsd-update.conf -r 7.0-RELEASE upgrade
# sh freebsd-update.sh -f freebsd-update.conf install
```

在继续之前，必须使用新安装的内核重新启动系统。

```sh
# shutdown -r now
```

接下来，需要再次运行 `freebsd-update.sh` 以安装新的用户空间组件，然后应重新编译所有 Ports 以链接到新的库：

```sh
# sh freebsd-update.sh -f freebsd-update.conf install
# portupgrade -faP
```

最后，需要最后一次运行 `freebsd-update.sh` 以移除旧的系统库，然后应重新启动系统，以便运行更新后的用户空间和 Ports：

```sh
# sh freebsd-update.sh -f freebsd-update.conf install
# shutdown -r now
```

更多信息，请参阅：

- <http://www.daemonology.net/blog/2007-11-11-freebsd-major-version-upgrade.html>
