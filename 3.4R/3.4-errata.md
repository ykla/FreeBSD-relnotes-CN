# FreeBSD 3.4-RELEASE 勘误

如果在安装此版本 FreeBSD 之前你没有阅读任何其他文档，那么至少务必**阅读本发行版的勘误**，以免在已经发现并修复的问题上绊倒。此 ERRATA.TXT 文件按定义显然已经过时，但网络上的其他副本会保持更新，应作为你所发行版的“当前勘误”查阅。这些其他勘误副本位于：

1. <http://www.FreeBSD.org/releases/>
2. **ftp://ftp.FreeBSD.org/pub/FreeBSD/releases/<your-release>/ERRATA.TXT**（以及任何保持此位置最新镜像的站点）。

对此文件的任何更改也会自动发送电子邮件至：

`freebsd-stable@FreeBSD.org`

有关所有 CERT 安全公告，请参见：

**ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT/**

以获取最新的安全事件信息。

---- 安全公告：

当前针对 3.4 的有效安全公告：

- FreeBSD-SA-00:01.make

---- 系统更新信息：

按原样发行时，3.4 中的“Custom”安装选项存在问题，Configure 等菜单项无法使用。

修复：“Novice”和“Express”安装路径仍然可用，且同样有效（即便不够简洁）。或者，你可以从“Index”菜单（Installation、Custom）调用自定义安装以及 Configuration 选项。你也可以直接从以下 URL 下载修复后的 mfsroot.flp 软盘镜像（如果你需要 2.88MB 引导介质，则下载 boot.flp）：

**ftp://ftp.FreeBSD.org/pub/FreeBSD/releases/i386/3.4-RELEASE/floppies/updates/**

3.4 ISO 安装镜像也已更新，包含所有这些勘误项的修复。

802.1Q VLAN 支持编译错误：包含“pseudo-device vlan #”的内核配置无法编译。

修复：将你的 sys/net/if_vlan.c 副本更新为 3-stable 分支版本（已修复），或对随 3.4-RELEASE 分发的 sys/net/if_vlan.c 应用以下补丁：

```sh
RCS file: /home/ncvs/src/sys/net/if_vlan.c,v
retrieving revision 1.4.2.2
retrieving revision 1.4.2.3
diff -C2 -r1.4.2.2 -r1.4.2.3
*** if_vlan.c   1999/12/13 02:02:23 1.4.2.2
--- if_vlan.c   1999/12/25 03:28:51 1.4.2.3
***************
*** 515,519 ****
            ifv->ifv_p = 0;
            if_down(ifp);
!           ifv->if_flags &= ~(IFF_UP|IFF_RUNNING);
            break;
        }
```
