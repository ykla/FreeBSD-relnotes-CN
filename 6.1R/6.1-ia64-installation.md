## 摘要

本文简要说明如何安装 FreeBSD/ia64 6.1-RELEASE，并着重介绍获取 FreeBSD 发行版的方法。同时提供了一些故障排除和常见问题解答。

## 1 安装 FreeBSD

本节介绍安装全新 FreeBSD 发行版的过程。这些说明着重于获取 FreeBSD 6.1-RELEASE 发行版以及开始安装过程。FreeBSD 手册中的 [“安装 FreeBSD”](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/install.html) 章节提供了关于安装程序本身的更深入信息，包括带截图的引导式操作演示。

如果你是从早期版本的 FreeBSD 升级，请参阅 [升级 FreeBSD](https://www.freebsd.org/releases/6.1R/installation-ia64.html#UPGRADING) 获取升级说明。

### 1.1 准备工作

最重要的预安装步骤可能是阅读 FreeBSD 提供的各种说明文档。关于本次 FreeBSD 发行版的文档路线图可在 **README.TXT** 中找到，该文件通常与本文档位于同一位置；大多数文档（如发行说明和硬件兼容列表）也可在安装程序的“文档”菜单中访问。

如果你有互联网连接，还可以从 [FreeBSD 项目网站](http://www.FreeBSD.org/) 获取 FreeBSD [FAQ](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/faq/) 和 [手册](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/) 的在线版本。

这套文档看起来可能令人生畏，但花时间阅读将在日后为你节省数倍的时间。熟悉可用的资源在安装过程中遇到问题时也会有所帮助。

即使最周密的计划有时也会出错，因此如果遇到问题，请参阅[故障排除](https://www.freebsd.org/releases/6.1R/installation-ia64.html#TROUBLE)，其中包含有价值的故障排除信息。你还应该在安装前阅读最新的 **ERRATA.TXT** 副本，因为它会提醒你注意特定发行版在发布后报告的任何问题。

> **重要**
>
> 尽管 FreeBSD 尽力防止数据意外丢失，但如果操作失误，仍然极有可能*清除整个磁盘*。在进入最终的 FreeBSD 安装菜单之前，请确保已充分备份所有重要数据。

### 1.5 各种安装类型的详细说明

一旦你以某种方式进入初始安装屏幕，应该能够按照各种菜单提示进行操作。如果你以前从未使用过 FreeBSD 安装程序，也建议阅读“文档”子菜单中的一些文档以及第一个菜单上的通用“Usage”说明。

> **注意**
>
> 如果在某个屏幕卡住，按 **F1** 键可获取该特定章节的在线文档。

如果你从未安装过 FreeBSD，或者即使安装过，也最推荐使用“Standard”安装模式，因为它确保你会经过所有重要的检查清单项。如果你对 FreeBSD 安装过程非常熟悉并且*确切*知道要做什么，可使用“Express”或“Custom”安装选项。如果你要升级现有系统，使用“Upgrade”选项。

FreeBSD 安装程序支持直接使用软盘、DOS、磁带、CDROM、FTP、NFS 和 UFS 分区作为安装媒体；以下列出每种媒体类型的安装提示。

#### 1.5.1 从网络 CDROM 安装

如果你只想从本地 CDROM 驱动器安装，请参阅 [第 1.4 节](#14-从-cdrom-或互联网安装-freebsd)。如果你的系统没有 CDROM 驱动器，想使用另一台你有网络连接的机器的 CDROM 驱动器中的 FreeBSD 发行版 CD，有几种方式：

- 如果你可以直接从某台 FreeBSD 机器的 CDROM 驱动器通过 FTP 安装 FreeBSD，操作很简单：确保 FTP 服务器正在运行，然后使用 [vipw(8)](http://www.freebsd.org/cgi/man.cgi?query=vipw&sektion=8&manpath=FreeBSD+6.1-RELEASE) 命令在密码文件中添加以下行：

```sh
ftp:*:99:99::0:0:FTP:/cdrom:/sbin/nologin
```

在运行安装的机器上，进入 Options 菜单并将 Release Name 设置为 `any`。然后选择 `FTP` 作为 Media 类型，并在 FTP 站点菜单中选择“URL”后输入 `ftp://machine`。

> **警告**
>
> 这可能允许本地网络（或互联网）上的任何人对此机器进行“匿名 FTP”连接，这可能并非所愿。

- 如果你更愿意使用 NFS 直接将 CDROM 导出给你要安装的机器，需要先在 **/etc/exports** 文件中（在有 CDROM 驱动器的机器上）添加条目。以下示例允许 **ziggy.foo.com** 在安装期间通过 NFS 直接挂载 CDROM：

```sh
/cdrom          -ro             ziggy.foo.com
```

有 CDROM 的机器还必须配置为 NFS 服务器。如果你不确定如何操作，那么 NFS 安装可能不是最佳选择，除非你愿意阅读 [rc.conf(5)](http://www.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&manpath=FreeBSD+6.1-RELEASE) 并进行适当配置。假设这部分顺利完成，在目标机器安装时应该能够输入 **cdrom-host:/cdrom** 作为 NFS 安装路径，例如 **wiggy:/cdrom**。

#### 1.5.2 从软盘安装

如果必须从软盘安装（无论是由于硬件不受支持还是因为你喜欢用困难的方式做事），必须先为安装准备一些软盘。

首先，按照 [软盘镜像说明](https://www.freebsd.org/releases/6.1R/installation-ia64.html#FLOPPIES) 的说明制作启动软盘。

其次，仔细阅读 [发行版格式](https://www.freebsd.org/releases/6.1R/installation-ia64.html#LAYOUT) 并特别注意“Distribution Format”部分，因为它描述了哪些文件需要放到软盘上，哪些可以安全跳过。

接下来，你至少需要足够多的 1.44 MB 软盘来容纳 **bin**（二进制发行版）目录中的所有文件。如果在 DOS 下准备这些软盘，这些软盘*必须*使用 MS-DOS `FORMAT` 命令格式化。如果使用 Windows，使用 Windows 文件管理器的格式化命令。

> **重要**
>
> 软盘通常以“出厂预格式化”形式销售。虽然方便，但过去用户报告的许多问题都源于使用不当格式化的媒体。请自行重新格式化以确保无误。

如果从另一台 FreeBSD 机器创建软盘，格式化仍然是个好主意，尽管不需要在每个软盘上放置 DOS 文件系统。可以使用 [disklabel(8)](http://www.freebsd.org/cgi/man.cgi?query=disklabel&sektion=8&manpath=FreeBSD+6.1-RELEASE) 和 [newfs(8)](http://www.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&manpath=FreeBSD+6.1-RELEASE) 命令在软盘上放置 UFS 文件系统，如下列命令序列所示：

```sh
# fdformat -f 1440 fd0
# disklabel -w fd0 floppy3
# newfs -i 65536 /dev/fd0
```

为 DOS 或 UFS 格式化软盘后，需要将文件复制到它们上面。发行版文件的大小使得一张软盘可容纳一个文件。每个发行版应放在软盘上各自的子目录中，例如：**a:\bin\bin.inf**、**a:\bin\bin.aa**、**a:\bin\bin.ab**……

> **重要**
>
> **bin.inf** 文件也需要放在 **bin** 集的第一张软盘上，因为安装程序会读取它以确定在获取和拼接发行版时还要查找多少个附加部分。将发行版放到软盘上时，**distname.inf** 文件*必须*占据每个发行版集的第一张软盘。

到达安装的 Media 屏幕时，选择“Floppy”，系统会提示你完成其余步骤。

#### 1.5.4 从 QIC/SCSI 磁带安装

从磁带安装时，安装程序期望文件简单地以 tar 格式存储在磁带上。因此，在获取你感兴趣的发行版的所有文件后，使用 [tar(1)](http://www.freebsd.org/cgi/man.cgi?query=tar&sektion=1&manpath=FreeBSD+6.1-RELEASE) 将它们写入磁带，命令类似于：

```sh
# cd /where/you/have/your/dists
# tar cvf /dev/sa0 dist1 .. dist2
```

进行安装时，还应确保在某个临时目录（你可以选择）中留出足够空间以容纳所创建磁带的*全部*内容。由于磁带的非随机访问特性，这种安装方法需要相当多的临时存储空间。你应该预期需要的临时存储空间与你写入磁带的内容量相当。

> **注意**
>
> 进行安装时，必须在从启动软盘启动*之前*将磁带放入驱动器。否则安装“探测”可能无法找到它。

现在按照 [软盘镜像说明](https://www.freebsd.org/releases/6.1R/installation-ia64.html#FLOPPIES) 的说明创建启动软盘，然后继续安装。

#### 1.5.5 通过网络使用 FTP 或 NFS 安装

按照第一节所述制作启动软盘后，可以使用三种连接类型之一通过网络加载其余安装内容：串口、并口或以太网。

##### 1.5.5.1 串口

SLIP 支持相当原始，主要限于硬连线链路，例如两台计算机之间的串行电缆。该链路必须是硬连线的，因为 SLIP 安装目前不提供拨号功能。如果需要使用调制解调器拨号，或在连接前需要以其他方式与链路进行交互，应使用 PPP 工具。

如果使用 PPP，确保手头有互联网服务提供商的 IP 地址和 DNS 信息，因为你在安装过程的早期就需要知道这些信息。你可能还需要知道自己的 IP 地址，不过 PPP 支持动态地址协商，如果你的 ISP 支持，可能可以直接从 ISP 获取此信息。

你还需要了解如何使用特定品牌调制解调器的各种“AT 命令”进行拨号，因为 PPP 拨号器仅提供非常简单的终端仿真程序。

##### 1.5.5.2 并口

如果有到另一台 FreeBSD 或 Linux 机器的硬连线连接，还可以考虑通过“laplink”式并口电缆安装。并口的数据传输率远高于串行线路通常可能的速率（最高 50k/sec），因此安装更快。以这种方式使用点对点并口电缆时，通常不需要使用“真实”IP 地址，一般可以只使用 RFC 1918 风格的地址作为链路两端（例如 **10.0.0.1**、**10.0.0.2** 等）。

> **重要**
>
> 如果使用 Linux 机器而非 FreeBSD 机器作为 PLIP 对端，还需在 TCP/IP 设置屏幕的“ifconfig 附加选项”字段中指定 `link0`，以兼容 Linux 略有差异的 PLIP 协议。

##### 1.5.5.3 以太网

FreeBSD 支持大多数常见的以太网卡；支持的网卡列表作为 FreeBSD 硬件说明的一部分提供（参见启动软盘上“文档”菜单或 CDROM 顶级目录中的 **HARDWARE.TXT**）。如果你使用的是受支持的 PCMCIA 以太网卡之一，还要确保在笔记本电脑开机*之前*插入网卡。遗憾的是，FreeBSD 目前不支持在安装期间“热插拔”PCMCIA 卡。

你还需要知道你在网络上的 IP 地址、子网的 `netmask` 值以及你的机器名称。系统管理员可以告诉你哪些值适合你的特定网络设置。如果你想通过名称而非 IP 地址引用其他主机，还需要名称服务器和可能的网关地址（如果使用 PPP，则是你的提供商的 IP 地址）来进行通信。如果你想通过 HTTP 代理进行 FTP 安装（见下文），还需要代理的地址。

如果你不知道这些问题的答案，那么在尝试此类安装之前应该先与系统管理员讨论。在活动网络上使用随机选择的 IP 地址或子网掩码几乎肯定会失败，并且可能会受到系统管理员的训斥。

一旦某种网络连接正常工作，安装就可以通过 NFS 或 FTP 继续。

##### 1.5.5.4 NFS 安装提示

NFS 安装相当直接：只需将你想要的 FreeBSD 发行版文件复制到某个服务器上，然后将 NFS 媒体选择指向它。

如果此服务器仅支持“特权端口”访问（这通常是 Sun 和 Linux 工作站的默认设置），你可能需要在 Options 菜单中设置此选项才能继续安装。

如果你的以太网卡质量较差，传输速率很慢，可能还需要切换相应的 Options 标志。

为使 NFS 安装正常工作，服务器还必须支持“子目录挂载”，例如如果你的 FreeBSD 发行版目录位于 **wiggy:/usr/archive/stuff/FreeBSD**，那么 `wiggy` 必须允许直接挂载 **/usr/archive/stuff/FreeBSD**，而不仅仅是 **/usr** 或 **/usr/archive/stuff**。

在 FreeBSD 的 **/etc/exports** 文件中，这由 `-alldirs` 选项控制。其他 NFS 服务器可能有不同的约定。如果你从服务器收到 `Permission Denied` 消息，很可能是你没有正确启用此功能。

##### 1.5.5.5 FTP 安装提示

可以从任何包含相当新的 FreeBSD 版本的镜像站点进行 FTP 安装。安装期间的 FTP 站点菜单提供了几乎适用于世界上任何位置的合理选择列表。

如果你要从此菜单中未列出的其他 FTP 站点安装，或者在配置名称服务器时遇到问题，还可以通过在该菜单中选择“URL”选项来指定自己的 URL。URL 可以包含主机名或 IP 地址，因此在没有名称服务器的情况下，类似以下的内容也可以工作：

```sh
ftp://216.66.64.162/pub/FreeBSD/releases/ia64/4.2-RELEASE
```

有三种 FTP 安装模式可供使用：

- **FTP**：此方法使用标准的“Active”模式进行传输，服务器向客户端发起连接。这在大多数防火墙下无法工作，但通常在不支持被动模式的旧 FTP 服务器上效果最佳。如果被动模式连接挂起，尝试此模式。
- **FTP Passive**：此模式设置 FTP“Passive”模式，阻止服务器向客户端打开连接。此选项最适合需要通过不允许在随机端口地址上进行传入连接的防火墙的用户。
- **通过 HTTP 代理的 FTP**：此选项指示 FreeBSD 使用 HTTP 连接到代理进行所有 FTP 操作。代理将转换请求并发送到 FTP 服务器。使用户能够穿越不允许 FTP 但提供 HTTP 代理的防火墙。除了 FTP 服务器外，你还必须指定代理的主机名。

在极少数情况下，如果你的 FTP 代理不通过 HTTP，可以将 URL 指定为类似以下形式：

```sh
ftp://foo.bar.com:port/pub/FreeBSD
```

在上面的 URL 中，`port` 是代理 FTP 服务器的端口号。

## 2 发行版格式

典型的 FreeBSD 发行版目录如下所示（具体细节可能因版本、架构和其他因素而异）：

```sh
ERRATA.HTM      README.TXT      compat1x        dict            kernel
ERRATA.TXT      RELNOTES.HTM    compat20        doc             manpages
HARDWARE.HTM    RELNOTES.TXT    compat21        docbook.css     packages
HARDWARE.TXT    base            compat22        filename.txt    ports
INSTALL.HTM     boot            compat3x        floppies        proflibs
INSTALL.TXT     catpages        compat4x        games           src
README.HTM      cdrom.inf       crypto          info            tools
```

这些相同文件也包含在多盘套装的第一张 CDROM 中，但在磁盘上的布局略有不同。在大多数架构上，安装 CDROM 除发行版文件外还包含“live filesystem”。live filesystem 在修复或故障排除现有 FreeBSD 安装时很有用（参见 [故障排除](https://www.freebsd.org/releases/6.1R/installation-ia64.html#TROUBLE)）。

**floppies** 目录对无法从 CDROM 媒体启动（但能通过其他方式读取 CDROM）的用户特别有用。从 **floppies** 目录生成一组 1.44 MB 启动软盘很容易（参见[软盘镜像说明](https://www.freebsd.org/releases/6.1R/installation-ia64.html#FLOPPIES)了解操作说明），并使用这些软盘从 CDROM、FTP 或 NFS 启动安装。安装期间所需的其余数据将根据你的选择自动获取。如果你从未安装过 FreeBSD，还应通读本文档（安装说明）文件的全部内容。

如果你要尝试其他类型的安装，或者只是好奇发行版是如何组织的，以下是对其中一些项目的更详细描述：

- **\*.TXT** 和 **\*.HTM** 文件包含文档（例如，本文档同时包含在 **INSTALL.TXT** 和 **INSTALL.HTM** 中），应在开始安装前阅读。**\*.TXT** 文件是纯文本，而 **\*.HTM** 文件是 HTML 文件，几乎任何 Web 浏览器都可以读取。某些发行版可能还包含其他格式的文档，如 PDF 或 PostScript。
- **docbook.css** 是层叠样式表（CSS）文件，某些 Web 浏览器用它来格式化 HTML 文档。
- **base**、**catpages**、**crypto**、**dict**、**doc**、**games**、**info**、**manpages**、**proflibs** 和 **src** 目录包含 FreeBSD 本身的主要发行版组件，并拆分为较小的文件以便装到软盘上（如有必要）。
- **compat1x**、**compat20**、**compat21**、**compat22**、**compat3x** 和 **compat4x** 目录包含与旧版本兼容的发行版，作为单个 gzip 压缩的 tar 文件分发——可在发行时或稍后通过运行其 **install.sh** 脚本安装。
- **floppies/** 子目录包含软盘安装镜像；有关使用它们的更多信息可在[软盘镜像说明](https://www.freebsd.org/releases/6.1R/installation-ia64.html#FLOPPIES)中找到。
- **packages** 和 **ports** 目录包含 FreeBSD 软件包和 Ports。可以通过运行以下命令从 packages 目录安装软件包：

  ```sh
  # /stand/sysinstall configPackages
  ```

  也可以将 **packages/** 中的各个文件名提供给 [pkg_add(1)](http://www.freebsd.org/cgi/man.cgi?query=pkg_add&sektion=1&manpath=FreeBSD+6.1-RELEASE) 命令来安装软件包。

  Ports 可以像其他发行版一样安装，解压后大约需要 190 MB 空间。有关 Ports 的更多信息可从 [http://www.FreeBSD.org/ports/](http://www.FreeBSD.org/ports/) 获取，或者在安装了 **doc** 发行版后从本地 **/usr/share/doc/handbook** 获取。
- **tools** 目录包含各种 DOS 工具，用于发现磁盘几何参数、安装启动管理器等。它完全是可选的，仅为用户方便而提供。

典型的发行版目录（例如 **info** 发行版）内部如下所示：

```sh
CHECKSUM.MD5    info.ab         info.ad         info.inf        install.sh
info.aa         info.ac         info.ae         info.mtree
```

**CHECKSUM.MD5** 文件包含每个文件的 MD5 签名，以防数据损坏之疑，仅供参考。实际安装不使用它，也不需要与发行版文件其余部分一起复制。**info.a\*** 文件是分割的 gzip 压缩 tar 文件，其内容可通过以下命令查看：

```sh
# cat info.a* | tar tvzf -
```

安装期间，安装程序会自动拼接并解压这些文件。

**info.inf** 文件也是必需的，因为安装程序会读取它以确定在获取和拼接发行版时要查找多少个部分。将发行版放到软盘上时，**.inf** 文件*必须*占据每个发行版集的第一张软盘！

**info.mtree** 文件是另一个非必要文件，仅供用户参考。它包含*解压后*发行版文件的 MD5 签名，可稍后与 [mtree(8)](http://www.freebsd.org/cgi/man.cgi?query=mtree&sektion=8&manpath=FreeBSD+6.1-RELEASE) 程序一起使用，以验证安装权限和校验和，检测文件是否遭到修改。与 **base** 发行版一起使用时，这是检测系统木马攻击的绝佳方法。

最后，**install.sh** 文件供那些想要在安装后安装发行版的用户使用。例如，要在系统安装后从 CDROM 安装 info 发行版，可以执行：

```sh
# cd /cdrom/info
# sh install.sh
```

## 3 升级 FreeBSD

这些说明描述从旧版本 FreeBSD 进行二进制升级的过程。

> **警告**
>
> 尽管 FreeBSD 升级过程尽力防止数据意外丢失，但仍然极有可能通过此安装*清除整个磁盘*！在未充分备份所有重要数据文件之前，请不要接受最终确认请求。

> **重要**
>
> 这些说明假设你使用的是与你要升级到的 FreeBSD 版本一起提供的 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 版本。使用不匹配的 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 版本几乎肯定会出问题，并且已知会使系统处于不可用状态。最常见的此类错误是使用现有安装中的旧 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 副本升级到较新版本的 FreeBSD。这*不*推荐。

> **警告**
>
> 目前不支持从 FreeBSD 4-STABLE 二进制升级到 FreeBSD 6.1-RELEASE。FreeBSD 4-STABLE 中存在某些文件可能会造成干扰，但二进制升级不会删除它们。一个值得注意的例子是旧的 **/usr/include/g++** 目录会导致 C++ 程序编译错误（或根本无法编译）。
>
> 这些升级说明供从相对较新的 FreeBSD 6.1-STABLE 快照升级的用户使用。

### 3.1 概述

升级过程将用户选择的发行版替换为与新版 FreeBSD 对应的发行版。它保留标准系统配置数据，以及用户数据、已安装的软件包和其他软件。

考虑升级的管理员应在开始升级前完整阅读本节。未能这样做可能导致升级失败或数据丢失。

#### 3.1.1 升级概览

通过在先前版本之上解压新版本的组件来执行发行版升级。属于旧发行版的文件不会被删除。

系统配置通过保留并恢复以下文件的先前版本来进行保护：**Xaccel.ini**、**XF86Config**、**adduser.conf**、**aliases**、**aliases.db**、**amd.map**、**crontab**、**csh.cshrc**、**csh.login**、**csh.logout**、**cvsupfile**、**dhclient.conf**、**disktab**、**dm.conf**、**dumpdates**、**exports**、**fbtab**、**fstab**、**ftpusers**、**gettytab**、**gnats**、**group**、**hosts**、**hosts.allow**、**hosts.equiv**、**hosts.lpd**、**inetd.conf**、**localtime**、**login.access**、**login.conf**、**mail**、**mail.rc**、**make.conf**、**manpath.config**、**master.passwd**、**motd**、**namedb**、**networks**、**newsyslog.conf**、**nsmb.conf**、**nsswitch.conf**、**pam.conf**、**passwd**、**periodic**、**ppp**、**printcap**、**profile**、**pwd.db**、**rc.conf**、**rc.conf.local**、**rc.firewall**、**rc.local**、**remote**、**resolv.conf**、**rmt**、**sendmail.cf**、**sendmail.cw**、**services**、**shells**、**skeykeys**、**spwd.db**、**ssh**、**syslog.conf**、**ttys**、**uucp**。

与新版本对应的这些文件版本被移动到 **/etc/upgrade/**。系统管理员可以审阅这些新版本并根据需要合并组件。请注意，这些文件中许多是相互依赖的，最佳合并过程是将所有站点特定数据从当前文件复制到新文件中。

在升级过程中，系统管理员会被提示指定一个位置，用于保存 **/etc/** 中的所有文件。如果对其他文件进行了本地修改，随后可从此位置检索这些文件。

### 3.2 升级步骤

本节详述升级过程。特别关注与正常安装有重大不同的项目。

#### 3.2.1 备份

在升级之前应备份用户数据和系统配置。虽然升级过程尽力防止意外失误，但仍有可能部分或完全破坏数据和配置信息。

#### 3.2.2 挂载文件系统

进入磁盘标签编辑器时，会列出指定磁盘的文件系统设备。在开始升级之前，管理员应记录设备名称和对应的挂载点。此处应输入这些挂载点。*不要*为任何文件系统设置“newfs flag”，因为这会导致数据丢失。

#### 3.2.3 选择发行版

选择发行版时，对选择哪些没有限制。作为一般规则，应选择 `base` 发行版进行更新，如果已安装手册页，则选择 `man` 发行版。如果管理员希望添加额外功能，可以选择超出最初安装范围的其他发行版。

#### 3.2.4 安装后

安装过程完成后，系统管理员会被提示检查新的配置文件。此时应进行检查以确保系统配置有效。特别应检查 **/etc/rc.conf** 和 **/etc/fstab** 文件。

### 3.3 从源代码升级

如果用户对更具灵活性和复杂性的升级方法感兴趣，可参阅 FreeBSD 手册中的 [The Cutting Edge](http://www.FreeBSD.org/doc/en_US.ISO8859-1/books/handbook/cutting-edge.html)。此过程涉及从源代码重新构建整个 FreeBSD。它需要可靠的网络连接、额外的磁盘空间和时间，但对于网络和其他更复杂的安装具有优势。这与用于跟踪 -STABLE 或 -CURRENT 开发分支的过程大致相同。

**/usr/src/UPDATING** 包含从源代码更新 FreeBSD 系统的重要信息。它列出了 FreeBSD 变更导致的可能影响升级的各种问题。

## 4 故障排除

### 4.1 修复现有 FreeBSD 安装

FreeBSD 在 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 安装程序的顶部菜单中提供“fixit”选项。它提供一个带有 FreeBSD 基本系统中常见程序的 shell；此环境对修复或故障排除现有 FreeBSD 安装很有用。要使用 fixit 模式，你还需要 **fixit.flp** 软盘（以与启动软盘相同的方式生成）或“live filesystem”CDROM。在多盘 FreeBSD 发行版中，live filesystem 镜像通常位于安装盘上。请注意，使用 fixit 选项需要一些 UNIX 系统管理经验。

通常有两种调用 fixit 模式的方式。能从 FreeBSD 安装 CDROM 启动的用户应这样做，然后从主 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 菜单中选择“fixit”项。然后从 fixit 菜单中选择“CDROM/DVD”选项。

无法从 CDROM 启动但能从软盘启动的用户需要更多步骤。除安装所需的 **boot.flp** 和 **kernX.flp** 软盘外，还需以与其他软盘相同的方式创建 **fixit.flp** 软盘。按照从软盘启动安装程序的说明操作，直到到达主 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 菜单。此时从主 [sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+6.1-RELEASE) 菜单中选择“fixit”项。然后从 fixit 菜单中选择“Floppy”选项，并在提示时插入 **fixit.flp** 软盘。

CDROM 和软盘 fixit 环境相似但不完全相同。两者都提供一个 shell，其中包含可用于检查、修复和查看文件系统及其内容的各种命令。CDROM 版本的 fixit 通过 live filesystem 提供 FreeBSD 基本系统中可用的所有命令和程序。相比之下，由于空间限制，软盘 fixit 环境只能提供命令的子集。

在软盘版本的 fixit 中，一些独立实用程序可在 **/stand** 或 **/mnt2/stand** 中找到。在 CDROM 版本的 fixit 中，这些相同程序可在 **/stand** 或 **/mnt2/rescue** 中找到（以及 live filesystem 中的其余程序，可在 **/mnt** 下找到）。
