### 2.2 在 Alpha 上运行 FreeBSD 的基本要求

显然，你需要一台 FreeBSD 了解的 Alpha 机器。Alpha 机器*不像* PC。各种核心逻辑芯片组和主板设计之间存在相当大的差异。这意味着内核在运行之前需要了解特定机器的详细信息。向未知硬件抛出某个 **GENERIC** 内核几乎肯定会惨败。

对于要考虑用于 FreeBSD 的机器，请确保已安装 SRM 控制台固件。或者至少确保特定机器类型有 SRM 控制台固件可用。如果 FreeBSD 当前不支持你的机器类型，假设 SRM 可用，这很可能在某个时候会改变。当 SRM 控制台固件不可用时，一切都不确定。

带有 ARC 或 AlphaBIOS 控制台固件的机器是为 WindowsNT 设计的。有些机器在系统 ROM 中有 SRM 控制台固件可用，你只需选择（通过 ARC 或 AlphaBIOS 菜单）。在其他情况下，你需要用 SRM 代码重新刷新 ROM。请查看 <http://ftp.digital.com/pub/DEC/Alpha/firmware> 了解特定系统可用的固件。无论如何：没有 SRM 就意味着*没有* FreeBSD（NetBSD、OpenBSD、Tru64 Unix 或 OpenVMS 也是如此）。随着 WindowsNT/alpha 的消亡，许多以前的 NT 机器在二手市场上出售。当它们仅支持 NT 时，几乎没有或没有折旧价值。所以，如果价格看起来太好的话，要小心。

如果你在选择 SRM 时遇到问题，换句话说，如果系统在选择 SRM 控制台后回退到 AlphaBIOS，是时候检查 CMOS 时钟/参数 RAM 的电池是否耗尽了。如果是这样，更换电池很可能会解决问题。

已知的非 SRM 机器有：

- Digital XL 系列
- Digital XLT 系列
- Samsung PC164UX（“Ruffian”）
- Samsung 164B

有 SRM 但 FreeBSD 不支持的机器有：

- DECpc 150（“Jensen”）
- DEC 2000/300（“Jensen”）
- DEC 2000/500（“Culzean”）
- AXPvme 系列（“Medulla”）

为了使事情更复杂一些：Digital 曾经有所谓的“白箱”Alpha 机器（专为 NT-only 设计）和“蓝箱”Alpha 机器（专为 OpenVMS 和 Digital Unix 设计）。这些名称基于机箱的颜色，分别为“FrostWhite”和“TopGunBlue”。虽然你可以在白箱上安装 SRM 控制台固件，但 OpenVMS 和 Digital Unix 会拒绝在它们上启动。FreeBSD 在 4.0-RELEASE 之后的版本可以在白箱和蓝箱变体上运行。在有人问之前：白箱有一个相当不同的（即更便宜的）Digital 价格标签。

作为 SRM 的一部分，你会得到所谓的 OSF/1 PAL 代码（OSF/1 是 Digital 在 Alpha 上的 UNIX 产品的初始名称）。PAL 代码可以被认为是硬件和操作系统之间的软件抽象层。它使用普通 CPU 指令以及少量专用于 PAL 使用的特权指令。PAL 不是微代码。ARC 控制台固件包含不同的 PAL 代码，面向 WinNT，完全不适合 FreeBSD 使用（更一般地说：Unix 或 OpenVMS）。在有人问之前：Linux/alpha 自带 PAL 代码，允许它在 ARC 和 AlphaBIOS 上启动。*BSD 人士认为这不是一个好主意，原因有很多。我不想在这里详述。如果你对那些复杂的细节感兴趣，请搜索 FreeBSD 和 NetBSD 网站。

还有另一个陷阱：你需要 SRM 控制台固件识别的磁盘适配器才能从磁盘启动。遗憾的是，SRM 接受什么作为启动适配器高度依赖于系统和 SRM 版本。对于较旧的基于 PCI 的机器，这意味着你需要基于 NCR/Symbios 53C810 的适配器，或基于 Qlogic 1020/1040 的适配器。有些机器在主板上嵌入了 SCSI 芯片。较新的机器设计和 SRM 版本将能够与更现代的 SCSI 芯片/适配器一起工作。查看下面的机器特定信息。请注意，本讨论的其余部分仅涉及 Symbios 芯片，这包括仍然印有 NCR 的较旧芯片。Symbios 在某个时候收购了 NCR。

这个问题可能会咬到那些机器最初是 WindowsNT 机器的人。ARC 或 AlphaBIOS 知道它可以从中启动的*其他*适配器类型，而 SRM 则不同。例如，你可以使用 ARC/AlphaBios 从 Adaptec 2940UW 启动，但（通常）不能使用 SRM。一些较新的机器类型引入了 Adaptec 启动支持。请查阅机器特定部分以获取详细信息。

大多数不能启动的适配器对于仅数据磁盘工作正常。SRM 和 ARC 之间的差异也可能在某些（前 WindowsNT）系统中为你带来预封装的 IDE CDROM 和硬盘。存在可以从 IDE 磁盘和 CDROM 启动的 SRM 版本（取决于机器类型）。查看机器特定部分以获取详细信息。

为了可启动，根分区（分区 a）必须位于磁盘驱动器的偏移 0 处。这意味着你必须使用安装程序的分区菜单，并从将分区 a 分配给根分区（偏移 0）开始。随后根据你的喜好布局其余分区。如果你不遵守此规则，安装将顺利进行，但系统将无法从新安装的磁盘启动。请注意，在 Alpha 上运行的 FreeBSD 不使用 fdisk，磁盘使用 disklabel 直接标记。

如果你没有/不想要本地磁盘驱动器，你可以通过以太网启动。这假定 SRM 控制台识别以太网适配器/芯片。一般来说，这意味着你需要带有 21x4x 以太网芯片的板卡，因为那是 Digital 使用的。这些芯片由 FreeBSD `de`（较旧的驱动程序）或 `dc`（较新的驱动程序）驱动。一些新的 SRM 版本已知可以识别由 FreeBSD `fxp` 驱动程序驱动的 Intel 8255x 以太网芯片。但要注意：据报道 `fxp` 驱动程序在 FreeBSD/alpha 上不能正确工作（虽然它在 FreeBSD/x86 上工作得很好）。

Alpha 机器可以在 SRM 上运行图形控制台或串行控制台。如果需要，ARC 也可以在串行控制台上运行。带有 8 位控件的 VT100 仿真至少应该允许你从 ARC/AlphaBIOS 切换到 SRM 模式，而无需先安装显卡。

如果你想在没动显示器/显卡的情况下运行 Alpha 机器，只需不连接键盘/鼠标到机器。相反，将串行终端 [模拟器] 连接到串行端口 #1。SRM 将以 9600N81 与你通信。这对于调试目的也非常实用。注意：一些/大多数（？）SRM 也会在串行端口 #2 上为你提供控制台提示。但是，启动内核将在串行端口 #1 上显示启动消息，并将控制台放在那里。*这可能会非常令人困惑。*

大多数基于 PCI 的 Alpha 可以使用普通 PC 型 VGA 卡。SRM 包含足够的智能使其工作。然而，这并不意味着街上的每个 PCI VGA 卡都能在 Alpha 机器中工作。像 S3 Trio64、Mach64 和 Matrox Millennium 这样的卡通常可以工作。旧的基于 ET4000 的 ISA 卡对我来说也工作过。但在购买之前先问一下。

来自 PC 世界的多数 PCI 设备也可以在 FreeBSD 基于 PCI 的机器中工作。查看 **/sys/alpha/conf/GENERIC** 文件以获取这方面的最新信息。如果你想使用上面有 PCI 桥芯片的 PCI 卡，请查看相应机器类型的讨论。在某些情况下，你可能会遇到 PCI 卡不能正确处理 PCI 奇偶校验的问题。这可能导致 panic。可以使用以下 SRM 命令禁用 PCI 奇偶校验：

```sh
>>> SET PCI_PARITY OFF
```

这不是 FreeBSD 的问题，在 Alpha 硬件上运行的所有操作系统都需要这种变通方法。

如果你的系统（也）包含 EISA 扩展槽，在安装 EISA 卡或升级控制台固件后，你需要运行 EISA 配置实用程序（ECU）。

对于 Alpha CPU，你会发现多代。原始 Alpha 设计是 21064。它是在称为 MOS4 的芯片工艺中生产的，在此工艺中制造的芯片被昵称为 EV4。较新的 CPU 有 21164、21264 等。你会看到像 EV4S、EV45、EV5、EV56、EV6、EV67、EV68 这样的代号。带有两位数字的 EV 是略有改进的版本。例如，EV45 与它基于的 EV4 相比，有改进的 FPU 和 16 kByte 片上独立的 I 和 D 缓存。经验法则：紧跟“EV”的数字越高，越可取（即：更快/更现代）。不建议在低于 EV5 的任何东西上花钱。当然，免费获得的硬件另当别论。

对于内存，你至少需要 32 Mbyte。几年前我曾经在 16 Mbyte 系统上运行 FreeBSD，但你不会喜欢那样。当我升级到 32 Mbyte 时，内核构建时间减半。请注意，SRM 控制台从总系统内存中窃取 2 Mbyte（并保留它）。对于更严肃的工作，推荐 128 Mbyte 或更多。虽然 Alpha 机器通常可以容纳大到非常大的物理内存大小，但 FreeBSD 限于 1 或 2 Gbyte（取决于核心芯片组）的 RAM。此限制是由于 VM 系统的当前实现。

在内存问题上：密切关注你的机器使用的内存类型。各种机器有非常不同的内存配置和要求。

最后的话：我期望上述对第一次使用 Alpha 的用户来说听起来有点令人生畏。不要太担心。如果阅读本文档后仍有不清楚的地方，请随时提问。

### 2.3 系统特定信息

下面是 FreeBSD 运行的硬件概述。此列表肯定会增长，查看 **/sys/alpha/conf/GENERIC** 可能会有所启发。

Alpha 机器通常最好通过其项目代号来了解。在已知的情况下，这些代号在下面括号中列出。

#### 2.3.1 AXPpci33 (“NoName”)

NoName 是一款基于 21066 LCA（低成本 Alpha）处理器的 baby-AT 主板。NoName 最初是为 OEM 使用而设计的。LCA 芯片包含几乎所有驱动 PCI 总线和内存子系统的逻辑。所有这些都使得设计价格低廉。

由于内存接口有限，在缓存未命中时系统不是特别快。只要你留在片上缓存内，CPU 就可与 21064（第一代 Alpha）相媲美。这些板现在应该非常便宜就能获得。它是一个完整的 64 位 CPU，只是不要期望在速度方面出现奇迹。

特性：

- 166 MHz 的 21066 Alpha CPU 或 233MHz 的 21066A CPU。21068 CPU 也有可能，但更慢。
- 板载 Bcache / L2 缓存：0、256k 或 1 Mbyte（使用 DIL 芯片）
- PS/2 鼠标和键盘端口或 5 针 DIN 键盘（2 种主板型号）
- 内存：
  - 总线宽度：64 位
  - PS/2 型 72 针 36 位快速页面模式 SIMM
  - 70ns 或更好
  - 成对安装
  - 4 个 SIMM 插槽
  - 使用 ECC
- 512kB 闪存 ROM 用于控制台代码
- 2 个 16550A 串口
- 1 个并口
- 软盘接口
- 1 个嵌入式 IDE 接口
- 扩展：
  - 3 个 32 位 PCI 插槽（1 个与 ISA 共享）
  - 5 个 ISA 插槽（1 个与 PCI 共享）
- 使用 Symbios 53C810 芯片的嵌入式快速 SCSI

NoNames 在其闪存 ROM 中可以有 SRM *或* ARC 控制台固件。闪存 ROM 不够大，无法同时容纳 ARC 和 SRM 并允许软件选择备用控制台代码。但你只需要 SRM。

NoNames 的缓存是 15 或 20 ns DIL 芯片。对于 256 kByte 缓存，你可以查看你丢弃的 486 主板。对于 1 Mbyte 缓存的芯片可惜是比较稀有的品种。至少获得 256 kByte 缓存在性能方面是推荐的。没有缓存它们真的很慢。

NoName 主板有一个 PC/AT 标准电源连接器。它还有一个 3.3 伏的电源连接器。不需要急于购买新电源。3.3 伏仅在运行 3.3 伏 PCI 扩展板时才需要。这些相当罕见。

IDE 接口由 FreeBSD 支持，需要在内核配置文件中添加如下行：

```ini
device ata
```

ATA 接口使用 irq 14。

遗憾的是，SRM 控制台 *无法从* IDE 磁盘启动。这意味着你必须使用 SCSI 磁盘作为启动设备。

NoName 在串行控制台方面有点固执。它需要

```sh
>>> SET CONSOLE SERIAL
```

才会使用串行控制台。从机器上拔下键盘是不够的，这与其他大多数 Alpha 型号不同。返回图形控制台需要

```sh
>>> SET CONSOLE GRAPHICS
```

在串行控制台上。

有报告说你有时需要按 Control-Alt-Del 来吸引 SRM 的注意。我自己从未见过这种情况，但如果在开机后被空白屏幕迎接，值得尝试。

确保你使用真正的 36 位 SIMM，并且只能是 FPM（快速页面模式）DRAM。EDO DRAM 或假奇偶校验 SIMM *将无法工作*。该板使用额外的 4 位进行 ECC。出于同样的原因，33 位 FPM SIMM 也无法工作。

如果有选择，请购买 PS/2 变体主板。除了给你一个鼠标端口作为奖励外，如果你曾经想或需要运行 Tru64 Unix，它可以直接支持。“DIN 插头”变体对于 FreeBSD 应该可以正常工作。

建议阅读 [OEM 手册](ftp://ftp.digital.com/pub/DEC/axppci/design_guide.ps)。

NoName 内核的内核配置文件必须包含：

```ini
options    DEC_AXPPCI_33
cpu EV4
```

#### 2.3.2 Personal Workstation (“Miata”)

Miata 是一台旨在放在桌子下面的小型塔式机器。有多种 Miata 变体。原始 Miata 是 MX5 型号。由于它受到许多硬件设计缺陷的困扰，进行了重新设计，产生了 MiataGL。可惜，从机箱外部不容易区分变体。一个简单的检查是看机器背面是否有两个 USB 连接器。如果有，就是 MiataGL。MX5 型号在二手系统市场上往往更常见。

系统型号看起来像“Personal Workstation 433a”。Personal Workstation 有点拗口，通常缩写为 PWS。这意味着它有 433 MHz CPU，并且最初是作为 WinNT 工作站（尾部的“a”）。从第一天起就指定运行 Tru64 Unix 或 OpenVMS 的系统将带有“433au”。WinNT-Miata 可能预装了 IDE CDROM 驱动器。因此，系统通常命名为 PWS[433,500,600]a[u]。

还有一种带有 Kryotech 特殊冷却系统的 Miata 型号。Kryotech 有特殊的冷却系统，装在不同的机箱中。

特性：

- 433、500 或 600MHz 的 21164A EV56 Alpha CPU
- 21174 核心逻辑（“Pyxis”）芯片
- 板载 Bcache / L3 缓存：0、2 或 4 Mbyte（使用缓存模块）
- 内存：
  - 总线宽度：128 位宽，ECC 保护
  - 无缓冲 72 位宽 SDRAM DIMM，成对安装
  - 6 个 DIMM 插槽
  - 最大内存 1.5 GByte
- 板载快速以太网：
  - MX5 使用 21142 或 21143 以太网芯片，取决于 PCI 转接卡的版本
  - MiataGL 有 21143 芯片
  - 批量头可以是 10/100 Mbit UTP，或 10 Mbit UTP/BNC
- 2 个板载 [E]IDE 磁盘接口，基于 CMD646（MX5）或 Cypress 82C693（MiataGL）
- 1 个 Ultra-Wide SCSI Qlogic 1040 [仅 MiataGL]
- 2 个 64 位 PCI 插槽
- 3 个 32 位 PCI 插槽（在 DEC PCI-PCI 桥芯片后面）
- 3 个 ISA 插槽（与 32 位 PCI 插槽物理共享，通过 Intel 82378IB PCI 到 ISA 桥芯片）
- 2 个 16550A 串口
- 1 个并口
- PS/2 键盘和鼠标端口
- USB 接口 [仅 MiataGL]
- 基于 ESS1888 芯片的嵌入式声卡

Miata 逻辑分为两块印刷电路板。机器底部的下板有 PCI 和 ISA 插槽以及声卡等。上板有 CPU、Pyxis 芯片、内存等。请注意，MX5 和 MiataGL 使用不同的 PCI 转接板。这意味着你不能只升级到 MiataGL CPU 板（带有较新的 Pyxis 芯片），还需要不同的转接板。显然，MX5 转接器与 MiataGL CPU 板可以工作，但这绝对不是受支持或经过测试的配置。其他所有东西（机箱、接线等）对于 MX5 和 MiataGL 都是相同的。

当 DMA 通过 2 个 64 位 PCI 插槽跨越页边界时，MX5 有问题。32 位插槽没有这个问题，因为 PCI-PCI 桥芯片不允许有问题的传输。SRM 代码知道这个问题，如果 64 位插槽中有它不知道的 PCI 卡，会拒绝启动系统。SRM“已知良好”的卡允许在 64 位插槽中使用。

如果你想欺骗 SRM，可以在 SRM 提示符下输入 `set pci_device_override`。只是不要抱怨你的数据被神秘地损坏。

完整命令是：

```sh
>>> SET PCI_DEVICE_OVERRIDE <vendor_id><device_id>
```

例如：

```sh
>>> SET PCI_DEVICE_OVERRIDE 88c15333
```

最激进的方法是使用：

```sh
>>> SET PCI_DEVICE_OVERRIDE -1
```

这完全禁用 PCI ID 检查，因此你可以安装任何 PCI 卡而不会检查其 ID。为此，你需要相当新的 SRM 版本。

> **重要：** 此操作风险自负。

FreeBSD 内核在看到有缺陷的 Pyxis 芯片时会报告：

```sh
Sep 16 18:39:43 miata /kernel: cia0: Pyxis, pass 1
Sep 16 18:39:43 miata /kernel: cia0: extended capabilities: 1<BWEN>
Sep 16 18:39:43 miata /kernel: cia0: WARNING: Pyxis pass 1 DMA bug; no bets...
```

MiataGL 探测为：

```sh
Jan  3 12:22:32 miata /kernel: cia0: Pyxis, pass 1
Jan  3 12:22:32 miata /kernel: cia0: extended capabilities: 1<BWEN>
Jan  3 12:22:32 miata /kernel: pcib0: <2117x PCI host bus adapter> on cia0
```

MiataGL 没有 MX5 的 DMA 问题。使 MX5 SRM 在 64 位插槽中安装时窒息的 PCI 卡被 MiataGL SRM 毫无问题地接受。

MX5 的最新主板修订版包含该 bug 的硬件修复。SRM 不知道 ECO，会像以前一样抱怨未知的卡。FreeBSD 内核也是如此。

Miata SRM 可以从 IDE CDROM 驱动器启动。IDE 硬盘启动已知对 MiataGL 和 MX5 磁盘都有效，因此你可以从 IDE 磁盘根启动 FreeBSD。假设有合适的驱动器，MX5 上的速度约为 14 Mbytes/sec。Miata 的 CMD646 芯片支持高达 WDMA2 模式，因为该芯片对于 UDMA 使用来说太有问题了。

Miata MX5 通常使用基于 Qlogic 1040 的 SCSI 适配器。这些可由 SRM 控制台启动。请注意，Adaptec 卡*不可*由 Miata SRM 控制台启动。

MiataGL 在 PCI 转接卡上有比某些 MX5 转接卡版本更快的 PCI-PCI 桥芯片。一些 MX5 转接器有与 MiataGL *相同*的芯片。总而言之，有很多变化。

并非所有 VGA 卡都能在 PCI-PCI 桥后面工作。这表现为根本没有视频。变通方法是将 VGA 卡放在桥“之前”的 64 位 PCI 插槽之一。使用 64 位插槽的图形性能通常明显更好。

MX5 和 MiataGL 都有板载声卡芯片 ESS1888。它模拟 SoundBlaster，可以通过在内核配置文件中放入

```ini
device pcm
device  sbc
```

来启用。

如果你的 Miata 安装了可选缓存板，请确保它牢固就位。观察到略微松动的缓存会导致奇怪的崩溃。缓存模块在 MX5 和 MiataGL 之间是相同的。

安装 2Mb 缓存模块可实现 10-15% 的速度提升（基于 buildworld 经 过的时间），但会*降低* 64 位 PCI 卡的 PCI DMA 读带宽。对 64 位 Myrinet 卡的基准测试结果显示，带宽从 149 Mbytes/sec 下降到 115 Mbytes/sec。在使用 64 位 PCI 适配器进行真正高速操作时，需要记住这一点。

虽然硬件允许你安装高达 1.5Gbyte 内存，但 FreeBSD 限于 1Gbyte，因为 DMA 代码不能正确处理 1Gbyte 以上的内存。

升级到更快的 CPU 非常简单，更换 CPU 芯片并将时钟倍频拨码开关设置为新 CPU 的速度即可。

如果你在停止 FreeBSD 后遇到类似如下的 SRM 错误：

```sh
ERROR: scancode 0xa3 not supported on PCXAL
```

你应该将 SRM 固件更新到 V7.2-1 或更高版本。此 SRM 版本首先在固件更新 CD V5.7 上提供，或在 <http://www.compaq.com/> 上提供。此 SRM 问题在 Miata MX5 和 Miata GL 上都已修复。

USB 由 FreeBSD 4.1 及更高版本支持。

拆卸机器前断开电源线，软电源开关在机器关闭时*甚至*保持部分逻辑通电。

Miata 内核的内核配置文件必须包含：

```ini
options    DEC_ST550
cpu EV5
```

#### 2.3.3 Evaluation Board 64 系列

在试图普及 Alpha CPU 时，DEC 生产了多款所谓的评估板。该系列的成员有 EB64、EB64+、AlphaPC64（代号“Cabriolet”）。该系列的非 DEC 成员是 Aspen Alpine。EB64 系列评估板具有以下特性集：

- 21064 或 21064A CPU，150 至 275 MHz
- 内存：
  - 内存总线宽度：128 位
  - PS/2 型 72 针 33 位快速页面模式 SIMM
  - 70ns 或更好
  - 成组安装（每组 4 个）
  - 8 个 SIMM 插槽
  - 使用奇偶校验内存
- Bcache / L2 缓存：0、512 kByte、1 Mbyte 或 2 Mbyte
- 21072（“APECS”）芯片组
- Intel 82378ZB PCI 到 ISA 桥芯片（“Saturn”）
- 双 16550A 串口
- 并行打印机端口
- Symbios 53C810 Fast-SCSI（AlphaPC64 上没有）
- IDE 接口（仅在 AlphaPC64 上）
- 嵌入式 10 Mbit 以太网（AlphaPC64 上没有）
- 2 个 PCI 插槽（AlphaPC64 上有 4 个插槽）
- 3 个 ISA 插槽

Aspen Alpine 略有不同，但与 EB64+ 足够接近，可以运行 EB64+ SRM EPROM（我的就是）。Aspen Alpine 没有嵌入式以太网，有 3 个而不是 2 个 PCI 插槽。它带有 2 Mbyte 已焊接在主板上的缓存。它有跳线可以选择使用 60、70 或 80ns SIMM 速度。

36 位 SIMM 工作正常，3 位简单地保持未使用。请注意，系统使用快速页面模式内存，而不是 EDO 内存。

EB64+ SRM 控制台代码 housed 在 UV 可擦除 EPROM 中。EB64+ 没有简单的闪存 SRM 升级。EB64+ 可用的最新 SRM 版本反正相当古老。

EB64+ SRM 可以启动 53C810 和 Qlogic1040 SCSI 适配器。Qlogic 的陷阱是 SRM 下载到 Qlogic 芯片的固件非常旧。EB64+ SRM 没有可用的更新。所以你也只能使用旧的 Qlogic 位。当我想在 Alpine 上使用 Qlogic 的 Ultra-SCSI 驱动器时遇到了很多问题。FreeBSD 内核可以编译为包含更新的 Qlogic 固件版本。这不是默认的，因为它为内核增加了数百 kByte 的膨胀。在 FreeBSD 4.1 及更高版本中，isp 固件包含在内核可加载模块中。所有这些可能意味着你需要使用非 Qlogic 适配器来启动。

AlphaPC64 板通常带有 ARC 控制台固件。SRM 控制台代码可以从软盘加载到闪存 ROM 中。

AlphaPC64 的 IDE 接口无法从 SRM 控制台启动。启用它需要在内核配置文件中添加以下行：

```ini
device ata
```

ATA 接口使用 irq 14。

请注意，这些板需要为 CPU 提供 3.3 伏的电源。

对于 EB64 系列机器，内核配置文件必须包含：

```ini
options    DEC_EB64PLUS
cpu EV4
```

#### 2.3.4 Evaluation Board 164（“EB164、PC164、PC164LX、PC164SX”）系列

EB164 是基于 21164A CPU 的较新设计评估板。此设计已用于“衍生”多个变体，其中一些由 OEM 制造商/组装店使用。Samsung 制造了自己的 PC164LX，只有 32 位 PCI，而 Digital 变体有 64 位 PCI。

- 21164A，多种速度变体 [EB164、PC164、PC164LX]
- 21164PC [仅在 PC164SX 上]
- 21171（Alcor）芯片组 [EB164]
- 21172（Alcor2）芯片组 [PC164]
- 21174（Pyxis）芯片 [164LX、164SX]
- Bcache / L3 缓存：EB164 使用特殊缓存 SIMM
- 内存总线：128 位 / 256 位
- 内存：
  - PS/2 型 SIMM，成组安装（每组 4 或 8 个）
  - 36 位，快速页面模式，使用 ECC [EB164 / PC164]
  - SDRAM DIMM，成组安装（每组 2 个），可使用非 ECC 或 ECC [PC164SX]
  - SDRAM DIMM，成组安装（每组 2 个），使用 ECC [PC164LX]
- 2 个 16550A 串口
- PS/2 型键盘和鼠标
- 软盘控制器
- 并口
- 32 位 PCI
- 64 位 PCI [某些型号]
- 通过 Intel 82378ZB PCI 到 ISA 桥芯片的 ISA 插槽

使用 8 个 SIMM 实现 256 位宽内存可以比 4 个 SIMM/128 位宽内存产生有趣的速度提升。显然，所有 8 个 SIMM 必须是相同类型才能使其工作。系统必须明确设置为使用 8 个 SIMM 内存排列。你必须有 8 个 SIMM，分布在 2 个存储库中的 4 个 SIMM 将无法工作。对于 AlphaPC164，你最多可以有 1Gbyte 的 RAM，使用 8 个 128Mbyte SIMM。手册指出最大为 512 Mbyte。

SRM 可以从 Qlogic 10xx 板或 Symbios 53C810[A] 启动。较新的 Symbios 810 修订版（如 Symbios 810AE）不被 PC164 上的 SRM 识别。PC164 SRM 似乎不能识别基于 Symbios 53C895 的主机适配器（用 Tekram DC-390U2W 测试）。另一方面，据报道某些无名 Symbios 53C985 板可以工作。像 Tekram DC-390F（基于 Symbios875）之类的卡已确认在 PC164 上工作正常。Symbios 53C896 已确认无法工作。

Symbios 53C825[a] 也可作为启动适配器。Diamond FirePort 虽然基于 Symbios 芯片，但不能由 PC164SX SRM 启动。据报道 PC164SX 可以与基于 Symbios825、Symbios875、Symbios895 和 Symbios876 的卡正常启动。此外，据报道 Adaptec 2940U 和 2940UW 可用于启动（在 SRM V5.7-1 上验证）。Adaptec 2930U2 和 2940U2[W] 无法工作。

带有 SRM 固件版本 5.8 或更高版本的 164LX 和 164SX 可以从 Adaptec 2940 系列适配器启动。使用 Adaptec 3940UW 进行的测试表明，该适配器不受支持。164SX SRM 识别 Intel 8255x 以太网卡，显示为 eia。使用这样的 NIC 允许网络启动。

总结：这个系列的机器在 SCSI 适配器方面“被赐予”了具有挑战性的兼容性。

在 164SX 上，你最多可以有 1 Gbyte 的 RAM。据报道，4 个常规（PC100 或 PC133）256MB DIMM 工作正常。512MB DIMM 是否也能工作目前未知。你可以使用 ECC 或非 ECC DIMM。非 ECC DIMM 与 PC 中常见的相同。可惜 164SX 对它喜欢的 DIMM 相当挑剔，所以准备好测试和实验。

PCI 桥芯片有时不被 164SX 欣赏，在这些情况下它们会导致 SRM 错误和内核 panic。这似乎取决于卡是否被 SRM 控制台识别并因此正确初始化。164SX 的板载 IDE 接口相当慢，Promise 卡可提供 3-4 倍的速度提升。

在 PC164 上，SRM 有时似乎会丢失其变量设置。“对于 PC164，当前迷信说，为了避免丢失设置，你需要先降级到 SRM 4.x，然后升级到 5.x。”观察到的一个示例错误是：

```sh
ERROR: ISA table corrupt!
```

降级到 SRM4.9，然后

```sh
>>> ISACFG -INIT
```

接着

```sh
>>> INIT
```

使问题消失。一些 PC164 所有者报告说他们从未见过该问题。

在 PC164SX 上，AlphaBIOS 允许你选择在下一次开机时使用 SRM 作为控制台。此选择似乎没有任何效果。换句话说，无论你选择什么，你都会得到 AlphaBIOS。解决方法是用 PC164SX 的 SRM 代码重新刷新控制台 ROM。这将覆盖 AlphaBIOS 并为你提供所需的 SRM 控制台。SRM 代码可以在 Compaq 网站上找到。

164LX 在其闪存 ROM 中可以有 SRM 控制台代码或 AlphaBIOS 代码，因为闪存 ROM 太小，无法同时容纳两者。

假设你的 SRM 版本足够新，PC164 可以从 IDE 磁盘启动。

EB164 需要提供 3.3 伏的电源。PC164 不实现 ATX 电源所需的 PS_ON 信号。一个简单的将此信号拉至接地的开关允许你使用标准 ATX 电源。

对于 EB164 类机器，内核配置文件必须包含：

```ini
options         DEC_EB164
cpu             EV5
```

#### 2.3.5 AlphaStation 200（“Mustang”）和 400（“Avanti”）系列

Digital AlphaStation 200 和 400 系列系统是早期的低端基于 PCI 的工作站。200 和 250 系列是桌面机箱，400 系列是桌边小型塔式机箱。

特性：

- 100 至 333 MHz 速度的 21064 或 21064A CPU
- DECchip 21071-AA 核心逻辑芯片组
- Bcache / L2 缓存：512 Kbytes（200 和 400 系列）或 2048KBytes（250 系列）
- 内存：
  - 64 位总线宽度
  - 8 至 384 MBytes RAM
  - 70 ns 或更好的快速页面 DRAM
  - 三对安装（200 和 400 系列）
  - 两个四联组，即四个一组的存储库（250 系列）
  - 内存子系统使用奇偶校验
- PS/2 键盘和鼠标端口
- 两个 16550 串口
- 并口
- 软盘驱动器接口
- 32 位 PCI 扩展插槽（AS400 系列有 3 个，AS200 和 250 系列有 2 个）
- ISA 扩展插槽（AS400 系列有 4 个，AS200 和 250 系列有 2 个）（一些 ISA/PCI 插槽物理共享）
- 嵌入式基于 21040 的以太网（200 和 250 系列）
- 嵌入式 Symbios 53c810 Fast SCSI-2 芯片
- Intel 82378IB（“Saturn”）PCI-ISA 桥芯片
- 图形为嵌入式 TGA 或 PCI VGA（取决于型号）
- 16 位声卡（200 和 250 系列）

系统使用奇偶校验内存 SIMM，但这些不需要 36 位宽 SIMM。33 位宽 SIMM 足够，36 位 SIMM 也可以接受。EDO 或 32 位 SIMM 无法工作。支持 4、8、16、32 和 64 Mbyte SIMM。

假设你的内核配置文件中有以下行，AS200 和 AS250 声音硬件据说工作正常：

```ini
device pcm
```

声音设备使用端口 0x530、IRQ 10 和 drq 0。你还需要在 **device.hints** 文件中指定 flags 0x10011。

AlphaStation 200 和 250 系列有自动 SCSI 终端器。这意味着一旦你将电缆插到外部 SCSI 连接器上，系统的内部终端器就会被禁用。这也意味着你不应该留下未终止的电缆插在机器上。

AlphaStation 400 系列有一个控制终端的 SRM 变量。如果连接了外部 SCSI 设备，必须使用以下命令设置此 SRM 变量：

```sh
>>> SET CONTROL_SCSI_TERM EXTERNAL
```

如果只有内部 SCSI 设备，请使用：

```sh
>>> SET CONTROL_SCSI_TERM INTERNAL
```

对于 AlphaStation-[24][05]00 机器，内核配置文件必须包含：

```ini
options    DEC_2100_A50
cpu EV4
```

#### 2.3.6 AlphaStation 500 和 600（EV5 用“Alcor”和“Maverick”，EV56 用“Bret”）

AS500 和 600 是基于 EV5 / PCI 的高端工作站。基于 EV6 的机器在此期间已取代它们成为前沿产品。AS500 是深蓝色机箱（TopGun blue）的台式机，AS600 是坚固的桌边机箱。AS600 有一个漂亮的 LCD 面板，用于观察 SRM 启动的早期阶段。

特性：

- 266、300、333、366、400、433、466 或 500 MHz 的 21164 EV5 CPU（AS500）或 266、300 或 333 MHz（AS600）
- 21171（Alcor）或 21172（Alcor2）核心逻辑芯片组
- 缓存：
  - 2 或 4 Mb L3 / Bcache（AS600 266 MHz）
  - 4 Mb L3 / Bcache（AS600 300 MHz）
  - 2 或 8 Mb L3 / Bcache（8 Mb 仅在 500 MHz 版本上）
  - 2 至 16 Mb L3 / Bcache（AS600；3 个缓存 SIMM 插槽）
- 内存总线宽度：256 位
- AS500 内存：
  - 工业标准 72 位宽缓冲快速页面模式 DIMM
  - 8 个 DIMM 插槽
  - 成组安装（每组 4 个）
  - 最大内存 1 GB（333 MHz CPU 上最大 512 Mb）
  - 使用 ECC
- AS600 内存：
  - 工业标准 36 位快速页面模式 SIMM
  - 32 个 SIMM 插槽
  - 成组安装（每组 8 个）
  - 最大内存 1 GB
  - 使用 ECC
- 基于 Qlogic 1020 的宽 SCSI 总线（AS500 有 1 条总线/芯片，AS600 有 2 条总线/芯片）
- 基于 21040 的 10 Mbit 以太网适配器，有 Thinwire 和 UTP 连接器
- 扩展：
  - AS500：
    - 3 个 32 位 PCI 插槽
    - 1 个 64 位 PCI 插槽
  - AS600：
    - 2 个 32 位 PCI 插槽
    - 3 个 64 位 PCI 插槽
    - 1 个 PCI/EISA 物理共享插槽
    - 3 个 EISA 插槽
    - 1 个 PCI 和 1 个 EISA 插槽默认被占用
- 21050 PCI-to-PCI 桥芯片
- Intel 82375EB PCI-EISA 桥（仅 AS600）
- 2 个 16550A 串口
- 1 个并口
- 16 位音频 Windows Sound System，在专用插槽中（AS500）或 EISA 插槽中（AS600，这是一块 ISA 卡）
- PS/2 键盘和鼠标端口

早期机器有 Fast SCSI 接口，后期机器支持 Ultra SCSI。AS500 将其单一 SCSI 总线与内部和外部设备共享。对于 Fast SCSI 总线，你被限制在机箱外部 1.8 米的总线长度。AS500 Qlogic ISP1020A 芯片可以通过设置 SRM 变量以 Ultra 模式运行。然而，FreeBSD 遵循 Qlogic 芯片勘误并将总线速度限制为 Fast。

小心 AS500 上的古老 SRM 版本。当 FreeBSD 报告奇怪 的 SCSI 速度时，如

```sh
cd0 at isp0 bus 0 target 4 lun 0
cd0: <DEC RRD45   DEC 0436> Removable CD-ROM SCSI-2 device
cd0: 250.000MB/s transfers (250.000MHz, offset 12)
```

是时候进行 SRM 控制台固件升级了。

AS600 有一个 Qlogic SCSI 芯片专用于内部设备，而另一个 Qlogic SCSI 芯片专用于外部 SCSI 设备。将卡保持在 DEC 推荐的最顶部 32 位 PCI 插槽中，除了两个 SCSI 接口外，该卡还装有以太网芯片。如果将其插入 64 位插槽之一，会出现机器检查。

在 AS500 中，DIMM 以“物理交错”布局成组安装（每组 4 个）。因此，4 个 DIMM 的一组 *不是* 4 个物理上相邻的 DIMM。请注意，DIMM *不是* SDRAM DIMM。

在 AS600 中，内存 SIMM 放置在两个内存子卡上。SIMM 成组安装（每组 8 个）。两个内存子卡必须相同地填充。

请注意，AS500 和 AS600 都是 EISA 机器。这意味着在添加 EISA 卡或更改板载 I/O 的配置设置后，你必须从软盘运行 EISA 配置实用程序（ECU）。对于没有物理 EISA 插槽的 AS500，ECU 用于配置板载声音接口等。

AS500 板载声音可以通过添加如下行来使用：

```ini
device pcm
```

到内核配置文件。

使用 ECU，我将我的 AS500 配置为使用 IRQ 10、端口 0x530、drq 0。相应的条目连同 flags 0x10011 必须放入 **device.hints** 文件。请注意，flags 值相当非标准。

AS600 的 PCI 插槽有一个特点。AS600（或者更确切地说是包含 SCSI 适配器的 PCI 扩展卡）不允许 I/O 端口映射，因此它后面的所有设备必须使用内存映射。如果在让 Qlogic SCSI 适配器工作时遇到问题，请将以下选项添加到 **/boot/loader.rc**：

```ini
set isp_mem_map=0xff
```

这可能需要在引导安装内核之前在引导加载程序提示符下输入。

对于 AlphaStation-[56]00 机器，内核配置文件必须包含：

```ini
options    DEC_KN20AA
cpu EV5
```

#### 2.3.7 AlphaServer 1000（“Mikasa”）、1000A（“Noritake”）和 800（“Corelle”）

AlphaServer 1000 和 800 系列机器旨在作为部门服务器。它们在包装和主板/CPU 方面有多种变体。一般来说，有基于 21064（EV4）CPU 的机器和基于 21164（EV5）的机器。CPU 在子卡上，CPU 类型（EV4 或 EV5）必须与使用的主板匹配。

AlphaServer 800 有一个小得多的迷你塔式机箱，它缺少 StorageWorks SCSI 热插拔机箱。AS1000 和 AS1000A 之间的主要区别在于 AS1000A 有 7 个 PCI 插槽，而 AS1000 只有 3 个 PCI 插槽并有 EISA 插槽。

带有 EV5/400 MHz CPU 的 AS800 后来被重新品牌化为“DIGITAL Server 3300[R]”，带有 EV5/500 MHz CPU 的 AS800 后来被重新品牌化为“DIGITAL Server 3305[R]”。

特性：

- 200、233 或 266 MHz 的 21064 EV4[5] CPU，300、333 或 400 MHz 的 21164 EV5[6] CPU（或仅 AS800 的 500 MHz）
- 内存：
  - 总线宽度：128 位带 ECC
  - AS1000[A]：
    - 72 针 36 位快速页面模式 SIMM，70ns 或更好
    - 16（EV5 机器）或 20（EV4 机器）个 SIMM 插槽
    - 最大内存 1 GB
    - 使用 ECC
  - AS800：使用 60ns 3.3 伏 EDO DIMM
- 嵌入式 VGA（在某些主板型号上）
- 3 个 PCI、2 个 EISA、1 个 64 位 PCI/EISA 组合（AS800）
- 7 个 PCI、2 个 EISA（AS1000A）
- 2 个 PCI、1 个 EISA/PCI、7 个 EISA（AS1000）
- 基于 Symbios 810 [AS1000] 或 Qlogic 1020 [AS1000A] 的嵌入式 SCSI

AS1000 系列机器有多种机箱类型。落地式、机架安装式、带或不带 StorageWorks SCSI 机箱等。电子设备是相同的。

AS1000 系统：所有基于 EV4 的机器使用标准 PS/2 型 36 位 72 针 SIMM，成组安装（每组 5 个）。第五个 SIMM 用于 ECC。所有基于 EV5 的机器使用标准 PS/2 型 36 位 72 针 SIMM，成组安装（每组 4 个）。ECC 基于 SIMM 的 4 个额外位（36 位中的 4 位）完成。EV5 主板有 16 个 SIMM 插槽，EV4 主板有 20 个插槽。


在串行控制台上。

#### 2.3.8 DS10/VS10/XP900 (“Webbrick”)/ XP1000 (“Monet”)/ DS10L (“Slate”)

Webbrick 和 Monet 是基于 EV6 CPU 和 Tsunami 芯片组的高性能工作站/服务器。Tsunami 也用于更高端的系统，因此有充足的性能可供提供。DS10、VS10 和 XP900 是本质上相同系统的不同名称。区别在于支持的软件和选项。DS10L 是 DS10 在 1U 高机架安装机箱中的衍生机器。DS10L 专为 ISP 和 HPTC 集群（例如 Beowulf）设计。

- 内存：
  - 工业标准 200 针 83 MHz 缓冲 ECC SDRAM DIMM
  - DS10 有 4 个 DIMM 插槽；最大内存 2GB
  - DS10L 有 2 个 DIMM 插槽；最大内存 1GB

- 21271 核心逻辑芯片组（“Tsunami”）
- 2 个板载 21143 快速以太网控制器
- AcerLabs M5237（Aladdin-V）USB 控制器（已禁用）

系统有一个智能电源控制器。这意味着当它关闭时，部分系统仍然通电（如 ATX 型 PC 电源）。维修机器前，请拔下电源线。

Webbrick 装在类似于旧 21164“Maverick”工作站的桌面型机箱中，但该机箱提供对组件更好的访问。如果你打算构建一个服务器农场，可以将它们机架安装在 19 英寸机架中；它们高 3U。Slate 高 1U 但只有一个 PCI 插槽。


##### 2.3.8.2 “Monet”

- 1 个并口

- 嵌入式 16 位 ESS ES1888 声卡芯片

Monet 装在与 Miata 机箱非常相似的迷你塔式机箱中。

板载 Qlogic UW-SCSI 芯片最多支持 4 个内部设备。板载 SCSI 没有外部连接器。

对于 500 MHz CPU，83 MHz DIMM 即可。Compaq 为所有 CPU 速度指定 PC100 DIMM。DIMM 成组安装（每组 4 个），从标记为“0”的 DIMM 插槽开始。内存容量最大 4 GB。DIMM“物理交错”安装，注意插槽的标记。Monet 的内存带宽是 Webbrick 的两倍。DIMM 位于 CPU 子卡上。请注意，系统使用 ECC RAM，因此你需要 72 位 DIMM（不是通用 PC 级 64 位 DIMM）。

EIDE 接口可用/可从 SRM 启动，因此 FreeBSD 可以根安装在 EIDE 磁盘上。虽然 Cypress 芯片有 2 个 EIDE 通道的潜力，但 Monet 只使用其中一个。

USB 接口由 FreeBSD 支持。如果在使用 USB 接口时遇到问题，请检查 SRM 变量 `usb_enable` 是否设置为 `on`。你可以通过执行以下操作来更改此设置：

```sh

```

> **重要：** 并非所有 VGA 卡都能在 PCI-PCI 桥后面工作（即在插槽 4 和 5 中）。只有正确实现 VGA 旧式寻址的卡才能工作。变通方法是将 VGA 卡放在桥“之前”。

声卡芯片目前不被 FreeBSD 支持。

内核配置文件必须包含：

```ini

cpu EV5
```

> **注意：**
>
> 与预期相反，内核配置文件中没有定义 `cpu EV6`。`cpu EV5` 是强制性的，以使 `config` 满意。

#### 2.3.11 AlphaServer 2000 (“DemiSable”), 2100 (“Sable”), 2100A (“Lynx”)

AlphaServer 2[01]00 机器旨在作为部门服务器。这是中型铁器。它们是多 CPU 机器，最多可安装 2 个 CPU（AS2000）或 4 个 CPU（2100[A]）。有落地式和 19“ 机架安装机箱。机架安装变体有不同的 I/O 扩展插槽数量、不同的最大 CPU 数量和不同的最大内存大小。其中一些机箱带有集成 StorageWorks 架，用于容纳热插拔 SCSI 磁盘。曾经有一个升级计划，可以通过更换 I/O 背板（C-bus 背板保持不变）将你的 Sable 机器转换为 Lynx。CPU 升级也可用。

- 200、233、275 MHz 的 21064 EV4[5] CPU 或 250、300、375、400 MHz 的 21164 EV5[6] CPU
- 缓存：随 CPU 型号而变化；每个 CPU 1、4 或 8 Mbyte
- 嵌入式软盘控制器，驱动 2.88 Mbyte 驱动器
- 嵌入式 10Mbit 21040 以太网 [仅 AS2100]
- 2 个串口
- 1 个并口
- PS/2 型键盘和鼠标端口

规格为 200 MHz 的 CPU 实际上以 190 MHz 运行。最大 CPU 数量为 4。所有 CPU 必须是相同类型/速度。

如果任何处理器被标记为失败，即使它们已被更换（或重新插拔），它们也将保持标记为失败状态，直到你在 SRM 控制台上发出命令

```sh
>>> CLEAR_ERROR ALL
```

并循环电源。这对于其他模块（IO 和内存）也可能是正确的，但尚未验证。

机器使用专用内存板。这些板位于与 CPU 板共享的 128 位 C-bus 上。DemiSable 最多支持 1GB，Sable 最多 2GB。其中一个内存总线插槽可以容纳 CPU 或内存卡。一台 4 CPU 机器最多可以有 2 块内存板。

一些内存板型号容纳 SIMM。这些被称为 SIMM 承载器。还有焊接内存芯片而不是 SIMM 的内存模块。这些被称为“扁平内存模块”。

SIMM 板成组使用（每组 8 个）72 针 36 位 FPM 内存，70ns 或更快。支持的 SIMM 类型有 1M x36 位（4 Mbyte）、2M x36 位（8 Mbyte）和 4M x36 位（16 Mbyte）。每块内存板可容纳 4 个 SIMM 存储库。单块内存板上不能混合 SIMM 尺寸。第一个内存模块必须先填满 SIMM，然后才能开始填充下一个内存模块。请注意，插槽之间的间距不大，所以请确保你的 SIMM 物理上合适（在购买之前）。

Lynx 和 Sable 在串行控制台方面有点固执。它们需要

```sh
>>> SET CONSOLE SERIAL
```

才会使用串行控制台。从机器上拔下键盘是不够的，这与其他许多 Alpha 型号不同。返回图形控制台需要

```sh
>>> SET CONSOLE GRAPHICS
```

在串行控制台上。在 Lynx 上，将 VGA 卡保持在主 PCI 插槽之一。EISA VGA 卡对插槽不敏感。

机器配备小型 OCP（操作员控制面板）LCD 屏幕。在系统初始化期间，此屏幕上会显示自检消息。你可以使用 SRM 在那里放置你自己的小文本：

```sh
>>> SET OCP_TEXT "FreeBSD"
```

SRM 命令

```sh
>>> SHOW FRU
```

产生配置概述，包括模块序列号、硬件修订版和错误日志计数。

Sable、DemiSable 和 Lynx 都有基于 Symbios 810 的板载 Fast SCSI。检查它是否设置为 Fast SCSI 速度：

```sh
>>> SHOW PKA0_FAST
```

设置为 1 时，它正在协商 Fast 速度。

```sh
>>> SET PKA0_FAST 1
```

启用 Fast SCSI 速度。

AS2100[A] 配备 StorageWorks 7 插槽 SCSI 机箱。机柜内可以添加第二个机箱。AS2000 有一个 7 插槽 SCSI 机箱，无法扩展另一个。请注意，这些机箱中的插槽位置与标准 StorageWorks 架的 SCSI ID 映射不同。使用单总线配置时，从上到下的插槽 ID 为 0、4、1、5、2、6、3。

机箱也可以设置为提供两条独立的 SCSI 总线。这用于嵌入式 RAID 控制器，如 KZPSC（Mylex DAC960）。分体总线插槽 ID 分配，从上到下：0A、0B、1A、1B、2A、2B、3A、3B。其中 A 和 B 表示 SCSI 总线。在单总线配置中，SCSI 机箱背面的终端器模块位于顶部。跳线模块位于底部。对于分体总线操作，这两个模块反转。通过注意终端器上的芯片，可以区分终端器和跳线。跳线上没有任何有源组件。

DemiSable 有 7 个 EISA 插槽和 3 个 PCI 插槽。Sable 有 8 个 EISA 和 3 个 PCI 插槽。Lynx 较新，有 8 个 PCI 和 3 个 EISA 插槽。Lynx PCI 插槽分为 4 组。最靠近 CPU/内存插槽的 4 个 PCI 插槽是主插槽，因此在 PCI 桥芯片之前。请注意，与预期相反，主 PCI 插槽是编号最高的插槽（PCI4 - PCI7）。

```sh
>>> RUNECU
```

来完成。

> **注意：**
>
> EISA 插槽目前不受支持，但 Compaq Qvision EISA VGA 适配器被视为 ISA 设备。因此，它作为控制台工作正常。

曾计划用于 C-bus 的特殊扩展 I/O 模块。它们是否见过天日尚不清楚。无论如何，FreeBSD 从未在 ExtIO 模块上验证过。


- 1 个并口
- PS/2 型键盘和鼠标端口

Rawhide 可以配备多种 CPU 模块。CPU 模块有带外部缓存和不带外部缓存的版本。在所有情况下，安装的 CPU 模块必须始终是相同速度的。NT-only 和完整 Tru64/VMS CPU 的混合工作正常。然而，这将导致系统向操作系统报告自己是 Digital Server 730x（即 NT-only 变体）。FreeBSD 不在乎，但这样的系统将不允许 Tru64 或 VMS 运行。

Rawhide 最多使用 8 个 RAM 模块。这些模块成对使用，并向总线提供 72 位（这包括 ECC 位）。内存可以是 EDO RAM 或同步 DRAM。完全填充的 AS4100 有 4 对内存模块。AS4000 型号限于 2 对内存模块。如果有选择，使用 SDRAM 以获得最佳性能。最高容量的内存板必须在标记为 MEM0L 和 MEM0H 的内存插槽中。允许混合内存板尺寸。EDO 和 SDRAM 的混合也可以工作（假设你不会在单个模块对中混合 EDO 和 SDRAM）。EDO 和 SDRAM 的混合会导致*整个*内存子系统以较慢的 EDO 时序运行。

Rawhide 有一个嵌入式 Symbios 810 芯片，为你提供窄型 fast-SCSI 总线。通常只有 SCSI CDROM 由该接口驱动。

Rawhide 有 8 个 64 位 PCI / 3 个 EISA 插槽扩展背板（称为“Saddle”模块）。有 2 条独立的 PCI 总线，PCI0 和 PCI1。PCI0 有 1 个专用 PCI 插槽和（共享的）3 个 PCI/EISA 插槽。PCI0 还有一个 PCI/EISA 桥，驱动串口和并口、键盘/鼠标等。PCI1 有 4 个 PCI 插槽和一个 Symbios 810 SCSI 芯片。VGA 控制台卡必须安装在连接到 PCI0 的插槽中。

当前 FreeBSD 实现在处理 PCI 桥方面有问题。目前有一个有限的修复，允许单级、单设备 PCI 桥。该修复允许使用 Digital 提供的 Qlogic SCSI 卡，该卡位于 21054 PCI 桥芯片后面。

> **注意：**
>
> EISA 插槽目前不受支持，但 Compaq Qvision EISA VGA 适配器被视为 ISA 设备。因此，它作为控制台工作。如果你在机器中使用 EISA 选项，必须从软盘运行 EISA 配置实用程序（ECU）。帮自己一个忙，使用 Tru64/OpenVMS ECU，而不是 WindowsNT ECU。

Rawhide 采用基于 I2C 的电源控制器系统。如果你想确保所有电源都已从系统中移除，请从系统中拔下所有主电源线。

Rawhide 具有 RCM 功能，这意味着你可以远程开关电源、重置等。另请参阅本文档 DS10 部分中 RMC 的描述。RCM 与 RMC 不是拼写错误，我查阅的各种文档交替使用这两个首字母缩略词。请注意，如果你希望远程开关电源工作，你需要将一个小型 DC 适配器连接到机器，以便为 RCM 逻辑供电。你需要向键盘连接器旁边的小型入口提供 9-12V DC。

内核配置文件必须包含：

```ini
options    DEC_KN300
cpu EV5
```

#### 2.3.13 AlphaServer 1200（“Tincup”）和 AlphaStation 1200（“DaVinci”）

AlphaServer 1200 机器是 AlphaServer 1000A 的继任者。它使用与 1000A 相同的机箱，但逻辑基于 AlphaServer 4000 设计。这些是多 CPU 机器，单台机器最多可有 2 个 CPU。基本磁盘存储位于 StorageWorks 架中。面向 NT 市场的 AS1200 被命名为 DIGITAL Server 5300（5/400 CPU）和 DIGITAL Server 5305（5/533 CPU）。

特性：

- 400 或 533 Mhz 的 21164A EV56 CPU
- 缓存：每个 CPU 4 Mbyte
- 内存总线：128 位带 ECC，两块内存子板上的 DIMM 内存
- 嵌入式软盘控制器
- 2 个串口
- 1 个并口
- PS/2 型键盘和鼠标端口

AS1200 使用 2 块内存子卡。每块卡上有 8 个 DIMM 插槽。DIMM 必须成对安装。最大内存大小为 4 GByte。插槽必须按顺序填充，如果使用不同尺寸的 DIMM，插槽 0 必须包含最大尺寸的 DIMM。AS1200 为 DIMM 采用固定起始地址，每对 DIMM 从 512 Mbyte 边界开始。这意味着如果使用小于 256 Mbyte 的 DIMM，系统的物理内存映射将包含“空洞”。支持的 DIMM 尺寸为 64 Mbyte 和 256 Mbyte。DIMM 基于 72 位 SDRAM，因为系统采用 ECC。

> **注意：**
>
> FreeBSD 目前最多支持 2GByte。

AS1200 有一个嵌入式 Symbios 810 驱动的 Fast SCSI 总线。

Tincup 有 5 个 64 位 PCI 插槽、1 个 32 位 PCI 插槽和 1 个 EISA 插槽（与其中一个 64 位 PCI 插槽物理共享）。有 2 条独立的 PCI 总线，PCI0 和 PCI1。PCI0 有 32 位 PCI 插槽和最顶部的 2 个 64 位 PCI 插槽。PCI0 还有一个 Intel 82375EB PCI/EISA 桥，驱动串口和并口、键盘/鼠标等。PCI1 有 4 个 64 位 PCI 插槽和一个 Symbios 810 SCSI 芯片。VGA 控制台卡必须安装在连接到 PCI0 的插槽中。

系统采用基于 I2C 的电源控制器系统。如果你想确保所有电源都已从系统中移除，请从系统中拔下主电源线。Tincup 使用双电源，以负载共享模式而不是冗余对方式工作。

内核配置文件必须包含：

```ini
options    DEC_KN300
cpu EV5
```

#### 2.3.14 Alpha Processor Inc. UP1000

UP1000 是一款基于 21264a CPU 的 ATX 主板，CPU 本身位于 Slot B 模块中。它通常装在 ATX 塔式机箱中。

特性：

- 600 或 700 MHz 的 21264a Alpha CPU，位于 Slot B 模块中（包括冷却风扇）
- 内存总线：到 L2 缓存 128 位，从 Slot B 到 AMD-751 64 位
- 板载 Bcache / L2 缓存：2MB（600Mhz）或 4MB（700Mhz）
- AMD AMD-751（“Irongate”）系统控制器芯片
- Acer Labs M1543C PCI-ISA 桥控制器/超级 I/O 芯片
- PS/2 鼠标和键盘端口
- 内存：168 针 PC100 无缓冲 SDRAM DIMM，3 个 DIMM 插槽，支持的 DIMM 尺寸为 64、128 或 256 Mb
- 2 个 16550A 串口
- 1 个 ECP/EPP 并口
- 软盘接口
- 2 个嵌入式 Ultra DMA33 IDE 接口
- 2 个 USB 端口
- 扩展：
  - 4 个 32 位 PCI 插槽
  - 2 个 ISA 插槽
  - 1 个 AGP 插槽

Slot B 是一个盒状机箱，容纳 CPU 和缓存的子板。它有 2 个小风扇用于冷却。风扇很响。

机器需要支持 ECC 的 DIMM，即 72 位的。这似乎没有在 UP1000 文档中记录。系统通过 SM 总线访问 DIMM 上的串行 EEPROM。请注意，如果只使用单个 DIMM，它必须安装在插槽 *2* 中。这有点反直觉。

根据制造商的说法，UP1000 需要 400Watt ATX 电源。从主板和 CPU 的功耗来看，这可能有点过于保守/悲观。但一如既往，你必须考虑你的扩展卡和外围设备。M1543C 芯片包含电源管理功能和温度监视（通过 I2C / SM 总线）。

你的 UP1000 很可能默认只带有 AlphaBios。SRM 控制台固件可从 Alpha Processor Inc. 网站获得。它目前有测试版，在将 FreeBSD 移植到 UP1000 期间成功使用。

嵌入式 Ultra DMA EIDE 端口可由 SRM 控制台启动。

UP1000 SRM 可以从 Adaptec 294x 适配器启动。在高 I/O 负载条件下，使用 Adaptec 294x 观察到机器锁定。基于 Symbios 875 的卡使用 sym 驱动程序工作正常。最有可能的是，sym 驱动程序支持的其他基于 Symbios 芯片的卡也可以工作。

USB 接口被 SRM 控制台禁用，尚未用 FreeBSD 测试。

对于 UP1000，内核配置文件必须包含：

```ini
options    API_UP1000  # UP1000, UP1100 (Nautilus)
cpu EV5
```

特性：

- 内存总线：100MHz 64 位（PC-100 SDRAM），800 MB/s 内存带宽
- 板载 Bcache / L2 缓存：2Mb

UP1100 标配 SRM 控制台代码。SRM 位于 2Mbyte 闪存 ROM 中。

#### 2.3.16 Alpha Processor Inc. CS20, Compaq DS20L

CS20 是一款基于 21264[ab] CPU 的 19“、1U 高机架安装服务器。最多可有 2 个 CPU。Compaq 将 CS20 重新品牌化为 AlphaServer DS20L。DS20L 有 833MHz CPU。

#### 2.3.17 Compaq AlphaServer ES40 (“Clipper”)

ES40 是一个 SMP 系统，可以有 1-4 个 21264 Alpha CPU。最大配置 32GB 内存，这些系统通常部署为重型数据库服务器，也用于 HPTC 计算场环境。

特性：

- 500（EV6）、667（EV67）或 833 MHz（EV68）的 21264 Alpha CPU（最多 4 个 CPU）
- 内存总线：256 位宽
- 21272 核心逻辑芯片组
- PS/2 鼠标和键盘端口
- 内存：200 针 JEDEC 标准 SDRAM DIMM，最大 32 GByte 内存
- 2 个 16550A 串口
- 1 个 ECP/EPP 并口
- ALI M1543C Ultra DMA66 IDE 接口
- 扩展：2 条 64 位 PCI 总线

ES40 标配 SRM 控制台代码。

ES40 带有 ATA CDROM 驱动器，但使用 SCSI 硬盘。通常的 Symbios 和 Qlogic 适配器可启动，KZPEA 又名 Adaptec 39160 双通道 LVD U160 适配器也可启动。

内存分为 4 个内存阵列，每个阵列包含一组 4 个 SDRAM DIMM。每个 DIMM 为 72 位宽，100MHz 速度变体。一个阵列可以包含 2 组，所以每个阵列最多 8 个 DIMM。DIMM 位于内存主板（MMB）上。有 2 种 MMB 型号，分别有 4 和 8 个 DIMM 插座。每块 MMB 为 CPU 提供 256 位内存总线宽度的一半。鉴于内存配置的众多选项，建议查阅系统文档以获取最佳内存配置。

根据型号变体，ES40 有 6 或 10 个 64 位 PCI 插槽。这基本上只是意味着相同的背板但安装的连接器更少。

ES40 具有与 DS10 和 DS20 相同的 RMC 远程电源控制。请参阅本文档 DS10 部分中 RMC 的描述。ES40 的大多数变体都有多个电源，允许 N+1 冗余。安装 CPU 卡时，你必须拔下所有电源线，CPU 卡从电源接收待机电源。最大内存配置需要多于默认数量的电源。

内核配置文件必须包含：

```ini
options   DEC_ST6600
cpu     EV5
```

与预期相反，内核配置文件中没有定义 `cpu EV6`。`cpu EV5` 是强制性的，以使 `config` 满意。

### 2.4 支持的硬件概述

提醒一下：FreeBSD 的装机量远不及 FreeBSD/Intel。这意味着市面上种类繁多的 PCI/ISA 扩展卡在 alpha 上测试的机会比在 Intel 上少得多。这并不意味着它们注定会失败，只是遇到从未测试过的东西的机会要高得多。**GENERIC** 只包含已知在 Alpha 上工作的东西。

PCI 和 ISA 扩展总线完全受支持。Turbo Channel 不在 **GENERIC** 中，支持有限（参见相关机器型号信息）。MCA 总线不受支持。EISA 总线不支持 EISA 扩展卡，因为缺少 EISA 支持代码。据报道，EISA 插槽中的 ISA 卡可以工作。Compaq Qvision EISA VGA 卡以 ISA 模式驱动，作为控制台工作正常。

支持 1.44 Mbyte 和 1.2 Mbyte 软盘驱动器。Alpha 机器中有时发现的 2.88 Mbyte 驱动器最多支持到 1.44 Mbyte。

ATA 和 ATAPI（IDE）设备通过 `ata` 驱动程序框架支持。由于大多数人使用 SCSI 磁盘运行他们的 Alpha，它没有像 SCSI 那样经过充分测试。请注意 IDE 磁盘的可启动性限制。参见机器特定信息。

通过 CAM 层为 Adaptec 2940x（基于 AIC7xxx 芯片）、Qlogic 系列和 Symbios 提供完整的 SCSI 支持。那些对 U160 SCSI 感兴趣的人可能想看看 Adaptec 39160 双通道 LVD U160 适配器。Compaq 称之为 KZPEA 适配器。较新的 Alpha 型号有可以从它们启动的 SRM 版本。通常要注意各种适配器型号的机器特定可启动性问题。在已知的情况下，它们列在各个机器描述中。

Qlogic QL2x00 FibreChannel 主机适配器完全受支持。

如果你想通过以太网启动你的 Alpha，你显然需要 SRM 控制台识别的以太网卡。这通常意味着你需要带有 21x4x 以太网芯片的板卡，因为那是 Digital 使用的。这些芯片由 FreeBSD `de`（较旧的驱动程序）或 `dc`（较新的驱动程序）驱动。一些新的 SRM 版本已知可以识别由 FreeBSD `fxp` 驱动程序驱动的 Intel 8255x 以太网芯片。但要注意：据报道 `fxp` 驱动程序在 FreeBSD/alpha 上不能正确工作（虽然它在 FreeBSD/x86 上工作得很好）。

通常，SRM 控制台在 PCI VGA 卡上模拟 VGA 兼容模式。然而，Compaq/DEC 并不保证这在每种卡类型上都能工作。当 SRM 认为 VGA 可接受时，FreeBSD 将能够使用它。控制台驱动程序与 FreeBSD/intel 机器上的工作方式相同。请注意，Alpha 上不支持 VESA 模式，因此你只能使用 80x25 控制台。

在一些 Alpha 机器中，你会发现基于 TGA 芯片的视频适配器。FreeBSD 中的 TGA 支持不够健壮。如果出现问题，建议尝试串行控制台或普通 VGA 卡。

大多数 Alpha 上发现的“PC 标准”串口受支持。

FreeBSD/alpha 不支持 ISDN（i4b）。
