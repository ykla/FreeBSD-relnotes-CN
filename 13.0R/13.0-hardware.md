# FreeBSD 13.0-RELEASE 硬件兼容列表

## 引言

本文档包含 FreeBSD 13.0-RELEASE 的硬件兼容性说明。其中列出了 FreeBSD 支持的硬件平台，以及各类硬件设备（存储控制器、网络接口等），并给出已知可正常工作的设备实例。

## 支持的处理器和系统板

本节提供各架构所支持的具体处理器和系统的相关信息。

### amd64

FreeBSD/amd64 支持 64 位 x86 架构，可在多种机器上运行。它是 Tier-1 平台（受完全支持的架构），应在 FreeBSD 操作系统的各个方面（包括安装和开发环境）均达到生产质量。由于此架构可用硬件范围广泛，无法穷尽列出 FreeBSD 支持的所有设备组合。但此处仍提供一些通用指引。

所有 64 位 x86 处理器均受支持，包括 AMD 的 AMD64 CPU 和 Intel 的 Intel 64 CPU（包括 Xeon 和 Core 处理器）。FreeBSD/amd64 支持多核和 SMT（Symmetric MultiThreading，对称多线程，在 Intel CPU 上又称 HyperThreading）处理器。

注意，此架构有多种名称，包括 x86-64、AMD64、Intel EM64T 和 IA-32e。

FreeBSD/amd64 需要高级配置与电源管理（ACPI）支持。ACPI 由 Intel 的 ACPI Component Architecture 参考实现提供支持，如 [acpi(4)](https://man.freebsd.org/cgi/man.cgi?query=acpi&sektion=4&format=html) 手册页所述。

FreeBSD/amd64 可在使用 UEFI 和传统 BIOS 的系统上引导。

FreeBSD 通常可在基于 amd64 的笔记本电脑、台式机和服务器上运行。对某些硬件特性（如声音、图形、无线和电源管理）的支持程度可能因所使用的具体硬件设备而异。

### i386

FreeBSD/i386 支持 32 位 x86 架构，可在多种 “IBM PC 兼容” 机器上运行。与 FreeBSD/amd64 一样，无法穷尽列出 FreeBSD 支持的所有设备组合，但此处仍提供一些通用指引。

所有带浮点单元且支持 CMPXCHG 指令（自 80486 起引入）的 32 位 x86 兼容处理器均受支持。包括 Intel 的 Pentium 系列处理器，以及 AMD、Cyrix、NexGen 和 VIA 的 x86 兼容处理器。

此架构有大量可选主板。使用 PCI 扩展总线的主板得到良好支持。对 AGP 设备仅有有限支持，ISA 支持主要面向现代 PCI 系统中 LPC 桥接器模拟的传统设备。

对称多处理器（SMP）系统通常受支持，包括多核和 SMT。

FreeBSD 会在支持此特性的 CPU 上利用物理地址扩展（PAE）支持。默认内核支持高达 24 GB 的内存，但设备驱动会使用反弹缓冲区来访问 4 GB 以上的 I/O 缓冲区。启用 `PAE` 特性的自定义内核可避免使用反弹缓冲区，但会禁用少量不兼容的设备驱动。此特性对 FreeBSD 其他可用的特性构成限制；详情请参阅 [pae(4)](https://man.freebsd.org/cgi/man.cgi?query=pae&sektion=4&format=html) 手册页。

FreeBSD 通常可在基于 i386 的笔记本电脑、台式机和服务器上运行。对某些硬件特性（如声音、图形、无线和电源管理）的支持程度可能因所使用的具体硬件设备而异。

FreeBSD/i386 支持现代机器使用的 ACPI，但也能在不支持 ACPI 的旧系统上引导。

FreeBSD/i386 仅支持从使用传统 BIOS 的系统引导，无法从 UEFI 引导。

### powerpc

所有内置 USB 的 Apple PowerPC 机器均受支持，以及少量非 Apple 机器，包括 POWER7 上的 KVM。

在所有拥有 1 个以上处理器的系统上支持 SMP。
