# FreeBSD 14.5-RELEASE 硬件兼容列表

## 引言

本硬件兼容性说明列出所有已知可在 FreeBSD 14.5 上工作的硬件。
其他配置也可能可用，但尚未经过测试。
欢迎对本列表提出反馈、更新和修正！

## 支持的平台

本节介绍 FreeBSD 平台项目当前支持的 CPU 架构、支持等级以及各平台所支持的系统。

### x86-64

[FreeBSD/amd64](/platforms/amd64) 将 x86-64 作为完全支持的 [Tier-1]({committers-guide}#archs) 架构。
发行介质预期可在所有至少配备 256 MiB 内存的 x86-64 机器上运行。
本发行版整合了多项优化和增强功能，以充分发挥基于 FreeBSD/amd64 系统的全部能力，在各种工作负载下提供更佳性能。

### RISC-V

[FreeBSD/riscv64](https://wiki.freebsd.org/riscv) 将 64 位 RISC-V 作为 [Tier-2]({committers-guide}#archs) 架构。

RISC-V 支持标志着这一开源操作系统发展中的重要里程碑。
FreeBSD 是首个内置支持 RISC-V 引导的操作系统，体现了其对新兴硬件架构的承诺。

FreeBSD 目前支持以下平台：

* HiFive Unleashed (SiFive FU540)
* HiFive Unmatched (SiFive FU740)
* StarFive VisionFive 2 (StarFive JH7110)
* Spike
* QEMU

[ahci(4)](https://man.freebsd.org/cgi/man.cgi?query=ahci&sektion=4&format=html)
驱动支持具有 PCI 类 1（海量存储）、子类 6（SATA）和编程接口 1（AHCI）的 AHCI 兼容控制器。

此外，与 ata(4) 的 atamarvell 和 atajmicron 驱动配合，它支持传统 PATA + AHCI-SATA 组合控制器的 AHCI 部分，例如 JMicron JMB36x 和 Marvell 88SE61xx。

[ahci(4)](https://man.freebsd.org/cgi/man.cgi?query=ahci&sektion=4&format=html)
驱动还支持作为 PCI 桥接
nvme(4)
的 AHCI 设备，使用 Intel Rapid Storage Technology（RST）。
要使用
nvme(4)
设备，需在 BIOS 中将 SATA 模式从 RST 设置为 AHCI，
否则需接受因 RST 启用中断共享而带来的性能影响。
FreeBSD 会自动检测处于 RST 模式且具有此扩展的 AHCI 设备。
当检测到时，
[ahci(4)](https://man.freebsd.org/cgi/man.cgi?query=ahci&sektion=4&format=html)
驱动会将
nvme(4)
子设备附加到
[ahci(4)](https://man.freebsd.org/cgi/man.cgi?query=ahci&sektion=4&format=html)
设备。
