# FreeBSD 4.0-RELEASE 勘误

如果在安装此版本 FreeBSD 之前你没有阅读任何其他文档，那么至少务必**阅读本发行版的勘误**，以免在已经发现并修复的问题上绊倒。此 ERRATA.TXT 文件按定义显然已经过时，但网络上的其他副本会保持更新，应作为你所发行版的“当前勘误”查阅。这些其他勘误副本位于：

1. <http://www.FreeBSD.org/releases/>
2. **ftp://ftp.FreeBSD.org/pub/FreeBSD/releases/<your-release>/ERRATA.TXT**（以及任何保持此位置最新镜像的站点）。

对此文件的任何更改也会自动发送电子邮件至：

`freebsd-current@FreeBSD.org`

有关所有 FreeBSD 安全公告，请参见：

<http://www.FreeBSD.org/security/>

以获取最新的安全事件信息。

---- 系统更新信息：

bin 发行版中的 tcpdump 二进制文件错误地链接到了 libcrypto.so 库，而该库仅存在于单独的 crypto 发行版中。

因此，如果你只安装了 bin 发行版而未安装 crypto 发行版，tcpdump 将无法正常工作。

修复：从以下位置下载新的 tcpdump 二进制文件：

**<http://people.FreeBSD.org/~kris/4.0R/i386/tcpdump>**（i386）
**<http://people.FreeBSD.org/~kris/4.0R/alpha/tcpdump>**（alpha）

此文件的 MD5 校验和为：

i386 版本：MD5 (tcpdump) = 0b3d32b367e7312d546ccae8f1824391
alpha 版本：MD5 (tcpdump) = 2d113fa4c38c8a0299d558acb5c6ad57

要验证你所下载副本的校验和，请执行以下命令：

```sh
/sbin/md5 /path/to/downloaded/tcpdump
```

并与上述值进行比较。

- install.sh 未安装工具源代码（sysinstall 之外）

如果你试图从 CDROM 中提取完整源代码（在 sysinstall 程序之外），最终会缺少工具源代码。

修复：如果你从 **/cdrom/src** 运行 install.sh，还需要运行：

```sh
cat stool.?? | tar --unlink -xpzf - -C /usr/src
```

以安装工具源代码（**/usr/src/tools** 层级目录）。这些源代码是成功构建 world 所必需的。
