# FreeBSD 2.2.5-RELEASE 勘误

文件 **ERRATA.TXT** 包含 2.2.5 的发布后勘误，在报告此版本的问题之前，应始终**首先**查阅此文件。此文件也会随着新问题的报告而定期更新，因此即使你最近已查阅过，在提交漏洞报告前请再次检查。对此文件的任何更改也会自动发送邮件至：

[freebsd-stable@FreeBSD.org](mailto:freebsd-stable@FreeBSD.org)

2.2.5 的安全公告请参见：

[ftp://FreeBSD.org/pub/CERT/](ftp://FreeBSD.org/pub/CERT/)

以获取最新信息（请仔细注意 URL——这不是 ftp.FreeBSD.org）。

## 安全公告

2.2.5 当前有效的安全公告：1 个

- FreeBSD-SA-97:05（可从 [ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT](ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT) 获取）。

  如果你未运行 X 服务器或其他需要访问 I/O 指令的专用工具，也可以直接删除 **/dev/io** 作为快速变通方法。

## 系统更新信息

- 2.2.5 中的 appletalk 协议栈已损坏。

  修复：如果你计划运行 appletalk，应当应用以下补丁：

  [ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.5-RELEASE/atalk.diff.2.2.gz](ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.5-RELEASE/atalk.diff.2.2.gz)

- 行式打印机后台程序 `lpd` 在向远程打印机发送任务时，会在 `ct` 功能指定的超时时间（默认 2 分钟）后终止发送至远程的子进程。

  修复：请将 `lpd` 子系统升级至 2.2-stable。作为变通方法，可将 `ct` 功能增大到一个不合理的较大秒数（如 3600）。

- Intel “F00F bug” 使用户能够在拥有 Pentium 处理器的机器上挂起系统，前提是他们有权限访问该机器并能执行程序。

  修复：更新至 2.2-stable 版本的内核，或应用以下补丁：

  [ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.5-RELEASE/f00f.diff.2.2.gz](ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.5-RELEASE/f00f.diff.2.2.gz)

- `ipfw` 代码中存在一个漏洞，使用 “reset tcp” 防火墙命令会导致内核将以太网包头写入随机的内核栈位置。

  修复：更新至 2.2-stable 版本的内核，或应用以下补丁：

  [ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.5-RELEASE/ipfw.diff.gz](ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.5-RELEASE/ipfw.diff.gz)

- `XF86Setup` 中存在一个漏洞，如果符号链接尚不存在，则无法从 **/usr/X11R6/bin/X** 创建指向正确 X 服务器的符号链接。当你输入 `startx` 时，会显示以下错误：

  ```sh
  xinit:  No such file or directory (errno 2):  no server "X" found in PATH
  ```

  修复：以 root 身份执行以下命令，然后重新运行 `XF86Setup`。

  ```sh
  # cd /usr/X11R6/bin
  # ln -s XF86_VGA16 X
  ```

  如果 `XF86Setup` 询问你是否要使用现有的 **XF86Config** 作为默认配置，请选择否。当它询问你是否要创建指向服务器的 “X” 链接时，选择是。

- 用户级 `ppp` 的状态图实现中存在一个漏洞，在关闭链路时会导致某些 ppp 实现出现问题。线路会断开，但调制解调器不会挂断，除非使用 `pppctl`（或开关）手动挂断。

  修复：可从 <http://www.FreeBSD.org/~brian> 获取源自 -current 源码的 ppp 版本。

  它应该能在 2.0.5 及以上版本的 FreeBSD 上构建。此代码可在 -current 树中获取，但（尚未）在 2.2-STABLE 树中可用。
