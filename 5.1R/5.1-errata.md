FreeBSD 是 Wind River Systems, Inc. 的注册商标。预计此情况将很快改变。

Intel、Celeron、EtherExpress、i386、i486、Itanium、Pentium 和 Xeon 是 Intel Corporation 或其子公司在美国及其他国家的商标或注册商标。

Sparc、Sparc64、SPARCEngine 和 UltraSPARC 是 SPARC International, Inc 在美国及其他国家的商标。带有 SPARC 商标的产品基于 Sun Microsystems, Inc. 开发的架构。

许多制造商和销售商用于区分其产品的名称被主张为商标。在本文件中出现这些名称时，如果 FreeBSD 项目知道该商标的主张，就会在名称后附上“™”或“®”符号。

本文档列出了 FreeBSD 5.1-RELEASE 的勘误项，包含发行后才发现或发行周期后期才确定而未能纳入发行文档的重要信息。这些信息包括安全公告，以及与可能影响其运行或可用性的软件或文档相关的消息。在安装此版本 FreeBSD 之前，应始终查阅本文档的最新版本。

FreeBSD 5.1-RELEASE 的勘误文档将维护至 FreeBSD 5.2-RELEASE 发布。

## 1 引言

本勘误文档包含有关 FreeBSD 5.1-RELEASE 的“最新消息”。在安装此版本之前，请务必查阅本文档，以了解发行后才发现的或已经发现并修复的问题。

随发行版实际分发的任何勘误文档（例如 CDROM 发行版中的）按定义已过时，但其他副本会在互联网上保持更新，应作为此版本的“当前勘误”查阅。这些其他勘误副本位于 [http://www.FreeBSD.org/releases/](http://www.freebsd.org/releases/)，以及任何保持此位置最新镜像的站点。

FreeBSD 5-CURRENT 的源代码和二进制快照也包含本文档的最新副本（截至快照时的版本）。

有关所有 FreeBSD CERT 安全公告的列表，请参见 [http://www.FreeBSD.org/security/](http://www.freebsd.org/security/) 或 [ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT/](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/)。

## 2 安全公告

[realpath(3)](http://www.freebsd.org/cgi/man.cgi?query=realpath&sektion=3&manpath=FreeBSD+5.1-current) 函数的实现中包含一个单字节缓冲区溢出 bug。此 bug 的影响取决于使用 [realpath(3)](http://www.freebsd.org/cgi/man.cgi?query=realpath&sektion=3&manpath=FreeBSD+5.1-current) 的应用程序及其他因素。此 bug 在 5.1-RELEASE 之前已在 5-CURRENT 开发分支上修复；因此 FreeBSD 5.1-RELEASE 不受影响。但是，此变更未在发行文档中注明。更多信息请参见安全公告 [FreeBSD-SA-03:08](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:08.realpath.asc)。

内核中存在一个 bug，可能导致其尝试传递无效信号，从而导致内核恐慌，或在某些情况下未经授权修改内核内存。此 bug 已在 5-CURRENT 开发分支和 5.1-RELEASE 安全修复分支上修复。更多信息请参见安全公告 [FreeBSD-SA-03:09](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:09.signal.asc)。

iBCS2 仿真模块中的一个 bug 可能导致泄露内核内存内容。（注意，此模块在 FreeBSD 中默认未启用。）此 bug 已在 5-CURRENT 开发分支和 5.1-RELEASE 安全修复分支上修复。更多信息请参见安全公告 [FreeBSD-SA-03:10](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:10.ibcs2.asc)。

**OpenSSH** 的缓冲区管理代码中包含一个 bug，可能导致其崩溃。此 bug 已通过厂商提供的补丁在 5-CURRENT 开发分支和 5.1-RELEASE 安全修复分支上修复。更多详情请参见安全公告 [FreeBSD-SA-03:12](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:12.openssh.asc)。

**sendmail** 包含一个可远程利用的缓冲区溢出。此 bug 已通过新版本导入在 5-CURRENT 开发分支上修复，并通过厂商提供的补丁在 5.1-RELEASE 安全修复分支上修复。更多详情请参见安全公告 [FreeBSD-SA-03:13](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:13.sendmail.asc)。

FreeBSD ARP 代码中存在一个 bug，可能导致内核造成资源耗尽，最终导致系统恐慌。此 bug 已在 5-CURRENT 开发分支和 5.1-RELEASE 安全修复分支上修复。更多信息请参见安全公告 [FreeBSD-SA-03:14](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:14.arp.asc)。

**OpenSSH** PAM 认证代码中的几个 bug 可能导致从认证错误到栈损坏等各种影响。这些已通过厂商提供的补丁修复；详情请参见安全公告 [FreeBSD-SA-03:15](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:15.openssh.asc)。

[procfs(5)](http://www.freebsd.org/cgi/man.cgi?query=procfs&sektion=5&manpath=FreeBSD+5.1-current) 和 [linprocfs(5)](http://www.freebsd.org/cgi/man.cgi?query=linprocfs&sektion=5&manpath=FreeBSD+5.1-current) 的实现中包含一个 bug，可能导致泄露内核内存内容。此 bug 已在 5-CURRENT 开发分支和 5.1-RELEASE 安全修复分支上修复。更多信息请参见安全公告 [FreeBSD-SA-03:17](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:17.procfs.asc)。

**OpenSSL** 包含几个 bug，可能允许远程攻击者使使用 **OpenSSL** 的应用程序崩溃或以应用程序的权限执行任意代码。这些 bug 已通过厂商提供的补丁在 5-CURRENT 开发分支和 5.1-RELEASE 安全修复分支上修复。注意，仅使用 **OpenSSL** 的 ASN.1 或 X.509 处理代码的应用程序受到影响（例如 **OpenSSH** 不受影响）。更多信息请参见安全公告 [FreeBSD-SA-03:18](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:18.openssl.asc)。

**BIND** 存在拒绝服务攻击的可能性。此漏洞已通过厂商补丁在 5.1-RELEASE 安全修复分支上解决，并通过导入新版本到 5-CURRENT 开发分支解决。更多信息请参见 [FreeBSD-SA-03:19](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-03:19.bind.asc)。

## 3 未解决问题

[raid(4)](http://www.freebsd.org/cgi/man.cgi?query=raid&sektion=4&manpath=FreeBSD+5.1-current) 中描述的 RAIDframe 磁盘驱动程序在此发行版中无法使用。

ACPI 似乎会使某些 i386™ 机器不稳定。关闭 ACPI 支持可能有助于解决其中一些问题；参见 [第 4 节](#4-最新消息) 中的一项。

整数溢出可能在具有特定内存大小的使用 PAE 的机器上导致内核恐慌。此 bug 已在 `RELENG_5_1` 和 `HEAD` 分支上修复。此问题的解决方法是移除一些内存，更新相关系统，然后重新安装内存。

尝试通过 `mount -u` 将已从只读升级为读写的 [msdosfs(5)](http://www.freebsd.org/cgi/man.cgi?query=msdosfs&sektion=5&manpath=FreeBSD+5.1-current) 文件系统写入数据会导致系统锁定。要解决此问题，请先卸载文件系统，然后使用适当的选项重新挂载，而不是使用 `mount -u`。

## 4 最新消息

[ipfw(4)](http://www.freebsd.org/cgi/man.cgi?query=ipfw&sektion=4&manpath=FreeBSD+5.1-current) 现在应该在严格对齐的 64 位架构（如 alpha 和 Sparc64®）上正常工作。

发行说明本应说明 `libthr` 库默认为 i386 平台构建。

FreeBSD 5.1-RELEASE 包含一些新的引导加载程序脚本，旨在使使用不同选项引导 FreeBSD 更容易。这可能有助于诊断引导问题。这些脚本基于现有的基于 Forth 的引导加载程序脚本构建（因此 **/boot/loader.conf** 和其他现有的加载程序配置文件仍然适用）。它们仅默认安装在 i386 机器上的新二进制安装中。新脚本提供了一个引导时菜单，用于控制 FreeBSD 的引导方式，包括关闭 ACPI、“安全模式”引导、单用户引导和详细引导等选项。当与系统硬件的兼容性不确定时，“安全模式”引导特别有用，它设置以下内核可调变量：

```sh
hint.acpi.0.disabled=1 # 禁用 ACPI（仅 i386）
hw.ata.ata_dma=0       # 禁用 IDE DMA
hw.ata.atapi_dma=0     # 禁用 ATAPI/IDE DMA
hw.ata.wc=0            # 禁用 IDE 磁盘写入缓存
hw.eisa_slots=0        # 禁用 EISA 设备探测
```

对于 i386 架构机器上的新安装，[sysinstall(8)](http://www.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+5.1-current) 会尝试确定 ACPI 是否通过上述新引导加载程序脚本禁用，如果是，则询问是否应将此更改设为永久。

发行说明本应提及以下功能的开发由美国国防高级研究计划局（DARPA）赞助：OpenPAM、NSS 支持、PAE 支持、各种 MAC 框架更新、GEOM 磁盘几何系统。
