## 系统更新信息

- 根挂载点变更，影响通过 `make world` 或 FreeBSD 2.2.6 升级的用户。

  修复：2.2.6 引入了根文件系统挂载设备命名的变更。此变更影响所有系统，但只有进行升级安装的系统才需要用户干预。

  此前，根文件系统始终从兼容切片挂载，而同一磁盘上的其他分区则从其真实切片挂载。例如，这可能导致 **/etc/fstab** 文件如下所示：

  ```sh
  # Device      Mountpoint      FStype  Options         Dump    Pass#
  /dev/wd0s2b   none            swap    sw              0       0
  /dev/wd0a     /               ufs     rw              1       1
  /dev/wd0s2f   /local0         ufs     rw              1       1
  /dev/wd0s2e   /usr            ufs     rw              1       1
  ```

  对于 FreeBSD 2.2.6 及更高版本，此格式发生变更，使得‘/’的设备与其他设备一致，即：

  ```sh
  # Device      Mountpoint      FStype  Options         Dump    Pass#
  /dev/wd0s2b   none            swap    sw              0       0
  /dev/wd0s2a   /               ufs     rw              1       1
  /dev/wd0s2f   /local0         ufs     rw              1       1
  /dev/wd0s2e   /usr            ufs     rw              1       1
  ```

  如果在这种情况下未手动更新 **/etc/fstab**，系统将在挂载 / 时（通常在启动时）发出警告消息，指出必须进行的更改。此外，如果根文件系统未正确卸载，可能会遇到问题，导致根文件系统在下一次重启时不会被标记为干净状态。

  此更改应在升级系统成功重启后立即进行。

- `ppp` 程序无法工作，提示缺少共享库“libdes.so.3.0”。

  修复：有三种可能的修复方法：

  1. 最简单的修复方法是使用 **/stand/sysinstall** 安装 des 发行版，记住如果你身在美国和加拿大以外地区，要选择一个允许你导出的站点（ftp.FreeBSD.org 和 ftp.internat.FreeBSD.org 都属于此类别）。

  2. 纯粹作为变通方案，如果 `ppp` 也是你连接到网络的唯一途径，你可能需要这样做（以 root 身份）：

     ```sh
     cp /usr/lib/libcrypt.so.2.0 /usr/lib/libdes.so.3.0
     ldconfig -m /usr/lib
     ```

  3. 另一种修复方法（无需获取 DES 组件）是将 ppp 源码安装到 **/usr/src/usr.sbin/ppp** 并重新构建。源码足够“智能”，能识别系统上没有 DES 库，因此不会创建依赖它的二进制文件。

  注意：如果你选择第 2 或第 3 种修复方法，你也将无法使用 MSCHAP（Microsoft Win*）风格的认证。

- XFree86 3.3.2 中的 `xterm` 程序在退出时不会删除 utmp 条目（例如，`xterm` 会话在退出后仍会出现在 `who` 或 `w` 中）。

  修复：获取更新的 xterm 二进制文件：

  [ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.6-RELEASE/xterm](ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.6-RELEASE/xterm)

  或者获取**最新**的 Ports 集合到你的机器上（参见 <http://www.FreeBSD.org/ports>），并使用 **x11/XFree86** 中的 port 来构建已应用此补丁的 xterm（截至 98/04/06）。补丁本身也可从 port 中获取：

  [ftp://ftp.FreeBSD.org/pub/FreeBSD/FreeBSD-current/ports/x11/XFree86/patches/patch-ag](ftp://ftp.FreeBSD.org/pub/FreeBSD/FreeBSD-current/ports/x11/XFree86/patches/patch-ag)

- 较旧的 Matsushita（Panasonic）、Sony CDU-31 和 Mitsumi（非 IDE）CD-ROM 光驱不再允许 CD-ROM 安装。

  修复：从 **updates/** 目录获取更新的启动软盘，例如：

  [ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.6-RELEASE/boot.flp](ftp://ftp.FreeBSD.org/pub/FreeBSD/updates/2.2.6-RELEASE/boot.flp)

  并使用它来安装 2.2.6。此问题在 2.2-stable 中修复完成，在下一个 FreeBSD 版本中将不再存在。
