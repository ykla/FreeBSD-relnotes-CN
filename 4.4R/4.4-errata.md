本文档列出了 FreeBSD 4.4-RELEASE 的勘误项，包含发行后才发现的重要信息。这些信息包括安全公告，以及与可能影响其运行或可用性的软件或文档相关的消息。在安装此版本 FreeBSD 之前，应始终查阅本文档的最新版本。

FreeBSD 4.4-RELEASE 的勘误文档将维护至 FreeBSD 4.5-RELEASE 发布。

## 1 引言

本勘误文档包含有关 FreeBSD 4.4-RELEASE 的“最新消息”。在安装此版本之前，请务必查阅本文档，以了解发行后才发现的或已经发现并修复的问题。

随发行版实际分发的任何勘误文档（例如 CDROM 发行版中的）按定义已过时，但其他副本会在互联网上保持更新，应作为此版本的“当前勘误”查阅。这些其他勘误副本位于 [http://www.FreeBSD.org/releases/](http://www.freebsd.org/releases/)，以及任何保持此位置最新镜像的站点。

FreeBSD 4-STABLE 的源代码和二进制快照也包含本文档的最新副本（截至快照时的版本）。

有关所有 FreeBSD CERT 安全公告的列表，请参见 [http://www.FreeBSD.org/security/](http://www.freebsd.org/security/) 或 [ftp://ftp.FreeBSD.org/pub/FreeBSD/CERT/](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/)。

## 2 安全公告

对每用户 [~/.login_conf](https://man.freebsd.org/cgi/man.cgi?query=login.conf&sektion=5&manpath=FreeBSD+4.4-stable) 文件的支持（见 [login.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=login.conf&sektion=5&manpath=FreeBSD+4.4-stable)）由于本地和全局设置的合并不正确而被禁用。此问题已在 FreeBSD 4.5-RC 中修复。

UUCP 中的本地 root 漏洞已在 FreeBSD 4.4-RELEASE 发布前修复，并记录在安全公告 [FreeBSD-SA-01:62](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-01:62.uucp.asc) 中。发行说明提到了修复，但未提及安全公告，因为该公告在 4.4-RELEASE 发布日期之后发布。

OpenSSH 中的安全漏洞已关闭，该漏洞在设置 `UseLogin yes` 时可能允许用户以任意权限执行代码。请注意此设置的默认值为 `UseLogin no`。更多信息（包括变通方案和修复）请参见安全公告 [FreeBSD-SA-01:63](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-01:63.openssh.asc)。

[pkg_add(1)](https://man.freebsd.org/cgi/man.cgi?query=pkg_add&sektion=1&manpath=FreeBSD+4.4-stable) 使用不安全的临时目录，可能允许本地攻击者在二进制软件包安装过程中修改其内容。此漏洞已关闭。更多信息请参见安全公告 [FreeBSD-SA-02:01](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:01.pkg_add.asc)。

[pw(8)](https://man.freebsd.org/cgi/man.cgi?query=pw&sektion=8&manpath=FreeBSD+4.4-stable) 中的竞态条件可能泄露 **/etc/master.passwd** 的内容，已消除。更多信息（包括变通方案和修复）请参见安全公告 [FreeBSD-SA-02:02](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:02.pw.asc)。

[k5su(8)](https://man.freebsd.org/cgi/man.cgi?query=k5su&sektion=8&manpath=FreeBSD+4.4-stable) 中的 bug 可能允许已放弃超级用户权限的进程重新获得这些权限。此 bug 已修复。（见安全公告 [FreeBSD-SA-02:07](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:07.k5su.asc)。）

[exec(3)](https://man.freebsd.org/cgi/man.cgi?query=exec&sektion=3&manpath=FreeBSD+4.4-stable) 系统调用中的竞态条件可能导致本地用户获得提升的权限，已修复。更多信息（包括修复）请参见安全公告 [FreeBSD-SA-02:08](ftp://ftp.freebsd.org/pub/FreeBSD/CERT/advisories/FreeBSD-SA-02:08.exec.asc)。

## 3 系统更新信息

[printcap(5)](https://man.freebsd.org/cgi/man.cgi?query=printcap&sektion=5&manpath=FreeBSD+4.4-stable) 文件中某些类型注释的处理存在缺陷。如果注释行以 `:\`（通常用于指示打印机定义续行的字符）结尾，注释后的行将被解析为前一个打印机定义的一部分。

发行说明中的一项提到了 **net.inet.tcp.tcp_seq_genscheme** sysctl 变量，用于在 TCP 初始序列号生成算法中启用改进的兼容性。由于采用了基于 RFC 1948 的算法，此变量已不再必要，该算法在所有情况下恢复正常运行，同时在初始序列号中保持高度不可预测性。

FreeBSD CDROM 发行版的 **tools/** 目录通常包含一些适用于 i386 机器的 DOS 程序，如 `fdimage.exe` 和 `fips.exe`。此目录在 FreeBSD 项目生成的初始 4.4-RELEASE ISO 镜像中被意外遗漏。`4.4-install.iso` 和 `4.4-mini.iso` ISO 镜像的更正版本已于 2001 年 9 月 22 日星期六重新生成。对于需要这些 DOS 模式工具但不愿重新获取 ISO 镜像的用户，**tools/** 目录的内容可通过匿名 FTP 从 [ftp://ftp.FreeBSD.org/pub/FreeBSD/tools](ftp://ftp.freebsd.org/pub/FreeBSD/tools) 获取（许多镜像也有此目录的副本）。

发行说明中的一项提供了 PCCARD 弹出时挂起的变通方案，但给出了错误的命令语法。正确的命令行是：

```sh
# pccardc power slot 0
```

在 i386 上，通过 [sysinstall(8)](https://man.freebsd.org/cgi/man.cgi?query=sysinstall&sektion=8&manpath=FreeBSD+4.4-stable) 安装 FreeBSD 的最低内存要求为 16MB，运行使用精简内核的 FreeBSD 需 4-8MB。**INSTALL.TXT**（及其他版本）中的安装说明引用了较低的数字。

发行说明中提到了两个用于密码认证哈希的库的合并，但名称有误。合并的库是 `libscrypt` 和 `libdescrypt`；新的统一库称为 `libcrypt`。

越来越多大量使用内核栈（有时会溢出）的设备驱动程序出现了问题。此问题有易于识别的特征；进程会随机收到 `SIGPROF` 信号，并可能因“`Profiling time alarm`”消息而终止。对于遇到此问题的用户，变通方案是编辑 **/src/sys/i386/include/param.h** 并将以下行：

```c
#define UPAGES  2
```

改为：

```c
#define UPAGES  3
```

需要重新编译内核并重启才能使更改生效。此问题已在 FreeBSD 4.5-RC 中修复。
