# FreeBSD 6.4-RELEASE 发行公告

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 6.4-RELEASE 现已发布。目前 6.4-RELEASE 预计是 6-STABLE 的最后一个发行版。部分亮点：

- 全新且大幅改进的 NFS 锁管理器（NLM）客户端
- 支持 Camellia 加密算法
- 引导加载程序进行了多项更改，其中包括允许从 USB 设备启动，以及在支持 GPT 的 BIOS 上从带 GPT 标签的设备启动
- 提供 amd64/i386 的 DVD 安装 ISO 镜像
- KDE 更新至 3.5.10，GNOME 更新至 2.22.3
- 更新了 BIND、sendmail、OpenPAM 等

有关新功能和已知问题的完整列表，请参见在线发行说明和勘误列表，地址如下：

- [http://www.FreeBSD.org/releases/6.4R/relnotes.html](http://www.freebsd.org/releases/6.4R/relnotes.html)
- [http://www.FreeBSD.org/releases/6.4R/errata.html](http://www.freebsd.org/releases/6.4R/errata.html)

有关 FreeBSD 发布工程活动的更多信息，请参见：

[http://www.FreeBSD.org/releng/](http://www.freebsd.org/releng/)

FreeBSD 安全团队计划对 6.4-RELEASE 提供支持至 2010 年 11 月 30 日。

## 可用性

FreeBSD 6.4-RELEASE 现可用于 amd64、i386、pc98 和 sparc64 架构。alpha 架构的构建尚未完成，将在稍后公布。FreeBSD 6.4-RELEASE 可从可引导 ISO 镜像安装或通过网络安装；所需文件可通过 FTP 或 BitTorrent 下载，如下文所述。虽然某些较小的 FTP 镜像可能不包含所有架构，但通常都会包含较常见的架构，如 i386 和 amd64。

发行版 ISO 镜像的 MD5 和 SHA256 哈希值附在本消息末尾。

大多数架构的发行版 ISO 镜像内容发生了变化。以 i386 架构为例，有名为“bootonly”、“disc1”、“disc2”、“disc3”、“docs”和“dvd1”的 ISO 镜像。“bootonly”镜像适用于引导机器进行基于 FTP 或 NFS 的网络安装。“disc1”、“disc2”和“disc3”镜像是 CDROM 尺寸（700MB 介质），用于进行包含基本软件包集合的完整安装，安装过程中无需访问 FTP 或 NFS 服务器。此外，“disc1”支持引导进入“基于 live CD 的文件系统”和系统救援模式。“docs”镜像包含所有支持语言的全部文档。“dvd1”镜像是 DVD 尺寸，包含上述 CDROM 光盘上的所有内容。因此，“dvd1”可用于进行包含基本软件包集合的完整安装，包含所有支持语言的全部文档，并可用于引导进入“基于 live CD 的文件系统”和系统救援模式。大多数用户会发现，如果其机器没有 DVD 驱动器，则仅需“disc1”、“disc2”和“disc3”即可。对于有 DVD 驱动器的机器，“dvd1”应足以满足需求。如果打算从源码安装 Ports 而非使用发行版附带的预编译软件包，则仅需“disc1”。

也可从多家供应商处购买 CD-ROM 形式的 FreeBSD 6.4-RELEASE。将提供基于 FreeBSD 6.4 的产品的供应商之一是：

## FreeBSD Update

[freebsd-update(8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&manpath=FreeBSD+6.4-RELEASE) 工具支持对运行早期 FreeBSD 发行版的 i386 和 amd64 系统进行二进制升级。运行 6.3-RELEASE、6.4-BETA、6.4-RC1 或 6.4-RC2 的系统可按如下方式升级：

```sh
# freebsd-update upgrade -r 6.4-RELEASE
```

在此过程中，FreeBSD Update 可能要求用户协助合并一些配置文件，或确认自动执行的合并是否正确。

```sh
# freebsd-update install
```

在继续之前，必须使用新安装的内核重启系统。

```sh
# shutdown -r now
```

重启后，需要再次运行 `freebsd-update` 以安装新的用户空间组件，并需要再次重启系统：

```sh
# freebsd-update install
# shutdown -r now
```

注意，FreeBSD Update 将下载的升级文件存储在 **/var/db/freebsd-update**，因此在运行 `freebsd-update` 之前，**/var** 中至少应有 400MB 的可用空间；如果 **/var** 分区太小，可使用 `freebsd-update` 的 `-d` 选项指定将升级文件存储到其他目录。

更多信息，请参见：

[http://www.daemonology.net/blog/2007-11-10-freebsd-minor-version-upgrade.html](http://www.daemonology.net/blog/2007-11-10-freebsd-minor-version-upgrade.html)
