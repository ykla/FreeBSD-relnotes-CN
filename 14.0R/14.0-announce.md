# FreeBSD 14.0-RELEASE 发行公告

日期：2023 年 11 月 17 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 14.0-RELEASE 现已发布。
这是 **stable/14** 分支的首个版本。

部分亮点：

* OpenSSH 升级到 9.5p1 版本。

* OpenSSL 升级到 3.0.12 版本，这是相对于 FreeBSD 13.2-RELEASE 中 OpenSSL 1.1.1t 的一次重大升级。

* bhyve 虚拟机监控器现在支持 TPM 和 GPU 直通。

* FreeBSD 在 amd64 和 arm64 平台上支持多达 1024 个核心。

* ZFS 升级到 OpenZFS 2.2 版本，提供显著的性能改进。

* 现在可以在运行带日志软更新的 UFS 文件系统上执行后台文件系统检查。

* AWS 和 Azure 现已提供实验性 ZFS 镜像。

* TCP 的默认拥塞控制机制现改为 CUBIC。

* 还有更多……

完整的新功能和已知问题列表，请参阅在线发行说明和勘误列表，地址如下：

* <https://www.FreeBSD.org/releases/14.0R/relnotes/>
* <https://www.FreeBSD.org/releases/14.0R/errata/>

关于 FreeBSD 发布工程活动的更多信息，请参阅：

* <https://www.FreeBSD.org/releng/>

## 献辞

FreeBSD 项目将 FreeBSD 14.0-RELEASE 献给 Hans Petter Selasky 以示纪念。

## 获取途径

FreeBSD 14.0-RELEASE 现已支持 amd64、aarch64、i386、powerpc、powerpc64、powerpc64le、powerpcspe、armv7 和 riscv64 架构。

FreeBSD 14.0-RELEASE 可从可启动 ISO 镜像或通过网络安装。
部分架构还支持从 USB 闪存盘安装。
所需文件可按以下说明下载。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/14.0R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/14.0R/announce.asc>

发行版所提供镜像的用途如下：

**dvd1**

包含安装 FreeBSD 基本系统、文档、调试发行版和少量预构建软件包（旨在启动图形工作站）所需的全部内容。还支持启动到基于“livefs”的救援模式。若你能刻录并使用 DVD 介质，这应是你所需的全部。

此外，可将其写入 USB 闪存盘（U 盘）用于 amd64 架构，在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-14.0-RELEASE-amd64-dvd1.iso \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**disc1**

包含 FreeBSD 基本系统。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

此外，可将其写入 USB 闪存盘（U 盘）用于 amd64 架构，在可从 USB 启动的机器上进行安装。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-14.0-RELEASE-amd64-disc1.iso \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**bootonly**

支持使用 CDROM 驱动器启动机器，但不包含从 CD 本身安装 FreeBSD 所需的安装发行版。从 CD 启动后，需要执行基于网络的安装（例如从 HTTP 或 FTP 服务器）。

此外，可将其写入 USB 闪存盘（U 盘）用于 amd64 架构，在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 bootonly 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-14.0-RELEASE-amd64-bootonly.iso \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**memstick**

可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-14.0-RELEASE-amd64-memstick.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**mini-memstick**

可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-14.0-RELEASE-amd64-mini-memstick.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**FreeBSD/arm SD 卡镜像**

可写入 SD 卡并用于启动受支持的 arm 系统。SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 5 GiB 的 SD 卡上。

为方便无控制台访问系统的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 **ssh(1)** 访问。此外，`root` 用户密码设为 `root`，强烈建议在获取系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，使用 **dd(1)** 工具，将 *KERNEL* 替换为系统适用的内核配置名称。

```sh
# dd if=FreeBSD-14.0-RELEASE-arm64-aarch64-KERNEL.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。
