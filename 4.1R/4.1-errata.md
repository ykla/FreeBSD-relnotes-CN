# FreeBSD 4.1-RELEASE 勘误

如果在安装此版本 FreeBSD 之前你没有阅读任何其他文档，那么至少务必**阅读本发行版的勘误**，以免在已经发现并修复的问题上绊倒。此 ERRATA.TXT 文件按定义显然已经过时，但网络上的其他副本会保持更新，应作为你所发行版的“当前勘误”查阅。这些其他勘误副本位于：

1. <http://www.FreeBSD.org/releases/>
2. **ftp://ftp.FreeBSD.org/pub/FreeBSD/releases/<your-release>/ERRATA.TXT**（以及任何保持此位置最新镜像的站点）。

对此文件的任何更改也会自动发送电子邮件至：

`freebsd-current@FreeBSD.org`

有关所有 FreeBSD 安全公告，请参见：

<http://www.FreeBSD.org/security/>

以获取最新的安全事件信息。

---- 系统更新信息：

FreeBSD 引导管理器（boot0）存在一个 bug，会导致机器在引导过程中挂起且无屏幕输出。

修复：通过引导软盘或 CD-ROM 将机器引导至 FreeBSD，然后从以下位置下载新的 boot0 二进制文件：

**<http://people.FreeBSD.org/~jhb/4.1R/i386/boot0>**

下载新二进制文件后，使用 `boot0cfg` 命令将其安装到硬盘上。例如，如果 boot0 位于磁盘 ad0 上，则运行以下命令：

```sh
/usr/sbin/boot0cfg -B -b /path/to/downloaded/boot0 ad0
```

你也可以使用 cvsup 更新源代码树，并从源代码构建新的 boot0 二进制文件。你需要 **src/sys/boot/i386/boot0/boot0.s** 的 1.14.2.3 版本或更高版本。

此文件的 MD5 校验和为：

```sh
MD5 (boot0) = 8770a386dba44f0aa06b15db72c1f624
```

要验证你所下载副本的校验和，请执行以下命令：

```sh
/sbin/md5 /path/to/downloaded/boot0
```

并与上述值进行比较。
