# FreeBSD 14.4-RELEASE 发行说明（2026 年 3 月 10 日）

- 原文地址：[FreeBSD 14.4-RELEASE Release Notes](https://www.freebsd.org/releases/14.4R/relnotes/)
- 版本：[Alexander Ziaee](https://cgit.freebsd.org/doc/commit/?id=97584c7947)
- 原文最后发布时间：2026 年 3 月 9 日

## 摘要

FreeBSD 14.4-RELEASE 发行说明是对在 14-STABLE 开发分支上，对 FreeBSD 基本系统所做变动的总结。本文件列出了自上回发行以来 RELEASE 的适用安全公告，以及 FreeBSD 内核和用户空间的重大变动。还有一些有关更新的简要说明。

## 介绍

本文件是 FreeBSD 14.4-RELEASE 的发行说明。它述说了 FreeBSD 最近新增、变动和删除的功能。它还提供了一些关于从 FreeBSD 旧版本进行更新的说明。

这些发行说明所适用的“RELEASE”发行版表示自 14-STABLE 创建以来，14-STABLE 开发分支上的最新节点。有关该分支上预构建二进制“RELEASE”发行版的信息，请参见 [https://www.FreeBSD.org/releases/](https://www.freebsd.org/releases/)。

这些发行说明所适用的“RELEASE”发行版意味着在 14-STABLE 开发分支上，介于 14.3-RELEASE 和后续 14.5-RELEASE 之间的一个节点。有关该分支上预构建二进制“RELEASE”发行版的信息，请参见 [https://www.FreeBSD.org/releases/](https://www.freebsd.org/releases/)。

FreeBSD 14.4-RELEASE 的这个发行版是一个“RELEASE”发行版。可以在 [https://www.FreeBSD.org/releases/](https://www.freebsd.org/releases/) 或其镜像站找到它。有关获取此（或其他）FreeBSD “RELEASE”发行版的更多信息，请参见 [FreeBSD Handbook](https://docs.freebsd.org/en/books/handbook/) 的 [获取 FreeBSD 附录](https://docs.freebsd.org/en/books/handbook/mirrors)。

建议用户在安装 FreeBSD 之前查阅发布勘误表。勘误文档会更新在发布周期后期或发布后发现的“最新突发”信息。通常，它包含了已知错误、安全公告以及文档更正信息。可在 FreeBSD 网站上找到 FreeBSD 14.4-RELEASE 的最新勘误表。

本文档描述了自 14.3-RELEASE 以来 FreeBSD 中用户最易察觉的新功能或功能变动。一般来说，除非某功能被特别标记为已合并（MERGED），否则此处所述的变动仅 14-STABLE 分支特有。

典型的发行说明条目记录了 14.3-RELEASE 之后发布的最新安全公告、新的驱动或硬件支持、新命令或选项、主要错误修复或第三方的软件升级。它们也可能列出主要 Ports/软件包的更改或发布工程实践。显然，发行说明不可能列出 FreeBSD 两个版本之间的每一项更改；本文档主要关注安全公告、用户可见的更改以及主要架构改进。

## 从旧版本 FreeBSD 进行更新

使用 [freebsd-update(8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 工具能在 RELEASE 版本之间（二进制）进行更新（以及各安全分支的快照）。请参阅针对特定版本的升级流程，[FreeBSD 14.4-RELEASE 更新信息](https://www.freebsd.org/releases/14.4R/installation/#upgrade-binary)，以及 FreeBSD 手册中的 [二进制更新流程](https://docs.freebsd.org/en/books/handbook/cutting-edge/#freebsdupdate-upgrade) 获取更多详细信息。此操作将更新未修改的用户空间实用程序，以及随官方 FreeBSD 发布的一部分来分发的未修改 GENERIC 内核。[freebsd-update(8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 工具要求进行更新的主机具有互联网连接。

支持基于源码的更新（即通过源代码重新编译 FreeBSD 基本系统进行的更新），请参照 `/usr/src/UPDATING` 中的说明操作。

>**警告**
>
>在更新 FreeBSD 前应先备份 **所有** 数据和配置文件。

## 用户空间

本节包含了用户空间应用程序、第三方软件和系统实用工具的更改和新增内容。

### 用户空间应用程序变动

现在工具 [diff(1)](https://man.freebsd.org/cgi/man.cgi?query=diff&sektion=1&format=html) 在 Stone 算法的文件比较阶段报告遇到的 I/O 错误时，将提供错误消息，而以前仅通过退出状态指示失败。[3c10ed2ba3aa](https://cgit.freebsd.org/src/commit/?id=3c10ed2ba3aa)。（由 Klara, Inc. 赞助）

工具 [diff(1)](https://man.freebsd.org/cgi/man.cgi?query=diff&sektion=1&format=html) 不再错误地将文件或目录与自身比较，修复了 diff 可能产生误导性输出的错误。此外，还进行了若干内部正确性和稳健性改进（见相关提交），包括修复分页代码中的资源泄漏、改进了文件描述符操作的错误处理，以及在使用非常大上下文窗口时防止潜在的整数溢出。还添加了额外测试以覆盖这些情况。[b4139147bbb7](https://cgit.freebsd.org/src/commit/?id=b4139147bbb7)、[6761e555376e](https://cgit.freebsd.org/src/commit/?id=6761e555376e)、[2434f3b279a9](https://cgit.freebsd.org/src/commit/?id=2434f3b279a9)、[238bf5ebf684](https://cgit.freebsd.org/src/commit/?id=238bf5ebf684)。（由 Klara, Inc. 赞助）

特权提升工具 [mdo(1)](https://man.freebsd.org/cgi/man.cgi?query=mdo&sektion=1&format=html) 增加了控制启动进程的用户和组 ID 的新选项，如保持当前用户的 `-k`，设置主组和附加组的 `-g` 和 `-G`，修改附加组的 `-s`，以及覆盖特定 ID 的 `--euid`/`--ruid`/`--svuid`/`--egid`/`--rgid`/`--svgid`。这提供了对进程凭据的更细粒度控制，同时保持与现有行为的兼容性。[58f55afb301b](https://cgit.freebsd.org/src/commit/?id=58f55afb301b)。（由 FreeBSD 基金会 | 谷歌公司（谷歌编程之夏 2025）赞助）

现在 [sockstat(1)](https://man.freebsd.org/cgi/man.cgi?query=sockstat&sektion=1&format=html) 工具将默认显示 UDP-Lite 端点，使这些套接字与其他网络连接一起可见。[23cda744e4da](https://cgit.freebsd.org/src/commit/?id=23cda744e4da)

[nuageinit(7)](https://man.freebsd.org/cgi/man.cgi?query=nuageinit&sektion=7&format=html) 工具支持了 **chpasswd** 命令，可通过列表或多行字符串更改密码，包括为兼容某些提供商的已弃用语法。[6c912470030b](https://cgit.freebsd.org/src/commit/?id=6c912470030b)。（由 OVHCloud 赞助）

现在工具 [pkg(7)](https://man.freebsd.org/cgi/man.cgi?query=pkg&sektion=7&format=html) 已与 [pkg(8)](https://man.freebsd.org/cgi/man.cgi?query=pkg&sektion=8&format=html) 使用相同的方式解析命令行参数，要求选项放置在相同位置。**注意**：这改变了一些以前可接受的命令序列行为，例如 `pkg -f bootstrap` 不再有效；用户必须改为使用 `pkg bootstrap -f`。[62947e508161](https://cgit.freebsd.org/src/commit/?id=62947e508161)。（由 FreeBSD 基金会 | FreeBSD 基金会赞助）

安装程序 [bsdinstall(8)](https://man.freebsd.org/cgi/man.cgi?query=bsdinstall&sektion=8&format=html) 不再支持使用 MBR 磁盘布局进行 ZFS 安装。移除了之前可能导致安装失败的错误方案。[220584471931](https://cgit.freebsd.org/src/commit/?id=220584471931)。（由 FreeBSD 基金会赞助）

工具 [freebsd-update(8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 现在将按特定顺序安装共享库（libsys、libc、libthr，然后是其他库），从而防止从 14.x 更新到 15.x 时出错。[e26928669f39](https://cgit.freebsd.org/src/commit/?id=e26928669f39)。（由 [https://www.patreon.com/cperciva](https://www.patreon.com/cperciva) 赞助）

文件系统创建工具 [newfs(8)](https://man.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&format=html) 增加了标志 `-u`，可禁用 UFS2 文件系统的默认软更新和软更新日志。[929ef0d36c6c](https://cgit.freebsd.org/src/commit/?id=929ef0d36c6c)。（由 Klara, Inc. | NetApp, Inc. 赞助）

工具 [ngctl(8)](https://man.freebsd.org/cgi/man.cgi?query=ngctl&sektion=8&format=html) 增加了标志 `-j`，可附加并在 jail 内运行，从而能在 [jail(8)](https://man.freebsd.org/cgi/man.cgi?query=jail&sektion=8&format=html) 内操作 netgraph 节点。这使管理员能够在 ngctl 可能无法直接使用的 jail 中管理 netgraph 配置。[04911babef1b](https://cgit.freebsd.org/src/commit/?id=04911babef1b)

新增控制声卡设备的工具 [sndctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sndctl&sektion=8&format=html)，其接口类似于 [mixer(8)](https://man.freebsd.org/cgi/man.cgi?query=mixer&sektion=8&format=html)。[00988d12bc37](https://cgit.freebsd.org/src/commit/?id=00988d12bc37)。（由 FreeBSD 基金会赞助）

[jail(8)](https://man.freebsd.org/cgi/man.cgi?query=jail&sektion=8&format=html) 子系统增加了参数 `meta` 和 `env`，可为每个 jail 关联任意字符串元数据和环境信息。可在创建 jail 时设置这些参数，也可使用 `jail -cm` 后期修改，并可通过 [jls(8)](https://man.freebsd.org/cgi/man.cgi?query=jls&sektion=8&format=html) 查看。[sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) `security.jail.meta_maxbufsize` 控制着这些参数的最大大小。[527027da391d](https://cgit.freebsd.org/src/commit/?id=527027da391d)。（由 SkunkWerks GmbH 赞助）

现在，蓝牙启动脚本 `rc.d/bluetooth` 将最多重试三次 `hccontrol reset` 来提高可靠性，还修复了可能生成多余文件的重定向错误。[53d1c328e912](https://cgit.freebsd.org/src/commit/?id=53d1c328e912)

工具 [swapon(8)](https://man.freebsd.org/cgi/man.cgi?query=swapon&sektion=8&format=html) 现在可使用 [md(4)](https://man.freebsd.org/cgi/man.cgi?query=md&sektion=4&format=html) 设备的加密交换文件（后缀带 `.eli`），可在 [fstab(5)](https://man.freebsd.org/cgi/man.cgi?query=fstab&sektion=5&format=html) 中配置加密交换，如之前文档所述。[9d80d681ee9d](https://cgit.freebsd.org/src/commit/?id=9d80d681ee9d)

[dumpon(8)](https://man.freebsd.org/cgi/man.cgi?query=dumpon&sektion=8&format=html) 手册记录了 [crash(8)](https://man.freebsd.org/cgi/man.cgi?query=crash&sektion=8&format=html) 可调转储行为的设置。[7c8717183536](https://cgit.freebsd.org/src/commit/?id=7c8717183536)

[ipfw(8)](https://man.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&format=html) 手册记录了如何删除 NAT 配置实例。[186ac4724746](https://cgit.freebsd.org/src/commit/?id=186ac4724746)

现在，[mtree(8)](https://man.freebsd.org/cgi/man.cgi?query=mtree&sektion=8&format=html) 手册明确指出关键字 `type` 仍然是必需的，并不会被 `-R all` 删除。这保证了行为一致性，并防止对命令的潜在误解。[f957857c4835](https://cgit.freebsd.org/src/commit/?id=f957857c4835)

现在，[pf.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=pf.conf&sektion=5&format=html) 手册说明，在列表宏中使用的网络地址范围必须使用额外的单引号进行引用。[0077daf9cdc4](https://cgit.freebsd.org/src/commit/?id=0077daf9cdc4)

现在 [pw(8)](https://man.freebsd.org/cgi/man.cgi?query=pw&sektion=8&format=html) 手册明确了选项 `groupadd` 和 `groupmod` 中 `-M`、`-m` 与 `-d` 标志的成员列表可接受的格式。[78343cd2a0f1](https://cgit.freebsd.org/src/commit/?id=78343cd2a0f1)

现在 [vt(4)](https://man.freebsd.org/cgi/man.cgi?query=vt&sektion=4&format=html) 手册新增了关于扩大滚动缓冲区的示例，还新增了一个有关控制台字体的章节，涵盖字体转换、支持情况及使用方法。[ce92b9d8332a](https://cgit.freebsd.org/src/commit/?id=ce92b9d8332a), [c330c43e58d7](https://cgit.freebsd.org/src/commit/?id=c330c43e58d7)
