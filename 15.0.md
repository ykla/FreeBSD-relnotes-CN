# FreeBSD 15.0-RELEASE 发行说明（2025 年 12 月 2 日）

- 原文：[FreeBSD 15.0-RELEASE Release Notes](https://www.freebsd.org/releases/15.0R/relnotes/)
- 版本撰写人： [Dave Cottlehuber](https://cgit.freebsd.org/doc/commit/?id=7743f2e569)
- 版本时间：2025 年 12 月 2 日

## 摘要

FreeBSD 15.0 - RELEASE 的发行注记包含了在 15 - STABLE 开发线上对 FreeBSD 基础系统所做变更的总结。本文档列出了自上次发行以来发布的适用安全公告，以及对 FreeBSD 内核和用户态的重要变更。同时也简要给出了一些关于升级的说明。

## 介绍

本文件包含 FreeBSD 15.0 - RELEASE 的发行注记。它描述了 FreeBSD 最近新增、变更或删除的特性，并提供了从先前版本的 FreeBSD 升级时的若干说明。

这些发行注记所对应的 “release” 发布版本，代表了位于 14.0 - RELEASE 与未来的 15.1 - RELEASE 之间、15 - STABLE 开发分支上的最新节点。关于该分支上预构建二进制 “release” 发布版本的信息可在 [https://www.FreeBSD.org/releases/](https://www.freebsd.org/releases/) 找到。关于获取此（或其他） FreeBSD “release” 发布版本的更多信息，可参阅 [FreeBSD 手册](https://docs.freebsd.org/en/books/handbook//) 的 [获取 FreeBSD 附录](https://docs.freebsd.org/en/books/handbook//mirrors)。

强烈建议所有用户在安装 FreeBSD 之前查阅发行勘误。发行勘误文档会随着发行周期末期或发行之后发现的 “最新” 信息而更新。通常，它包含已知缺陷、安全公告以及文档修正。FreeBSD 15.0 - RELEASE 的最新发行勘误可在 FreeBSD 网站上找到。

本文档描述了自 14.0 - RELEASE 以来 FreeBSD 中对用户最为可见的新特性或变更。一般而言，这里描述的变更为 15 - STABLE 分支所独有，除非特别标记为 MERGED 特性。

典型的发行注记项目包括：在 14.0 - RELEASE 之后发布的近期安全公告、新的驱动或硬件支持、新的命令或选项、主要缺陷修复或第三方软件升级。它们也可能列出对主要 Ports / 包或发行工程实践的变更。当然，发行注记不可能列出 FreeBSD 在两个发行版本之间的每一项变更；本文主要聚焦安全公告、用户可见变更以及主要架构改进。

## 从先前版本的 FreeBSD 升级

在 RELEASE 版本之间（以及各安全分支的快照之间）进行二进制升级，可使用 [freebsd - update (8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 工具。请参阅与发行版本相关的升级流程 —— [FreeBSD 15.0 - RELEASE 升级信息](https://www.freebsd.org/releases/15.0R/installation/#upgrade-binary)，并结合 FreeBSD 手册中的 [二进制升级流程](https://docs.freebsd.org/en/books/handbook/cutting-edge/#freebsdupdate-upgrade) 了解更多细节。该方式会更新未修改的用户态工具，以及作为官方 FreeBSD 发行版一部分提供的未修改 GENERIC 内核。[freebsd - update (8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 工具要求被升级的主机具备 Internet 连接。

基于源码的升级（即通过重新编译 FreeBSD 基础系统进行升级）也受支持，其步骤可参阅 **/usr/src/UPDATING** 中的说明。

## 从已有的预发行基础系统包安装进行升级

|   | 对于使用 FreeBSD 15.0 的 PRERELEASE、ALPHA 和 BETA 构建版本的用户，由于 FreeBSD.org 基础设施在后期发生变更，无法直接使用 [pkg - upgrade (8)](https://man.freebsd.org/cgi/man.cgi?query=pkg-upgrade&sektion=8&format=html) 工具进行升级。 |
| - | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

用户应手动从 `15.0-RELEASE` 标签的源码树检出，或从 `STABLE` 或 `CURRENT` 分支的更晚提交中复制所需文件；也可以从官方发行版基础系统包中强制安装 `FreeBSD-pkg-bootstrap` 包。

推荐且最安全的方式，是使用 2025 - 11 - 27 22:00 UTC 之后的 head、stable / 15 或 releng / 15.0 分支的源码树检出。

```
# cp /usr/src/usr.sbin/pkg/FreeBSD.conf.quarterly-release \
       /etc/pkg/FreeBSD.conf
# cp -R /usr/src/share/keys/pkgbase-15 /usr/share/keys/pkgbase-15
```

没有安装最新源码的用户，可采用安全性稍低但更简单的方式：在安装后验证校验和。由于这些文件与架构无关，因此其校验和在所有平台上均一致。在使用 [pkg (8)](https://man.freebsd.org/cgi/man.cgi?query=pkg&sektion=8&format=html) 进行升级前，需要启用 FreeBSD - base 包仓库。

```
# pkg add -f https://pkg.freebsd.org/FreeBSD:15:$(uname -p)/base_release_0/FreeBSD-pkg-bootstrap-15.0.pkg
```

## 架构

久经考验的 32 位硬件平台 i386、armv6 以及 32 位 powerpc 已经退役。32 位应用支持仍然存在，通过它们各自 64 位平台的 32 位兼容模式提供。armv7 平台仍是最后支持的 32 位平台。我们感谢这些平台多年来的服务。

### 用户空间配置更改

Kerberos v5 认证服务 [krb5kdc(8)](https://man.freebsd.org/cgi/man.cgi?query=krb5kdc&sektion=8&format=html) 在 [daemon(8)](https://man.freebsd.org/cgi/man.cgi?query=daemon&sektion=8&format=html) 下增加了新的 `kdc_restart` 变量。在 [rc.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&format=html) 中设置 `kdc_restart="YES"` 可在 kdc 异常终止时自动重启。设置 `kdc_restart_delay="N"` 可指定重启 kdc 前的延迟秒数。[abc4b3088941](https://cgit.freebsd.org/src/commit/?id=abc4b3088941)

`daily` [periodic(8)](https://man.freebsd.org/cgi/man.cgi?query=periodic&sektion=8&format=html) 脚本默认在邮件中显示的上下文减少，以减小输出量。此行为可通过 [periodic.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=periodic.conf&sektion=5&format=html) 中的 `daily_diff_flags` 变量控制。同样，安全脚本显示的变更上下文也比以前少，可通过 `security_status_diff_flags` 变量控制。[538994626b9f](https://cgit.freebsd.org/src/commit/?id=538994626b9f), [37dc394170a5](https://cgit.freebsd.org/src/commit/?id=37dc394170a5), [128e78ffb084](https://cgit.freebsd.org/src/commit/?id=128e78ffb084)

`FreeBSD-base` 仓库现在在 `/etc/pkg/FreeBSD.conf` 中定义，默认禁用。使用 pkgbase 在 15.0-RC1 之前安装（如果运行 `releng/15.0`）或 11 月 15 日之前安装（如果使用 `stable`/`main` 快照）的系统，需要从 `/usr/local/etc/pkg/repos/` 中删除 `FreeBSD-base` 仓库定义，并替换为一行 `FreeBSD-base: { enabled: yes }`。[5d832135a971](https://cgit.freebsd.org/src/commit/?id=5d832135a971)

[powerd(8)](https://man.freebsd.org/cgi/man.cgi?query=powerd&sektion=8&format=html) 工具现在在 arm64 Raspberry Pi 镜像（`arm64-aarch64-RPI` 文件）中默认在 `/etc/rc.conf` 启用。这可以防止 CPU 时钟长期运行缓慢。[4347ef60501f](https://cgit.freebsd.org/src/commit/?id=4347ef60501f)

[ps(1)](https://man.freebsd.org/cgi/man.cgi?query=ps&sektion=1&format=html) 工具现在会自动移除“预设显示”中与显式请求的列重复的数据列。在此之前，如果用户请求添加某个“预设显示”（选项 `-j`、`-l`、`-u` 或 `-v`），与命令行前面显式请求列重复的列会被省略，但反向情况（预设显示出现在显式请求列之前）不会生效。此外，不同关键词但作为同一关键词别名的列也被视为持有相同数据，而不仅限于相同关键词的列。[cd768a840644](https://cgit.freebsd.org/src/commit/?id=cd768a840644) **（由 The FreeBSD Foundation 赞助）**

[ps(1)](https://man.freebsd.org/cgi/man.cgi?query=ps&sektion=1&format=html) 工具的 `-O` 选项现在更加灵活且可预测。[ps(1)] 的显示列列表现在首先构建时不考虑 `-O` 选项。第二步中，通过 `-O` 传入的所有列会按照其在 `-O` 参数中出现的顺序，插入到已构建显示的第一个 PID 列之后（如果存在，否则插入到开头）。[5dad61d9b949](https://cgit.freebsd.org/src/commit/?id=5dad61d9b949) **（由 The FreeBSD Foundation 赞助）**

[ps(1)] 工具的 `-a` 和 `-A` 选项现在总是显示所有进程。之前，当与影响进程选择的其他选项组合使用时，除了 `-X` 和 `-x`，`-a` 选项不会生效（`-A` 会退化为仅 `-x`），这与其他选择选项的规则相矛盾——一旦指定任何选择选项并选择了某个进程，该进程应被列出，这是 POSIX 所要求的，也是用户的自然预期。现在，指定 `-a` 或 `-A` 将列出所有进程，而不受诸如 `-U`、`-p`、`-G` 等其他选择选项影响，但 `-X` 和 `-x` 过滤选项仍然有效。特别地，如果只想列出特定 jail 的进程，不应将 `-a` 与 `-J` 一起使用。实际上，`-J` 从未像其初衷那样作为过滤器起作用，之前只是由于 bug 与 `-a` 偶然结合才生效。[93a94ce731a8](https://cgit.freebsd.org/src/commit/?id=93a94ce731a8) **（由 The FreeBSD Foundation 赞助）**

[ps(1)] 工具现在使用有效用户 ID 匹配当前用户的进程。此前使用的是真实用户 ID。这一更改使 [ps(1)] 在此行为上符合 POSIX 标准。[1aabbb25c9f9](https://cgit.freebsd.org/src/commit/?id=1aabbb25c9f9c4372) **（由 The FreeBSD Foundation 赞助）**

[ps(1)] 工具的 `-U` 标志现在按真实用户 ID 选择进程。这符合 POSIX 对 `-U` 的要求，也是大多数用户在大多数情况下真正需要的行为。此前，`-U` 会按有效用户 ID 选择进程（这是 POSIX 对 `-u` 选项的要求）。[995b690d1398](https://cgit.freebsd.org/src/commit/?id=995b690d1398) **（由 The FreeBSD Foundation 赞助）**

[sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) 工具新增了过滤 jail、prison 和 vnet 变量的标志，使用户无需查看源代码即可判断变量是否属于 jail/prison/vnet。[615c9ce250ee](https://cgit.freebsd.org/src/commit/?id=615c9ce250ee)

### 通用内核变更

ktrace(2) 现在会记录有关 capability 模式违规的详细信息。kdump(1) 工具已更新，可显示这些信息。[9bec84131215](https://cgit.freebsd.org/src/commit/?id=9bec84131215), [96c8b3e50988](https://cgit.freebsd.org/src/commit/?id=96c8b3e50988), [05296a0ff616](https://cgit.freebsd.org/src/commit/?id=05296a0ff616), [6a4616a529c1](https://cgit.freebsd.org/src/commit/?id=6a4616a529c1), [0cd9cde767c3](https://cgit.freebsd.org/src/commit/?id=0cd9cde767c3), [aa32d7cbc92c](https://cgit.freebsd.org/src/commit/?id=aa32d7cbc92c)

FreeBSD 现原生实现了 Linux [inotify(2)](https://man.freebsd.org/cgi/man.cgi?query=inotify&sektion=2&format=html) 接口。系统调用本身并非 API 兼容，但 libc 提供了兼容接口，因此依赖 inotify 的软件可无需修改直接运行。[f1f230439fa4](https://cgit.freebsd.org/src/commit/?id=f1f230439fa4) **（由 Klara, Inc. 赞助）**

`fpu_kern_enter` 和 `fpu_kern_leave` 例程已在 powerpc 上实现，允许在内核中使用依赖浮点和向量寄存器的 [ossl(4)](https://man.freebsd.org/cgi/man.cgi?query=ossl&sektion=4&format=html) 加密函数。[91e53779b4fc](https://cgit.freebsd.org/src/commit/?id=91e53779b4fc)

支持 arm64 上的传统 PCI 热插拔。[355f02cddbf0](https://cgit.freebsd.org/src/commit/?id=355f02cddbf0) **（由 Arm Ltd 赞助）**

Jails 现在可以通过 [jail_set(2)](https://man.freebsd.org/cgi/man.cgi?query=jail_set&sektion=2&format=html) 和 [jail_get(2)](https://man.freebsd.org/cgi/man.cgi?query=jail_get&sektion=2&format=html) 中的 jail 描述符访问，也可以通过新的 `jail_attach_jd(2)` 和 `jail_remove_jd(2)` 系统调用操作。它们允许通过文件描述符接口管理 jails，从而避免 jail ID 固有的竞态条件，并可选择控制 jail 生命周期。[851dc7f859c2](https://cgit.freebsd.org/src/commit/?id=851dc7f859c2)

Jails 和 jail 描述符现在有关联的 [kevent(2)](https://man.freebsd.org/cgi/man.cgi?query=kevent&sektion=2&format=html) 过滤器，可跟踪 jail 的创建、变更、附加和移除。[1bd74d201a53](https://cgit.freebsd.org/src/commit/?id=1bd74d201a53) [9d7f89ef2607](https://cgit.freebsd.org/src/commit/?id=9d7f89ef2607)

为 MAC 模块的 jail 参数创建了新的通用 'mac' 节点。未来所有 MAC 模块的 jail 参数都会显示在该节点下。有关 MAC 的介绍，请参见 [mac(4)](https://man.freebsd.org/cgi/man.cgi?query=mac&sektion=4&format=html)。首个使用该节点的模块是 [mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html)。[5041b20503db](https://cgit.freebsd.org/src/commit/?id=5041b20503db), [f3a06ced2568](https://cgit.freebsd.org/src/commit/?id=f3a06ced2568) **（由 FreeBSD Foundation 赞助）**

[mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html) 在经过多项重要修复后，现在被认为已达到生产就绪水平。[bbf8af664dc9](https://cgit.freebsd.org/src/commit/?id=bbf8af664dc9), [292c814931d9](https://cgit.freebsd.org/src/commit/?id=292c814931d9), [53d2e0d48549](https://cgit.freebsd.org/src/commit/?id=53d2e0d48549), [add521c1a5d2](https://cgit.freebsd.org/src/commit/?id=add521c1a5d2), [2a20ce91dc29](https://cgit.freebsd.org/src/commit/?id=2a20ce91dc29), [fa4352b74580](https://cgit.freebsd.org/src/commit/?id=fa4352b74580), [3d8d91a5b32c](https://cgit.freebsd.org/src/commit/?id=3d8d91a5b32c), [8f7e8726e3f5](https://cgit.freebsd.org/src/commit/?id=8f7e8726e3f5), [89958992b618](https://cgit.freebsd.org/src/commit/?id=89958992b618) **（由 FreeBSD Foundation 赞助）**

[mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html) 现在支持通过 `security.mac.do.rules` [sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) 控件在 jails 内修改规则。[b3f93680e39b](https://cgit.freebsd.org/src/commit/?id=b3f93680e39b) **（由 FreeBSD Foundation 赞助）**

引入 [setcred(2)](https://man.freebsd.org/cgi/man.cgi?query=setcred&sektion=2&format=html) 系统调用及相关 MAC 钩子。该系统调用允许一次性设置进程所需的所有凭证：有效、真实及保存的用户 ID，有效、真实及保存的组 ID，辅助组以及 MAC 标签。除了提供原子性之外，它相较于标准凭证设置系统调用（如 `setuid()`、`seteuid()` 等）的优势在于，使 MAC 模块（如 [mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html)）能够以更精细的方式限制进程可能获得的凭证集合，因为它们现在可以看到最终期望状态并与初始状态进行比较。[ddb3eb4efe55](https://cgit.freebsd.org/src/commit/?id=ddb3eb4efe55) **（由 FreeBSD Foundation 赞助）**

在 [mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html) 中支持将多个用户和组作为单条规则的目标。支持组目标是 [mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html) 能够在目标凭证中强制执行有限的有效新组集并允许仅限组的凭证转换的前提条件。允许的组与一个或多个用户 ID 关联。规则目标部分的多个用户和组被视为可选项（包含或关系），但表达辅助组的强制存在或缺失的子句除外。规则语法发生了不兼容更改，但迁移现有规则仅需在目标部分前添加 `uid=`，将逗号（`,`）替换为分号（`;`），冒号（`:`）替换为大于号（`>`）。详情请参见 [mac_do(4)](https://man.freebsd.org/cgi/man.cgi?query=mac_do&sektion=4&format=html) 手册页。[83ffc412b2e9](https://cgit.freebsd.org/src/commit/?id=83ffc412b2e9), [8f7e8726e3f5](https://cgit.freebsd.org/src/commit/?id=8f7e8726e3f5), [f01d26dec67f](https://cgit.freebsd.org/src/commit/?id=f01d26dec67f) **（由 FreeBSD Foundation 赞助）**

[sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) 现在可在 jail 中附加并运行自身。这允许父 jail 在子 jail 未安装 [sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) 的情况下（例如轻量级 OCI 容器或精简 jail）获取或设置内核状态。这在操作 jail prison 或 vnet sysctl 时尤为有用。例如，`sysctl -j foo -Ja` 或 `sysctl -j foo net.fibs=2`。[8d5d7e2ba3a6](https://cgit.freebsd.org/src/commit/?id=8d5d7e2ba3a6)

使 vnet [sysctl(9)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=9&format=html) 变量可通过 loader 调整。在 [3da1cf1e88f8](https://cgit.freebsd.org/src/commit/?id=3da1cf1e88f8) 中，`CTLFLAG_TUN` 标志的含义被扩展为自动检查是否存在用于在早期启动时初始化 SYSCTL 的内核环境变量。该功能适用于所有静态和动态创建的 SYSCTL 类型，但不包括属于 VNET 的 SYSCTL。需要注意的是，该实现有一个限制：它的行为与非 vnet loader 调整项相同，即内核或模块初始化后，通过 `kenv` 等方式修改内核环境变量不会影响随后创建的 VNET 的对应变量。为解决此问题，可在 VNET 构建过程中使用 `TUNABLE_XXX_FETCH` 将内核环境变量提取到这些 vnet 变量中。[894efae09de4](https://cgit.freebsd.org/src/commit/?id=894efae09de4)

[sound(4)](https://man.freebsd.org/cgi/man.cgi?query=sound&sektion=4&format=html)：按需分配 vchans。重构了 `pcm_chnalloc()`，并将 `vchan_setnew()`（现已移除）和 `dsp_open()` 的通道创建部分合并为新的 `dsp_chn_alloc()` 函数。该函数负责使用空闲的硬件通道（当 vchans 被禁用时）或分配新的 vchan。`hw.snd.vchans_enable`（之前为 `hw.snd.maxautovchans`）和 `dev.pcm.X.{play|rec}.vchans` 现在作为可调项，仅用于启用或禁用 vchans，而不是设置数量或分配/释放 vchans。由于这些 sysctl 不再触发分配/释放操作，其效果是即时的，而之前在将 `dev.pcm.X.{play|rec}.vchans` 设置为非常大值时可能导致系统冻结。[960ee8094913](https://cgit.freebsd.org/src/commit/?id=960ee8094913) **（由 FreeBSD Foundation 赞助）**

移除了 `hw.snd.version` [sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) 控件。[7398d1ece5cf](https://cgit.freebsd.org/src/commit/?id=7398d1ece5cf) **（由 FreeBSD Foundation 赞助）**

[sound(4)](https://man.freebsd.org/cgi/man.cgi?query=sound&sektion=4&format=html) 中的 `unit.*` 代码已废弃，作为一部分，移除了 `hw.snd.maxunit` [loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 可调项。[25723d66369f](https://cgit.freebsd.org/src/commit/?id=25723d66369f) **（由 FreeBSD Foundation 赞助）**

修复了部分 AMD GPU 用户使用 `drm-kmod` Ports 中 amdgpu DRM 驱动（自 v5.15 起，`graphics/drm-515-kmod` Port）时出现的逐渐变慢和冻结问题。尤其是搭载 Green Sardine、Polaris 10/20 和 Vega 芯片的显卡用户受到影响。近期 Intel GPU（13 代及以上）用户可能也受到影响。[718d1928f874](https://cgit.freebsd.org/src/commit/?id=718d1928f874), [4ca9190251bb](https://cgit.freebsd.org/src/commit/?id=4ca9190251bb), [986edb19a49c](https://cgit.freebsd.org/src/commit/?id=986edb19a49c), [9d1f3ce79d85](https://cgit.freebsd.org/src/commit/?id=9d1f3ce79d85), [da257e519bc0](https://cgit.freebsd.org/src/commit/?id=da257e519bc0) **（由 FreeBSD Foundation 赞助）**

改进并修复了内存域（NUMA）迭代代码，特别是降低了部分 DRM 驱动的图形操作延迟。[da257e519bc0](https://cgit.freebsd.org/src/commit/?id=da257e519bc0), [83ad6d8d8eee](https://cgit.freebsd.org/src/commit/?id=83ad6d8d8eee), [b15ff7214020](https://cgit.freebsd.org/src/commit/?id=b15ff7214020) **（由 FreeBSD Foundation 赞助）**

有效组 ID 现在存储在 `struct cred` 的新字段 `cr_gid` 中，并已从 `cr_groups[]` 的第一个元素移除，`cr_groups[]` 现在仅包含辅助组。所有使用 `cr_groups[0]` 的下游及外部模块必须修改为使用 `cr_gid`，而循环遍历 `cr_groups[]` 元素（排除 `cr_groups[0]`，即仅处理辅助组）的相关代码也需调整，因为辅助组现在从 `&cr_groups[0]` 开始，而不是 `&cr_groups[1]`。需要同时兼容 15.0 及更早版本的代码可以继续使用宏 `cr_gid`，并可通过测试 `&cr_groups[0] != &cr_gid` 来正确遍历辅助组。[be1f7435ef21](https://cgit.freebsd.org/src/commit/?id=be1f7435ef218b1df35) **（由 FreeBSD Foundation 赞助）**

## 存储

本节涵盖文件系统及其他存储子系统（本地和网络）的更改与新增功能。

### 通用存储

增加了 Solaris 风格的扩展属性（在 NFSv4 中称为 named attributes）。目前，只有 ZFS（且文件系统数据集的 `xattr` 属性设置为 `dir`）和 NFSv4 支持它们。这些属性在目录中以普通文件形式呈现。详情参见 [named_attribute(7)](https://man.freebsd.org/cgi/man.cgi?query=named_attribute&sektion=7&format=html)。[2ec2ba7e232d](https://cgit.freebsd.org/src/commit/?id=2ec2ba7e232d), [df58e8b1506f](https://cgit.freebsd.org/src/commit/?id=df58e8b1506f), [f61844833ee8](https://cgit.freebsd.org/src/commit/?id=f61844833ee8), [b1b607bd200f](https://cgit.freebsd.org/src/commit/?id=b1b607bd200f), [ee95e4d02dbd](https://cgit.freebsd.org/src/commit/?id=ee95e4d02dbd)

增加了对通过 TCP 传输访问远程 NVMe over Fabrics 控制器的支持。可通过 [nvmecontrol(8)](https://man.freebsd.org/cgi/man.cgi?query=nvmecontrol&sektion=8&format=html) 中的新命令建立与远程控制器的连接。连接建立后，由 [nvmf(4)](https://man.freebsd.org/cgi/man.cgi?query=nvmf&sektion=4&format=html) 内核模块处理，创建 `nvmeX` 设备并将远程命名空间导出为 [nda(4)](https://man.freebsd.org/cgi/man.cgi?query=nda&sektion=4&format=html) 磁盘。[a1eda74167b5](https://cgit.freebsd.org/src/commit/?id=a1eda74167b5), [1058c12197ab](https://cgit.freebsd.org/src/commit/?id=1058c12197ab) **（由 Chelsio Communications 赞助）**

增加了对将命名空间导出到远程 NVMe over Fabrics 主机（通过 TCP 传输）的支持。[nvmft(4)](https://man.freebsd.org/cgi/man.cgi?query=nvmft&sektion=4&format=html) 内核模块为 CAM target 层增加了新的前端，可将 [ctl(4)](https://man.freebsd.org/cgi/man.cgi?query=ctl&sektion=4&format=html) LUN 导出为 NVMe 命名空间到远程主机。[ctld(8)](https://man.freebsd.org/cgi/man.cgi?query=ctld&sektion=8&format=html) 守护进程现在支持 NVMe 控制器，负责接受连接请求并将连接的队列对交给 [nvmft(4)](https://man.freebsd.org/cgi/man.cgi?query=nvmft&sektion=4&format=html) 处理。[a15f7c96a276](https://cgit.freebsd.org/src/commit/?id=a15f7c96a276), [66b5296f1b29](https://cgit.freebsd.org/src/commit/?id=66b5296f1b29) **（由 Chelsio Communications 赞助）**

增加了对 NVMe 命名空间动态调整大小的支持。[nvd(4)](https://man.freebsd.org/cgi/man.cgi?query=nvd&sektion=4&format=html) 和 [nda(4)](https://man.freebsd.org/cgi/man.cgi?query=nda&sektion=4&format=html) 驱动现在能够实时通知 geom 设备大小的变化。[86d3ec359a56](https://cgit.freebsd.org/src/commit/?id=86d3ec359a56) **（由 Netflix 赞助）**

### NFS

`nfs_reserved_port_only` [rc.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&format=html) 设置的默认值已更改。FreeBSD NFS 服务器现在要求请求的源端口必须位于特权端口范围内（即 ≤ 1023），这通常要求客户端在本地系统上具有提升权限。若希望恢复以前的行为，可在 [rc.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=rc.conf&sektion=5&format=html) 中设置 `nfs_reserved_port_only=NO`。[6d5ce2bb6344](https://cgit.freebsd.org/src/commit/?id=6d5ce2bb6344) **（由 FreeBSD 基金会赞助）**

为 [mountd(8)](https://man.freebsd.org/cgi/man.cgi?query=mountd&sektion=8&format=html) 定义了新的 `-a` 命令行选项，如果目录路径不是服务器文件系统的挂载点，该选项可防止使用 `-alldirs` 标志导出文件系统。[07cd69e272da](https://cgit.freebsd.org/src/commit/?id=07cd69e272da)

[tarfs(4)](https://man.freebsd.org/cgi/man.cgi?query=tarfs&sektion=4&format=html)、[tmpfs(4)](https://man.freebsd.org/cgi/man.cgi?query=tmpfs&sektion=4&format=html)、[cd9660(4)](https://man.freebsd.org/cgi/man.cgi?query=cd9660&sektion=4&format=html) 和 [ext2fs(4)](https://man.freebsd.org/cgi/man.cgi?query=ext2fs&sektion=4&format=html) 文件系统的 NFS 文件句柄布局已更改。导出这些文件系统的 NFS 服务器，其客户端需要先卸载然后重新挂载这些导出。[4db1b113b151](https://cgit.freebsd.org/src/commit/?id=4db1b113b151), [1ccbdf561f41](https://cgit.freebsd.org/src/commit/?id=1ccbdf561f41), [205659c43d87](https://cgit.freebsd.org/src/commit/?id=205659c43d87), [cf0ede720391](https://cgit.freebsd.org/src/commit/?id=cf0ede720391), [8ae6247aa966](https://cgit.freebsd.org/src/commit/?id=8ae6247aa966) **（由 FreeBSD 基金会赞助）**

[mountd(8)](https://man.freebsd.org/cgi/man.cgi?query=mountd&sektion=8&format=html) 服务器已修改为使用 [strunvis(3)](https://man.freebsd.org/cgi/man.cgi?query=strunvis&sektion=3&format=html) 来解码 [exports(5)](https://man.freebsd.org/cgi/man.cgi?query=exports&sektion=5&format=html) 文件中的目录名，从而允许目录名中包含空格等特殊字符。可使用 `vis -M` 对这些目录名进行编码；详情参见 [vis(1)](https://man.freebsd.org/cgi/man.cgi?query=vis&sektion=1&format=html)。[2c83f1ada435](https://cgit.freebsd.org/src/commit/?id=2c83f1ada435)

允许在 [exports(5)](https://man.freebsd.org/cgi/man.cgi?query=exports&sektion=5&format=html) 的 `-maproot` 或 `-mapall` 选项中指定系统配置支持的任意数量的组。此前的上限为 `NGROUPS_MAX + 1`，其中 `NGROUPS_MAX` 仅表示允许的辅助组数量的最小最大值。现在改为使用正确的 `{NGROUPS_MAX} + 1` 值，`{NGROUPS_MAX}` 在运行时通过 [sysconf(3)](https://man.freebsd.org/cgi/man.cgi?query=sysconf&sektion=3&format=html) 获取。[e87848a8150e](https://cgit.freebsd.org/src/commit/?id=e87848a8150e) **（由 FreeBSD 基金会赞助）**

在 `kern.rpc.unenc` 和 `kern.rpc.tls` 下新增了 [sysctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&format=html) 变量，使 NFS 服务器管理员能够监控 NFS-over-TLS 的使用情况。大量握手失败可能表明 NFS 配置存在问题。[b8e137d8d32d](https://cgit.freebsd.org/src/commit/?id=b8e137d8d32d)

使用 `nocto` 挂载选项时，NFSv4.1/4.2 的 delegations 利用率有所提升。这需要启用了 delegations 的最新 NFSv4.1/4.2 服务器。例如，在构建同时挂载 `src` 和 `obj` 的 FreeBSD 内核时，总 RPC 调用次数从 5461286 降至 945643，整体耗时降低约 20%。[171f66b0c2ca](https://cgit.freebsd.org/src/commit/?id=171f66b0c2ca), [50e733f19b37](https://cgit.freebsd.org/src/commit/?id=50e733f19b37)

新增对 NFSv4.2 Clone 操作的支持，该操作使用块克隆在 NFS 服务器上进行“写时复制”文件。当前仅适用于已启用块克隆的导出 ZFS 文件系统。[cce64f2e6851](https://cgit.freebsd.org/src/commit/?id=cce64f2e6851)

### UFS

创建新的 UFS 文件系统时，[newfs(8)](https://man.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&format=html) 默认启用软更新（soft updates）。[6b2af2d88ffd](https://cgit.freebsd.org/src/commit/?id=6b2af2d88ffd)

对于拥有超过 2G inode 的卷，UFS 的可靠性显著提升。根本问题在于 32 位 inode 编号被错误地解释为有符号值，导致符号扩展到 `ino_t`。[c069ca085bd1](https://cgit.freebsd.org/src/commit/?id=c069ca085bd1), [e36f069ecb47](https://cgit.freebsd.org/src/commit/?id=e36f069ecb47) **（由 FreeBSD 基金会赞助）**

UFS1 文件系统的 2038 年 1 月 19 日日期限制已延迟至 2106 年 2 月 7 日。仅影响旧 UFS1 格式的文件系统。详情请参阅提交说明。[1111a44301da](https://cgit.freebsd.org/src/commit/?id=1111a44301da)

为 `VOP_COPY_FILE_RANGE()` 添加块克隆支持。目前，只有启用块克隆的 ZFS 文件系统支持此功能，NFSv4.2 也支持。详细信息请参见 `pathconf(2)` 和 `copy_file_range(2)`。[37b2cb5ecb0f](https://cgit.freebsd.org/src/commit/?id=37b2cb5ecb0f)

## 启动加载器更改

ASCII [loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 现在可以在图形系统上通过 [loader.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=loader.conf&sektion=5&format=html) 中的可选变量 `loader_gfx` 再次启用。 [bef6d85b6de5](https://cgit.freebsd.org/src/commit/?id=bef6d85b6de5)

[loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 现在在读取其他配置文件后，会读取 `local_loader_conf_files` 变量中列出的本地配置文件，默认值为 **/boot/loader.conf.local**。[a25531db0fc2](https://cgit.freebsd.org/src/commit/?id=a25531db0fc2)

[loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 现在可以根据 SMBIOS 中的 planar maker、planar product、system product 和 uboot m_product 变量来读取特定配置文件。当前最详细的文档是 git 提交信息，[3eb3a802a31b](https://cgit.freebsd.org/src/commit/?id=3eb3a802a31b)。

在 EFI 系统上，[loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 的控制台检测已改进。如果没有 ConOut 变量，则检查 ConIn；如果发现多个设备，则优先使用串口设备。[20a6f4779ac6](https://cgit.freebsd.org/src/commit/?id=20a6f4779ac6) **（由 Netflix 赞助）**

[loader(8)](https://man.freebsd.org/cgi/man.cgi?query=loader&sektion=8&format=html) 的帧缓冲支持现在可以使用仅文本的视频驱动，从而节省空间。[57ca2848c0aa](https://cgit.freebsd.org/src/commit/?id=57ca2848c0aa) **（由 Netflix 赞助）**

在 arm64 系统上，[loader.efi(8)](https://man.freebsd.org/cgi/man.cgi?query=loader.efi&sektion=8&format=html) 对 ACPI 的检测现在更早完成。使用 ACPI 的 arm64 系统应更新 EFI 分区上的 **loader.efi**。[05cf4dda599a](https://cgit.freebsd.org/src/commit/?id=05cf4dda599a), [16c09de80135](https://cgit.freebsd.org/src/commit/?id=16c09de80135)

LinuxBoot 启动加载器现在可以在 aarch64 和 amd64 上从 Linux 启动 FreeBSD。[46010641267](https://cgit.freebsd.org/src/commit/?id=46010641267) **（由 Netflix 赞助）**

BIOS 启动加载器重新增加了对 gzip 和 bzip2 的支持，但默认移除了图形模式支持，以解决体积问题。（EFI 启动加载器保持不变，仍支持所有模式）[4d3b05a8530e](https://cgit.freebsd.org/src/commit/?id=4d3b05a8530e) **（由 Netflix 赞助）**

BIOS 启动加载器现在可以在表位于 4GB 以下时使用 SMBIOS v3（64 位）入口点。由于 BIOS 启动加载器作为 BTX 客户端在 amd64 上仍为 32 位，因此无法访问 4GB 以上地址，但 64 位入口点可能引用 4GB 以下的结构表，可在 BIOS 未提供 32 位入口点时使用，例如 Hetzner 虚拟机。[7f005c6699f4](https://cgit.freebsd.org/src/commit/?id=7f005c6699f4) **（由 FreeBSD 基金会赞助）**

BIOS 启动加载器现在优先使用 SMBIOS v3（64 位）入口点。当 32 位和 64 位入口点同时存在时，SMBIOS 规范规定 64 位入口点至少包含 32 位入口点引用的所有结构，即 32 位入口点仅为兼容性提供，而 64 位入口点更可能包含完整有效值。[3f744fb8b2c5](https://cgit.freebsd.org/src/commit/?id=3f744fb8b2c5) **（由 FreeBSD 基金会赞助）**

EFI 启动加载器现在同样优先使用 SMBIOS v3（64 位）入口点，与 BIOS 启动保持一致。不同之处在于，EFI 加载器在 64 位平台上运行，因此 v3 入口点的结构表不受 4GB 限制。[96f77576e9ea](https://cgit.freebsd.org/src/commit/?id=96f77576e9ea) **（由 FreeBSD 基金会赞助）**

## 网络（Networking）

本节描述影响 FreeBSD 网络的变更。

### 通用网络（General Network）

FreeBSD 现在实现了最初来自 OpenBSD 的 `SO_SPLICE` 接口。该功能允许用户空间应用将两个已连接的 TCP 套接字拼接在一起，此后到达一个套接字的数据会自动通过拼接的另一个套接字转发，而不是交付给应用程序。[a1da7dc1cdad](https://cgit.freebsd.org/src/commit/?id=a1da7dc1cdad) **（由 Klara, Inc. 赞助）** **（由 Stormshield 赞助）**

已恢复对 802 标准网络的 ARP（[arp(4)](https://man.freebsd.org/cgi/man.cgi?query=arp&sektion=4&format=html)）支持；此前在移除 FDDI 支持时被意外删除。（这与以太网标准封装不同。）[d776dd5fbd48](https://cgit.freebsd.org/src/commit/?id=d776dd5fbd48)

现在可以构建支持 IPv6（INET6）但不包含 IPv4（INET）的内核。[6df9fa1c6b83](https://cgit.freebsd.org/src/commit/?id=6df9fa1c6b83) 等

netgraph 的 [ng_ipfw(4)](https://man.freebsd.org/cgi/man.cgi?query=ng_ipfw&sektion=4&format=html) 模块不再将 cookie 截断为 16 位，现在支持完整 32 位。[dadf64c5586e](https://cgit.freebsd.org/src/commit/?id=dadf64c5586e)

在 [igc(4)](https://man.freebsd.org/cgi/man.cgi?query=igc&sektion=4&format=html) 驱动中新增了 AIM（自适应中断调节）支持。[472a0ccf847a](https://cgit.freebsd.org/src/commit/?id=472a0ccf847a) **（由 Rubicon Communications, LLC（“Netgate”）和 BBOX.io 赞助）**

该功能也已添加到 [lem(4)](https://man.freebsd.org/cgi/man.cgi?query=lem&sektion=4&format=html)、[em(4)](https://man.freebsd.org/cgi/man.cgi?query=em&sektion=4&format=html) 和 [igb(4)](https://man.freebsd.org/cgi/man.cgi?query=igb&sektion=4&format=html) 驱动中。FreeBSD 12.0 中引入的 UDP 性能重大回退（包括通过 UDP 的 NFS）被认为已通过此更改修复。[49f12d5b38f6](https://cgit.freebsd.org/src/commit/?id=49f12d5b38f6) **（由 Rubicon Communications, LLC（“Netgate”）和 BBOX.io 赞助）**

让 [ip6addrctl(8)](https://man.freebsd.org/cgi/man.cgi?query=ip6addrctl&sektion=8&format=html) 能够附加并在 jail 中运行。这将使管理 vnet jail 的地址选择策略更加容易，尤其适用于轻量级 OCI 容器或精简 jail。[b709f7b38cc4](https://cgit.freebsd.org/src/commit/?id=b709f7b38cc4)

[pf(4)](https://man.freebsd.org/cgi/man.cgi?query=pf&sektion=4&format=html) 包过滤器增加了新的运行时 [loader.conf(5)](https://man.freebsd.org/cgi/man.cgi?query=loader.conf&sektion=5&format=html) 可调参数 `net.pf.default_to_drop`，以及编译时选项 `PF_DEFAULT_TO_DROP`，使默认规则为丢弃。[7f7ef494f11d](https://cgit.freebsd.org/src/commit/?id=7f7ef494f11d), [3965be101c43](https://cgit.freebsd.org/src/commit/?id=3965be101c43)

在 [pf(4)](https://man.freebsd.org/cgi/man.cgi?query=pf&sektion=4&format=html) 中新增了路由到池（route-to pool）选项 “prefer-ipv6-nexthop”，允许通过 IPv6 网关路由 IPv4 数据包。[65c318630123](https://cgit.freebsd.org/src/commit/?id=65c318630123) [d2761422eb0a](https://cgit.freebsd.org/src/commit/?id=d2761422eb0a) **（由 InnoGames GmbH 赞助）**

[pf(4)](https://man.freebsd.org/cgi/man.cgi?query=pf&sektion=4&format=html) 现在支持 OpenBSD 风格的 NAT 语法。可以在 “pass” 和 “match” 规则中使用 “nat-to”、 “rdr-to” 和 “binat-to”。旧的 “nat on …” 语法仍可使用。[e0fe26691fc9](https://cgit.freebsd.org/src/commit/?id=e0fe26691fc9) **（由 InnoGames GmbH 赞助）**

[pfsync(4)](https://man.freebsd.org/cgi/man.cgi?query=pfsync&sektion=4&format=html) 协议已更新，可同步多个缺失属性。这修复了带有 route-to、af-to、rtable、dummynet、tags 和 scrub 选项的状态同步问题。如果需要与旧版本 FreeBSD 进行同步，可通过 `ifconfig pfsync0 version $VERSION` 配置协议版本，其中 $VERSION 为 13.X 版本使用 1301，14.X 版本使用 1400。对于运行 FreeBSD 15.0 的主机之间的同步，默认值为 1500。[99475087d63b](https://cgit.freebsd.org/src/commit/?id=99475087d63b) **（由 InnoGames GmbH 赞助）**

内核 TLS 支持现在在 aarch64、amd64、powerpc64 和 powerpc64le 的 `GENERIC`（默认）内核中默认启用。[b2f7c53430c3](https://cgit.freebsd.org/src/commit/?id=b2f7c53430c3) **（由 Chelsio Communications 赞助）**

新增了 `net.inet.{tcp,udp,raw}.bind_all_fibs` 可调参数。为了向后兼容，它们默认值为 1。设置为 0 会修改对应协议的套接字行为，使得非源自与套接字同一 FIB 的接口的数据包被忽略。在这种情况下，不同 FIB 的 TCP 和 UDP 套接字也可以绑定到同一地址。默认行为保持不变。[5dc99e9bb985](https://cgit.freebsd.org/src/commit/?id=5dc99e9bb985), [08e638c089ab](https://cgit.freebsd.org/src/commit/?id=08e638c089ab), [4009a98fe80b](https://cgit.freebsd.org/src/commit/?id=4009a98fe80b) **（由 Klara, Inc. 赞助）** **（由 Stormshield 赞助）**

对 `INADDR_ANY` 的连接（即将其作为 `localhost` 的别名使用）默认已禁用。可以通过将 `net.inet.ip.connect_inaddr_wild` sysctl 设置为 1 来重新启用此功能。[cd240957d7ba](https://cgit.freebsd.org/src/commit/?id=cd240957d7ba) **（由 The FreeBSD Foundation 赞助）**

新增内核内联 IPSEC 卸载基础设施。另见支持该功能的 [mlx5(4)](https://man.freebsd.org/cgi/man.cgi?query=mlx5&sektion=4&format=html) 驱动说明。[ef2a572bf6](https://cgit.freebsd.org/src/commit/?id=ef2a572bf6) **（由 NVIDIA networking 赞助）**

[ngctl(8)](https://man.freebsd.org/cgi/man.cgi?query=ngctl&sektion=8&format=html) 新增 `-j` 标志，允许其附加并在 jail 内运行，使得即使 jail 内未安装 [ngctl(8)](https://man.freebsd.org/cgi/man.cgi?query=ngctl&sektion=8&format=html)，也能操作 netgraph 节点。[72d01e62b082](https://cgit.freebsd.org/src/commit/?id=72d01e62b082)

[sockstat(4)](https://man.freebsd.org/cgi/man.cgi?query=sockstat&sektion=4&format=html) 默认显示 UDP-Lite 端点。[978615d7bf7c](https://cgit.freebsd.org/src/commit/?id=978615d7bf7c)

支持 FreeBSD 7 和 8 的 [ipfw(8)](https://man.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&format=html) 二进制兼容代码已被移除。[660255be1ed9](https://cgit.freebsd.org/src/commit/?id=660255be1ed9) **（由 The FreeBSD Foundation 赞助）**

### 网络协议（Network Protocols）

网络栈进行了大量改进，包括 [sctp(4)](https://man.freebsd.org/cgi/man.cgi?query=sctp&sektion=4&format=html) 栈的性能提升和错误修复。

通过 [sctp_peeloff(2)](https://man.freebsd.org/cgi/man.cgi?query=sctp_peeloff&sektion=2&format=html) 返回的描述符现在会继承父套接字的 Capsicum 能力 [rights(4)](https://man.freebsd.org/cgi/man.cgi?query=rights&sektion=4&format=html)。[ae3d7e27abc9](https://cgit.freebsd.org/src/commit/?id=ae3d7e27abc9) **（由 The FreeBSD Foundation 赞助）**

sysctl 变量 `net.inet.tcp.nolocaltimewait` 的默认值已从 1 改为 0。这意味着 FreeBSD 不再跳过远程地址为本地的端点的 `TIME_WAIT` 状态。新的 sysctl 变量 `net.inet.tcp.msl_local` 可用于控制这些端点在 `TIME_WAIT` 状态下的停留时间。`net.inet.tcp.nolocaltimewait` 已被弃用，计划在 FreeBSD 16 中移除。[c3fc0db3bc50](https://cgit.freebsd.org/src/commit/?id=c3fc0db3bc50) **（由 Netflix 赞助）**

本地流套接字（AF_UNIX/SOCK_STREAM）和有序数据包流套接字（AF_UNIX/SOCK_SEQPACKET）已改进，以提升批量传输和往返时间性能。SOCK_SEQPACKET 套接字已符合规范，现在表现为真正的流套接字，而在以前的 FreeBSD 版本中，它可能表现出数据报套接字的特性。依赖旧实现漏洞的应用可能需要调整。[d15792780760](https://cgit.freebsd.org/src/commit/?id=d15792780760)

### 无线网络（Wireless Networking）

LinuxKPI 802.11 兼容层 [linuxkpi_wlan(4)](https://man.freebsd.org/cgi/man.cgi?query=linuxkpi_wlan&sektion=4&format=html) 现支持来自 [wlan_gcmp(4)](https://man.freebsd.org/cgi/man.cgi?query=wlan_gcmp&sektion=4&format=html) 的 Galois/Counter 模式协议（GCMP）。**（由 The FreeBSD Foundation 赞助）**

继其他驱动之后，[iwlwififw(4)](https://man.freebsd.org/cgi/man.cgi?query=iwlwififw&sektion=4&format=html) 固件已从基础系统中移除，改为使用 Ports 方式并支持 [fwget(8)](https://man.freebsd.org/cgi/man.cgi?query=fwget&sektion=8&format=html)。在从早期版本升级时，用户必须先安装固件包。**（由 The FreeBSD Foundation 赞助）**

[iwlwifi(4)](https://man.freebsd.org/cgi/man.cgi?query=iwlwifi&sektion=4&format=html) 无线驱动支持部分 Intel Wi-Fi 5，以及全部 Intel Wi-Fi 6 和 Wi-Fi 7 硬件的 802.11ac（VHT）。**（由 The FreeBSD Foundation 赞助）**
[iwx(4)](https://man.freebsd.org/cgi/man.cgi?query=iwx&sektion=4&format=html) 无线驱动支持 Intel Wi-Fi 6 硬件的 802.11ac（VHT）。**（由 The FreeBSD Foundation 赞助）**
[rtwn(4)](https://man.freebsd.org/cgi/man.cgi?query=rtwn&sektion=4&format=html) 无线驱动支持 RTL8812A 和 RTL8821A 芯片组的 802.11ac（VHT）。
[rtw89(4)](https://man.freebsd.org/cgi/man.cgi?query=rtw89&sektion=4&format=html) 无线驱动支持部分 Realtek Wi-Fi 6 和 Wi-Fi 7 硬件的 802.11g。[a2d1e07f6451](https://cgit.freebsd.org/src/commit/?id=a2d1e07f6451) **（由 The FreeBSD Foundation 赞助）**

### 虚拟化支持（Virtualization Support）

[bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 和 [vmm(4)](https://man.freebsd.org/cgi/man.cgi?query=vmm&sektion=4&format=html) 现在支持 arm64 和 riscv 平台。`sysutils/u-boot-bhyve-arm64` 和 `sysutils/u-boot-bhyve-riscv` Ports 提供这些平台使用的引导加载程序。[47e073941f4e](https://cgit.freebsd.org/src/commit/?id=47e073941f4e) [d3916eace506](https://cgit.freebsd.org/src/commit/?id=d3916eace506) **（由 Arm Ltd 赞助）** **（由 Innovate UK 赞助）** **（由 The FreeBSD Foundation 赞助）** **（由布加勒斯特理工大学赞助）**

[bhyve(4)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=4&format=html) 现在支持 “slirp” 网络后端，从而启用非特权用户网络。目前仅支持对虚拟机的入站连接，虚拟机的出站连接尚不支持。此功能需要 `net/libslirp` Port。[c5359e2af5ab](https://cgit.freebsd.org/src/commit/?id=c5359e2af5ab) **（由 Innovate UK 赞助）**

[bhyve(4)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=4&format=html) 现在可以为虚拟机内存配置 NUMA 拓扑。此外，可以为每个虚拟机 NUMA 域定义 [domainset(9)](https://man.freebsd.org/cgi/man.cgi?query=domainset&sektion=9&format=html) 策略，用于指定主机内存用于支撑该虚拟机 NUMA 域的物理内存，类似于 [cpuset(1)](https://man.freebsd.org/cgi/man.cgi?query=cpuset&sektion=1&format=html) 的 `-n` 选项。目前仅支持 amd64 虚拟机。[f1d705d4f431](https://cgit.freebsd.org/src/commit/?id=f1d705d4f431)

[bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 的 VNC 服务器在使用 [www/novnc](https://cgit.freebsd.org/ports/tree/www/novnc/) 客户端时将显示正确的颜色。[f9e09dc5b1d5](https://cgit.freebsd.org/src/commit/?id=f9e09dc5b1d5)

在使用启动 ROM（即不使用 bhyveload(8)）运行 [bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 虚拟机时，bhyve 现在假设启动 ROM 会启用 PCI BAR 解码。这与某些启动 ROM（尤其是过时的 `edk2-bhyve` 构建）不兼容。要恢复旧行为，请在 bhyve 配置中添加 `pci.enable_bars='true'`。注意，`uefi-edk2-bhyve` 包已重命名为 `edk2-bhyve`。[e962b37bf0ff](https://cgit.freebsd.org/src/commit/?id=e962b37bf0ff) **（由 Innovate UK 赞助）**

amd64 [bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 的 `lpc.bootrom` 和 `lpc.bootvars` 选项已被弃用，请改用顶层 `bootrom` 和 `bootvars` 选项。[43caa2e805c2](https://cgit.freebsd.org/src/commit/?id=43caa2e805c2) **（由 Innovate UK 赞助）**

NVMM 虚拟机监控程序现在已被检测到。[34f40baca641](https://cgit.freebsd.org/src/commit/?id=34f40baca641)

在 Hyper-V 下，TLB 刷新现在使用 hypercall 而非 IPI，从而提升最多 40% 的 TLB 性能。[7ece5993b787](https://cgit.freebsd.org/src/commit/?id=7ece5993b787) **（由 Microsoft 赞助）**

## 多媒体（Multimedia）

音频栈进行了多项改进，包括在 [mixer(8)](https://man.freebsd.org/cgi/man.cgi?query=mixer&sektion=8&format=html) 中支持热插拔，以及新增 [mididump(1)](https://man.freebsd.org/cgi/man.cgi?query=mididump&sektion=1&format=html) 工具。[cf9d2fb18433](https://cgit.freebsd.org/src/commit/?id=cf9d2fb18433) **（由 The FreeBSD Foundation 赞助）** [7224e9f2d4af](https://cgit.freebsd.org/src/commit/?id=7224e9f2d4af) **（由 The FreeBSD Foundation 赞助）**

新增实用工具 [sndctl(8)](https://man.freebsd.org/cgi/man.cgi?query=sndctl&sektion=8&format=html)，将用于查看和操作音频设备设置（sysctls、`/dev/sndstat`）的各类接口集中到一个工具中，其控制界面类似于 `mixer(8)`。[44e5a0150835](https://cgit.freebsd.org/src/commit/?id=44e5a0150835), [9a37f1024ceb](https://cgit.freebsd.org/src/commit/?id=9a37f1024ceb) **（由 The FreeBSD Foundation 赞助）**

`virtual_oss` 已被导入到基础系统。从 FreeBSD 15.0 开始，`audio/virtual_oss` Port 将不再构建。对于用户而言，唯一实际变化是安装过程：基础系统已提供所有功能，以下可选组件仍可通过 Ports 安装：

* sndio 后端支持：`audio/virtual_oss_sndio`
* 蓝牙后端支持：`audio/virtual_oss_bluetooth`
* `virtual_equalizer(8)`：`audio/virtual_oss_equalizer`

除此之外，`virtual_oss` 应正常工作。`virtual_oss` 用户可以卸载 `audio/virtual_oss`，改用基础系统提供的版本。[5a31c623143f](https://cgit.freebsd.org/src/commit/?id=5a31c623143f) **（由 The FreeBSD Foundation 赞助）**

## Ports Collection 和 Package 基础设施

本节介绍 FreeBSD Ports、Package 基础设施以及包管理和安装工具的变化。

默认 `/etc/pkg/FreeBSD.conf` [pkg(8)](https://man.freebsd.org/cgi/man.cgi?query=pkg&sektion=8&format=html) 配置文件中新增了 `FreeBSD-kmods` 仓库。该仓库包含专门为 15.0-RELEASE 编译的内核模块，而非 15-STABLE 分支。使用此仓库安装内核模块可以让接口不稳定的驱动程序（尤其是显卡驱动）在主 15-STABLE 仓库的包基于旧版本构建时仍能正常工作。[a47542f71511](https://cgit.freebsd.org/src/commit/?id=a47542f71511)

`/etc/pkg/FreeBSD.conf` 中定义的 `FreeBSD` 和 `FreeBSD-kmods` 仓库已分别重命名为 `FreeBSD-ports` 和 `FreeBSD-ports-kmods`。若用户在 `/usr/local/etc/pkg/repos` 中覆盖了这些仓库，需要调整配置以匹配新名称。

### 安装器

FreeBSD 安装器 [bsdinstall(8)](https://man.freebsd.org/cgi/man.cgi?query=bsdinstall&sektion=8&format=html) 现在支持在 FreeBSD 基础系统安装完成后下载并安装固件包。[03c07bdc8b31](https://cgit.freebsd.org/src/commit/?id=03c07bdc8b31) **（由 The FreeBSD Foundation 赞助）**

### 包管理变更

bootonly ISO 和 mini-memstick 镜像现在包含 [net/wifi-firmware-iwlwifi-kmod](https://cgit.freebsd.org/ports/tree/net/wifi-firmware-iwlwifi-kmod/) 和 [net/wifi-firmware-rtw88-kmod](https://cgit.freebsd.org/ports/tree/net/wifi-firmware-rtw88-kmod/) 包，使得在支持这些固件包的系统上可以通过无线连接进行安装。[655fcdde1aff](https://cgit.freebsd.org/src/commit/?id=655fcdde1aff) **（由 The FreeBSD Foundation 赞助）**

DVD ISO 中增加了 [net/wifi-firmware-kmod@release](https://cgit.freebsd.org/ports/tree/net/wifi-firmware-kmod/) 包，为更多 Wi-Fi 驱动提供固件支持。[8c6df7ead19c](https://cgit.freebsd.org/src/commit/?id=8c6df7ead19c) **（由 The FreeBSD Foundation 赞助）**
