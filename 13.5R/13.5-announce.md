# FreeBSD 13.5-RELEASE 发行公告

日期：2025 年 3 月 11 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 13.5-RELEASE 现已发布。
这是 stable/13 分支的第六个也是最终版本。

由于本次发布处于旧版稳定分支的后期，新增功能较少；重点主要在于维护。
因此，自 13.4-RELEASE 以来的更改主要由缺陷修复、驱动程序更新和外部维护软件的新版本组成。

完整的新功能、支持的硬件和已知问题列表，请参阅在线发行说明、硬件兼容性说明和勘误列表，地址如下：

* <https://www.FreeBSD.org/releases/13.5R/relnotes/>
* <https://www.FreeBSD.org/releases/13.5R/hardware/>
* <https://www.FreeBSD.org/releases/13.5R/errata/>

关于 FreeBSD 发布工程活动的更多信息，请参阅：

* <https://www.FreeBSD.org/releng/>

## 获取途径

FreeBSD 13.5-RELEASE 现已支持 amd64、i386、aarch64、armv6、armv7、powerpc、powerpc64、powerpc64le、powerpcspe 和 riscv64 架构。

FreeBSD 13.5-RELEASE 可从可启动 ISO 镜像或通过网络安装。
部分架构还支持从 USB 闪存盘安装。
所需文件可按以下说明下载。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/13.5R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/13.5R/announce.asc>

发行版所提供镜像的用途如下：

**dvd1**

包含安装 FreeBSD 基本系统、文档、调试发行版和少量预构建软件包（旨在启动图形工作站）所需的全部内容。还支持启动到基于“livefs”的救援模式。若你能刻录并使用 DVD 介质，这应是你所需的全部。

此外，可将其写入 USB 闪存盘（U 盘）用于 amd64 架构，在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。

作为使用 dvd1 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-13.5-RELEASE-amd64-dvd1.iso \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**disc1**

包含 FreeBSD 基本系统。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

此外，可将其写入 USB 闪存盘（U 盘）用于 amd64 架构，在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 disc1 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-13.5-RELEASE-amd64-disc1.iso \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**bootonly**

支持使用 CDROM 驱动器启动机器，但不包含从 CD 本身安装 FreeBSD 所需的安装发行版。从 CD 启动后，需要执行基于网络的安装（例如从 HTTP 或 FTP 服务器）。

此外，可将其写入 USB 闪存盘（U 盘）用于 amd64 架构，在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 bootonly 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-13.5-RELEASE-amd64-bootonly.iso \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**memstick**

可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-13.5-RELEASE-amd64-memstick.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**mini-memstick**

可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-13.5-RELEASE-amd64-mini-memstick.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**FreeBSD/arm SD 卡镜像**

可写入 SD 卡并用于启动受支持的 arm 系统。SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 5 GB 的 SD 卡上。

为方便无控制台访问系统的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 `ssh(1)` 访问。此外，`root` 用户密码设为 `root`，强烈建议在获取系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，使用 `dd(1)` 工具，将 _KERNEL_ 替换为系统适用的内核配置名称。

```sh
# dd if=FreeBSD-13.5-RELEASE-arm64-aarch64-RPI.img \
    of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

预安装的虚拟机镜像也可用于 amd64 (x86_64)、i386 (x86_32)、AArch64 (arm64) 和 RISCV (riscv64) 架构，提供 `QCOW2`、`VHD` 和 `VMDK` 磁盘镜像格式，以及 raw（未格式化）镜像。
