# FreeBSD 9.0-RELEASE 发行说明（2012 年 1 月 12 日）

- 原文链接：[FreeBSD 9.0-RELEASE Release Notes](https://www.freebsd.org/releases/9.0R/relnotes/)

这是 FreeBSD 9.0-RELEASE 发行说明的简化版本。

>**重要**
>
> 如果从之前的 FreeBSD 版本升级，请仔细阅读 [发行说明中的升级部分](http://www.FreeBSD.org/releases/9.0R/relnotes-detailed.html#UPGRADE)，了解显著的不兼容性。

## 发布亮点

9.0-RELEASE 的亮点包括以下内容。更多详情，请参阅 [详细发行说明](../relnotes-detailed/)。

- FreeBSD 内核现在支持 Capsicum 能力模式。Capsicum 是一套用于沙箱支持的特性，采用能力模型，其中能力为文件描述符。已经向 **GENERIC** 内核添加了两个新选项 `CAPABILITIES` 和 `CAPABILITY_MODE`。Capsicum 详见 [http://www.cl.cam.ac.uk/research/security/capsicum/](http://www.cl.cam.ac.uk/research/security/capsicum/)。[r219129](http://svn.freebsd.org/viewvc/base?view=revision&revision=219129)
- 现已实现 [hhook(9)](http://www.freebsd.org/cgi/man.cgi?query=hhook&sektion=9&manpath=FreeBSD+9.0-RELEASE)（帮助钩子）和 [khelp(9)](http://www.freebsd.org/cgi/man.cgi?query=khelp&sektion=9&manpath=FreeBSD+9.0-RELEASE)（内核帮助）KPI。这些是 [pfil(9)](http://www.freebsd.org/cgi/man.cgi?query=pfil&sektion=9&manpath=FreeBSD+9.0-RELEASE) 框架的超集，用于在内核中进行更通用的使用。[hhook(9)](http://www.freebsd.org/cgi/man.cgi?query=hhook&sektion=9&manpath=FreeBSD+9.0-RELEASE) KPI 提供了一种方式，使得内核子系统能够导出钩子点，供 [khelp(9)](http://www.freebsd.org/cgi/man.cgi?query=khelp&sektion=9&manpath=FreeBSD+9.0-RELEASE) 模块挂接，从而为内核提供增强或新增的功能。[khelp(9)](http://www.freebsd.org/cgi/man.cgi?query=khelp&sektion=9&manpath=FreeBSD+9.0-RELEASE) KPI 提供了一个管理 [khelp(9)](http://www.freebsd.org/cgi/man.cgi?query=khelp&sektion=9&manpath=FreeBSD+9.0-RELEASE) 模块的框架，这些模块间接使用 [hhook(9)](http://www.freebsd.org/cgi/man.cgi?query=hhook&sektion=9&manpath=FreeBSD+9.0-RELEASE) KPI 在内核中的钩子点注册其钩子函数。这些功能为内核提供了以结构化的方式在运行时动态扩展的能力，且能够保持 ABI 兼容性。[r216758](http://svn.freebsd.org/viewvc/base?view=revision&revision=216758), [r216615](http://svn.freebsd.org/viewvc/base?view=revision&revision=216615)
- 现已实现新的资源计费 API。它可以记录每个进程、每个 Jail 和每个登录类的资源使用信息。请注意，该功能默认并未构建或安装。如需构建并安装该功能，请在内核配置文件中指定选项 `RACCT`，并按照 [FreeBSD 手册](http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/makeworld.html) 中的说明重新构建基本系统。[r220137](http://svn.freebsd.org/viewvc/base?view=revision&revision=220137)
- 现已实现新的资源限制 API。它与 `RACCT` 资源计费实现协同工作，并基于其维护的规则集和当前的资源使用情况采取用户可配置的操作。新增了 [rctl(8)](http://www.freebsd.org/cgi/man.cgi?query=rctl&sektion=8&manpath=FreeBSD+9.0-RELEASE) 工具来管理用户空间中的规则。请注意，该功能默认并未构建或安装。如需构建并安装该功能，请在内核配置文件中指定选项 `RCTL`，并按照 [FreeBSD 手册](http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/makeworld.html) 中的说明重新构建基本系统。[r220163](http://svn.freebsd.org/viewvc/base?view=revision&revision=220163)
- [powerpc] FreeBSD/powerpc 现在使用 OtherOS 功能支持 Sony Playstation 3（该功能在固件版本 3.15 及更早版本中可用）。[r217044](http://svn.freebsd.org/viewvc/base?view=revision&revision=217044)
- [amd64, i386] FreeBSD [usb(4)](http://www.freebsd.org/cgi/man.cgi?query=usb&sektion=4&manpath=FreeBSD+9.0-RELEASE) 子系统现在默认支持 USB 3.0。[r223098](http://svn.freebsd.org/viewvc/base?view=revision&revision=223098)

- [ipfw(8)](http://www.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&manpath=FreeBSD+9.0-RELEASE) 现在在 `fwd` 动作中支持 IPv6。[r225044](http://svn.freebsd.org/viewvc/base?view=revision&revision=225044)
- [ipfw(8)](http://www.freebsd.org/cgi/man.cgi?query=ipfw&sektion=8&manpath=FreeBSD+9.0-RELEASE) 现在支持 `call` 和 `return` 动作。执行 `call number` 动作时，当前规则编号会被保存到内部栈中，规则集处理将从编号大于等于 `number` 的第一条规则继续。`return` 动作会取出最近一次 `call` 动作保存到内部栈的规则编号，并将规则集处理返回到编号大于该保存编号的第一条规则。[r223666](http://svn.freebsd.org/viewvc/base?view=revision&revision=223666)
- 为了支持 Infiniband，已将 OFED（OpenFabrics Enterprise Distribution）版本 1.5.3 导入到基本系统中。请注意，该功能默认并未构建或安装。如需构建并安装该功能，请在 **/etc/src.conf** 中指定 `WITH_OFED=yes`，并按照 [FreeBSD 手册](http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/makeworld.html) 中的说明重新构建基本系统。[r219820](http://svn.freebsd.org/viewvc/base?view=revision&revision=219820)
- FreeBSD TCP/IP 网络协议栈现在支持 RFC 3021“Using 31-Bit Prefixes on IPv4 Point-to-Point Links”中所述的 /31 IPv4 前缀。[r226572](http://svn.freebsd.org/viewvc/base?view=revision&revision=226572)
- FreeBSD TCP/IP 网络协议栈现在支持 [mod_cc(9)](http://www.freebsd.org/cgi/man.cgi?query=mod_cc&sektion=9&manpath=FreeBSD+9.0-RELEASE) 可插拔拥塞控制框架。这允许将 TCP 拥塞控制算法实现为可动态加载的内核模块。截至 9.0-RELEASE，可用的内核模块包括：用于 CAIA-Hamilton-Delay 算法的 [cc_chd(4)](http://www.freebsd.org/cgi/man.cgi?query=cc_chd&sektion=4&manpath=FreeBSD+9.0-RELEASE)、用于 CUBIC 算法的 [cc_cubic(4)](http://www.freebsd.org/cgi/man.cgi?query=cc_cubic&sektion=4&manpath=FreeBSD+9.0-RELEASE)、用于 Hamilton-Delay 算法的 [cc_hd(4)](http://www.freebsd.org/cgi/man.cgi?query=cc_hd&sektion=4&manpath=FreeBSD+9.0-RELEASE)、用于 H-TCP 算法的 [cc_htcp(4)](http://www.freebsd.org/cgi/man.cgi?query=cc_htcp&sektion=4&manpath=FreeBSD+9.0-RELEASE)、用于 NewReno 算法的 [cc_newreno(4)](http://www.freebsd.org/cgi/man.cgi?query=cc_newreno&sektion=4&manpath=FreeBSD+9.0-RELEASE) 以及用于 Vegas 算法的 [cc_vegas(4)](http://www.freebsd.org/cgi/man.cgi?query=cc_vegas&sektion=4&manpath=FreeBSD+9.0-RELEASE)。默认算法可以通过新的 [sysctl(8)](http://www.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&manpath=FreeBSD+9.0-RELEASE) 变量 `net.inet.tcp.cc.algorithm` 来设置。该值必须设置为 `net.inet.tcp.cc.available` 列出的名称之一，启动时默认值为 `newreno`。更多详情，请参阅 [mod_cc(4)](http://www.freebsd.org/cgi/man.cgi?query=mod_cc&sektion=4&manpath=FreeBSD+9.0-RELEASE) 和 [mod_cc(9)](http://www.freebsd.org/cgi/man.cgi?query=mod_cc&sektion=9&manpath=FreeBSD+9.0-RELEASE) 手册页。[r216109](http://svn.freebsd.org/viewvc/base?view=revision&revision=216109), [r216114](http://svn.freebsd.org/viewvc/base?view=revision&revision=216114), [r216115](http://svn.freebsd.org/viewvc/base?view=revision&revision=216115), [r218152](http://svn.freebsd.org/viewvc/base?view=revision&revision=218152), [r218153](http://svn.freebsd.org/viewvc/base?view=revision&revision=218153), [r218155](http://svn.freebsd.org/viewvc/base?view=revision&revision=218155)
- 添加了 [h_ertt(4)](http://www.freebsd.org/cgi/man.cgi?query=h_ertt&sektion=4&manpath=FreeBSD+9.0-RELEASE)（增强往返时间）[khelp(9)](http://www.freebsd.org/cgi/man.cgi?query=khelp&sektion=9&manpath=FreeBSD+9.0-RELEASE) 模块。该模块允许在 TCP/IP 网络协议栈中对每个连接进行低噪声的瞬时 RTT 估算，即使在连接使用了延迟确认和/或 TSO（TCP 分段卸载）的情况下也能保持稳健的实现。[r217806](http://svn.freebsd.org/viewvc/base?view=revision&revision=217806)
- 添加了新的 [tcp(4)](http://www.freebsd.org/cgi/man.cgi?query=tcp&sektion=4&manpath=FreeBSD+9.0-RELEASE) 套接字选项 `TCP_CONGESTION`。这允许选择或查询 TCP/IP 网络协议栈将用于套接字上连接的拥塞控制算法。[r218912](http://svn.freebsd.org/viewvc/base?view=revision&revision=218912)
- [ng_netflow(4)](http://www.freebsd.org/cgi/man.cgi?query=ng_netflow&sektion=4&manpath=FreeBSD+9.0-RELEASE) [netgraph(4)](http://www.freebsd.org/cgi/man.cgi?query=netgraph&sektion=4&manpath=FreeBSD+9.0-RELEASE) 节点现在支持 NetFlow 版本 9。新增了 `export9` 钩子用于 NetFlow v9 数据。请注意，数据导出可以同时在版本 5 和版本 9 中进行。[r219183](http://svn.freebsd.org/viewvc/base?view=revision&revision=219183)
- 添加了 [geom_map(4)](http://www.freebsd.org/cgi/man.cgi?query=geom_map&sektion=4&manpath=FreeBSD+9.0-RELEASE) GEOM 类。这允许基于设备的硬编码布局（例如嵌入式闪存存储）生成多个 geom 提供者，而无需显式分区表。更多信息，请参阅 [geom_map(4)](http://www.freebsd.org/cgi/man.cgi?query=geom_map&sektion=4&manpath=FreeBSD+9.0-RELEASE) 手册页。[r220559](http://svn.freebsd.org/viewvc/base?view=revision&revision=220559)
- 添加了 [graid(8)](http://www.freebsd.org/cgi/man.cgi?query=graid&sektion=8&manpath=FreeBSD+9.0-RELEASE) GEOM 类。这是 [ataraid(4)](http://www.freebsd.org/cgi/man.cgi?query=ataraid&sektion=4&manpath=FreeBSD+9.0-RELEASE) 驱动程序的替代品，支持各种基于 BIOS 的软件 RAID。[r219974](http://svn.freebsd.org/viewvc/base?view=revision&revision=219974)
- 添加了 [tws(4)](http://www.freebsd.org/cgi/man.cgi?query=tws&sektion=4&manpath=FreeBSD+9.0-RELEASE) 驱动程序，支持 3ware 9750 SATA+SAS 6Gb/s RAID 控制器。[r226115](http://svn.freebsd.org/viewvc/base?view=revision&revision=226115)
- FreeBSD 快速文件系统现在支持 softupdates 日志记录。它向启用了 softupdates 的文件系统引入了意图日志，从而消除了即使在非正常关机时也进行后台 [fsck(8)](http://www.freebsd.org/cgi/man.cgi?query=fsck&sektion=8&manpath=FreeBSD+9.0-RELEASE) 的需要。可以使用 [newfs(8)](http://www.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&manpath=FreeBSD+9.0-RELEASE) 工具的 `-j` 选项或 [tunefs(8)](http://www.freebsd.org/cgi/man.cgi?query=tunefs&sektion=8&manpath=FreeBSD+9.0-RELEASE) 工具的 `-j enable` 选项，按文件系统启用该功能。请注意，9.0-RELEASE 安装程序会自动为新创建的 UFS 文件系统启用 softupdates 日志记录。[r207141](http://svn.freebsd.org/viewvc/base?view=revision&revision=207141), [r218726](http://svn.freebsd.org/viewvc/base?view=revision&revision=218726)
- FreeBSD 快速文件系统现在在释放数据块时支持 `TRIM` 命令。[newfs(8)](http://www.freebsd.org/cgi/man.cgi?query=newfs&sektion=8&manpath=FreeBSD+9.0-RELEASE) 和 [tunefs(8)](http://www.freebsd.org/cgi/man.cgi?query=tunefs&sektion=8&manpath=FreeBSD+9.0-RELEASE) 工具中的新 `-t` 标志用于为文件系统设置 TRIM 启用标志。TRIM 启用标志使文件系统在释放每个块时向底层设备发送删除请求。`TRIM` 命令在 ATA8-ACS2 标准中被指定为数据集管理命令，用于将与已删除数据块相关的信息传递给设备，特别是用于 SSD（固态硬盘）的优化。[r216796](http://svn.freebsd.org/viewvc/base?view=revision&revision=216796)
- FreeBSD NFS 子系统已更新。新实现除了支持 NFS 版本 2 和 3 外，还支持 NFS 版本 4。NFS 服务器和客户端的内核选项从 `NFSSERVER` 和 `NFSCLIENT` 更改为 `NFSD` 和 `NFSCL`。以 `vfs.nfssrv.` 开头的 [sysctl(8)](http://www.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&manpath=FreeBSD+9.0-RELEASE) 变量已重命名为 `vfs.nfsd.`。NFS 服务器现在支持 `vfs.nfsd.server_max_nfsvers` 和 `vfs.nfsd.server_min_nfsvers` [sysctl(8)](http://www.freebsd.org/cgi/man.cgi?query=sysctl&sektion=8&manpath=FreeBSD+9.0-RELEASE) 变量，用于指定服务器接受的最大和最小 NFS 版本号。默认值分别设置为 `3` 和 `2`。[r221124](http://svn.freebsd.org/viewvc/base?view=revision&revision=221124)
- FreeBSD ZFS 子系统已更新至 SPA（Storage Pool Allocator，也称为 zpool）版本 28。现在支持数据去重、三重奇偶校验 RAIDZ（raidz3）、快照保留、日志设备移除、zfs diff、zpool split、`zpool import -F` 以及只读 zpool import。[r219089](http://svn.freebsd.org/viewvc/base?view=revision&revision=219089)

- 已将 [readline(3)](http://www.freebsd.org/cgi/man.cgi?query=readline&sektion=3&manpath=FreeBSD+9.0-RELEASE) API 集导入到 **libedit** 中。这基于 NetBSD 的实现，BSD 许可的工具现在使用它而不是 GNU **libreadline**。[r220370](http://svn.freebsd.org/viewvc/base?view=revision&revision=220370)
- [rtsold(8)](http://www.freebsd.org/cgi/man.cgi?query=rtsold&sektion=8&manpath=FreeBSD+9.0-RELEASE) 和 [rtadvd(8)](http://www.freebsd.org/cgi/man.cgi?query=rtadvd&sektion=8&manpath=FreeBSD+9.0-RELEASE) 守护进程现在支持 RFC 6106“IPv6 Router Advertisement Options for DNS Configuration”中所述的 RDNSS 和 DNSSL 选项。新增了 [rtadvctl(8)](http://www.freebsd.org/cgi/man.cgi?query=rtadvctl&sektion=8&manpath=FreeBSD+9.0-RELEASE) 工具用于控制 [rtadvd(8)](http://www.freebsd.org/cgi/man.cgi?query=rtadvd&sektion=8&manpath=FreeBSD+9.0-RELEASE) 守护进程。[r222732](http://svn.freebsd.org/viewvc/base?view=revision&revision=222732), [r224006](http://svn.freebsd.org/viewvc/base?view=revision&revision=224006)

- 修复了 [tftpd(8)](http://www.freebsd.org/cgi/man.cgi?query=tftpd&sektion=8&manpath=FreeBSD+9.0-RELEASE) 守护进程中的一个错误。该错误在传输大文件时会导致互操作性问题。[r224536](http://svn.freebsd.org/viewvc/base?view=revision&revision=224536)
- [utmp(5)](http://www.freebsd.org/cgi/man.cgi?query=utmp&sektion=5&manpath=FreeBSD+9.0-RELEASE) 用户账目数据库已被 [utmpx(3)](http://www.freebsd.org/cgi/man.cgi?query=utmpx&sektion=3&manpath=FreeBSD+9.0-RELEASE) 替代。用户账目工具现在将专门使用 **utmpx** 数据库文件。可以使用 [wtmpcvt(1)](http://www.freebsd.org/cgi/man.cgi?query=wtmpcvt&sektion=1&manpath=FreeBSD+9.0-RELEASE) 工具将 **wtmp** 文件转换为新格式，使得更新后的工具能够读取它们。[r202188](http://svn.freebsd.org/viewvc/base?view=revision&revision=202188)
- [zpool(8)](http://www.freebsd.org/cgi/man.cgi?query=zpool&sektion=8&manpath=FreeBSD+9.0-RELEASE) 工具现在支持 `zpool labelclear` 命令。这允许擦除池中非活动驱动器上的标签数据。[r224171](http://svn.freebsd.org/viewvc/base?view=revision&revision=224171)

当前正在开发的所有平台列表可以在 [支持的平台](../../../platforms/index.html) 页面找到。
