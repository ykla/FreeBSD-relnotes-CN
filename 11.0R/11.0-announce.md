# FreeBSD 11.0-RELEASE 发行公告

日期：2016 年 10 月 10 日

FreeBSD 发布工程团队很高兴地宣布 FreeBSD 11.0-RELEASE 现已发布。这是 stable/11 分支的首个版本。

部分亮点如下：

* OpenSSH `DSA` 密钥生成默认已禁用。在升级前更新 OpenSSH 密钥非常重要。此外，Protocol 1 支持已移除。
* OpenSSH 已更新至 7.2p2。
* 多个驱动新增了对 802.11n 的额外无线支持。
* 默认情况下，[ifconfig(8)](https://man.freebsd.org/cgi/man.cgi?query=ifconfig&sektion=8&format=html) 实用工具会将无线接口的默认监管域设置为 FCC。因此，使用默认设置新创建的无线接口违反各国特定法规的可能性降低。
* [svnlite(1)](https://man.freebsd.org/cgi/man.cgi?query=svnlite&sektion=1&format=html) 实用工具已更新至版本 1.9.4。
* [libblacklist(3)](https://man.freebsd.org/cgi/man.cgi?query=libblacklist&sektion=3&format=html) 库及应用程序已从 NetBSD 项目移植。
* 新增对 AArch64（arm64）架构的支持。
* [bhyve(8)](https://man.freebsd.org/cgi/man.cgi?query=bhyve&sektion=8&format=html) 虚拟机监控器新增原生图形支持。
* 新增更广泛的无线网络驱动支持。

完整的新功能和已知问题列表，请参阅在线发行说明和勘误列表，地址如下：

* <https://www.FreeBSD.org/releases/11.0R/relnotes/>
* <https://www.FreeBSD.org/releases/11.0R/errata/>

关于 FreeBSD 发布工程活动的更多信息，请参阅：

* <https://www.FreeBSD.org/releng/>

## 重要说明

请注意，由于撤回的 FreeBSD 11.0-RELEASE 镜像在官方公告发布之前已可用，以及若干最后一刻才发现的问题，[uname(1)](https://man.freebsd.org/cgi/man.cgi?query=uname&sektion=1&format=html) 将显示 FreeBSD 11.0-RELEASE-p1，因为这些镜像由 `releng/11.0` 分支的补丁级修订版生成。

从镜像上最初提供的镜像或通过 [freebsd-update(8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 在最终版本重新构建前安装 FreeBSD 11.0-RELEASE 的用户，请立即将系统升级至 FreeBSD 11.0-RELEASE-p1。

从系统源代码升级的用户应使用 `releng/11.0` 修订版 `r306420`。

通过 [freebsd-update(8)](https://man.freebsd.org/cgi/man.cgi?query=freebsd-update&sektion=8&format=html) 升级的用户应按以下说明将系统更新至 FreeBSD 11.0-RELEASE-p1：

### 从 FreeBSD 11.0-RELEASE 升级

```sh
# : > /usr/bin/bspatch
# freebsd-update fetch
# freebsd-update install
```

### 从 FreeBSD 10.3-RELEASE 及更早版本升级

```sh
# : > /usr/bin/bspatch
# freebsd-update upgrade -r 11.0-RELEASE
# freebsd-update install
<重新启动系统>
# freebsd-update install
<重新构建第三方软件>
# freebsd-update install
```

EC2(TM) 用户请阅读 FreeBSD 11.0-RELEASE 的勘误说明，其中涉及在发布周期后期发现的某项问题，从早期 FreeBSD 版本升级时可能导致系统在引导过程中挂起。新的 EC2(TM) 安装不受影响，但运行早期版本的现有安装建议等待该问题在勘误通知中解决后再升级。

## 获取途径

FreeBSD 11.0-RELEASE 现已支持 amd64、i386、powerpc、powerpc64、sparc64、armv6 和 aarch64 架构。

FreeBSD 11.0-RELEASE 可从可启动 ISO 镜像或通过网络安装。部分架构还支持从 USB 闪存盘安装。所需文件可按以下说明通过 FTP 下载。虽然部分较小的 FTP 镜像可能不包含所有架构，但通常都会包含 amd64 和 i386 等较常见的架构。

本消息底部包含发行版 ISO、闪存盘和 SD 卡镜像的 SHA512 和 SHA256 校验和。

发行版镜像的 PGP 签名校验和也可从以下地址获取：

* <https://www.FreeBSD.org/releases/11.0R/signatures/>

本公告的 PGP 签名版本可在以下地址获取：

* <https://www.FreeBSD.org/releases/11.0R/announce.asc>

发行版所提供镜像的用途如下：

**dvd1**

包含安装 FreeBSD 基本系统、文档以及少量预构建软件包（旨在启动图形工作站）所需的全部内容。还支持启动到基于“livefs”的救援模式。若你能刻录并使用 DVD 介质，这应是你所需的全部。

**disc1**

包含 FreeBSD 基本系统。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

**bootonly**

支持使用 CDROM 驱动器启动机器，但不包含从 CD 本身安装 FreeBSD 所需的安装发行版。从 CD 启动后，需要执行基于网络的安装（例如从 FTP 服务器）。

**memstick**

可写入 USB 闪存盘（U 盘），用于在可从 USB 启动的机器上进行安装。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-11.0-RELEASE-amd64-memstick.img of=/dev/da0 bs=10240 conv=sync
```

务必确保目标（`of=`）正确。

**mini-memstick**

可写入 USB 闪存盘（U 盘）用于启动机器，但介质本身不包含安装发行版，与 bootonly 镜像类似。还支持启动到基于“livefs”的救援模式。不含预构建的软件包。

作为使用 mini-memstick 镜像的示例，假设 USB 驱动器在你的机器上显示为 **/dev/da0**，类似如下命令应可工作：

```sh
# dd if=FreeBSD-11.0-RELEASE-amd64-mini-memstick.img of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。

**FreeBSD/arm SD 卡镜像**

可写入 SD 卡并用于启动受支持的 arm 系统。SD 卡镜像包含完整的 FreeBSD 安装，可安装到小至 512Mb 的 SD 卡上。

为方便没有控制台访问权限的用户，默认提供 `freebsd` 用户（密码为 `freebsd`）用于 [ssh(1)](https://man.freebsd.org/cgi/man.cgi?query=ssh&sektion=1&format=html) 访问。此外，`root` 用户密码设置为 `root`，强烈建议在获得系统访问权限后立即更改这两个用户的密码。

要将 FreeBSD/arm 镜像写入 SD 卡，请使用 [dd(1)](https://man.freebsd.org/cgi/man.cgi?query=dd&sektion=1&format=html) 工具，将 *KERNEL* 替换为系统对应的内核配置名称。

```sh
# dd if=FreeBSD-11.0-RELEASE-arm-armv6-KERNEL.img of=/dev/da0 bs=1m conv=sync
```

务必确保目标（`of=`）正确。
