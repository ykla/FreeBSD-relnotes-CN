# FreeBSD 15.0-RELEASE 硬件兼容列表

## 引言

本硬件兼容性说明列出所有已知可在 FreeBSD 15.0 上工作的硬件。
其他配置也可能可用，但尚未经过测试。
欢迎对本列表提出反馈、更新和修正！

## 支持的平台

本节介绍 FreeBSD 平台项目当前支持的 CPU 架构、支持等级以及各平台所支持的系统。

### x86-64

[FreeBSD/amd64](/platforms/amd64) 将 x86-64 作为完全支持的 [Tier-1](https://docs.freebsd.org/en/articles/committers-guide/#archs) 架构。
发行介质预期可在所有至少配备 256 MiB 内存的 x86-64 机器上运行。
本发行版整合了多项优化和增强功能，以充分发挥基于 FreeBSD/amd64 系统的全部能力，在各种工作负载下提供更佳性能。

### RISC-V

[FreeBSD/riscv64](https://wiki.freebsd.org/riscv) 将 64 位 RISC-V 作为 [Tier-2](https://docs.freebsd.org/en/articles/committers-guide/#archs) 架构。

RISC-V 支持标志着这一开源操作系统发展中的重要里程碑。
FreeBSD 是首个内置支持 RISC-V 引导的操作系统，体现了其对新兴硬件架构的承诺。

* HiFive Unleashed (SiFive FU540)
* HiFive Unmatched (SiFive FU740)
* StarFive VisionFive 2 (StarFive JH7110)
* Spike
* QEMU

**ahci**
驱动还支持使用 Intel Rapid Storage Technology (RST) 充当 nvme(4)
PCI 桥的 AHCI 设备。
若要使用
nvme(4)
设备，必须在 BIOS 中将 SATA 模式从 RST 改为 AHCI，
否则须接受因中断共享导致 RST 启用时的性能。
FreeBSD
会自动检测启用此扩展且处于 RST 模式的 AHCI 设备。
此时，
**ahci**
会为
**ahci**
设备附加
nvme(4)
子设备。

当设备通过
fdt(4)
或
acpi(4)
配置时，最常需要 quirks 和自定义配置。
